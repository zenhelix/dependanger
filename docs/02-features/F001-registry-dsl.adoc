Kotlin DSL для определения библиотек, версий, бандлов, плагинов, импорта BOM, целевых платформ, профилей генерации и правил совместимости.

== Описание

Registry DSL предоставляет Kotlin DSL (`DependangerDsl`) для определения централизованного каталога зависимостей.
Пользователь описывает библиотеки, версии, бандлы, импорты BOM, целевые платформы и правила совместимости в декларативном стиле.

== User Stories

=== US-001: Определение библиотеки

[quote]
____
Как разработчик, я хочу определить библиотеку с Maven координатами и метаданными, чтобы она была включена в генерируемый каталог.
____

**Acceptance Criteria:**

[cols="1,3,3,3",options="header"]
|===
| ID | Given | When | Then

| AC-001-1
| DSL с пустым блоком `libraries { }`
| Добавляю `library("my-lib", "com.example:my-lib:1.0.0")`
| Библиотека создана с `alias=my-lib`, `group=com.example`, `name=my-lib`, `version=1.0.0`

| AC-001-2
| DSL с библиотекой без версии
| Добавляю `library("spring-web", "org.springframework:spring-web")`
| Библиотека создана с `version=null`, версия будет резолвиться из BOM или останется null

| AC-001-3
| DSL с библиотекой
| Добавляю `tags("web", "spring")` в блок библиотеки
| Библиотека имеет `tags=["web", "spring"]`

| AC-001-4
| DSL с библиотекой
| Добавляю `requires { jdk { min = 17 } }`
| Библиотека имеет `requires.jdk.min=17`

| AC-001-5
| DSL с библиотекой
| Добавляю `deprecated(replacedBy = "new-lib", message = "Use new-lib")`
| Библиотека помечена как deprecated с указанием замены
|===

=== US-009: Определение Platform Library

[quote]
____
Как разработчик, я хочу определить Platform Library (BOM) для экспорта в TOML, чтобы потребители могли подключить её через `platform()`.
____

**Acceptance Criteria:**

[cols="1,3,3,3",options="header"]
|===
| ID | Given | When | Then

| AC-009-1
| DSL с пустым блоком `libraries { }`
| Добавляю `platformLibrary("spring-bom", "org.springframework.boot:spring-boot-dependencies", version = versionRef("spring-boot"))`
| Platform library создана с `type = PLATFORM`

| AC-009-2
| DSL с platformLibrary `spring-bom`
| Генерирую TOML
| В TOML добавлена запись `spring-bom = \{ group = "org.springframework.boot", name = "spring-boot-dependencies", version.ref = "spring-boot" \}` без явного указания type (Gradle определяет автоматически при использовании через `platform()`)

| DSL с platformLibrary и bomImport с одинаковыми координатами

| DSL с platformLibrary без версии
| Запускаю валидацию

== Спецификация DSL

=== Базовая структура

[source,kotlin]
----
val dsl = DependangerDsl()

dsl.versions {
    version("kotlin", "2.1.20")
    version("spring-boot", "3.4.0") {
        fallback("2.7.18") { jdkBelow(17) }
    }
}

dsl.libraries {
    library("kotlin-stdlib", "org.jetbrains.kotlin:kotlin-stdlib",
        version = versionRef("kotlin")) {
        tags("kotlin", "core")
    }

    library("spring-web", "org.springframework:spring-web") {
        tags("spring", "web")
        requires { jdk { min = 17 } }
    }
}

dsl.bundles {
    bundle("kotlin-core") {
        libraries("kotlin-stdlib", "kotlin-reflect")
    }

    bundle("spring-backend") {
        libraries("spring-web", "spring-data-jpa")
        extends("kotlin-core")
    }
}

dsl.plugins {
    plugin("kotlin-jvm", "org.jetbrains.kotlin.jvm",
        version = versionRef("kotlin"))
}

dsl.bomImports {
    bom("spring-boot-bom", "org.springframework.boot:spring-boot-dependencies",
        version = versionRef("spring-boot"))
    bom("kotlin-bom", "org.jetbrains.kotlin:kotlin-bom:2.1.20")
}

dsl.targetPlatforms {
    platform("jvm17") {
        jdk { min = 17; max = 21 }
    }
    platform("android-min21") {
        android { minSdk = 21; targetSdk = 34 }
    }
}

dsl.distributions {
    distribution("android") {
        spec {
            filter {
                byTags { include { anyOf("android", "mobile") } }
            }
        }
    }

    distribution("backend") {
        spec {
            filter {
                byBundles { include("spring-backend") }
            }
        }
    }
}

dsl.compatibility {
    jdkRequirement("spring-jdk17") {
        matches = "org.springframework:*"
        minJdk = 17
    }

    mutualExclusion("json-libs") {
        libraries("jackson-core", "gson")
        severity = WARNING
    }
}

dsl.settings {
    validation {
        onCompatibilityViolation = ValidationAction.FAIL
        onUnresolvedVersion = ValidationAction.WARN
    }
    defaultDistribution = "core"
    toml {
        filename = "libs.versions.toml"
        includeComments = true
    }
    bom {
        groupId = "com.example"
        artifactId = "my-bom"
        version = "1.0.0"
    }
}
----

== Блок versions

=== Синтаксис

[source,kotlin]
----
versions {
    version("kotlin", "2.1.20")

    version("spring-boot", "3.4.0") {
        fallback("2.7.18") { jdkBelow(17) }
        fallback("3.2.0") { kotlinVersionBelow("1.9") }
    }
}
----

=== Доступные условия

[cols="1,3",options="header"]
|===
| Условие | Описание

| `jdkBelow(version)`
| JDK версия < указанной

| `jdkAtLeast(version)`
| JDK версия >= указанной

| `jdkBetween(min, max)`
| JDK версия в диапазоне

| `kotlinVersionBelow(version)`
| Kotlin версия < указанной

| `kotlinVersionAtLeast(version)`
| Kotlin версия >= указанной

| `gradleVersionAtLeast(version)`
| Gradle версия >= указанной

| `gradleVersionBelow(version)`
| Gradle версия < указанной

| `distribution(name)`
| Активен указанный distribution profile

| `environment(key, value)`
| Переменная окружения равна значению

| `all(conditions...)`
| Все условия выполнены

| `any(conditions...)`
| Хотя бы одно условие выполнено

| `not(condition)`
| Условие не выполнено
|===

=== Комбинирование условий

Условия можно комбинировать для создания сложных правил:

[source,kotlin]
----
versions {
    version("spring-boot", "3.4.0") {
        // Одно условие
        fallback("2.7.18") { jdkBelow(17) }

        // AND: все условия должны выполниться
        fallback("3.2.0") {
            all(
                jdkAtLeast(17),
                kotlinVersionBelow("2.0")
            )
        }

        // OR: хотя бы одно условие
        fallback("3.0.0") {
            any(
                jdkBelow(11),
                environment("LEGACY_MODE", "true")
            )
        }

        // NOT: инверсия условия
        fallback("3.3.0") {
            not(distribution("production"))
        }

        // Комбинация AND, OR, NOT
        fallback("2.5.0") {
            all(
                jdkBelow(17),
                any(
                    environment("CI", "true"),
                    not(distribution("local"))
                )
            )
        }
    }
}
----

NOTE: Fallback условия проверяются **по порядку определения**.
Первое matching условие выигрывает.

== Блок libraries

=== Синтаксис

[source,kotlin]
----
libraries {
    // С ссылкой на version alias
    library("kotlin-stdlib", "org.jetbrains.kotlin:kotlin-stdlib",
        version = versionRef("kotlin")) {
        tags("core", "kotlin")
    }

    // С inline версией
    library("slf4j-api", "org.slf4j:slf4j-api:2.0.9") {
        tags("logging")
    }

    // Без версии (будет взята из BomImport)
    library("spring-web", "org.springframework:spring-web") {
        tags("web", "spring")
        requires { jdk { min = 17 } }
    }

    // Platform library для экспорта в TOML
    platformLibrary("spring-boot-bom", "org.springframework.boot:spring-boot-dependencies",
        version = versionRef("spring-boot"))

    // Deprecated библиотека — полная форма
    library("old-http", "com.example:old-http:1.0") {
        tags("http", "legacy")
        deprecated {
            replacedBy = "new-http"
            message = "Use new-http which supports HTTP/2"
            since = "1.2.0"
            removalVersion = "2.0.0"
        }
    }

    // Deprecated библиотека — короткая форма
    library("legacy-json", "com.example:legacy-json:1.0") {
        deprecated(replacedBy = "modern-json")
    }

    // Deprecated без замены (abandoned)
    library("abandoned-lib", "com.example:abandoned:1.0") {
        deprecated(message = "Library is no longer maintained")
    }
}
----

=== Блок deprecated

Параметры блока `deprecated`:

[cols="1,1,3",options="header"]
|===
| Параметр | Тип | Описание

| `replacedBy`
| String?
| Alias библиотеки-замены. Должен указывать на существующую библиотеку

| `message`
| String?
| Сообщение для разработчиков с пояснением и инструкциями по миграции

| `since`
| String?
| Версия каталога Dependanger, начиная с которой библиотека помечена как deprecated

| `removalVersion`
| String?
| Версия каталога, в которой библиотека будет удалена. Если текущая версия >= removalVersion, генерируется ERROR
|===

[[bom-imports]]
== Блок bomImports

=== Синтаксис

[source,kotlin]
----
bomImports {
    // С ссылкой на version alias
    bom("spring-boot-bom", "org.springframework.boot:spring-boot-dependencies",
        version = versionRef("spring-boot"))

    // С inline версией
    bom("kotlin-bom", "org.jetbrains.kotlin:kotlin-bom:2.1.20")
}
----

=== Поведение

* Версии из BOM импортируются во время processing pipeline (BomImportProcessor)
* Приоритет: DECLARED (явная версия в DSL) > BOM_IMPORT (из bomImports)
* Порядок BomImport имеет значение: первый BOM с версией для библиотеки выигрывает

[[version-source]]
=== VersionSource и приоритет версий

При резолвинге версий система определяет источник (`VersionSource`) для каждой библиотеки:

[cols="1,1,3",options="header"]
|===
| Приоритет | Source | Когда используется

| 1 (высший)
| DECLARED
| Версия явно указана в DSL любым из способов (см. ниже)

| 2 (низший)
| BOM_IMPORT
| Версия импортирована из BomImport через BomImportProcessor
|===

**Что считается DECLARED:**

[source,kotlin]
----
libraries {
    // 1. Inline версия в coordinates — DECLARED
    library("slf4j", "org.slf4j:slf4j-api:2.0.9")

    // 2. Явный параметр version с литералом — DECLARED
    library("guava", "com.google.guava:guava") {
        version = "33.0.0-jre"
    }

    // 3. Ссылка на version alias через versionRef() — DECLARED
    library("kotlin-stdlib", "org.jetbrains.kotlin:kotlin-stdlib",
        version = versionRef("kotlin"))

    // 4. Без версии — версия будет BOM_IMPORT (если найдена в BOM)
    //    или останется null (WARNING/ERROR в зависимости от strictVersionResolution)
    library("spring-web", "org.springframework:spring-web")
}
----

IMPORTANT: Даже если библиотека с DECLARED версией присутствует в BOM — используется DECLARED версия.
BOM_IMPORT применяется **только** когда версия не указана явно.

== Блок distributions

=== Синтаксис

[source,kotlin]
----
distributions {
    distribution("android") {
        spec {
            filter {
                byTags { include { anyOf("android", "mobile") } }
                byGroups { exclude { matching("org.springframework:*") } }
            }
        }
    }

    distribution("backend") {
        spec {
            filter {
                byTags { include { anyOf("server", "web", "database") } }
                byBundles { include("spring-backend", "kotlin-core") }
            }
        }
    }

    // Production без deprecated библиотек
    distribution("production") {
        spec {
            filter {
                byDeprecated { exclude() }
            }
        }
    }
}
----

=== Типы фильтров

[cols="1,3",options="header"]
|===
| Фильтр | Описание

| `byTags`
| Фильтрация по тегам библиотек

| `byGroups`
| Фильтрация по Maven group pattern

| `byAliases`
| Фильтрация по alias библиотек

| `byBundles`
| Включение содержимого бандлов

| `byDeprecated`
| Фильтрация deprecated библиотек. По умолчанию deprecated **включаются** (с WARNING). Используйте `exclude()` для исключения

== Блок compatibility

}

== Модель данных

=== DependangerDsl

Корневая точка входа DSL, содержащая блоки:

* `versions { }` — именованные версии с fallback
* `libraries { }` — библиотеки с Maven координатами
* `plugins { }` — Gradle plugins
* `bundles { }` — группы библиотек
* `bomImports { }` — импорт версий из внешних BOM
* `targetPlatforms { }` — целевые платформы
* `distributions { }` — профили генерации с фильтрацией
* `compatibility { }` — правила совместимости
* `settings { }` — глобальные настройки
* `presets { }` — именованные пресеты конфигурации
* `processing { }` — настройки пайплайна обработки

=== Ключевые сущности

* **Library** — `alias`, `group:name`, `version`, `tags`, `requires`
* **Plugin** — `alias`, `id`, `version`
* **Version** — `name`, `value`, `fallbacks`
* **Bundle** — `name`, `libraries`, `extends`
* **BomImport** — `alias`, `group:name:version`
* **TargetPlatform** — `name`, `jdk`, `android`, `targets`
* **Distribution** — `name`, `spec` (фильтры)
* **Settings** — `validation`, `toml`, `bom`, `defaultDistribution`

См. xref:../01-concepts/domain-model.adoc[Доменная модель] для полного описания.

== Edge Cases

[cols="1,3,2",options="header"]
|===
| Сценарий | Описание | Ожидаемое поведение

| Дубликат alias
| Две библиотеки с одинаковым alias `my-lib`
| ERROR: "Duplicate alias 'my-lib'"

| Невалидный alias
| Alias содержит недопустимые символы: `My_Lib`, `123lib`, `lib.name`
| ERROR: "Invalid alias format. Expected: [a-z][a-z0-9-]*"

| Несуществующий versionRef
| `versionRef("unknown")` ссылается на несуществующую версию
| ERROR: "Unknown version reference 'unknown'"

| Циклическая зависимость бандлов
| Бандл A extends B, бандл B extends A
| ERROR: "Circular bundle dependency detected: A → B → A"

| Несуществующая библиотека в бандле
| `bundle("test") { libraries("unknown-lib") }`
| ERROR: "Unknown library 'unknown-lib' in bundle 'test'"

| Несуществующий бандл в extends
| `bundle("test") { extends("unknown-bundle") }`
| ERROR: "Unknown bundle 'unknown-bundle' in extends"

| Пустой alias
| `library("", "com.example:lib:1.0")`
| ERROR: "Alias cannot be empty"

| Невалидные Maven координаты
| `library("lib", "invalid-coordinates")`
| ERROR: "Invalid Maven coordinates format. Expected: group:artifact or group:artifact:version"

| Deprecated без replacedBy
| `deprecated(message = "Old lib")` без указания замены
| WARNING: допустимо, но рекомендуется указать replacedBy

| Deprecated replacedBy несуществующий
| `deprecated(replacedBy = "unknown-lib")`
| WARNING: "Replacement library 'unknown-lib' not found"

| Deprecated replacedBy указывает на deprecated
| `deprecated(replacedBy = "also-deprecated-lib")`
| WARNING: "Replacement library 'also-deprecated-lib' is also deprecated"

| Циклическая замена deprecated
| A replacedBy B, B replacedBy A
| ERROR: "Circular deprecation replacement: A → B → A"

| removalVersion в прошлом
| `removalVersion = "1.0.0"`, текущая версия каталога `2.0.0`
| ERROR: "Library 'xxx' should have been removed in version 1.0.0"

| Бандл содержит deprecated
| Бандл включает deprecated библиотеку
| WARNING: "Bundle 'xxx' contains deprecated library 'yyy'"

| byDeprecated exclude с replacedBy в том же distribution
| Distribution исключает deprecated, но включает replacedBy
| Корректное поведение: deprecated исключена, replacedBy включена

| Пустой DSL
| DSL без определений (пустые блоки)
| Валидно, создаётся пустой DependangerMetadata

| Дубликат fallback условия
| Две одинаковые fallback условия для версии
| WARNING: первый fallback выигрывает

| Конфликт fallback условий
| Fallback условия перекрываются (jdkBelow(17) и jdkBelow(11))
| Используется первый matching fallback по порядку определения

| Отрицательные JDK constraints
| `jdk { min = -1 }`
| ERROR: "JDK version must be positive"

| min > max в constraints
| `jdk { min = 21; max = 17 }`
| ERROR: "Invalid JDK range: min (21) > max (17)"

| Пустой distribution spec
| `distribution("empty") { }` без фильтров
| Все библиотеки включаются (эквивалент отсутствия фильтрации)

3+h| Presets

| Несуществующий пресет
| `applyPreset("unknown")`
| ERROR: "Unknown preset 'unknown'"

| Пресет ссылается на несуществующий бандл
| `preset("p") { bundles("unknown-bundle") }`
| ERROR: "Unknown bundle 'unknown-bundle' in preset 'p'"

| Пресет ссылается на несуществующий distribution
| `preset("p") { distributions("unknown-dist") }`
| ERROR: "Unknown distribution 'unknown-dist' in preset 'p'"

| Два пресета с конфликтующими settings
| `applyPreset("a")` (strictVersionResolution=true), `applyPreset("b")` (strictVersionResolution=false)
| Используется значение из последнего пресета: `strictVersionResolution=false`

| Явные settings после applyPreset
| `applyPreset("base")` затем `settings { strictVersionResolution = true }`
| Явные settings переопределяют значения из пресета

| Циклические пресеты (если поддерживается extends)
| Пресет A extends B, пресет B extends A
| ERROR: "Circular preset dependency: A → B → A"

| Пустой пресет
| `preset("empty") { }` без bundles/distributions/settings
| Валидно, пресет не вносит изменений при применении

3+h| Platform Library

| platformLibrary без версии
| `platformLibrary("bom", "group:artifact")`
| ERROR: "Platform library 'bom' must have explicit version"

| platformLibrary с теми же координатами что и bomImport
| `platformLibrary("spring-bom", ...)` и `bom("spring-bom", ...)` с одинаковыми coordinates
| WARNING: "Platform library and BOM import have same coordinates"

| platformLibrary в бандле
| Бандл содержит platformLibrary
| WARNING: "Bundle 'xxx' contains platform library 'yyy'. Platform libraries are typically not bundled"

3+h| Processing

| Отключение обязательного процессора
| `disabledProcessors = listOf("validation")`
| ERROR: "Cannot disable required processor 'validation'"

| Конфликт preset и явных настроек
| `preset = MINIMAL` и `withBomImport()` вместе
| `withBomImport()` добавляется к MINIMAL preset

| Неизвестный процессор в disabledProcessors
| `disabledProcessors = listOf("unknown-processor")`
| WARNING: "Unknown processor 'unknown-processor' in disabledProcessors"

| importBoms=true без bomImports
| `processing { importBoms = true }`, блок `bomImports` пуст
| INFO: "BOM import enabled but no BOM imports defined"
|===

== Трансформация DSL → Metadata

[[to-metadata]]
=== Метод toMetadata()

`DependangerDsl` конвертируется в `DependangerMetadata` через метод `toMetadata()`:

[source,kotlin]
----
val dsl = DependangerDsl()

dsl.versions { version("kotlin", "2.1.20") }
dsl.libraries { library("kotlin-stdlib", "org.jetbrains.kotlin:kotlin-stdlib", version = versionRef("kotlin")) }

// Конвертация в сериализуемую модель
val metadata: DependangerMetadata = dsl.toMetadata()

// Сериализация в JSON
val json = SerializationFormat.JSON.write(metadata)
----

=== Когда вызывается toMetadata()

[cols="1,2",options="header"]
|===
| Контекст | Поведение

| Gradle Plugin
| Вызывается автоматически в задаче `dependangerGenerateMetadata`

| Library API
| Вызывается явно пользователем: `dsl.toMetadata()`

| CLI
| CLI работает с уже сериализованным `metadata.json`, не с DSL напрямую
|===

=== Что происходит при toMetadata()

1. **Валидация DSL** — проверка уникальности alias, существования ссылок, отсутствия циклов
2. **Применение пресетов** — если вызван `applyPreset()`, настройки мержатся
3. **Создание Metadata** — все сущности конвертируются в сериализуемые DTO
4. **НЕ резолвятся версии** — резолвинг происходит позже в Processing Pipeline

NOTE: `processing { }` блок **НЕ сериализуется** в metadata.json.
Это runtime-only конфигурация.
Каждый интерфейс (Gradle Plugin, CLI) настраивает pipeline самостоятельно.
