:toc: left
:toclevels: 3

[abstract]
--
Импорт версий из внешних Maven BOM (Spring Boot BOM, Kotlin BOM и др.).
Опциональный процессор в pipeline (order: 15).
--

| Приоритет | P2 (важно)

== Описание

BOM Import позволяет импортировать версии из внешних Maven BOM (Bill of Materials).
Это избавляет от необходимости дублировать версии, которые уже определены в популярных BOM:
Spring Boot Dependencies, Kotlin BOM, AWS SDK BOM и др.

== User Stories

=== US-001: Импорт версий из Spring Boot BOM

[quote]
____
Как разработчик Spring приложения, я хочу импортировать версии из Spring Boot Dependencies BOM, чтобы не указывать версии для каждой Spring библиотеки вручную.
____

**Acceptance Criteria:**

[cols="1,3,3,3",options="header"]
|===
| ID | Given | When | Then

| AC-001-1
| BOM `spring-boot-dependencies:3.4.0` указан в `bomImports { }`
| Pipeline обрабатывает библиотеку `org.springframework:spring-web` без явной версии
| Библиотека получает версию из BOM (например, `6.2.1`), `versionSource = BOM_IMPORT`

| AC-001-2
| BOM указан в `bomImports { }`, библиотека имеет явную версию `5.0.0`
| Pipeline обрабатывает библиотеку
| Используется явная версия `5.0.0`, `versionSource = DECLARED`

| AC-001-3
| BOM указан в `bomImports { }`, библиотека `com.example:unknown` отсутствует в BOM
| Pipeline обрабатывает библиотеку без явной версии
| WARNING в логе, версия остаётся null (или ERROR в strict mode)
|===

=== US-002: Несколько BOM с приоритетом

[quote]
____
Как разработчик, я хочу использовать несколько BOM одновременно, с предсказуемым порядком приоритета.
____

**Acceptance Criteria:**

[cols="1,3,3,3",options="header"]
|===
| ID | Given | When | Then

| AC-002-1
| BOM-1 (Spring Boot) и BOM-2 (AWS) указаны в порядке: Spring, AWS
| Библиотека `com.fasterxml.jackson.core:jackson-core` есть в обоих BOM
| Версия берётся из Spring Boot BOM (первый в списке)

| AC-002-2
| BOM-1 (Spring Boot) и BOM-2 (AWS) указаны
| Библиотека `software.amazon.awssdk:s3` есть только в AWS BOM
| Версия берётся из AWS BOM

| AC-002-3
| Три BOM указаны: A, B, C
| Библиотека есть в B и C, но не в A
| Версия берётся из B (первый содержащий)
|===

=== US-003: Кеширование BOM

[quote]
____
Как разработчик, я хочу чтобы BOM файлы кешировались локально, чтобы ускорить повторные сборки и работать офлайн.
____

**Acceptance Criteria:**

[cols="1,3,3,3",options="header"]
|===
| ID | Given | When | Then

| AC-003-1
| BOM скачан менее 24 часов назад
| Pipeline запускается повторно
| BOM читается из кеша, сетевой запрос не выполняется

| AC-003-2
| BOM в кеше старше 24 часов, сеть доступна
| Pipeline запускается
| BOM скачивается заново, кеш обновляется

| AC-003-3
| BOM в кеше старше 24 часов, сеть недоступна
| Pipeline запускается
| Используется устаревший кеш, WARNING в логе
|===

== DSL синтаксис

[source,kotlin]
----
dsl.bomImports {
    // Spring Boot BOM - высший приоритет
    bom("spring-boot-bom", "org.springframework.boot:spring-boot-dependencies:3.4.0")

    // Kotlin BOM
    bom("kotlin-bom", "org.jetbrains.kotlin:kotlin-bom:2.1.20")

    // AWS BOM с ссылкой на version alias
    bom("aws-bom", "software.amazon.awssdk:bom", version = versionRef("aws"))
}

dsl.libraries {
    // Версия будет из Spring Boot BOM
    library("spring-web", "org.springframework:spring-web") {
        tags("spring", "web")
    }

    // Явная версия имеет приоритет над BOM
    library("jackson-core", "com.fasterxml.jackson.core:jackson-core:2.16.0") {
        tags("json")
    }
}
----

== Функциональные требования

[cols="1,1,4",options="header"]
|===
| ID | Приоритет | Требование

| FR-001
| MUST
| Система ДОЛЖНА импортировать версии из Maven BOM, указанных в `bomImports { }`

| FR-002
| MUST
| При наличии явной версии (`DECLARED`) система ДОЛЖНА игнорировать версию из BOM

| FR-003
| MUST
| При нескольких BOM система ДОЛЖНА использовать версию из первого BOM, содержащего библиотеку

| FR-004
| MUST
| Система ДОЛЖНА корректно парсить секцию `<dependencyManagement>` Maven POM

| FR-005
| SHOULD
| Система ДОЛЖНА кешировать скачанные BOM файлы

| FR-006
| SHOULD
| Система ДОЛЖНА использовать кеш при недоступности сети

| FR-007
| MAY
| TTL кеша МОЖЕТ быть настраиваемым (по умолчанию 24 часа)

| FR-008
| MUST
| В strict mode система ДОЛЖНА завершаться с ERROR при неразрешённой версии
|===

== Поведение

=== Version Source Priority

[source]
----
DECLARED > BOM_IMPORT
----

=== Алгоритм резолвинга

1. Если библиотека имеет явную версию (`DECLARED`, включая inline в coordinates) — использовать её
2. Иначе искать в BOM по порядку их объявления
3. Первый BOM, содержащий `<dependency>` с matching `groupId:artifactId`, даёт версию
4. Если версия не найдена — WARNING (или ERROR в strict mode)

=== Resolved Version

Версия, полученная из BOM, имеет `source = BOM_IMPORT` и хранит ссылку на исходный BOM в `originalRef`.

== Edge Cases

[cols="1,3,2",options="header"]
|===
| Сценарий | Описание | Ожидаемое поведение

3+h| Ошибки парсинга

| Невалидный BOM
| BOM POM файл содержит синтаксическую ошибку XML
| ERROR с указанием BOM и причины, pipeline останавливается

| BOM без dependencyManagement
| BOM POM не содержит секции `<dependencyManagement>`
| WARNING "BOM has no dependencyManagement section", BOM игнорируется

| Property placeholder в версии
| BOM содержит версию `${spring.version}`
| Резолвить property из `<properties>` того же POM

| Неразрешённый property
| BOM содержит `${unknown.version}`, property не определён
| ERROR "Unresolved property 'unknown.version' in BOM"

| Parent BOM
| BOM наследует `<dependencyManagement>` от parent
| Резолвить parent и объединить dependencyManagement

| Глубокая иерархия parent
| BOM → parent → grandparent → ... (10+ уровней)
| Резолвить до корня, WARNING если > 10 уровней

| Циклическая зависимость BOM
| BOM-A импортирует BOM-B, который импортирует BOM-A
| ERROR "Circular BOM dependency detected", pipeline останавливается

3+h| Сетевые ошибки

| BOM недоступен
| Сетевая ошибка при скачивании BOM, кеш пуст
| ERROR "Cannot fetch BOM: {coordinates}", pipeline останавливается

| BOM недоступен, есть кеш
| Сетевая ошибка, но есть закешированный BOM
| WARNING, используется кеш, pipeline продолжается

| Connection timeout
| Репозиторий не отвечает в течение таймаута (30s default)
| ERROR "Connection timeout for BOM: {coordinates}"

| Read timeout
| Соединение установлено, но данные не приходят
| ERROR "Read timeout for BOM: {coordinates}"

| DNS resolution failure
| Не удаётся разрешить hostname репозитория
| ERROR "Cannot resolve host: {hostname}"

| SSL/TLS ошибка
| Невалидный сертификат, expired certificate, hostname mismatch
| ERROR "SSL error for {url}: {причина}"

| Self-signed certificate
| Репозиторий использует self-signed сертификат
| ERROR по умолчанию; можно отключить проверку через `settings { insecureHosts = [...] }`

| HTTP 404 Not Found
| BOM не существует в репозитории
| ERROR "BOM not found: {coordinates} (HTTP 404)"

| HTTP 401/403 Unauthorized
| Репозиторий требует аутентификации
| ERROR "Authentication required for {url}. Configure credentials in settings"

| HTTP 500 Server Error
| Внутренняя ошибка репозитория
| Retry 3 раза с exponential backoff, затем ERROR

| HTTP 429 Too Many Requests
| Rate limiting от репозитория
| Retry с Retry-After header, WARNING в логе

| Redirect loop
| HTTP redirect зацикливается (A → B → A)
| ERROR "Redirect loop detected for {url}" после 10 редиректов

| Redirect to different host
| Репозиторий редиректит на другой домен
| Следовать редиректу, WARNING если протокол меняется (https → http)

| Partial download
| Соединение разорвано посреди загрузки
| Retry, при повторной неудаче — ERROR "Incomplete download"

| Corrupted download
| Скачанный файл повреждён (checksum mismatch)
| ERROR "Checksum mismatch for BOM: expected {x}, got {y}"

| Very large BOM
| BOM файл > 10MB
| WARNING "Large BOM file ({size})", загрузка продолжается

3+h| Кеширование

| Пустой bomImports
| Блок `bomImports { }` пуст или отсутствует
| Процессор пропускается, никаких изменений

| Кеш повреждён
| Файл в кеше не читается (corrupted)
| WARNING, удалить из кеша, скачать заново

| Кеш заполнен
| Директория кеша заполнена (disk full)
| WARNING, работать без кеша

| Нет прав на запись в кеш
| Директория `~/.dependanger/cache/` read-only
| WARNING, работать без кеша

| Concurrent cache access
| Два процесса одновременно пишут в кеш
| File locking, второй процесс ждёт или читает после записи

| Кеш с другой версией BOM
| В кеше `spring-boot:3.3.0`, запрошен `3.4.0`
| Скачать новую версию, обновить кеш

3+h| Специфичные случаи

| Несколько репозиториев
| BOM есть в repository-2, но не в repository-1
| Пробовать репозитории по порядку, использовать первый успешный

| Snapshot версия BOM
| BOM версия `3.4.0-SNAPSHOT`
| WARNING "Snapshot BOM versions may be unstable", TTL кеша = 1 час

| Private repository
| BOM в корпоративном Nexus/Artifactory
| Использовать credentials из `settings { repositories { maven { credentials { } } } }`

| BOM с classifier
| BOM координаты включают classifier: `...:bom:pom`
| Корректно обработать classifier при запросе

| Дубликаты в BOM
| BOM содержит два `<dependency>` с одинаковыми `groupId:artifactId`
| WARNING, использовать последний (как Maven)

| Scope/type в BOM dependency
| `<dependency>` в BOM имеет `<scope>import</scope>`
| Рекурсивно импортировать вложенный BOM
|===

== SPI: BomImportProcessor

`BomImportProcessor` — опциональный процессор (order: 15), который:

1. Получает список `bomImports` из metadata
2. Для каждого BOM скачивает и парсит секцию `<dependencyManagement>`
3. Для библиотек без версии находит соответствующую версию в BOM
4. Устанавливает `ResolvedVersion` с `source = BOM_IMPORT`

Включается через `withBomImport()` в builder API.

== API использование

[source,kotlin]
----
val result = Dependanger.fromDsl {
    bomImports {
        bom("spring-boot-bom", "org.springframework.boot:spring-boot-dependencies:3.4.0")
    }
    libraries {
        library("spring-web", "org.springframework:spring-web")
    }
}
.withBomImport()  // Включить BOM import процессор
.build()

// spring-web будет иметь версию из Spring Boot BOM
----

== Кеширование

* BOM файлы кешируются в `~/.dependanger/cache/bom/`
* TTL кеша: 24 часа (настраивается через `settings { bomCache { ttl = 48.hours } }`)
* При отсутствии сети используется кеш с WARNING в логе
* Кеш инвалидируется при изменении версии BOM

== Связи

* **Входные данные:** `bomImports` из DependangerMetadata
* **Выходные данные:** Обогащённые версии в ProcessingContext
* **Предыдущий этап:** F002 (Metadata), ExtractedVersions processor (order: 12)
* **Следующий этап:** LibraryFilter processor (order: 20)
