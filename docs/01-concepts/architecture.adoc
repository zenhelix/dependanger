= Архитектура системы
:toc: left
:toclevels: 3
:sectnums:
:icons: font

[abstract]
--
Документ описывает архитектуру системы Dependanger:
архитектурные слои, модули, потоки данных и ключевые SPI интерфейсы.
--

== Принципы архитектуры

[cols="1,3",options="header"]
|===
| Принцип | Описание

| Направление зависимостей
| Зависимости направлены от периферии к ядру: interfaces → api → features → core

| Изоляция домена
| Core модуль не зависит от инфраструктуры (HTTP, файловая система, форматы)

| Расширяемость через SPI
| Новый функционал добавляется через реализацию интерфейсов, без изменения core

| Единая точка входа
| Весь функционал доступен через API фасад, интерфейсы (CLI, Gradle) — тонкие обёртки

| Абстракция форматов
| Сериализация модели абстрагирована через `SerializationFormat<T>` SPI

| Расширяемая генерация
| Генераторы артефактов добавляются через `ArtifactGenerator<T>` SPI

| Экспортный слой на границе
| Генераторы — точка экспорта. Артефакты (TOML, BOM) публикуются для использования в других проектах
|===

== Архитектурные слои

Система организована в 5 архитектурных слоёв:

[source]
----
┌─────────────────────────────────────────────────────────────────┐
│ 1. INTERFACES — точки входа (CLI, Gradle Plugin)                │
├─────────────────────────────────────────────────────────────────┤
│ 2. API — фасад Dependanger, координация                         │
├─────────────────────────────────────────────────────────────────┤
│ 3. MODEL — core (DSL), metadata (сериализация), effective       │
├─────────────────────────────────────────────────────────────────┤
│ 4. FEATURES — опциональные процессоры (resolver, updates, etc.) │
├─────────────────────────────────────────────────────────────────┤
│ 5. EXPORT — генераторы артефактов для внешних проектов (SPI)    │
└─────────────────────────────────────────────────────────────────┘
----

=== Описание слоёв

[cols="1,2,2",options="header"]
|===
| Слой | Описание | Модули

| INTERFACES
| Два равноправных входа для пользователей: CLI выполняет CRUD операции напрямую с metadata.json, Gradle Plugin парсит Kotlin DSL и создаёт metadata.json.
| `dependanger-gradle-plugin`, `dependanger-cli`

| API
| Фасад и координация. Единая точка входа для всего функционала.
| `dependanger-api`

| MODEL
| Доменные модели: Core (DSL, сущности), metadata.json (ВСЕГДА создаётся как промежуточный артефакт), EffectiveMetadata (результат pipeline с updateReport и compatibilityReport).
| `dependanger-core`, `dependanger-metadata-json`, `dependanger-effective`

| FEATURES
| Опциональные процессоры pipeline. Могут быть отключены.
| `dependanger-maven-resolver`, `dependanger-updates`, `dependanger-analysis`

| EXPORT
| Экспортный слой — точка выхода артефактов для внешних проектов. Генераторы преобразуют `EffectiveMetadata` в стандартные форматы (TOML, BOM) через `ArtifactGenerator` SPI. Артефакты публикуются и используются в других Gradle/Maven проектах.
| `dependanger-generator-toml`, `dependanger-generator-bom`
|===

== SPI Интерфейсы

Система использует Service Provider Interface (SPI) для расширяемости.

=== SerializationFormat

Абстракция формата сериализации `DependangerMetadata`.

[source,kotlin]
----
interface SerializationFormat<T> {
    val formatId: String  // "json", "yaml"
    val fileExtension: String  // ".json", ".yaml"

    fun serialize(metadata: DependangerMetadata): T
    fun deserialize(input: T): DependangerMetadata
    fun write(metadata: DependangerMetadata, path: Path)
    fun read(path: Path): DependangerMetadata
}
----

**Встроенные реализации:** `JsonSerializationFormat` (`dependanger-metadata-json`)

---

=== EffectiveMetadataProcessor

Процессор в pipeline обработки.

[source,kotlin]
----
interface EffectiveMetadataProcessor {
    val order: Int          // Порядок выполнения (меньше = раньше)
    val processorId: String // Уникальный идентификатор
    val phase: ProcessingPhase
    val isOptional: Boolean // Можно ли отключить

    suspend fun process(context: ProcessingContext): ProcessingContext
}
----

**Встроенные процессоры:** см. xref:../02-features/F003-processing-pipeline.adoc[F003: Processing Pipeline]

---

=== ArtifactGenerator

Генератор выходных артефактов.

[source,kotlin]
----
interface ArtifactGenerator<T> {
    val generatorId: String  // "toml", "bom"

    fun generate(effective: EffectiveMetadata): T
    fun write(artifact: T, output: Path)
}
----

**Встроенные реализации:** `TomlGenerator`, `BomGenerator`

---

=== LibraryFilter

Стратегия фильтрации библиотек.

[source,kotlin]
----
interface LibraryFilter {
    fun filter(libraries: Collection<Library>): Collection<Library>
}
----

**Встроенные реализации:** `TagFilter`, `CompositeLibraryFilter`

== Программное использование

=== Базовое использование

[source,kotlin]
----
val result = Dependanger.fromDsl {
    versions {
        version("kotlin", "2.1.20")
    }
    libraries {
        library("kotlin-stdlib", "org.jetbrains.kotlin:kotlin-stdlib",
            version = versionRef("kotlin"))
    }
}.build()

// Генерация артефактов
val toml = result.toToml()
val bom = result.toBom()

// Запись в файлы
result.writeTomlTo(Path.of("gradle/libs.versions.toml"))
result.writeBomTo(Path.of("build/dependanger-bom.pom"))
----

=== Builder Pattern

[source,kotlin]
----
val result = Dependanger.fromDsl {
    // DSL конфигурация...
}
.withBomImport()                         // Включить BOM import
.withUpdateCheck()                       // Включить проверку обновлений
.withCompatibilityAnalysis(targetJdk = 17) // Включить анализ совместимости
.forDistribution("android")              // Для конкретного distribution
.build()

// Проверка результата
if (result.diagnostics.hasErrors()) {
    result.diagnostics.errors.forEach { println(it.message) }
}
----

=== Загрузка из файла

[source,kotlin]
----
val result = Dependanger.fromFile(Path.of("metadata.json"))
    .forDistribution("backend")
    .build()
----
