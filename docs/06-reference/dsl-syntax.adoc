== Корневой DSL

=== DependangerDsl

Корневая точка входа Kotlin DSL.

==== Доступные блоки

[cols="1,2,1,2",options="header"]
|===
| Блок | Тип | Источник | Описание

| `versions { }`
| VersionsDsl
| xref:../02-features/F001-registry-dsl.adoc[F001]
| Именованные версии с fallback условиями

| `libraries { }`
| LibrariesDsl
| xref:../02-features/F001-registry-dsl.adoc[F001]
| Библиотеки с Maven координатами

| `plugins { }`
| PluginsDsl
| xref:../02-features/F001-registry-dsl.adoc[F001]
| Gradle plugins

| `bundles { }`
| BundlesDsl
| xref:../02-features/F001-registry-dsl.adoc[F001]
| Группы библиотек

| `bomImports { }`
| BomImportsDsl
| xref:../02-features/F001-registry-dsl.adoc[F001], xref:../02-features/F006-bom-import.adoc[F006]
| Импорт версий из внешних Maven BOM

| `targetPlatforms { }`
| TargetPlatformsDsl
| xref:../02-features/F001-registry-dsl.adoc[F001]
| Целевые платформы с ограничениями

| `distributions { }`
| DistributionsDsl
| xref:../02-features/F001-registry-dsl.adoc[F001]
| Профили генерации с фильтрацией

| `compatibility { }`
| CompatibilityDsl
| xref:../02-features/F001-registry-dsl.adoc[F001], xref:../02-features/F008-compatibility-analysis.adoc[F008]
| Правила совместимости

| `settings { }`
| SettingsDsl
| xref:../02-features/F001-registry-dsl.adoc[F001], F004-F012
| Глобальные настройки

| `presets { }`
| PresetsDsl
| xref:../02-features/F001-registry-dsl.adoc[F001]
| Именованные пресеты конфигурации

| `processing { }`
| ProcessingDsl
| xref:../02-features/F003-processing-pipeline.adoc[F003]
| Runtime-only конфигурация pipeline
|===

NOTE: Канонический порядок блоков: `versions` -> `libraries` -> `plugins` -> `bundles` -> `bomImports` -> `targetPlatforms` -> `distributions` -> `compatibility` -> `settings` -> `presets` -> `processing`

==== Полный пример

[source,kotlin]
----
dependanger {
    versions {
        version("kotlin", "2.1.20")
        version("spring-boot", "3.4.0") {
            fallback("2.7.18") { jdkBelow(17) }
        }
    }

    libraries {
        library("kotlin-stdlib", "org.jetbrains.kotlin:kotlin-stdlib",
            version = versionRef("kotlin")) {
            tags("core", "kotlin")
        }

        library("spring-web", "org.springframework:spring-web") {
            tags("web", "spring")
            requires { jdk { min = 17 } }
            license { id = "Apache-2.0" }
        }
    }

    bundles {
        bundle("kotlin-core") {
            libraries("kotlin-stdlib", "kotlin-reflect")
        }
    }

    plugins {
        plugin("kotlin-jvm", "org.jetbrains.kotlin.jvm",
            version = versionRef("kotlin"))
    }

    bomImports {
        bom("spring-boot-bom", "org.springframework.boot:spring-boot-dependencies",
            version = versionRef("spring-boot"))
    }

    targetPlatforms {
        platform("jvm17") {
            jdk { min = 17; max = 21 }
        }
    }

    distributions {
        distribution("backend") {
            spec {
                filter {
                    byTags { include { anyOf("server", "web") } }
                    byDeprecated { exclude() }
                }
            }
        }
    }

    compatibility {
        jdkRequirement("spring-jdk17") {
            matches = "org.springframework:*"
            minJdk = 17
        }
    }

    settings {
        defaultDistribution = "backend"
        strictVersionResolution = true

        validation {
            onCompatibilityViolation = ValidationAction.FAIL
        }

        toml {
            filename = "libs.versions.toml"
            includeComments = true
        }
    }
}
----

[[dsl-versions]]
== versions { }

Блок для определения именованных версий.

**Источник:** xref:../02-features/F001-registry-dsl.adoc[F001: Registry DSL]

=== version()

Определяет именованную версию (version alias).

==== Сигнатуры

[source,kotlin]
----
// Простая версия
fun version(alias: String, value: String)

// Версия с fallback условиями
fun version(alias: String, value: String, block: VersionDsl.() -> Unit)
----

==== Параметры

[cols="1,1,1,3",options="header"]
|===
| Параметр | Тип | Обязательный | Описание

| `alias`
| String
| Да
| Уникальный идентификатор версии. Формат: `[a-z][a-z0-9-]*`

| `value`
| String
| Да
| Значение версии (например: `2.1.20`, `3.4.0-SNAPSHOT`)
|===

==== Примеры

[source,kotlin]
----
versions {
    // Простая версия
    version("kotlin", "2.1.20")
    version("guava", "33.0.0-jre")

    // Версия с одним fallback
    version("spring-boot", "3.4.0") {
        fallback("2.7.18") { jdkBelow(17) }
    }

    // Версия с несколькими fallback
    version("ktor", "2.3.8") {
        fallback("2.2.4") { kotlinVersionBelow("1.9") }
        fallback("1.6.8") { jdkBelow(11) }
    }

    // Сложные условия
    version("serialization", "1.6.3") {
        fallback("1.5.1") {
            all(
                jdkBelow(17),
                kotlinVersionBelow("1.9")
            )
        }
        fallback("1.4.1") {
            any(
                jdkBelow(11),
                distribution("legacy")
            )
        }
    }
}
----

=== fallback()

Определяет альтернативную версию при выполнении условия.

==== Сигнатура

[source,kotlin]
----
fun fallback(value: String, condition: FallbackConditionDsl.() -> FallbackCondition)
----

=== Доступные условия для fallback

[cols="1,2,2",options="header"]
|===
| Условие | Описание | Пример

| `jdkBelow(version)`
| JDK версия < указанной
| `jdkBelow(17)`

| `jdkAtLeast(version)`
| JDK версия >= указанной
| `jdkAtLeast(21)`

| `jdkBetween(min, max)`
| JDK версия в диапазоне [min, max]
| `jdkBetween(11, 17)`

| `kotlinVersionBelow(version)`
| Kotlin версия < указанной
| `kotlinVersionBelow("1.9")`

| `kotlinVersionAtLeast(version)`
| Kotlin версия >= указанной
| `kotlinVersionAtLeast("2.0")`

| `gradleVersionAtLeast(version)`
| Gradle версия >= указанной
| `gradleVersionAtLeast("8.0")`

| `gradleVersionBelow(version)`
| Gradle версия < указанной
| `gradleVersionBelow("7.0")`

| `distribution(name)`
| Активен указанный distribution
| `distribution("android")`

| `environment(key, value)`
| Переменная окружения равна значению
| `environment("ENV", "prod")`

| `all(conditions...)`
| Все условия выполнены (AND)
| `all(jdkBelow(17), kotlinVersionBelow("1.9"))`

| `any(conditions...)`
| Хотя бы одно выполнено (OR)
| `any(jdkBelow(11), distribution("legacy"))`

| `not(condition)`
| Условие не выполнено (NOT)
| `not(jdkAtLeast(17))`
|===

[[dsl-libraries]]
== libraries { }

Блок для определения библиотек.

**Источник:** xref:../02-features/F001-registry-dsl.adoc[F001: Registry DSL], xref:../02-features/F011-license-compliance.adoc[F011: License Compliance], xref:../02-features/F012-transitive-resolution.adoc[F012: Transitive Resolution]

=== library()

Определяет библиотеку с Maven координатами.

==== Сигнатуры

[source,kotlin]
----
// Минимальная форма (версия из BomImport или inline)
fun library(alias: String, coordinates: String)

// С version reference
fun library(alias: String, coordinates: String, version: VersionReference)

// С отдельными group/artifact
fun library(alias: String, group: String, artifact: String, version: VersionReference)

// С блоком настроек
fun library(alias: String, coordinates: String, block: LibraryDsl.() -> Unit)
fun library(alias: String, coordinates: String, version: VersionReference, block: LibraryDsl.() -> Unit)
----

==== Параметры

[cols="1,1,1,3",options="header"]
|===
| Параметр | Тип | Обязательный | Описание

| `alias`
| String
| Да
| Уникальный идентификатор библиотеки

| `coordinates`
| String
| Да
| Maven координаты: `group:artifact` или `group:artifact:version`

| `version`
| VersionReference
| Нет
| Ссылка на версию: `versionRef("alias")` или `"1.0.0"`
|===

==== Примеры

[source,kotlin]
----
libraries {
    // Минимальная форма (версия inline)
    library("slf4j-api", "org.slf4j:slf4j-api:2.0.9")

    // С version reference
    library("kotlin-stdlib", "org.jetbrains.kotlin:kotlin-stdlib",
        version = versionRef("kotlin"))

    // Без версии (будет из BomImport)
    library("spring-web", "org.springframework:spring-web")

    // С настройками
    library("guava", "com.google.guava:guava", version = "33.0.0-jre") {
        description = "Google core libraries for Java"
        tags("core", "utils", "google")

        requires {
            jdk { min = 8 }
        }

        license {
            id = "Apache-2.0"
            url = "https://www.apache.org/licenses/LICENSE-2.0"
        }
    }

    // Игнорировать обновления для конкретной библиотеки
    library("internal-utils", "com.example:internal-utils:1.0.0") {
        description = "Internal utilities pinned to specific version"
        ignoreUpdates = true
    }

    // Deprecated библиотека
    library("old-http", "com.example:old-http:1.0") {
        deprecated {
            replacedBy = "new-http"
            message = "Use new-http which supports HTTP/2"
            since = "1.2.0"
            removalVersion = "2.0.0"
        }
    }

    // Deprecated — короткая форма
    library("legacy-json", "com.example:legacy-json:1.0") {
        deprecated(replacedBy = "modern-json")
    }

    // С constraints для транзитивных зависимостей
    library("spring-core", "org.springframework:spring-core",
        version = versionRef("spring")) {
        constraints {
            constraint("org.springframework:spring-beans", versionRef("spring"))
            exclude("commons-logging:commons-logging")
            substitute("javax.annotation:javax.annotation-api",
                "jakarta.annotation:jakarta.annotation-api:2.1.1")
        }
    }
}
----

=== LibraryDsl — внутренний блок

[cols="1,1,3",options="header"]
|===
| Метод/Свойство | Тип | Описание

| `description`
| String?
| Описание библиотеки (по умолчанию `null`)

| `ignoreUpdates`
| Boolean
| Пропускать проверку обновлений (по умолчанию `false`)

| `tags(vararg tags: String)`
| Unit
| Добавить теги для фильтрации

| `requires { }`
| RequiresDsl
| Требования к окружению

| `deprecated { }`
| DeprecationDsl
| Информация об устаревании

| `deprecated(replacedBy, message)`
| Unit
| Короткая форма deprecated

| `license { }`
| LicenseDsl
| Информация о лицензии (F011)

| `constraints { }`
| ConstraintsDsl
| Constraints для транзитивных (F012)
|===

=== platformLibrary()

Определяет platform-зависимость (BOM) для экспорта в TOML.

==== Сигнатура

[source,kotlin]
----
fun platformLibrary(alias: String, coordinates: String, version: VersionReference)
----

==== Пример

[source,kotlin]
----
libraries {
    // Platform library — экспортируется в TOML как library с type=pom
    platformLibrary("spring-boot-bom", "org.springframework.boot:spring-boot-dependencies",
        version = versionRef("spring-boot"))

    // В сгенерированном TOML:
    // [libraries]
    // spring-boot-bom = { group = "org.springframework.boot", name = "spring-boot-dependencies", version.ref = "spring-boot" }
}
----

NOTE: Не путать с `bomImports {}`, который используется для *импорта* версий из внешних BOM. `platformLibrary()` — для *экспорта* BOM-зависимости в генерируемый TOML/BOM.

=== requires { }

Требования к окружению для библиотеки.

[cols="1,1,1,3",options="header"]
|===
| Блок | Свойство | Тип | Описание

| `jdk { }`
| `min`
| Int?
| Минимальная версия JDK (например, 17)

|
| `max`
| Int?
| Максимальная версия JDK (например, 21)

| `kotlin { }`
| `min`
| String?
| Минимальная версия Kotlin (например, "1.9")

|
| `max`
| String?
| Максимальная версия Kotlin
|===

[source,kotlin]
----
requires {
    jdk {
        min = 17
        max = 21
    }
    kotlin {
        min = "1.9"
    }
}
----

NOTE: Требования используются для валидации совместимости с `TargetPlatform` и при анализе совместимости (F008).

=== deprecated { }

Информация об устаревшей библиотеке.

[cols="1,1,1,3",options="header"]
|===
| Свойство | Тип | Обязательный | Описание

| `replacedBy`
| String?
| Нет
| Alias библиотеки-замены

| `message`
| String?
| Нет
| Сообщение с причиной и инструкциями

| `since`
| String?
| Нет
| Версия каталога, с которой deprecated

| `removalVersion`
| String?
| Нет
| Версия каталога, в которой будет удалена
|===

=== license { }

Информация о лицензии (xref:../02-features/F011-license-compliance.adoc[F011]).

[source,kotlin]
----
license {
    id = "Apache-2.0"      // SPDX ID
    url = "https://..."    // URL лицензии (опционально)
}
----

=== constraints { }

Constraints для транзитивных зависимостей (xref:../02-features/F012-transitive-resolution.adoc[F012]).

[source,kotlin]
----
constraints {
    // Зафиксировать версию транзитивной зависимости
    constraint("org.slf4j:slf4j-api", "2.0.9")
    constraint("org.slf4j:slf4j-api") {
        version = "2.0.9"
        because = "Security fix"
    }

    // Исключить транзитивную зависимость
    exclude("commons-logging:commons-logging")
    exclude(group = "commons-logging", artifact = "commons-logging")

    // Заменить зависимость
    substitute("javax.annotation:javax.annotation-api",
        "jakarta.annotation:jakarta.annotation-api:2.1.1")
    substitute("javax.annotation:javax.annotation-api",
        "jakarta.annotation:jakarta.annotation-api:2.1.1") {
        because = "Jakarta namespace migration"
    }
}
----

== plugins { }

Блок для определения Gradle plugins.

**Источник:** xref:../02-features/F001-registry-dsl.adoc[F001: Registry DSL]

=== plugin()

==== Сигнатуры

[source,kotlin]
----
fun plugin(alias: String, id: String)
fun plugin(alias: String, id: String, version: VersionReference)
fun plugin(alias: String, id: String, block: PluginDsl.() -> Unit)
fun plugin(alias: String, id: String, version: VersionReference, block: PluginDsl.() -> Unit)

==== Параметры

[cols="1,1,1,3",options="header"]

| String
| Да

| `version`
| VersionReference

[source,kotlin]

    // Без версии (встроенный Gradle plugin)
    plugin("java-library", "java-library")

    // С тегами
    plugin("spring-boot", "org.springframework.boot",
        version = versionRef("spring-boot")) {
        tags("spring", "backend")
    }
}
----

== bundles { }

Блок для определения групп библиотек.

**Источник:** xref:../02-features/F001-registry-dsl.adoc[F001: Registry DSL]

=== bundle()

==== Сигнатура

[source,kotlin]
----
fun bundle(name: String, block: BundleDsl.() -> Unit)
----

==== Методы BundleDsl

[cols="1,3",options="header"]
|===
| Метод | Описание

| `libraries(vararg aliases: String)`
| Добавить библиотеки в бандл

| `extends(vararg bundles: String)`
| Наследовать библиотеки из других бандлов
|===

==== Примеры

[source,kotlin]
----
bundles {
    // Простой бандл
    bundle("kotlin-core") {
        libraries("kotlin-stdlib", "kotlin-reflect")
    }

    // Бандл с наследованием
    bundle("kotlin-serialization") {
        libraries("kotlinx-serialization-core", "kotlinx-serialization-json")
        extends("kotlin-core")
    }

    // Комплексный бандл
    bundle("spring-backend") {
        libraries("spring-web", "spring-data-jpa", "spring-security")
        extends("kotlin-core", "kotlin-serialization")
    }

    // В сгенерированном TOML:
    // [bundles]
    // kotlin-core = ["kotlin-stdlib", "kotlin-reflect"]
    // kotlin-serialization = ["kotlinx-serialization-core", "kotlinx-serialization-json", "kotlin-stdlib", "kotlin-reflect"]
}
----

IMPORTANT: При `extends` библиотеки из родительских бандлов добавляются в дочерний.
Циклические зависимости запрещены.

[[dsl-bom-imports]]
== bomImports { }

Блок для импорта версий из внешних Maven BOM.

**Источник:** xref:../02-features/F001-registry-dsl.adoc[F001: Registry DSL], xref:../02-features/F006-bom-import.adoc[F006: BOM Import]

=== bom()

==== Сигнатуры

[source,kotlin]
----
fun bom(alias: String, coordinates: String)
fun bom(alias: String, coordinates: String, version: VersionReference)
fun bom(alias: String, group: String, artifact: String, version: VersionReference)
----

==== Примеры

[source,kotlin]
----
bomImports {
    // С version reference
    bom("spring-boot-bom", "org.springframework.boot:spring-boot-dependencies",
        version = versionRef("spring-boot"))

    // С inline версией
    bom("kotlin-bom", "org.jetbrains.kotlin:kotlin-bom:2.1.20")

    // Cloud BOM
    bom("spring-cloud-bom", "org.springframework.cloud:spring-cloud-dependencies",
        version = versionRef("spring-cloud"))
}

// Библиотеки без версий получат версии из BOM
libraries {
    library("spring-web", "org.springframework:spring-web")
    library("spring-data-jpa", "org.springframework.data:spring-data-jpa")
}
----

=== Порядок приоритета версий

1. **DECLARED** — явно указана в библиотеке
2. **BOM_IMPORT** — из импортированного BOM

Если библиотека без версии и не найдена в BOM — ошибка (или warning в зависимости от `settings.validation.onUnresolvedVersion`).

[[dsl-target-platforms]]
== targetPlatforms { }

Блок для определения целевых платформ.

**Источник:** xref:../02-features/F001-registry-dsl.adoc[F001: Registry DSL]

=== platform()

==== Сигнатура

[source,kotlin]
----
fun platform(name: String, block: PlatformDsl.() -> Unit)
----

==== Методы PlatformDsl

[cols="1,3",options="header"]
|===
| Метод | Описание

| `jdk { min; max }`
| Ограничения по JDK версии

| `android { minSdk; targetSdk }`
| Ограничения для Android

| `kotlin { min; max }`
| Ограничения по Kotlin версии

| `targets { ... }`
| KMP targets
|===

==== Примеры

[source,kotlin]
----
targetPlatforms {
    // JVM платформа
    platform("jvm17") {
        jdk { min = 17; max = 21 }
        targets { jvm() }
    }

    // Android платформа
    platform("android-min21") {
        android { minSdk = 21; targetSdk = 34 }
        targets { android() }
    }

    // KMP мобильная платформа
    platform("kmp-mobile") {
        kotlin { min = "1.9" }
        targets {
            android()
            ios()
        }
    }

    // KMP полная
    platform("kmp-full") {
        targets {
            jvm()
            js()
            android()
            ios()
            macos()
            linux()
            mingw()
            wasmJs()
        }
    }
}
----

=== Доступные targets

[cols="1,2",options="header"]
|===
| Target | Описание

| `jvm()`
| JVM target

| `js()`
| JavaScript (Browser, Node)

| `android()`
| Android

| `ios()`
| iOS (arm64, x64, simulatorArm64)

| `macos()`
| macOS (arm64, x64)

| `linux()`
| Linux (x64, arm64)

| `mingw()`
| Windows (MinGW x64)

| `native()`
| Все native targets

| `wasmJs()`
| WebAssembly JS

| `wasmWasi()`
| WebAssembly WASI
|===

[[dsl-distributions]]
== distributions { }

Блок для определения профилей генерации с фильтрацией.

**Источник:** xref:../02-features/F001-registry-dsl.adoc[F001: Registry DSL]

=== distribution()

==== Сигнатура

[source,kotlin]
----
fun distribution(name: String, block: DistributionDsl.() -> Unit)
----

==== Структура DistributionDsl

[source,kotlin]
----
distribution("name") {
    spec {
        filter {
            byTags { ... }
            byGroups { ... }
            byAliases { ... }
            byBundles { ... }
            byDeprecated { ... }
        }
    }
}
----

==== Примеры

[source,kotlin]
----
distributions {
    // Фильтрация по тегам
    distribution("android") {
        spec {
            filter {
                byTags {
                    include { anyOf("android", "mobile", "kmp") }
                    exclude { anyOf("server-only", "jvm-only") }
                }
            }
        }
    }

    // Фильтрация по Maven group
    distribution("no-spring") {
        spec {
            filter {
                byGroups {
                    exclude { matching("org.springframework*") }
                    exclude { matching("io.spring*") }
                }
            }
        }
    }

    // Фильтрация по alias
    distribution("minimal") {
        spec {
            filter {
                byAliases {
                    include("kotlin-stdlib", "slf4j-api", "logback-classic")
                }
            }
        }
    }

    // Включение бандлов
    distribution("backend") {
        spec {
            filter {
                byBundles {
                    include("spring-backend", "kotlin-core")
                }
            }
        }
    }

    // Исключение deprecated
    distribution("production") {
        spec {
            filter {
                byDeprecated { exclude() }
            }
        }
    }

    // Комбинированный фильтр
    distribution("android-prod") {
        spec {
            filter {
                byTags { include { anyOf("android", "mobile") } }
                byGroups { exclude { matching("org.springframework*") } }
                byDeprecated { exclude() }
            }
        }
    }
}
----

=== Типы фильтров

==== byTags

[source,kotlin]
----
byTags {
    include { anyOf("tag1", "tag2") }  // Любой из тегов
    include { allOf("tag1", "tag2") }  // Все теги
    exclude { anyOf("tag3", "tag4") }  // Исключить с любым из тегов
}
----

==== byGroups

[source,kotlin]
----
byGroups {
    include { matching("org.example*") }    // Glob pattern
    include { matching("com.company.*") }
    exclude { matching("org.legacy*") }
}
----

==== byAliases

[source,kotlin]
----
byAliases {
    include("kotlin-stdlib", "guava", "slf4j-api")
    exclude("deprecated-lib", "legacy-lib")
}
----

==== byBundles

[source,kotlin]
----
byBundles {
    include("kotlin-core", "spring-backend")
    exclude("legacy-bundle")
}
----

==== byDeprecated

[source,kotlin]
----
byDeprecated {
    include()   // Включить deprecated (по умолчанию)
    exclude()   // Исключить deprecated
}
----

=== Логика комбинирования фильтров

[cols="1,3",options="header"]
|===
| Комбинация | Логика

| Несколько `include` в одном фильтре
| **OR** — библиотека включается, если подходит под любой include

| Несколько `exclude` в одном фильтре
| **OR** — библиотека исключается, если подходит под любой exclude

| `include` + `exclude` в одном фильтре
| `include` применяется первым, затем из результата удаляются `exclude`

| Несколько разных фильтров (`byTags` + `byGroups`)
| **AND** — библиотека должна пройти все фильтры
|===

==== Алгоритм фильтрации

1. Если нет `include` фильтров -> все библиотеки потенциально включены
2. Применяются `include` фильтры -> только matching библиотеки остаются
3. Применяются `exclude` фильтры -> matching библиотеки удаляются
4. Результаты всех фильтров объединяются через **AND**

[[dsl-compatibility]]
== compatibility { }

Блок для определения правил совместимости.

**Источник:** xref:../02-features/F001-registry-dsl.adoc[F001: Registry DSL], xref:../02-features/F008-compatibility-analysis.adoc[F008: Compatibility Analysis]

=== jdkRequirement()

Правило требования к версии JDK.

[source,kotlin]
----
compatibility {
    jdkRequirement("spring-jdk17") {
        matches = "org.springframework:*"    // Glob pattern
        minJdk = 17
        maxJdk = 21                          // Опционально
        message = "Spring 6.x requires JDK 17+"
        severity = Severity.ERROR            // ERROR, WARNING, INFO
    }

    jdkRequirement("legacy-jdk8") {
        matches = "com.legacy:*"
        minJdk = 8
        maxJdk = 11
        message = "Legacy libraries require JDK 8-11"
    }
}
----

=== mutualExclusion()

Правило взаимоисключающих библиотек.

[source,kotlin]
----
compatibility {
    mutualExclusion("json-libs") {
        libraries("jackson-core", "gson")
        severity = Severity.WARNING
        message = "Consider using only one JSON library"
    }

    mutualExclusion("logging-impls") {
        libraries("logback-classic", "log4j-core", "jul-to-slf4j")
        severity = Severity.ERROR
        message = "Only one SLF4J implementation allowed"
    }
}
----

=== versionConstraint()

Правило ограничения версий между библиотеками.

[source,kotlin]
----
compatibility {
    versionConstraint("kotlin-version-sync") {
        libraries("kotlin-stdlib", "kotlin-reflect", "kotlinx-coroutines-core")
        constraint = VersionConstraintType.SAME_MAJOR_MINOR
        message = "Kotlin libraries should have matching versions"
    }

    versionConstraint("spring-version-sync") {
        libraries("spring-web", "spring-data-jpa", "spring-security-core")
        constraint = VersionConstraintType.SAME_MAJOR
    }
}
----

==== VersionConstraintType

[cols="1,3",options="header"]
|===
| Значение | Описание

| `SAME_VERSION`
| Все библиотеки должны иметь одинаковую версию

| `SAME_MAJOR`
| Все библиотеки должны иметь одинаковую major версию

| `SAME_MAJOR_MINOR`
| Все библиотеки должны иметь одинаковые major.minor
|===

=== customRule()

Пользовательское правило совместимости.

[source,kotlin]
----
compatibility {
    customRule("no-junit4-with-junit5") {
        condition = { metadata ->
            val hasJunit4 = metadata.libraries.any { it.alias == "junit4" }
            val hasJunit5 = metadata.libraries.any { it.alias.startsWith("junit-jupiter") }
            hasJunit4 && hasJunit5
        }
        message = "JUnit 4 and JUnit 5 should not be mixed"
        severity = Severity.WARNING
    }
}
----

[[dsl-processing]]
== processing { }

Runtime-only конфигурация пайплайна обработки.

**Источник:** xref:../02-features/F003-processing-pipeline.adoc[F003: Processing Pipeline]

IMPORTANT: `processing {}` — это **runtime-only** конфигурация.
Она задаётся в DSL или через CLI параметры, но **НЕ сохраняется** в `metadata.json`.
Каждый интерфейс (Gradle Plugin, CLI) настраивает pipeline самостоятельно.

=== Основные настройки

[source,kotlin]
----
processing {
    // Использовать preset конфигурации pipeline
    preset = ProcessingPreset.DEFAULT

    // Включить импорт версий из BOM
    importBoms = true

    // Включить проверку обновлений
    checkUpdates = false

    // Настройки анализа совместимости
    analyzeCompatibility {
        enabled = true
        targetJdk = 17
        failOnErrors = true
    }

    // Список отключённых процессоров
    disabledProcessors = listOf("update-check", "security-check")
}
----

=== Processing Presets

[cols="1,3",options="header"]
|===
| Preset | Включённые процессоры

| `DEFAULT`

| `MINIMAL`
| profile, metadata-conversion, extracted-versions, library-filter, version-fallback, version-resolver, bundle-filter, plugin-filter, plugin, used-versions

| `STRICT`
| Все процессоры + strictVersionResolution

| `DISTRIBUTION`

=== Builder методы

[[dsl-version-references]]
== Version References

Способы указания версий в DSL.

=== versionRef()

Ссылка на именованную версию.

[source,kotlin]
----
// Определение версии
versions {
    version("kotlin", "2.1.20")
}

// Использование ссылки
libraries {
    library("kotlin-stdlib", "org.jetbrains.kotlin:kotlin-stdlib",
        version = versionRef("kotlin"))
}
----

=== Literal версия

Прямое указание версии.

[source,kotlin]
----
libraries {
    // Через параметр version
    library("guava", "com.google.guava:guava", version = "33.0.0-jre")

    // Inline в coordinates
    library("slf4j-api", "org.slf4j:slf4j-api:2.0.9")
}
----

=== versionRange()

Диапазон версий (для constraints).

[source,kotlin]
----
// Maven-style ranges
versionRange("[1.0, 2.0)")   // >= 1.0 и < 2.0
versionRange("[1.0,)")       // >= 1.0
versionRange("(,2.0]")       // <= 2.0
versionRange("[1.0]")        // Точно 1.0
----
