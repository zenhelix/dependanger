[[overview]]
== Обзор

Модуль `dependanger-security` проверяет библиотеки на известные уязвимости (CVE) через OSV API (https://api.osv.dev).
Результаты хранятся в `EffectiveMetadata.extensions` через `VulnerabilitiesExtensionKey`.

Приоритет: **P3** (желательно).

[[spec-security-check]]
== SPEC-10.1: SecurityCheckProcessor.process()

[[purpose-security]]
=== Назначение

Собрать все библиотеки из EffectiveMetadata, batch-запросом отправить в OSV API, получить уязвимости, классифицировать по severity и сохранить в extensions.

[[algorithm-security]]
=== Алгоритм

[source,text]
----
1. Получить настройки securityCheck из context.settings
2. Собрать библиотеки для проверки:
   a. Все библиотеки из metadata.libraries с version != null
   b. Исключить библиотеки из ignoreLibraries
   c. Если includeTransitives=true -> добавить транзитивные из extensions
3. Проверить кеш: для каждой библиотеки проверить ~/.dependanger/cache/security/
4. Библиотеки без кеша -> batch-запрос в OSV API:
   a. Разбить на batches по 1000 (лимит OSV API)
   b. POST https://api.osv.dev/v1/querybatch
   c. Парсить ответ: OSV batch API гарантирует `results[i]` соответствует `queries[i]` (1:1 по индексу). Если пакет не найден, `results[i].vulns` будет `null` или пустой список
5. Для каждой найденной уязвимости:
   a. Извлечь CVE ID из aliases
   b. Парсить CVSS score из severity
   c. Определить severity (CRITICAL/HIGH/MEDIUM/LOW) по CVSS score
   d. Извлечь fixedVersion из affected[].ranges
   e. Создать VulnerabilityInfo
6. Отфильтровать по minSeverity и minCvssScore
7. Отфильтровать ignoreCves
8. Записать в кеш
9. Записать в metadata: withExtension(VulnerabilitiesExtensionKey, vulns)
10. Добавить diagnostics:
    a. INFO: "Scanned X libraries, found Y vulnerabilities"
    b. Если failOnCritical && есть CRITICAL -> ERROR
    c. Если failOnHigh && есть HIGH -> ERROR
----

[[spec-osv-client]]
== SPEC-10.2: OsvApiClient.queryVulnerabilities()

[[algorithm-osv]]
=== Алгоритм

[source,text]
----
1. Проверить кеш для group:artifact:version
2. Если нет в кеше:
   a. Сформировать запрос для batch endpoint
   b. POST https://api.osv.dev/v1/querybatch с JSON body
   c. Парсить ответ: results[i].vulns[]
   d. Для каждой vulns[]:
      i.   Извлечь id (OSV ID, например GHSA-xxx)
      ii.  Извлечь aliases (CVE-xxx)
      iii. Извлечь summary
      iv.  Парсить severity[].score (CVSS v3 vector string)
      v.   Рассчитать cvssScore из CVSS vector
      vi.  Определить VulnerabilitySeverity по score
      vii. Извлечь fixedVersion из affected[].ranges[].events[].fixed
      viii.Извлечь references[].url
   e. Создать VulnerabilityInfo для каждой уязвимости
   f. Сохранить в кеш
3. Вернуть List<VulnerabilityInfo>
----

==== OsvApiClient

[source,kotlin]
----
public class OsvApiClient(
    public val apiUrl: String = "https://api.osv.dev",
    private val httpClient: HttpClient = createDefaultClient(),
    private val batchSize: Int = 1000,
    private val retries: Int = 3,
    private val timeout: Long = 30_000,
) {
    public suspend fun queryBatch(
        packages: List<PackageQuery>
    ): List<List<VulnerabilityInfo>> {
        val results = mutableListOf<List<VulnerabilityInfo>>()

        for (batch in packages.chunked(batchSize)) {
            val response = executeWithRetry {
                httpClient.post("$apiUrl/v1/querybatch") {
                    contentType(ContentType.Application.Json)
                    timeout { requestTimeoutMillis = this@OsvApiClient.timeout }
                    setBody(OsvBatchRequest(
                        queries = batch.map { pkg ->
                            OsvQuery(
                                version = pkg.version,
                                `package` = OsvPackage(
                                    name = "${pkg.group}:${pkg.artifact}",
                                    ecosystem = "Maven"
                                )
                            )
                        }
                    ))
                }
            }

            val osvResponse = response.body<OsvBatchResponse>()
            results.addAll(osvResponse.results.map { result ->
                result.vulns?.map { vuln -> mapToVulnerabilityInfo(vuln) } ?: emptyList()
            })
        }
        return results
    }

    private fun mapToVulnerabilityInfo(vuln: OsvVulnerability): VulnerabilityInfo {
        val cveId = vuln.aliases?.firstOrNull { it.startsWith("CVE-") }
        val cvssScore = vuln.severity
            ?.firstOrNull { it.type == "CVSS_V3" }
            ?.let { parseCvssScore(it.score) }
        val severity = cvssScoreToSeverity(cvssScore)
        val fixedVersion = vuln.affected
            ?.flatMap { it.ranges ?: emptyList() }
            ?.flatMap { it.events ?: emptyList() }
            ?.firstOrNull { it.fixed != null }
            ?.fixed

        return VulnerabilityInfo(
            id = vuln.id,
            aliases = vuln.aliases ?: emptyList(),
            summary = vuln.summary ?: "",
            severity = severity,
            cvssScore = cvssScore,
            fixedVersion = fixedVersion,
            url = vuln.references?.firstOrNull()?.url,
            affectedGroup = "", // заполняется вызывающим кодом
            affectedArtifact = "",
            affectedVersion = "",
        )
    }
}
----

==== CVSS Score парсинг

[source,kotlin]
----
/**
 * Парсит CVSS v3 vector string и извлекает score.
 * Пример: "CVSS:3.1/AV:N/AC:L/PR:N/UI:N/S:C/C:H/I:H/A:H" -> 10.0
 *
 * Для упрощения используется приблизительный расчёт по Base Score metrics.
 * В production рекомендуется использовать библиотеку для точного расчёта.
 */
private fun parseCvssScore(vector: String): Double? {
    // Упрощённый парсинг: если вектор содержит числовой score
    // OSV иногда возвращает score напрямую
    vector.toDoubleOrNull()?.let { return it }

    // Парсинг CVSS:3.x/... vector
    if (!vector.startsWith("CVSS:3")) return null

    // Приблизительный расчёт по ключевым метрикам
    val metrics = vector.substringAfter("/").split("/").associate {
        val (key, value) = it.split(":")
        key to value
    }
    // Используем lookup таблицу для основных комбинаций
    // или делегируем к CVSS calculator library
    return approximateCvssScore(metrics)
}

private fun cvssScoreToSeverity(score: Double?): VulnerabilitySeverity = when {
    score == null -> VulnerabilitySeverity.UNKNOWN
    score >= 9.0 -> VulnerabilitySeverity.CRITICAL
    score >= 7.0 -> VulnerabilitySeverity.HIGH
    score >= 4.0 -> VulnerabilitySeverity.MEDIUM
    score >= 0.1 -> VulnerabilitySeverity.LOW
    else -> VulnerabilitySeverity.NONE
}
----

==== CVSS Fallback: Severity Lookup Table

Если CVSS_V3 score недоступен, используется lookup table по severity string из OSV response:

[source,kotlin]
----
private fun severityStringToScore(severity: String): Double = when (severity.uppercase()) {
    "CRITICAL" -> 9.0
    "HIGH" -> 7.0
    "MEDIUM" -> 4.0
    "LOW" -> 1.0
    else -> 0.0
}
----

Приоритет: точный CVSS v3 score > lookup по severity string.

[[edge-cases]]
== Edge Cases и обработка ошибок

[cols="1,2,2"]
|===
| Ситуация | Поведение | Диагностический код

| OSV API недоступен, кеш пуст | WARNING, проверка пропускается | `SECURITY_API_UNREACHABLE`
| OSV API недоступен, кеш есть | Используется кеш + WARNING | `SECURITY_STALE_CACHE`
| Таймаут запроса | Retry 3 раза с exponential backoff (1s, 2s, 4s), затем WARNING | `SECURITY_TIMEOUT`
| Rate limiting (429) | HTTP 429 -> retry с exponential backoff (1s, 2s, 4s). Максимум 3 попытки | `SECURITY_RATE_LIMITED`
| Библиотека не найдена в OSV | Считается безопасной (нет данных != уязвима) | --
| Уязвимость без CVE | Используется OSV ID, cveId=null | --
| fixedVersion недоступен | fixedVersion=null, рекомендация "check manually" | --
| Множественные уязвимости | Отдельный VulnerabilityInfo для каждой | --
| Пустой каталог | 0 уязвимостей, INFO | `SECURITY_NO_VULNS`
| Невалидный CVSS | WARNING, severity по текстовому полю | `SECURITY_INVALID_CVSS`
| Игнорируемая CRITICAL + failOnCritical | Pipeline продолжается (игнорируется) | --
| > 1000 библиотек | Batch splitting по 1000 | --
| CVSS v2 only | Конвертация -> approximate v3 + WARNING | `SECURITY_CVSS_V2`
|===

[[diagnostic-codes]]
== Диагностические коды

[cols="1,2,1"]
|===
| Код | Описание | Уровень

| `SECURITY_SCAN_COMPLETE` | Сканирование завершено с результатами | INFO
| `SECURITY_NO_VULNS` | Уязвимости не найдены | INFO
| `SECURITY_CRITICAL_FOUND` | Найдены CRITICAL уязвимости | ERROR (если failOnCritical)
| `SECURITY_HIGH_FOUND` | Найдены HIGH уязвимости | ERROR (если failOnHigh)
| `SECURITY_API_UNREACHABLE` | OSV API недоступен | WARNING
| `SECURITY_STALE_CACHE` | Используется устаревший кеш | WARNING
| `SECURITY_TIMEOUT` | Таймаут запроса к OSV | WARNING
| `SECURITY_RATE_LIMITED` | Rate limiting от OSV | WARNING
| `SECURITY_INVALID_CVSS` | Невалидный CVSS score | WARNING
| `SECURITY_CVSS_V2` | Только CVSS v2 доступен | WARNING
|===
