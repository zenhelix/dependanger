== Иерархия правил

[source]
----
Constitution (MUST)     ─── Нарушение = блокировка merge
       │
       ▼
Guidelines (SHOULD)     ─── Нарушение = требует обоснования в PR
       │
       ▼
Fitness Functions       ─── Автоматические проверки в CI
----

== Ключевые принципы

Из xref:constitution.adoc[Constitution]:

* **M1**: Зависимости направлены от периферии к ядру
* **M2**: Доменная модель изолирована от инфраструктуры
* **E1**: Расширяемость через SPI, не через наследование
* **T1**: Все зависимости инъектируются через конструктор
* **C1**: Explicit API mode везде
