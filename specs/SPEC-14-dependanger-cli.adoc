[[overview]]
== Обзор

Модуль `dependanger-cli` предоставляет standalone command-line интерфейс для Dependanger.
Построен на базе Clikt 5.0.3 с Mordant 3.0.2 для цветного вывода.
Содержит **29 файлов**, из которых 25 команд и 2 утилиты имеют TODO-заглушки в методах `run()`.

Уже реализовано: `DependangerCli.kt` (main entry, маршрутизация subcommands), `CliConfig.kt` (data classes конфигурации).

[[pattern-metadata-io]]
=== Паттерн: Загрузка/сохранение metadata

Все команды, работающие с metadata.json, используют один и тот же паттерн I/O:

[source,kotlin]
----
// Загрузка metadata
private fun loadMetadata(inputPath: String): DependangerMetadata {
    val path = Path.of(inputPath)
    if (!path.exists()) {
        throw PrintMessage(
            formatError("File not found: $inputPath"),
            statusCode = EXIT_FILE_NOT_FOUND,
        )
    }
    val format = JsonSerializationFormat()
    return try {
        format.read(path)
    } catch (e: Exception) {
        throw PrintMessage(
            formatError("Failed to read metadata: ${e.message}"),
            statusCode = EXIT_ERROR,
        )
    }
}

// Сохранение metadata
private fun saveMetadata(metadata: DependangerMetadata, outputPath: String) {
    val format = JsonSerializationFormat()
    try {
        format.write(metadata, Path.of(outputPath))
    } catch (e: Exception) {
        throw PrintMessage(
            formatError("Failed to write metadata: ${e.message}"),
            statusCode = EXIT_ERROR,
        )
    }
}

[[purpose-output-formatter]]
=== Назначение

Форматирование вывода команд с поддержкой цветов (через Mordant) и JSON-вывода.

[[implementation-output-formatter]]
=== Детальная реализация

[source,kotlin]
----
import com.github.ajalt.mordant.rendering.TextColors.*
import com.github.ajalt.mordant.rendering.TextStyles.*
import com.github.ajalt.mordant.terminal.Terminal
import com.github.ajalt.mordant.table.table

public class OutputFormatter(
    public val useColor: Boolean = true,
    public val jsonOutput: Boolean = false,
) {
    private val terminal: Terminal = Terminal()

    public fun formatSuccess(message: String): String {
        if (jsonOutput) return """{"status":"success","message":"$message"}"""
        if (!useColor) return "[OK] $message"
        return (green + bold)("✓ $message")
    }

    public fun formatError(message: String): String {
        if (jsonOutput) return """{"status":"error","message":"$message"}"""
        if (!useColor) return "[ERROR] $message"
        return (red + bold)("✗ $message")
    }

    public fun formatWarning(message: String): String {
        if (jsonOutput) return """{"status":"warning","message":"$message"}"""
        if (!useColor) return "[WARN] $message"
        return (yellow)("⚠ $message")
    }

    public fun formatInfo(message: String): String {
        if (jsonOutput) return """{"status":"info","message":"$message"}"""
        if (!useColor) return "[INFO] $message"
        return (cyan)("ℹ $message")
    }

    public fun formatTable(headers: List<String>, rows: List<List<String>>): String {
        if (jsonOutput) {
            // Конвертировать в JSON массив объектов
            val jsonRows = rows.map { row ->
                headers.zip(row).joinToString(",") { (h, v) -> "\"$h\":\"$v\"" }
            }
            return "[${jsonRows.joinToString(",") { "{$it}" }}]"
        }

        return terminal.render(table {
            header {
                row(*headers.toTypedArray())
            }
            body {
                rows.forEach { row ->
                    row(*row.toTypedArray())
                }
            }
        })
    }
}

[[spec-config-loader]]
== SPEC-14.2: ConfigLoader

[[purpose-config-loader]]
=== Назначение

Загрузка конфигурации CLI из YAML-файла с поиском в стандартных местах.

[[algorithm-config-loader]]
=== Алгоритм

[source,text]
----
load(configPath):
1. Если configPath задан -> загрузить из него
2. Иначе -> findConfigFile() -> загрузить
3. Если файл не найден -> вернуть CliConfig() с дефолтами
4. Парсить YAML/JSON в CliConfig

findConfigFile():
1. Проверить текущую директорию: ./.dependanger.yaml
2. Проверить текущую директорию: ./.dependanger.yml
3. Проверить текущую директорию: ./.dependanger.json
4. Проверить домашнюю: ~/.dependanger/config.yaml
5. Проверить домашнюю: ~/.dependanger/config.json
6. Проверить переменную окружения DEPENDANGER_CONFIG
7. Если ничего не найдено -> вернуть null
----

[[implementation-config-loader]]
=== Детальная реализация

[source,kotlin]
----
import kotlinx.serialization.json.Json
import java.nio.file.Path
import kotlin.io.path.exists
import kotlin.io.path.readText

public class ConfigLoader {
    private val json = Json {
        ignoreUnknownKeys = true
        isLenient = true
    }

    public fun load(configPath: Path? = null): CliConfig {
        val path = configPath ?: findConfigFile() ?: return CliConfig()

        if (!path.exists()) return CliConfig()

        return try {
            val content = path.readText()
            json.decodeFromString<CliConfig>(content)
        } catch (e: Exception) {
            // Если конфиг-файл невалиден, логируем warning и используем дефолты
            System.err.println("Warning: Failed to parse config file $path: ${e.message}")
            CliConfig()
        }
    }

    public fun findConfigFile(): Path? {
        val candidates = buildList {
            // Текущая директория
            add(Path.of(".dependanger.yaml"))
            add(Path.of(".dependanger.yml"))
            add(Path.of(".dependanger.json"))

            // Домашняя директория
            val home = System.getProperty("user.home")
            if (home != null) {
                add(Path.of(home, ".dependanger", "config.yaml"))
                add(Path.of(home, ".dependanger", "config.yml"))
                add(Path.of(home, ".dependanger", "config.json"))
            }

            // Переменная окружения
            System.getenv("DEPENDANGER_CONFIG")?.let { add(Path.of(it)) }
        }

        return candidates.firstOrNull { it.exists() }
    }

    public companion object {
        public const val DEFAULT_CONFIG_FILENAME: String = ".dependanger.yaml"
    }
}
----

NOTE: Для поддержки YAML формата используется библиотека `com.charleskorn.kaml:kaml` -- kotlinx-serialization backend для YAML (см. <<dependencies>>).
Она позволяет десериализовать YAML-конфигурацию через `Yaml.default.decodeFromString<CliConfig>(content)` без дополнительных адаптеров.

=== Реализация

[source,kotlin]
----
override fun run() {
    val outputPath = Path.of(output)
    if (outputPath.exists() && !force) {
        echo(formatError("File already exists: $output. Use --force to overwrite."))
        throw ProgramResult(ExitCodes.ERROR)
    }

    val metadata = DependangerMetadata(schemaVersion = "1.0")
    val format = JsonSerializationFormat()

    try {
        format.write(metadata, outputPath)
        echo(formatSuccess("Created $output"))
    } catch (e: Exception) {
        echo(formatError("Failed to create metadata file: ${e.message}"))
        throw ProgramResult(ExitCodes.ERROR)
    }
}

=== SPEC-14.4.1: AddVersionCommand

[source,kotlin]
----
override fun run() {
    val metadata = MetadataIO.load(Path.of(input))
    val effectiveOutput = output ?: input

    // Проверка дубликатов
    if (metadata.versions.any { it.name == alias }) {
        echo(formatError("Version '$alias' already exists. Use update-version to modify."))
        throw ProgramResult(ExitCodes.ERROR)
    }

    // Парсинг fallback
    val fallbacks = parseFallback(fallback)

    // Создание новой версии
    val newVersion = Version(
        name = alias,
        value = value,
        fallbacks = fallbacks,
    )

    // Обновление metadata
    val updatedMetadata = metadata.copy(
        versions = metadata.versions + newVersion
    )

    MetadataIO.save(updatedMetadata, Path.of(effectiveOutput))
    echo(formatSuccess("Added version '$alias' = '$value'"))
}

private fun parseFallback(fallbackStr: String?): List<VersionFallback> {
    if (fallbackStr == null) return emptyList()
    // Формат: "condition=value", например "jdkBelow(17)=2.7.18"
    val parts = fallbackStr.split("=", limit = 2)
    if (parts.size != 2) {
        echo(formatError("Invalid fallback format. Expected: condition=value"))
        throw ProgramResult(ExitCodes.ERROR)
    }
    val condition = parseFallbackCondition(parts[0].trim())
    val fallbackValue = parts[1].trim()
    return listOf(VersionFallback(fallbackValue, condition))
}

[[spec-remove-commands]]
== SPEC-14.5: Remove-команды

=== Общий паттерн

[source,text]
----
1. Загрузить metadata из input
2. Найти entity по alias/name
3. Если не найден -> ошибка "Not found"
4. Если !force -> проверить зависимости (ссылки из бандлов, библиотек)
5. Удалить entity из списка
6. Сохранить обновлённый metadata
7. Вывести formatSuccess("Removed ...")
----

=== Проверка зависимостей (если не --force)

[source,kotlin]
----
// RemoveVersionCommand: проверить, что ни одна библиотека/плагин не ссылается на версию
private fun checkVersionReferences(metadata: DependangerMetadata, versionAlias: String): List<String> {
    val refs = mutableListOf<String>()
    metadata.libraries.forEach { lib ->
        if (lib.version is VersionReference.Reference && lib.version.name == versionAlias) {
            refs.add("library '${lib.alias}'")
        }
    }
    metadata.plugins.forEach { plugin ->
        if (plugin.version is VersionReference.Reference && plugin.version.name == versionAlias) {
            refs.add("plugin '${plugin.alias}'")
        }
    }
    return refs
}
----

override fun run() {

}

=== RemoveLibraryCommand, RemovePluginCommand, RemoveBundleCommand, RemoveBomCommand, RemoveDistributionCommand

Аналогичная структура с соответствующими проверками зависимостей:

- **RemoveLibraryCommand**: проверить ссылки из bundles
- **RemovePluginCommand**: нет зависимостей, простое удаление
- **RemoveBundleCommand**: проверить ссылки из других bundles (extends) и distributions
- **RemoveBomCommand**: нет прямых зависимостей, простое удаление
- **RemoveDistributionCommand**: проверить ссылки из presets

[[spec-migrate-deprecated]]
== SPEC-14.7: MigrateDeprecatedCommand

=== Алгоритм

[source,text]
----
1. Загрузить metadata
2. Найти все библиотеки с deprecation != null
3. Для каждой deprecated библиотеки:
   a. Определить replacement (deprecation.replacedBy)
   b. Найти бандлы, содержащие deprecated алиас
4. Если --dry-run:
   a. Вывести план миграции
   b. Завершить без изменений
5. Если --backup -> скопировать input в input.bak
6. Если --replace:
   a. В каждом бандле заменить deprecated алиас на replacement
   b. Если replacement отсутствует -> warning
7. Если --remove:
   a. Удалить deprecated библиотеки из списка libraries
   b. Если --remove-from-bundles -> удалить алиасы из бандлов вместо замены
8. Сохранить обновлённый metadata
9. Вывести отчёт
----

=== Реализация

[source,kotlin]
----
override fun run() {
    val metadata = MetadataIO.load(Path.of(input))
    val effectiveOutput = output ?: input

    val deprecated = metadata.libraries.filter { it.deprecation != null }
    if (deprecated.isEmpty()) {
        echo(formatSuccess("No deprecated libraries found."))
        return
    }

    // Построить план миграции
    val plan = deprecated.map { lib ->
        MigrationAction(
            library = lib,
            replacement = lib.deprecation?.replacedBy,
            usedInBundles = metadata.bundles.filter { it.libraries.contains(lib.alias) },
        )
    }

    if (dryRun) {
        printMigrationPlan(plan)
        return
    }

    // Создание backup
    if (backup) {
        val backupPath = Path.of("$input.bak")
        Path.of(input).copyTo(backupPath, overwrite = true)
        echo(formatInfo("Backup created: $backupPath"))
    }

    // Применение миграции
    var updatedMetadata = metadata
    val actions = mutableListOf<String>()

    for (action in plan) {
        if (replace && action.replacement != null) {
            // Заменить в бандлах
            updatedMetadata = replaceInBundles(updatedMetadata, action.library.alias, action.replacement)
            action.usedInBundles.forEach { bundle ->
                actions.add("Replaced '${action.library.alias}' with '${action.replacement}' in bundle '${bundle.name}'")
            }
        }

        if (removeFromBundles && action.replacement == null) {
            // Удалить из бандлов без замены
            updatedMetadata = removeFromBundles(updatedMetadata, action.library.alias)
        }

        if (remove) {
            // Удалить определение библиотеки
            updatedMetadata = updatedMetadata.copy(
                libraries = updatedMetadata.libraries.filter { it.alias != action.library.alias }
            )
            actions.add("Removed library '${action.library.alias}'")
        }
    }

    MetadataIO.save(updatedMetadata, Path.of(effectiveOutput))

    actions.forEach { echo(formatSuccess(it)) }
    echo(formatSuccess("Migration completed. Updated: $effectiveOutput"))
}

[[spec-validate-command]]
== SPEC-14.8: ValidateCommand

=== Алгоритм

[source,text]
----
1. Загрузить metadata из input
2. Создать Dependanger через fromJson/fromMetadata
3. Вызвать validate() (suspend -> runBlocking)
4. Форматировать diagnostics (errors, warnings, infos)
5. Вывести результат в выбранном формате (text/json)
6. Выставить exit code:
   - strict && warnings > 0 -> EXIT_WARNINGS
   - errors > 0 -> EXIT_ERROR
   - else -> EXIT_SUCCESS

=== Реализация

[source,kotlin]
----
override fun run() = runBlocking {
    val metadata = MetadataIO.load(Path.of(input))

    val dependanger = Dependanger.fromMetadata(metadata).build()
    val result = dependanger.validate()

    when (format) {
        "json" -> printJsonResult(result)
        else   -> printTextResult(result)
    }

    val exitCode = when {
        result.diagnostics.hasErrors -> ExitCodes.ERROR
        strict && result.diagnostics.warnings.isNotEmpty() -> ExitCodes.WARNINGS
        else -> ExitCodes.SUCCESS

    if (exitCode != ExitCodes.SUCCESS) {

    }

private fun printTextResult(result: DependangerResult) {
    val d = result.diagnostics
    d.errors.forEach { echo(formatError(it.message)) }
    d.warnings.forEach { echo(formatWarning(it.message)) }
    d.infos.forEach { echo(formatInfo(it.message)) }

    echo("")
    echo("Validation completed: ${d.errors.size} errors, ${d.warnings.size} warnings")
}

[[helper-extract-extensions]]
=== Паттерн: extension-функции извлечения данных из EffectiveMetadata

Все функции вида `extractUpdateData()`, `extractSecurityData()`, `extractLicenseData()`, `extractTransitiveData()` и `extractCompatibilityIssues()` определены как `internal` extension-функции на `EffectiveMetadata` в пакете `cli/util/`.
Они инкапсулируют логику обращения к `extensions` и обеспечивают type-safe доступ к данным, заполненным соответствующими feature-процессорами.

[source,kotlin]
----
// cli/util/EffectiveMetadataExtensions.kt
package io.github.zenhelix.dependanger.cli.util

import io.github.zenhelix.dependanger.effective.model.EffectiveMetadata
import io.github.zenhelix.dependanger.features.updates.model.UpdatesExtensionKey
import io.github.zenhelix.dependanger.features.security.model.VulnerabilitiesExtensionKey
import io.github.zenhelix.dependanger.features.license.model.LicenseViolationsExtensionKey
import io.github.zenhelix.dependanger.features.transitive.model.TransitivesExtensionKey
import io.github.zenhelix.dependanger.effective.model.getExtension
import io.github.zenhelix.dependanger.features.updates.model.UpdateAvailableInfo
import io.github.zenhelix.dependanger.features.security.model.VulnerabilityInfo
import io.github.zenhelix.dependanger.features.license.model.LicenseViolation

internal fun EffectiveMetadata.extractUpdateData(): List<UpdateAvailableInfo> =
    getExtension(UpdatesExtensionKey) ?: emptyList()

internal fun EffectiveMetadata.extractSecurityData(): List<VulnerabilityInfo> =
    getExtension(VulnerabilitiesExtensionKey) ?: emptyList()

internal fun EffectiveMetadata.extractLicenseData(): List<LicenseViolation> =
    getExtension(LicenseViolationsExtensionKey) ?: emptyList()

internal fun EffectiveMetadata.extractTransitiveData(): List<TransitiveTree> =
    getExtension(TransitivesExtensionKey) ?: emptyList()
----

NOTE: Благодаря этому паттерну, команды анализа (<<spec-analysis-commands>>) вызывают extraction-методы как `result.effective!!.extractUpdateData()` вместо прямого обращения к `getExtension(...)`, что повышает читаемость и изолирует знание о ключах extensions в одном месте.

----
// cli/util/VersionParser.kt
// Парсинг строки версии: "1.0.0" -> Literal, "ref:kotlin" -> Reference
public fun parseVersionOption(value: String?): VersionReference? {
    if (value == null) return null
    return if (value.startsWith("ref:")) {
        VersionReference.Reference(value.removePrefix("ref:"))
    } else {
        VersionReference.Literal(value)
    }
}

1. **Утилиты**: ExitCodes, MetadataIO, OutputFormatter, ConfigLoader -- базовая инфраструктура
2. **CRUD-команды**: InitCommand -> Add* -> Remove* -> Update* -> MigrateDeprecated -- не требуют API фасада
3. **Processing-команды**: ValidateCommand, ProcessCommand, GenerateCommand -- требуют Dependanger API
4. **Analysis-команды**: CheckUpdates, Analyze, SecurityCheck, LicenseCheck, ResolveTransitives -- требуют feature-модули
5. **ReportCommand** -- требует все предыдущие
