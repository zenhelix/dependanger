== Обзор

Gradle Plugin предоставляет интеграцию Dependanger в Gradle build через Kotlin DSL.
Это основной способ использования Dependanger для проектов на Kotlin/Java.

[cols="1,2",options="header"]
|===
| Характеристика | Значение

| Plugin ID
| `io.github.zenhelix.dependanger`

| Минимальная версия Gradle
| 8.0+

| Целевая аудитория
| Kotlin/Java разработчики
|===

== Подключение

[source,kotlin]
----
// settings.gradle.kts
pluginManagement {
    repositories {
        gradlePluginPortal()
        mavenCentral()
    }
}

// build.gradle.kts
plugins {
    id("io.github.zenhelix.dependanger") version "<version>"
}
----

== DSL Конфигурация

NOTE: Полная документация по DSL синтаксису: xref:../02-features/F001-registry-dsl.adoc[F001: Registry DSL].
Ниже приведены примеры использования в контексте Gradle плагина.

=== Базовая структура

[source,kotlin]
----
dependanger {
    // Версии зависимостей
    versions { ... }
    
    // Библиотеки
    libraries { ... }
    
    // Gradle плагины
    plugins { ... }
    
    // Группы библиотек
    bundles { ... }
    
    // Внешние BOM для импорта версий
    platforms { ... }
    
    // Профили генерации с фильтрацией
    distributions { ... }
    
    // Правила совместимости
    compatibility { ... }
    
    // Глобальные настройки
    settings { ... }
    
    // Именованные presets
    presets { ... }
    
    // Настройки пайплайна обработки
    processing { ... }
}
----

=== Блок versions

Объявление именованных версий для переиспользования.

[source,kotlin]
----
versions {
    // Простая версия
    version("kotlin", "2.1.20")
    version("spring-boot", "3.4.1")
    
    // Версия с условным fallback
    version("spring-boot", "3.4.1") {
        fallback("2.7.18") {
            jdkBelow(17)
        }
    }
    
    // Версия с метаданными
    version("junit", "5.11.0") {
        description = "JUnit 5 testing framework"
        stability = STABLE
    }
}
----

==== Условия для fallback

[cols="1,2",options="header"]
|===
| Условие | Описание

| `jdkBelow(version)`
| JDK ниже указанной версии

| `jdkAtLeast(version)`
| JDK не ниже указанной версии

| `jdkBetween(min, max)`
| JDK в диапазоне [min, max]

| `kotlinVersionAtLeast(version)`
| Kotlin не ниже указанной версии

| `kotlinVersionBelow(version)`
| Kotlin ниже указанной версии

| `gradleVersionAtLeast(version)`
| Gradle не ниже указанной версии

| `gradleVersionBelow(version)`
| Gradle ниже указанной версии

| `distribution(name)`
| Активная дистрибуция

| `environment(name, value)`
| Переменная окружения

| `all { ... }`
| Все условия должны быть истинны

| `any { ... }`
| Хотя бы одно условие истинно

| `not { ... }`
| Инверсия условия
|===

=== Блок libraries

Объявление библиотек с координатами Maven.

[source,kotlin]
----
libraries {
    // Библиотека с версией по ссылке
    library("kotlin-stdlib", "org.jetbrains.kotlin:kotlin-stdlib") {
        version = versionRef("kotlin")
    }
    
    // Библиотека с inline версией
    library("guava", "com.google.guava:guava:33.0.0-jre")
    
    // Библиотека с тегами для фильтрации
    library("android-core", "androidx.core:core-ktx") {
        version = "1.12.0"
        tags("android", "core")
    }
    
    // Библиотека без версии (resolved from BOM)
    library("spring-web", "org.springframework:spring-web") {
        version = versionRef("spring")  // или будет резолвиться из BOM
    }
}
----

=== Блок plugins

Объявление Gradle плагинов.

[source,kotlin]
----
plugins {
    plugin("kotlin-jvm", "org.jetbrains.kotlin.jvm") {
        version = versionRef("kotlin")
    }
    
    plugin("spring-boot", "org.springframework.boot") {
        version = versionRef("spring-boot")
    }
}
----

=== Блок bundles

Группировка библиотек в именованные бандлы.

[source,kotlin]
----
bundles {
    bundle("kotlin-core") {
        libraries("kotlin-stdlib", "kotlin-reflect")
    }
    
    bundle("testing") {
        libraries("junit-jupiter", "assertj-core", "mockk")
    }
    
    // Бандл с условиями
    bundle("android-compose") {
        libraries("compose-ui", "compose-material3", "compose-runtime")
        tags("android", "compose")
    }
}
----

=== Блок platforms

Импорт версий из внешних Maven BOM.

[source,kotlin]
----
platforms {
    // Spring Boot BOM
    platform("spring-boot-bom", "org.springframework.boot:spring-boot-dependencies") {
        version = versionRef("spring-boot")
    }
    
    // Kotlin BOM
    platform("kotlin-bom", "org.jetbrains.kotlin:kotlin-bom") {
        version = versionRef("kotlin")
    }
    
    // Platform с конкретной версией
    platform("junit-bom", "org.junit:junit-bom:5.11.0")
}
----

=== Блок distributions

Профили генерации с фильтрацией библиотек.

[source,kotlin]
----
distributions {
    // Дистрибуция для core библиотек
    distribution("core") {
        filter {
            includeTags("core")
            excludeTags("android", "ios")
        }
    }
    
    // Дистрибуция для Android
    distribution("android") {
        filter {
            includeTags("android", "core")
            excludeLibraries("desktop-only-lib")
        }
    }
    
    // Дистрибуция со всеми библиотеками (без фильтрации)
    distribution("full") {
        // Пустой фильтр = все библиотеки
    }
}
----

=== Блок compatibility

Правила проверки совместимости.

[source,kotlin]
----
compatibility {
    // Требование к JDK
    jdkRequirement("spring-boot-3x") {
        libraries("spring-boot-starter")
        minJdk = 17
    }
    
    // Взаимоисключающие библиотеки
    mutualExclusion("logging") {
        libraries("slf4j-simple", "logback-classic", "log4j-core")
        message = "Only one logging implementation should be used"
    }
    
    // Ограничение версий
    versionConstraint("kotlin-compat") {
        library = "kotlinx-coroutines-core"
        versionRange = "[1.7.0,)"  // >= 1.7.0
    }
    
    // Кастомное правило
    customRule("security-check") {
        condition { metadata ->
            // Проверка условия
            true
        }
        message = "Security check failed"
        severity = WARNING
    }
}
----

=== Блок settings

Глобальные настройки.

[source,kotlin]
----
settings {
    // Дефолтная дистрибуция
    defaultDistribution = "core"
    
    // Строгий режим резолюции версий
    strictVersionResolution = true
    
    // Настройки генерации TOML
    toml {
        filename = "libs.versions.toml"
        includeComments = true
    }
    
    // Настройки генерации BOM
    bom {
        groupId = "com.example"
        artifactId = "my-bom"
    }
}
----

[[processing-block]]
=== Блок processing

Настройки пайплайна обработки.

[source,kotlin]
----
processing {
    // Включить импорт версий из BOM
    importBoms = true
    
    // Включить проверку обновлений
    checkUpdates = true
    
    // Настройки анализа совместимости
    analyzeCompatibility {
        enabled = true
        targetJdk = 17
        failOnErrors = true
    }
}
----

== Gradle Tasks

=== Основные задачи

[cols="1,2,1",options="header"]
|===
| Задача | Описание | Зависимости

| `dependangerGenerateMetadata`
| DSL → metadata.json
| —

| `dependangerGenerateEffective`
| metadata.json → effective-*.json
| `generateMetadata`

| `dependangerGenerateToml`
| effective → TOML catalog
| `generateEffective`

| `dependangerGenerateBom`
| effective → Maven BOM
| `generateEffective`

| `dependangerGenerate`
| Полный пайплайн генерации
| `generateToml`, `generateBom`
|===

=== Задачи анализа

[cols="1,2,1",options="header"]
|===
| Задача | Описание | Зависимости

| `dependangerValidate`
| Валидация DSL конфигурации
| `generateMetadata`

| `dependangerCheckUpdates`
| Проверка доступных обновлений
| `generateEffective`

| `dependangerAnalyze`
| Анализ совместимости
| `generateEffective`

| `dependangerReport`
| Генерация unified JSON отчёта
| `checkUpdates`, `analyze`

| `dependangerListVersions`
| Вывод всех версий в консоль
| `generateEffective`
|===

=== Примеры использования

[source,bash]
----
# Полная генерация
./gradlew dependangerGenerate

# Только TOML
./gradlew dependangerGenerateToml

# Валидация
./gradlew dependangerValidate

# Проверка обновлений
./gradlew dependangerCheckUpdates

# Генерация отчёта
./gradlew dependangerReport
----

[cols="1,1,2,1",options="header"]

== Интеграция с Version Catalogs

Сгенерированный TOML файл можно подключить как Version Catalog:

[source,kotlin]
----
// settings.gradle.kts
dependencyResolutionManagement {
    versionCatalogs {
        create("libs") {
            from(files("build/dependanger/libs.versions.toml"))
        }
    }
}
----

Или использовать как source of truth для генерации:

[source,kotlin]
----
// build.gradle.kts
tasks.named("generateToml") {
    doLast {
        copy {
            from(layout.buildDirectory.file("dependanger/libs.versions.toml"))
            into(rootProject.file("gradle"))
        }
    }
}
----

== Кэширование

Все задачи Dependanger поддерживают Gradle Build Cache:

* `generateMetadata` — кэшируется по DSL конфигурации
* `generateEffective` — кэшируется по metadata.json
* `generateToml`/`generateBom` — кэшируются по effective-*.json
