[[overview]]
== Обзор

Модуль `dependanger-analysis` проверяет совместимость библиотек на основе правил, определённых в блоке `compatibility { }` DSL.
Поддерживает четыре типа правил: `jdkRequirement`, `mutualExclusion`, `versionConstraint`, `customRule`.
Результаты сохраняются в `EffectiveMetadata.extensions` через `CompatibilityIssuesExtensionKey`.

Приоритет: **P3** (желательно).

[[spec-compatibility-check]]
== SPEC-08.1: CompatibilityCheckProcessor.process()

[[purpose-compat]]
=== Назначение

Выполнить расширенный анализ совместимости, который дополняет базовый `compat-rules` процессор (order=66).
Включает: pattern matching по координатам, проверку constraints SAME_VERSION/SAME_MAJOR/SAME_MAJOR_MINOR, расширенный JDK анализ, и вызов customRule handlers.

[[algorithm-compat]]
=== Алгоритм

[source,text]
----
1. Получить правила совместимости из context.originalMetadata.compatibility
2. Получить настройки из context.settings.compatibilityAnalysis
3. Получить targetJdk:
      i.   `context.environment.jdkVersion` (explicit) -- приоритет
      ii.  `Runtime.version().feature()` (auto-detect)
      iii. `null` -> пропуск JDK-проверок с WARNING
   Получить targetKotlin:
      i.   `context.environment.kotlinVersion` (explicit)
      ii.  `null` -> пропуск Kotlin-проверок с WARNING
4. Если правил нет -> вернуть metadata, INFO "No compatibility rules defined"
5. Для каждого правила:
   a. JdkRequirement:
      i.   Найти все библиотеки matching `matches` glob pattern
      ii.  Если targetJdk < minJdk -> CompatibilityIssue(severity, message, affectedLibraries)
      iii. Если targetJdk > maxJdk (если задан) -> аналогично
   b. MutualExclusion:
      i.   Проверить наличие библиотек из списка в effective metadata
      ii.  Если 2+ присутствуют -> CompatibilityIssue(affectedLibraries = присутствующие)
   c. VersionConstraint:
      i.   Получить версии всех указанных библиотек
      ii.  Проверить constraint:
           - SAME_VERSION: все версии идентичны
           - SAME_MAJOR: все major совпадают
           - SAME_MAJOR_MINOR: все major.minor совпадают
      iii. При несовпадении -> CompatibilityIssue
   d. CustomRule:
      i.   Найти handler по ruleId через `CustomRuleHandler` SPI из xref:SPEC-01-dependanger-core.adoc#core-utilities[SPEC-01]. ServiceLoader: `META-INF/services/io.github.zenhelix.dependanger.core.spi.CustomRuleHandler`
      ii.  Вызвать `handler.evaluate(rule, libraries, context)` в try-catch
      iii. Обработать результат -> CompatibilityIssue
6. Собрать все CompatibilityIssue
7. Записать в metadata через withExtension(CompatibilityIssuesExtensionKey, issues)
8. Добавить diagnostics: INFO/WARNING/ERROR в зависимости от severity
9. В strict mode при severity=ERROR -> отметить в diagnostics как ERROR
----

=== Детальная реализация

==== Pattern Matching (Glob)

NOTE: `GlobMatcher` определён в `dependanger-core` (см. xref:SPEC-01-dependanger-core.adoc#core-utilities[SPEC-01: Общие утилиты]).
Модуль `dependanger-analysis` использует каноническую реализацию из `io.github.zenhelix.dependanger.core.util.GlobMatcher`.

Используемые методы:

* `GlobMatcher.matches(pattern, group, artifact)` — проверка координат "group:artifact" против glob-паттерна
* `GlobMatcher.matchesGlob(pattern, value)` — проверка строки против glob-паттерна

Поддерживаемые паттерны:

* `"org.springframework:*"` — все артефакты группы org.springframework
* `"org.springframework*:*"` — org.springframework И org.springframework.boot
* `"*:spring-*"` — все группы, артефакты начинающиеся с "spring-"

==== Version Constraint проверка

NOTE: Используется `VersionComparator` из `dependanger-core` (см. xref:SPEC-01-dependanger-core.adoc#core-utilities[SPEC-01: Общие утилиты]).

[source,kotlin]
----
private fun checkVersionConstraint(
    rule: CompatibilityRule.VersionConstraint,
    libraries: Map<String, EffectiveLibrary>
): CompatibilityIssue? {
    val involvedLibs = rule.libraries.mapNotNull { alias -> libraries[alias] }
    if (involvedLibs.size < 2) return null

    val versions = involvedLibs.mapNotNull { it.version?.value }
    if (versions.size < 2) return null

    val parsed = versions.map { VersionComparator.parse(it) ?: return null }

    val compatible = when (rule.constraint) {
        ConstraintType.SAME_VERSION -> parsed.distinct().size == 1
        ConstraintType.SAME_MAJOR -> parsed.map { it.major }.distinct().size == 1
        ConstraintType.SAME_MAJOR_MINOR ->
            parsed.map { it.major to it.minor }.distinct().size == 1
    }

    return if (!compatible) {
        CompatibilityIssue(
            ruleId = rule.id,
            message = rule.message ?: "Version constraint violated: ${rule.constraint}",
            severity = rule.severity,
            affectedLibraries = involvedLibs.map { it.alias },
            suggestion = "Align versions of: ${involvedLibs.joinToString { "${it.alias}:${it.version?.value}" }}"
        )
    } else null
}
----

==== JDK Requirement проверка

[source,kotlin]
----
private fun checkJdkRequirement(
    rule: CompatibilityRule.JdkRequirement,
    libraries: Map<String, EffectiveLibrary>,
    targetJdk: Int?
): CompatibilityIssue? {
    if (targetJdk == null) {
        logger.warn { "targetJdk not specified, skipping jdkRequirement rule '${rule.id}'" }
        return null
    }

    val matchingLibs = libraries.values.filter { lib ->
        GlobMatcher.matches(rule.matches, lib.group, lib.artifact)
    }

    if (matchingLibs.isEmpty()) return null

    val violated = when {
        rule.minJdk != null && targetJdk < rule.minJdk -> true
        rule.maxJdk != null && targetJdk > rule.maxJdk -> true
        else -> false
    }

    return if (violated) {
        CompatibilityIssue(
            ruleId = rule.id,
            message = rule.message ?: "Requires JDK ${rule.minJdk}+, but target is $targetJdk",
            severity = rule.severity,
            affectedLibraries = matchingLibs.map { it.alias },
            suggestion = "Upgrade to JDK ${rule.minJdk} or use compatible library versions"
        )
    } else null
}
----

==== Mutual Exclusion проверка

[source,kotlin]
----
private fun checkMutualExclusion(
    rule: CompatibilityRule.MutualExclusion,
    libraries: Map<String, EffectiveLibrary>
): CompatibilityIssue? {
    val present = rule.libraries.filter { it in libraries }
    return if (present.size >= 2) {
        CompatibilityIssue(
            ruleId = rule.id,
            message = rule.message ?: "Mutually exclusive libraries found: ${present.joinToString()}",
            severity = rule.severity,
            affectedLibraries = present,
            suggestion = "Remove one of: ${present.joinToString()}"
        )
    } else null
}
----

[[edge-cases]]
== Edge Cases и обработка ошибок

[cols="1,2,2"]
|===
| Ситуация | Поведение | Диагностический код

| Нет правил совместимости | Процессор пропускается, INFO | `COMPAT_NO_RULES`
| Все правила проходят | Пустой список issues, INFO | `COMPAT_ALL_PASSED`
| targetJdk не указан | jdkRequirement правила пропускаются с WARNING | `COMPAT_NO_TARGET_JDK`
| Невалидный glob pattern | ERROR для конкретного правила | `COMPAT_INVALID_PATTERN`
| Exception в customRule | ERROR с message, pipeline продолжается | `COMPAT_CUSTOM_RULE_FAILED`
| Дубликат ruleId | ERROR | `COMPAT_DUPLICATE_RULE`
| Неизвестная библиотека в правиле | WARNING | `COMPAT_UNKNOWN_LIB`
| Одна библиотека в mutualExclusion | WARNING "rule has only one library" | `COMPAT_SINGLE_LIB_RULE`
| Версии с разным форматом | Нормализация перед сравнением | --
| Strict mode + ERROR severity | ERROR в diagnostics | --
|===
