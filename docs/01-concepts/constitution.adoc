== Как изменить конституцию

[WARNING]
====
Изменение конституции — серьезный шаг.
Процесс:

1. Создать ADR с обоснованием изменения
2. Провести ревью всеми участниками проекта
3. Достичь консенсуса (или большинства голосов)
4. Обновить конституцию и связанные fitness functions
====

== Модульность

=== M1: Направление зависимостей

[IMPORTANT]
====
Зависимости между модулями ДОЛЖНЫ быть направлены от периферии к ядру.

```
interfaces → api → features → core
     ↓        ↓        ↓
   (CLI)   (facade) (processors)
```
====

**Запрещено:**

* [ ] Core модуль НЕ МОЖЕТ зависеть от api, features или interfaces
* [ ] API модуль НЕ МОЖЕТ зависеть от конкретных интерфейсов (CLI, Gradle Plugin)
* [ ] Циклические зависимости между модулями

**Проверка:** `ArchUnit: LayeredArchitectureTest`

---

=== M2: Изоляция доменной модели

[IMPORTANT]
====
Доменная модель (core) НЕ ДОЛЖНА содержать зависимостей от инфраструктуры.
====

**Запрещено в модуле core:**

* [ ] Зависимости от HTTP клиентов
* [ ] Зависимости от файловой системы (кроме абстракций)
* [ ] Зависимости от конкретных форматов сериализации
* [ ] Аннотации фреймворков (Spring, Ktor и т.д.)

**Разрешено:**

* [x] Kotlin stdlib
* [x] kotlinx-serialization (только аннотации `@Serializable`)
* [x] Собственные абстракции и интерфейсы

**Проверка:** `ArchUnit: DomainModelIsolationTest`

---

=== M3: Граница модуля = публичный API

[IMPORTANT]
====
Каждый модуль ДОЛЖЕН иметь явно определенный публичный API.
Внутренние классы ДОЛЖНЫ быть в пакете `internal`.
====

**Структура пакетов:**

```
io.github.zenhelix.dependanger.{module}/
├── api/           # Публичные интерфейсы
├── impl/          # Реализации (internal visibility)
├── internal/      # Внутренние классы
└── spi/           # Service Provider Interface (для расширений)
```

**Проверка:** `ArchUnit: PublicApiSurfaceTest`

== Расширяемость

=== E1: Точки расширения через SPI

[IMPORTANT]
====
Расширяемость ДОЛЖНА быть реализована через Service Provider Interface, а не через наследование конкретных классов.
====

**Точки расширения:**

[cols="1,2,1",options="header"]
|===
| Компонент | SPI интерфейс | Примеры реализаций

| Генераторы
| `ArtifactGenerator<T>`
| TomlGenerator, BomGenerator

| Процессоры
| `EffectiveMetadataProcessor`
| VersionResolver, BomImporter

| Фильтры
| `LibraryFilter`
| TagFilter, BundleFilter

| Резолверы
| `VersionResolver`
| MavenResolver, CustomResolver
|===

**Запрещено:**

* [ ] Наследование от конкретных классов для добавления функциональности
* [ ] Жестко закодированные списки реализаций

**Проверка:** `ArchUnit: ExtensionPointsTest`

---

=== E2: Открыт для расширения, закрыт для модификации

[IMPORTANT]
====
Добавление нового генератора/процессора НЕ ДОЛЖНО требовать изменения существующего кода в api или core модулях.
====

**Способ добавления нового функционала:**

. Создать новый модуль
. Реализовать SPI интерфейс
. Зарегистрировать через ServiceLoader или явную конфигурацию

**Проверка:** Ревью PR на отсутствие изменений в core для новых features

== Тестируемость

=== T1: Инъекция зависимостей

[IMPORTANT]
====
Все зависимости ДОЛЖНЫ быть инъектируемы через конструктор.
Запрещены статические зависимости и синглтоны.
====

**Паттерн:**

[source,kotlin]
----
// ✅ Правильно
class VersionChecker(
    private val repository: MavenRepository,
    private val cache: VersionCache
)

// ❌ Запрещено
class VersionChecker {
    private val repository = MavenCentralRepository.INSTANCE
}
----

**Проверка:** `ArchUnit: DependencyInjectionTest`

---

=== T2: Абстракции для внешних систем

[IMPORTANT]
====
Взаимодействие с внешними системами (сеть, файлы, время) ДОЛЖНО быть через абстракции, позволяющие замену в тестах.
====

**Обязательные абстракции:**

[cols="1,2",options="header"]
|===
| Внешняя система | Абстракция

| HTTP запросы
| `MavenRepository` interface

| Файловая система
| `FileSystem` interface (или Java NIO)

| Текущее время
| `Clock` interface

| Логирование
| SLF4J API (не конкретная реализация)
|===

**Проверка:** `ArchUnit: ExternalSystemAbstractionsTest`

---

=== T3: Пирамида тестирования

[IMPORTANT]
====
Проект ДОЛЖЕН следовать пирамиде тестирования:

* Unit тесты: 70%+ покрытия
* Integration тесты: ключевые сценарии
* E2E тесты: критические пути пользователя
====

**Структура тестов:**

```
src/test/kotlin/
├── unit/           # Быстрые, изолированные
├── integration/    # С реальными зависимостями
└── functional/     # E2E сценарии
```

**Проверка:** `JaCoCo coverage > 70%`, CI pipeline

== Код и стиль

=== C1: Явный API (Explicit API Mode)

[IMPORTANT]
====
Все модули ДОЛЖНЫ использовать Kotlin Explicit API mode.
Видимость каждого публичного элемента ДОЛЖНА быть явно указана.
====

[source,kotlin]
----
// build.gradle.kts
kotlin {
    explicitApi()
}
----

**Проверка:** Компиляция в CI

---

=== C2: Иммутабельность по умолчанию

[IMPORTANT]
====
Доменные модели ДОЛЖНЫ быть иммутабельными (`data class`, `val`).
Мутабельное состояние разрешено только в builder'ах и тестах.
====

**Паттерн:**

[source,kotlin]
----
// ✅ Иммутабельная модель
data class Library(
    val alias: String,
    val coordinates: MavenCoordinates,
    val version: Version
)

// ✅ Мутабельный builder
class LibraryBuilder {
    var alias: String = ""
    // ...
    fun build(): Library = Library(...)
}
----

**Проверка:** `Detekt: MutableDataClass`

== Запрещенные практики

=== X1: Глобальный mutable state

[IMPORTANT]
====
ЗАПРЕЩЕНО использование глобального мутабельного состояния.
====

**Запрещено:**

[source,kotlin]
----
// ❌ Глобальный mutable state
object GlobalConfig {
    var debugMode = false  // ЗАПРЕЩЕНО
}

// ❌ Статический mutable cache
companion object {
    private val cache = mutableMapOf<String, Any>()  // ЗАПРЕЩЕНО
}
----

---

=== X2: Блокирующий I/O без маркировки

[IMPORTANT]
====
Блокирующий I/O ДОЛЖЕН быть явно помечен и выполняться в соответствующем диспетчере.
====

[source,kotlin]
----
// ✅ Явно помечено
suspend fun fetchVersions(): List<Version> = withContext(Dispatchers.IO) {
    httpClient.get(...)
}

// ❌ Запрещено
fun fetchVersions(): List<Version> {
    return httpClient.get(...)  // Блокирующий вызов без контекста
}
----

---

=== X3: Жестко закодированные пути и URL

[IMPORTANT]
====
ЗАПРЕЩЕНО хардкодить пути к файлам, URL репозиториев, credentials.
Все ДОЛЖНО быть конфигурируемо.
====

== Чеклист для ревью

При каждом PR проверять:

* [ ] Направление зависимостей соблюдено (M1)
* [ ] Доменная модель не загрязнена инфраструктурой (M2)
* [ ] Новые публичные API осознанны (M3)
* [ ] Расширения через SPI (E1)
* [ ] Зависимости инъектируются (T1)
* [ ] Внешние системы за абстракциями (T2)
* [ ] Есть тесты (T3)
* [ ] Explicit API соблюден (C1)
* [ ] Модели иммутабельны (C2)
* [ ] Нет глобального состояния (X1)

== Примеры нарушений и исправлений

=== M1: Нарушение направления зависимостей

[source,kotlin]
----
// ❌ НАРУШЕНИЕ: core зависит от cli
// core/build.gradle.kts
dependencies {
    implementation(project(":cli"))  // ЗАПРЕЩЕНО
}

// ✅ ПРАВИЛЬНО: cli зависит от core
// cli/build.gradle.kts
dependencies {
    implementation(project(":core"))
}
----

=== E1: Нарушение SPI

[source,kotlin]
----
// ❌ НАРУШЕНИЕ: наследование от конкретного класса
class CustomTomlGenerator : EffectiveTomlGenerator() {
    override fun generate(...) { ... }
}

// ✅ ПРАВИЛЬНО: реализация интерфейса
class CustomTomlGenerator : ArtifactGenerator {
    override fun generate(...) { ... }
}
----

=== X1: Глобальное состояние

[source,kotlin]
----
// ❌ НАРУШЕНИЕ: mutable state в object
object VersionCache {
    var enabled = true  // ЗАПРЕЩЕНО
    private val cache = mutableMapOf<String, Version>()  // ЗАПРЕЩЕНО
}

// ✅ ПРАВИЛЬНО: инъекция через конструктор
class VersionCache(
    private val enabled: Boolean,
    private val storage: CacheStorage
)
----
