:toc: left
:toclevels: 3

[abstract]
--
Генерация Maven BOM (Bill of Materials) из EffectiveMetadata.
Альтернативный выходной формат для Maven-экосистемы.
Артефакт публикуется для использования в других Maven/Gradle-проектах организации.
--

== Описание

BOM Generation преобразует `EffectiveMetadata` в Maven BOM — POM-файл для централизованного управления версиями в Maven и Gradle (через platform).

== User Stories

=== US-001: Генерация Maven BOM

[quote]
____
Как Maven разработчик, я хочу получить BOM из моего каталога зависимостей, чтобы использовать централизованное управление версиями в Maven проектах.
____

**Acceptance Criteria:**

[cols="1,3,3,3",options="header"]
|===
| ID | Given | When | Then

| AC-001-1
| EffectiveMetadata с 5 библиотеками (версии резолвлены)
| Вызываю `BomGenerator.generate(metadata)`
| Создаётся POM с `<dependencyManagement>` содержащим 5 `<dependency>`

| AC-001-2
| Настройка `bom { groupId = "com.example"; artifactId = "my-bom"; version = "1.0.0" }`
| Генерирую BOM
| POM содержит указанные координаты в корневом элементе

| AC-001-3
| Сгенерированный BOM файл
| Валидирую через `mvn validate`
| Maven успешно парсит POM, ошибок валидации нет

| AC-001-4
| Gradle task `dependangerGenerateBom`
| Запускаю task
| Файл создан в `build/dependanger/bom.pom.xml`

| AC-001-5
| EffectiveMetadata с библиотекой `kotlin-stdlib:2.1.20`
| Генерирую BOM
| `<dependency>` содержит `<groupId>`, `<artifactId>`, `<version>` (не versionRef)
|===

== SPI: ArtifactGenerator

=== BomGenerator

`BomGenerator` реализует генерацию Maven BOM (Bill of Materials). Принимает `EffectiveMetadata` и создаёт POM файл с секцией `<dependencyManagement>` содержащей все библиотеки с версиями.

**Параметры:**

* `groupId` — Maven groupId для генерируемого BOM
* `artifactId` — Maven artifactId для генерируемого BOM
* `version` — версия BOM

== Формат BOM

=== Структура

[source,xml]
----
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 
                             http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>
    
    <groupId>com.example</groupId>
    <artifactId>dependanger-bom</artifactId>
    <version>1.0.0</version>
    <packaging>pom</packaging>
    
    <name>Dependanger BOM</name>
    <description>Generated by Dependanger</description>
    
    <dependencyManagement>
        <dependencies>
            <dependency>
                <groupId>org.jetbrains.kotlin</groupId>
                <artifactId>kotlin-stdlib</artifactId>
                <version>2.1.0</version>
            </dependency>
            <dependency>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-starter</artifactId>
                <version>3.2.0</version>
            </dependency>

            <!-- DEPRECATED: Use new-http which supports HTTP/2 -->
            <!-- Replacement: new-http -->
            <!-- Will be removed in: 2.0.0 -->
            <dependency>
                <groupId>com.example</groupId>
                <artifactId>old-http</artifactId>
                <version>1.0.0</version>
            </dependency>
            <!-- ... остальные зависимости ... -->
        </dependencies>
    </dependencyManagement>
</project>
----

== Gradle Task

=== dependangerGenerateBom

Задача `dependangerGenerateBom` генерирует Maven BOM из EffectiveMetadata.

**Зависимости:**

* `dependangerGenerateEffective` — требуется для получения EffectiveMetadata

**Inputs:**

* `effective-\{distribution\}.json` от `dependangerGenerateEffective`

**Outputs:**

* `build/dependanger/bom.pom.xml` (или `build/dependanger/bom-\{distribution\}.pom.xml` для конкретного профиля)

**Опции:**

* `bomGroupId`, `bomArtifactId`, `bomVersion` — координаты генерируемого BOM
* `includeName`, `includeDescription` — добавлять name/description в POM
* `distribution` — профиль для генерации (опционально)

== CLI Command

=== dependanger generate --bom

Команда генерации BOM через CLI.

[source,bash]
----
dependanger generate --bom [options] <input>
----

**Аргументы:**

[cols="1,2,1",options="header"]
|===
| Аргумент | Описание | Default

| `<input>`
| Путь к `effective-*.json` файлу
| `effective-metadata.json`
|===

**Опции:**

[cols="1,2,1",options="header"]
|===
| Опция | Описание | Default

| `--output`, `-o`
| Путь для выходного файла
| `./bom.pom.xml`

| `--group-id`
| Maven groupId для BOM
| (обязательный)

| `--artifact-id`
| Maven artifactId для BOM
| (обязательный)

| `--version`
| Версия BOM
| (обязательный)

| `--name`
| Название для `<name>` элемента
| —

| `--description`
| Описание для `<description>` элемента
| "Generated by Dependanger"

| `--distribution`, `-d`
| Профиль для генерации
| —

| `--sort`
| Сортировка зависимостей: `alpha`, `none`
| `alpha`

| `--include-deprecated`
| Включать deprecated библиотеки с комментариями
| `true`

| `--format`
| Форматирование XML: `pretty`, `compact`
| `pretty`
|===

**Примеры:**

[source,bash]
----
# Базовая генерация
dependanger generate --bom \
  --group-id com.example \
  --artifact-id my-bom \
  --version 1.0.0 \
  effective-metadata.json

# Генерация для профиля
dependanger generate --bom \
  --group-id com.example \
  --artifact-id my-bom-backend \
  --version 1.0.0 \
  --distribution backend \
  effective-backend.json

# С кастомным выводом
dependanger generate --bom \
  --group-id com.example \
  --artifact-id my-bom \
  --version 1.0.0 \
  --output /path/to/bom.pom.xml \
  --name "My Project BOM" \
  effective-metadata.json
----

== Settings DSL

Полная конфигурация BOM генерации в Gradle DSL.

[source,kotlin]
----
dependanger {
    settings {
        bom {
            // Координаты (обязательные)
            groupId = "com.example"
            artifactId = "dependanger-bom"
            version = project.version.toString()

            // Метаданные (опциональные)
            name = "My Project BOM"
            description = "Centralized dependency versions"

            // Сортировка зависимостей
            sortDependencies = true  // default: true

            // Deprecated библиотеки
            includeDeprecated = true  // default: true
            includeDeprecatedComments = true  // default: true

            // Форматирование
            prettyPrint = true  // default: true
            indent = "    "  // default: 4 spaces

            // Выходной файл
            filename = "bom.pom.xml"  // default

            // Placeholder support (для distribution-aware artifactId)
            // {distribution} → имя профиля
            artifactId = "my-bom-{distribution}"
        }
    }
}

== Использование

=== В Maven

[source,xml]
----
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>com.example</groupId>
            <artifactId>dependanger-bom</artifactId>
            <version>1.0.0</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>

<dependencies>
    <dependency>
        <groupId>org.jetbrains.kotlin</groupId>
        <artifactId>kotlin-stdlib</artifactId>
        <!-- версия из BOM -->
    </dependency>
</dependencies>
----

=== В Gradle

[source,kotlin]
----
dependencies {
    // Импорт BOM как platform
    implementation(platform("com.example:dependanger-bom:1.0.0"))
    
    // Версии автоматически из BOM
    implementation("org.jetbrains.kotlin:kotlin-stdlib")
    implementation("org.springframework.boot:spring-boot-starter")
}
----

== Публикация BOM

=== Gradle Publishing

[source,kotlin]
----
publishing {
    publications {
        create<MavenPublication>("bom") {
            // Используем сгенерированный POM
            pom.withXml {
                val bomFile = tasks.named("dependangerGenerateBom").get().outputFile.get().asFile
                asNode().replaceNode(XmlParser().parse(bomFile))
            }
            
            groupId = "com.example"
            artifactId = "dependanger-bom"
            version = project.version.toString()
        }
    }
}
----

| SHOULD

| FR-008

| Система ДОЛЖНА сортировать зависимости алфавитно по groupId:artifactId

== Edge Cases

[cols="1,3,2",options="header"]
|===
| Сценарий | Описание | Ожидаемое поведение

| Пустой каталог
| EffectiveMetadata без библиотек
| Генерируется BOM с пустым `<dependencyManagement>` (валидный POM)

| Библиотека без версии
| Библиотека с `version = null` после pipeline
| ERROR: "Library 'xxx' has no resolved version for BOM generation"

| Невалидный groupId
| groupId содержит недопустимые символы: `com.example!`
| ERROR: "Invalid Maven groupId 'com.example!'"

| Невалидный artifactId
| artifactId содержит пробелы: `my lib`
| ERROR: "Invalid Maven artifactId 'my lib'"

| Дубликаты координат
| Две библиотеки с одинаковыми group:artifact но разными alias
| ERROR: "Duplicate coordinates 'com.example:lib' from aliases 'lib1' and 'lib2'"

| Очень длинная версия
| Версия длиной 200+ символов
| WARNING, генерация продолжается

| Специальные символы в версии
| Версия `1.0.0-SNAPSHOT+build.123`
| Корректная сериализация в XML: `<version>1.0.0-SNAPSHOT+build.123</version>`

| XML-unsafe символы
| Description содержит `<`, `>`, `&`
| Корректное экранирование: `&lt;`, `&gt;`, `&amp;`

| Плагины в metadata
| EffectiveMetadata содержит плагины
| Плагины игнорируются (BOM не поддерживает plugins)

| Бандлы в metadata
| EffectiveMetadata содержит бандлы
| Бандлы игнорируются (BOM не поддерживает bundles)

| Нет координат BOM
| Не указаны groupId/artifactId/version для BOM
| ERROR: "BOM coordinates not configured. Set bom { groupId, artifactId, version }"

| Файл уже существует
| `bom.pom.xml` уже есть
| Перезаписывается

| Нет прав на запись
| Директория read-only
| ERROR: "Cannot write to [path]: Permission denied"

| Maven < 3.0
| Пользователь использует Maven 2.x
| WARNING: "BOM import requires Maven 3.0+", генерация продолжается

| Classifier в библиотеке
| Библиотека с classifier (например, `sources`, `javadoc`)
| Включается `<classifier>` в `<dependency>`

| Deprecated библиотека
| Библиотека помечена как deprecated
| Добавляется XML комментарий перед `<dependency>`: `<!-- DEPRECATED: ... -->`

| Deprecated с XML-unsafe символами в message
| `message` содержит `<`, `>`, `&`
| Символы экранируются в комментарии
|===
