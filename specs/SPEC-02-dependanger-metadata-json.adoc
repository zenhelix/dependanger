[[overview]]
== Обзор

Модуль `dependanger-metadata-json` отвечает за сериализацию/десериализацию `DependangerMetadata` в формат JSON через `kotlinx-serialization`.
Он является реализацией SPI интерфейса `SerializationFormat<String>`, определённого в `dependanger-core`.

Модуль содержит **один файл** с **четырьмя TODO-заглушками**:

* `serialize(metadata: DependangerMetadata): String`
* `deserialize(input: String): DependangerMetadata`
* `write(metadata: DependangerMetadata, path: Path)`
* `read(path: Path): DependangerMetadata`

Модуль критически важен для:

* Хранения конфигурации между запусками (Gradle task `dependangerGenerateMetadata`)
* Передачи данных между этапами Processing Pipeline
* CLI команд (`init`, `add-library`, `remove-library`, `update-version`)
* Кеширования для инкрементальной обработки

[[spec-serialize]]
== SPEC-02.1: serialize()

[[purpose-serialize]]
=== Назначение

Преобразовать объект `DependangerMetadata` в JSON-строку с pretty-print форматированием.
Результат должен быть валидным JSON, пригодным для записи в файл и обратной десериализации без потерь.

[[algorithm-serialize]]
=== Алгоритм

[source,text]
----
1. Принять metadata: DependangerMetadata
2. Использовать сконфигурированный Json инстанс (prettyPrint, ignoreUnknownKeys, encodeDefaults)
3. Вызвать Json.encodeToString(DependangerMetadata.serializer(), metadata)
4. Вернуть результирующую строку
----

[[implementation-serialize]]
=== Детальная реализация

[source,kotlin]
----
override fun serialize(metadata: DependangerMetadata): String =
    json.encodeToString(DependangerMetadata.serializer(), metadata)
----

=== Поведение Json-конфигурации

[cols="1,3",options="header"]
|===
| Параметр | Влияние на serialize()

| `prettyPrint = true`
| JSON форматируется с отступами и переносами строк для человекочитаемости

| `encodeDefaults = true`
| Все поля с default-значениями включаются в JSON (пустые списки, default Settings, и т.д.)

| `ignoreUnknownKeys = true`
| Не влияет на сериализацию (только на десериализацию)
|===

=== Пример вывода

[source,json]
----
{
    "schemaVersion": "1.0",
    "versions": [
        {
            "name": "kotlin",
            "value": "2.1.20",
            "fallbacks": [
                {
                    "value": "1.9.25",
                    "condition": {
                        "type": "io.github.zenhelix.dependanger.core.model.FallbackCondition.JdkBelow",
                        "version": 17
                    }
                }
            ]
        }
    ],
    "libraries": [
        {
            "alias": "kotlin-stdlib",
            "group": "org.jetbrains.kotlin",
            "artifact": "kotlin-stdlib",
            "version": {
                "type": "io.github.zenhelix.dependanger.core.model.VersionReference.Reference",
                "name": "kotlin"
            },
            "description": null,
            "tags": ["kotlin", "stdlib"],
            "requires": null,
            "deprecation": null,
            "license": null,
            "constraints": [],
            "isPlatform": false
        }
    ],
    "plugins": [],
    "bundles": [],
    "bomImports": [],
    "targetPlatforms": [],
    "distributions": [],
    "compatibility": [],
    "settings": {
        "defaultDistribution": null,
        "strictVersionResolution": false,
        "validation": { ... },
        ...
    },
    "presets": [],
    "extensions": {}
}
----

NOTE: Точный формат дискриминатора sealed-классов (`type`) зависит от стратегии `classDiscriminator` в `kotlinx-serialization`.
По умолчанию используется полное квалифицированное имя класса.
При необходимости можно настроить через `@SerialName` аннотации на подтипах (рекомендуется для стабильного API).

=== Edge cases (serialize)

[cols="1,3",options="header"]
|===
| Ситуация | Поведение

| Пустой `DependangerMetadata()`
| Валидный JSON со всеми default-значениями (пустые списки, default Settings)

| Metadata с Unicode и спецсимволами
| `kotlinx-serialization` автоматически экранирует в JSON-формате

| Metadata с `extensions` содержащими вложенные `JsonElement`
| Сериализуется корректно, `JsonElement` сериализуется как есть

| Очень большой metadata (10000+ библиотек)
| Работает корректно, результирующая строка может быть большой (мегабайты)
|===

[[spec-deserialize]]
== SPEC-02.2: deserialize()

[[purpose-deserialize]]
=== Назначение

Преобразовать JSON-строку обратно в объект `DependangerMetadata`.
Обеспечить корректную обработку ошибок при невалидном вводе.

[[algorithm-deserialize]]
=== Алгоритм

[source,text]
----
1. Принять input: String (JSON-строка)
2. Валидировать, что input не пустой
3. Вызвать Json.decodeFromString(DependangerMetadata.serializer(), input)
4. В случае ошибки парсинга — обернуть в понятное исключение
5. Вернуть результирующий DependangerMetadata
----

[[implementation-deserialize]]
=== Детальная реализация

[source,kotlin]
----
override fun deserialize(input: String): DependangerMetadata {
    require(input.isNotBlank()) {
        "Input JSON string must not be blank"
    }

    return try {
        json.decodeFromString(DependangerMetadata.serializer(), input)
    } catch (e: kotlinx.serialization.SerializationException) {
        throw MetadataDeserializationException(
            "Failed to deserialize DependangerMetadata from JSON: ${e.message}",
            e
        )
    } catch (e: IllegalArgumentException) {
        throw MetadataDeserializationException(
            "Invalid JSON structure for DependangerMetadata: ${e.message}",
            e
        )
    }
}
----

=== Класс исключения

Необходимо создать класс `MetadataDeserializationException` для типизированной обработки ошибок десериализации:

[source,kotlin]
----
public class MetadataDeserializationException(
    message: String,
    cause: Throwable? = null,
) : RuntimeException(message, cause)
----

=== Поведение Json-конфигурации при десериализации

[cols="1,3",options="header"]
|===
| Параметр | Влияние на deserialize()

| `ignoreUnknownKeys = true`
| Неизвестные поля в JSON тихо пропускаются (forward compatibility)

| `encodeDefaults = true`
| Не влияет на десериализацию

| `prettyPrint = true`
| Не влияет на десериализацию (парсит любой валидный JSON)
|===

=== Edge cases (deserialize)

[cols="1,3,2",options="header"]
|===
| Ситуация | Описание | Ожидаемое поведение

| Пустая строка `""`
| `input.isNotBlank()` = false
| `IllegalArgumentException: "Input JSON string must not be blank"`

| Строка с пробелами `"   "`
| `input.isNotBlank()` = false
| `IllegalArgumentException: "Input JSON string must not be blank"`

| Невалидный JSON `"{broken"`
| Синтаксическая ошибка JSON
| `MetadataDeserializationException` с деталями ошибки парсинга

| Отсутствует `schemaVersion`
| Поле имеет default `"1.0"` в data class
| Успешная десериализация с `schemaVersion = "1.0"` (default)

| Пустой JSON объект `{}`
| Все поля имеют default-значения
| Успешная десериализация: `DependangerMetadata()` с defaults

| Неизвестные поля `{"unknownField": "value", "schemaVersion": "1.0"}`
| `ignoreUnknownKeys = true`
| Неизвестное поле игнорируется, десериализация успешна

| Неверный тип поля `{"schemaVersion": 42}`
| Ожидается String, получен Int
| `MetadataDeserializationException`

| Неизвестный подтип sealed class
| `{"type": "UnknownSubtype", ...}` в versions/libraries
| `MetadataDeserializationException` с деталями о неизвестном подтипе

| Дублирующиеся ключи в JSON
| `{"versions": [...], "versions": [...]}`
| Поведение определяется JSON-парсером (обычно последний побеждает)

| UTF-8 с BOM
| Файл начинается с UTF-8 BOM (\uFEFF)
| `kotlinx-serialization` обрабатывает корректно

| `null` вместо списка `{"versions": null}`
| Ожидается `List<Version>`
| `MetadataDeserializationException` (non-nullable list с default не принимает null)
|===

[[spec-write]]
== SPEC-02.3: write()

[[purpose-write]]
=== Назначение

Сериализовать `DependangerMetadata` и записать результат в файл по указанному пути.
Создать родительские директории, если они не существуют.

[[algorithm-write]]
=== Алгоритм

[source,text]
----
1. Принять metadata: DependangerMetadata и path: Path
2. Сериализовать metadata в JSON-строку через serialize()
3. Создать родительские директории (path.parent), если не существуют
4. Записать JSON-строку в файл с кодировкой UTF-8
5. В случае ошибки I/O — обернуть в понятное исключение
----

[[implementation-write]]
=== Детальная реализация

[source,kotlin]
----
override fun write(metadata: DependangerMetadata, path: Path) {
    val jsonString = serialize(metadata)

    try {
        // Создать родительские директории, если не существуют
        path.parent?.let { parent ->
            if (!parent.exists()) {
                parent.createDirectories()
            }
        }

        // Записать в файл с UTF-8
        path.writeText(jsonString, Charsets.UTF_8)
    } catch (e: java.io.IOException) {
        throw MetadataWriteException(
            "Failed to write metadata to '$path': ${e.message}",
            e
        )
    } catch (e: SecurityException) {
        throw MetadataWriteException(
            "Permission denied writing metadata to '$path': ${e.message}",
            e
        )
    }
}
----

=== Класс исключения

[source,kotlin]
----
public class MetadataWriteException(
    message: String,
    cause: Throwable? = null,
) : RuntimeException(message, cause)
----

=== Требования к imports

[source,kotlin]
----
import kotlin.io.path.createDirectories
import kotlin.io.path.exists
import kotlin.io.path.writeText
----

=== Edge cases (write)

[cols="1,3,2",options="header"]
|===
| Ситуация | Описание | Ожидаемое поведение

| Родительская директория не существует
| `path = /tmp/a/b/c/metadata.json`, директории `a/b/c` нет
| Создаёт `/tmp/a/b/c/` и записывает файл

| Файл уже существует
| По `path` уже есть файл
| Файл перезаписывается (overwrite)

| Нет прав на запись
| Директория read-only
| `MetadataWriteException: "Permission denied writing metadata to '...'"`

| Диск полон
| `IOException` при записи
| `MetadataWriteException: "Failed to write metadata to '...'"`

| Путь указывает на директорию
| `path` -- это существующая директория, а не файл
| `MetadataWriteException` (IOException от writeText)

| Символическая ссылка
| `path` -- это symlink
| Запись по целевому пути symlink (стандартное поведение NIO)
|===

=== Атомарность записи

IMPORTANT: Текущая реализация использует прямую запись через `writeText`, что не является атомарной операцией.
Для production-ready решения рекомендуется:

[source,kotlin]
----
// Атомарная запись через temp-файл + rename
val tempFile = path.resolveSibling("${path.fileName}.tmp")
tempFile.writeText(jsonString, Charsets.UTF_8)
tempFile.moveTo(path, overwrite = true)  // атомарная операция на большинстве FS
----

Эта оптимизация может быть добавлена позже.
Для MVP прямая запись через `writeText` достаточна.

[[spec-read]]
== SPEC-02.4: read()

[[purpose-read]]
=== Назначение

Прочитать файл по указанному пути и десериализовать его содержимое в `DependangerMetadata`.

[[algorithm-read]]
=== Алгоритм

[source,text]
----
1. Принять path: Path
2. Проверить, что файл существует
3. Прочитать содержимое файла как String (UTF-8)
4. Десериализовать через deserialize()
5. В случае ошибки I/O — обернуть в понятное исключение
----

[[implementation-read]]
=== Детальная реализация

[source,kotlin]
----
override fun read(path: Path): DependangerMetadata {
    if (!path.exists()) {
        throw MetadataReadException(
            "Metadata file not found: '$path'"
        )
    }

    val content = try {
        path.readText(Charsets.UTF_8)
    } catch (e: java.io.IOException) {
        throw MetadataReadException(
            "Failed to read metadata file '$path': ${e.message}",
            e
        )
    } catch (e: SecurityException) {
        throw MetadataReadException(
            "Permission denied reading metadata file '$path': ${e.message}",
            e
        )
    }

    return try {
        deserialize(content)
    } catch (e: MetadataDeserializationException) {
        throw MetadataReadException(
            "Failed to parse metadata file '$path': ${e.message}",
            e
        )
    }
}
----

=== Класс исключения

[source,kotlin]
----

----

[source,kotlin]

=== Edge cases (read)

[cols="1,3,2",options="header"]
|===
| Ситуация | Описание | Ожидаемое поведение

| Файл не существует
| `path` указывает на несуществующий файл
| `MetadataReadException: "Metadata file not found: '...'"`

| Файл существует, но размер 0 байт

| Другой процесс пишет в этот файл
| Может прочитать частичный JSON -> `MetadataReadException`

| Очень большой файл (100+ МБ)
| Metadata с тысячами библиотек
| Работает, но может потребовать много памяти (String in-memory)

| Нет прав на чтение
| Файл read-protected
| `MetadataReadException: "Permission denied reading metadata file '...'"`

| Путь -- symlink на файл
| Символическая ссылка на metadata.json

[[spec-effective-json]]
== SPEC-02.5: EffectiveJsonFormat

=== Назначение

Сериализация/десериализация `EffectiveMetadata` в JSON.
Необходимо для:

* CLI — сохранение результатов `process` команды
* Gradle Plugin — задачи `dependangerGenerateEffective`, передача данных между тасками
* Report — включение effective metadata в отчёт
* Отладка и интроспекция pipeline

=== Проблема

`EffectiveMetadata` содержит поле `extensions: Map<ExtensionKey<*>, Any>` с аннотацией `@Transient`, что означает, что стандартная kotlinx-serialization его игнорирует.
Для полной сериализации необходим отдельный формат, который обрабатывает extensions вручную.

=== Архитектурное решение

`EffectiveJsonFormat` размещается в модуле `dependanger-effective` (не в `dependanger-metadata-json`), так как зависит от типов `EffectiveMetadata`, `ExtensionKey` и моделей extensions.

=== Модуль

`dependanger-effective` — добавить файл `EffectiveJsonFormat.kt` в пакет `io.github.zenhelix.dependanger.effective.serialization`.

=== Реализация

[source,kotlin]
----
package io.github.zenhelix.dependanger.effective.serialization

import io.github.zenhelix.dependanger.effective.model.*
import io.github.zenhelix.dependanger.effective.spi.ExtensionSerializerProvider
import kotlinx.serialization.json.*
import kotlinx.serialization.modules.SerializersModule
import kotlinx.serialization.modules.plus
import java.util.ServiceLoader

/**
 * Сериализация/десериализация EffectiveMetadata в JSON
 * с поддержкой extensions через ServiceLoader-based discovery.
 *
 * Feature-модули регистрируют свои ExtensionSerializerProvider через
 * META-INF/services/io.github.zenhelix.dependanger.effective.spi.ExtensionSerializerProvider
 */
public class EffectiveJsonFormat {

    public val formatId: String = "effective-json"
    public val fileExtension: String = ".effective.json"

    private val providers: List<ExtensionSerializerProvider> =
        ServiceLoader.load(ExtensionSerializerProvider::class.java).toList()

    private val combinedModule: SerializersModule =
        providers.fold(SerializersModule { }) { acc, provider -> acc + provider.serializersModule() }

    private val knownKeys: Map<String, ExtensionKey<*>> =
        providers.flatMap { it.knownKeys().entries }.associate { it.key to it.value }

    private val json: Json = Json {
        prettyPrint = true
        ignoreUnknownKeys = true
        encodeDefaults = true
        serializersModule = combinedModule
    }

    /**
     * Сериализует EffectiveMetadata в JSON-строку.
     * Extensions конвертируются через зарегистрированные провайдеры.
     */
    public fun serialize(metadata: EffectiveMetadata): String {
        val baseJson = json.encodeToJsonElement(EffectiveMetadata.serializer(), metadata)
            .jsonObject.toMutableMap()

        if (metadata.extensions.isNotEmpty()) {
            val extensionsJson = buildJsonObject {
                for ((key, value) in metadata.extensions) {
                    val serializer = combinedModule.getPolymorphic(Any::class, value)
                    if (serializer != null) {
                        @Suppress("UNCHECKED_CAST")
                        put(key.name, json.encodeToJsonElement(serializer as kotlinx.serialization.KSerializer<Any>, value))
                    } else {
                        logger.warn { "No serializer found for extension key '${key.name}', saving as JsonNull" }
                        put(key.name, JsonNull)
                    }
                }
            }
            baseJson["extensions"] = extensionsJson
        }

        return json.encodeToString(JsonElement.serializer(), JsonObject(baseJson))
    }

    /**
     * Десериализует JSON-строку в EffectiveMetadata.
     * Извлекает extensions через зарегистрированные провайдеры.
     */
    public fun deserialize(input: String): EffectiveMetadata {
        require(input.isNotBlank()) { "Input JSON string must not be blank" }

        val jsonObject = json.parseToJsonElement(input).jsonObject
        val base = json.decodeFromJsonElement(EffectiveMetadata.serializer(), jsonObject)

        val extensionsJson = jsonObject["extensions"]?.jsonObject ?: return base
        val extensions = mutableMapOf<ExtensionKey<*>, Any>()

        for ((keyName, element) in extensionsJson) {
            val extensionKey = knownKeys[keyName]
            if (extensionKey == null) {
                logger.warn { "Unknown extension key '$keyName', skipping" }
                continue
            }
            try {
                val serializer = combinedModule.getPolymorphic(Any::class, keyName)
                if (serializer != null) {
                    extensions[extensionKey] = json.decodeFromJsonElement(serializer, element)
                }
            } catch (e: Exception) {
                logger.warn { "Failed to deserialize extension '$keyName': ${e.message}" }
            }
        }

        return base.copy(extensions = extensions)
    }

    /**
     * Записывает EffectiveMetadata в файл.
     */
    public fun write(metadata: EffectiveMetadata, path: java.nio.file.Path) {
        val jsonString = serialize(metadata)
        path.parent?.let { parent ->
            if (!parent.toFile().exists()) parent.toFile().mkdirs()
        }
        path.toFile().writeText(jsonString, Charsets.UTF_8)
    }

    /**
     * Читает EffectiveMetadata из файла.
     */
    public fun read(path: java.nio.file.Path): EffectiveMetadata {
        if (!path.toFile().exists()) {
            throw IllegalArgumentException("Effective metadata file not found: '$path'")
        }
        val content = path.toFile().readText(Charsets.UTF_8)
        return deserialize(content)
    }
}

== Полная реализация файла

Ниже представлен полный код `JsonSerializationFormat.kt` после реализации всех TODO:

[source,kotlin]
----
package io.github.zenhelix.dependanger.metadata

import io.github.zenhelix.dependanger.core.model.metadata.DependangerMetadata
import io.github.zenhelix.dependanger.core.spi.SerializationFormat
import kotlinx.serialization.SerializationException
import kotlinx.serialization.json.Json
import java.io.IOException
import java.nio.file.Path
import kotlin.io.path.createDirectories
import kotlin.io.path.exists
import kotlin.io.path.readText
import kotlin.io.path.writeText

public class JsonSerializationFormat : SerializationFormat<String> {
    override val formatId: String = "json"
    override val fileExtension: String = ".json"
    override val description: String = "JSON format via kotlinx-serialization"

    private val json: Json = Json {
        prettyPrint = true
        ignoreUnknownKeys = true
        encodeDefaults = true
    }

    override fun serialize(metadata: DependangerMetadata): String =
        json.encodeToString(DependangerMetadata.serializer(), metadata)

    override fun deserialize(input: String): DependangerMetadata {
        require(input.isNotBlank()) {
            "Input JSON string must not be blank"
        }

        return try {
            json.decodeFromString(DependangerMetadata.serializer(), input)
        } catch (e: SerializationException) {
            throw MetadataDeserializationException(
                "Failed to deserialize DependangerMetadata from JSON: ${e.message}",
                e
            )
        } catch (e: IllegalArgumentException) {
            throw MetadataDeserializationException(
                "Invalid JSON structure for DependangerMetadata: ${e.message}",
                e
            )
        }
    }

    override fun write(metadata: DependangerMetadata, path: Path) {
        val jsonString = serialize(metadata)

        try {
            path.parent?.let { parent ->
                if (!parent.exists()) {
                    parent.createDirectories()
                }
            }

            path.writeText(jsonString, Charsets.UTF_8)
        } catch (e: IOException) {
            throw MetadataWriteException(
                "Failed to write metadata to '$path': ${e.message}",
                e
            )
        } catch (e: SecurityException) {
            throw MetadataWriteException(
                "Permission denied writing metadata to '$path': ${e.message}",
                e
            )
        }
    }

    override fun read(path: Path): DependangerMetadata {
        if (!path.exists()) {
            throw MetadataReadException(
                "Metadata file not found: '$path'"
            )
        }

        val content = try {
            path.readText(Charsets.UTF_8)
        } catch (e: IOException) {
            throw MetadataReadException(
                "Failed to read metadata file '$path': ${e.message}",
                e
            )
        } catch (e: SecurityException) {
            throw MetadataReadException(
                "Permission denied reading metadata file '$path': ${e.message}",
                e
            )
        }

        return try {
            deserialize(content)
        } catch (e: MetadataDeserializationException) {
            throw MetadataReadException(
                "Failed to parse metadata file '$path': ${e.message}",
                e
            )
        }
    }
}

[[round-trip-guarantee]]
== Гарантия Round-trip

=== Определение

Round-trip означает, что для любого валидного `DependangerMetadata`:

[source,text]
----
deserialize(serialize(metadata)) == metadata
----

А для файлового round-trip:

[source,text]
----
write(metadata, path)
read(path) == metadata
----

=== Что обеспечивает round-trip

1. **`encodeDefaults = true`** -- все поля с default-значениями записываются в JSON, не теряются при десериализации
2. **`@Serializable` на всех моделях** -- kotlinx-serialization генерирует корректные сериализаторы
3. **Sealed class discriminator** -- подтипы VersionReference, FallbackCondition, CompatibilityRule, Constraint сохраняют информацию о типе
4. **`data class` == structural equality** -- сравнение через `==` проверяет все поля
5. **Immutable collections** -- `List`, `Set`, `Map` не изменяются между сериализацией и десериализацией

=== Потенциальные нарушения round-trip

[cols="1,3,2",options="header"]
|===
| Проблема | Описание | Решение

| Порядок элементов в Set
| `Set<String>` может десериализоваться в другом порядке
| Использовать `LinkedHashSet` или сравнивать содержимое без порядка

| Порядок ключей в Map
| JSON объекты не гарантируют порядок ключей
| `kotlinx-serialization` сохраняет порядок при десериализации в `LinkedHashMap`

| Floating point precision
| Не применимо -- нет Float/Double полей в модели
| --

| `JsonElement` в extensions
| `JsonElement` сериализуется/десериализуется as-is
| Round-trip гарантирован для JsonElement
|===

[[implementation-notes]]
== Заметки по реализации

=== Почему используется `DependangerMetadata.serializer()` явно

Вместо reified `Json.encodeToString<DependangerMetadata>(metadata)` используется явный вызов `DependangerMetadata.serializer()`:

* Явный вызов гарантирует использование правильного сериализатора
* Не зависит от reified type resolution
* Совместимо с `explicitApi()` режимом

=== Конфигурация Json

Текущая конфигурация:

[cols="1,3,1",options="header"]
|===
| Параметр | Обоснование | Значение

| `prettyPrint`
| Человекочитаемый вывод для CLI и отладки. FR-008.
| `true`

| `ignoreUnknownKeys`
| Forward compatibility при чтении JSON более новой версии схемы. FR-006.
| `true`

| `encodeDefaults`
| Гарантия round-trip: все поля записываются, включая default. FR-002.
| `true`

Для стабильного JSON API рекомендуется добавить `@SerialName` аннотации на подтипы sealed-классов в `dependanger-core`:

    @Serializable
    @SerialName("literal")
    public data class Literal(val version: String) : VersionReference()

    @Serializable
    @SerialName("reference")

    @Serializable

Это обеспечит:

* Короткие имена дискриминатора (`"literal"` вместо `"io.github.zenhelix.dependanger.core.model.VersionReference.Literal"`)
* Стабильность при рефакторинге (переименование/перемещение класса не ломает JSON)
* Читаемость JSON-файлов

NOTE: Это изменение в `dependanger-core`, а не в `dependanger-metadata-json`.
Может быть выполнено отдельно.

* **Тестирование остальных модулей** -- для интеграционных тестов нужна возможность сериализовать/десериализовать metadata

=== Порядок реализации

1. Создать файл `Exceptions.kt` с иерархией исключений
2. Реализовать `serialize()` (самый простой, одна строка)
3. Реализовать `deserialize()` (зависит от исключений)
4. Реализовать `write()` (зависит от `serialize()`)
5. Реализовать `read()` (зависит от `deserialize()`)
6. Написать тесты round-trip
7. (Опционально) Добавить `@SerialName` аннотации в core
