[[cli-overview]]
== Обзор

Dependanger CLI предоставляет командный интерфейс для управления зависимостями без использования Gradle.

=== Установка

[source,bash]
----
# macOS / Linux
brew install dependanger

# или через zip
curl -L https://releases.dependanger.io/latest/dependanger.zip -o dependanger.zip
unzip dependanger.zip
export PATH=$PATH:$(pwd)/dependanger/bin
----

=== Общий синтаксис

[source,bash]
----
dependanger <command> [subcommand] [OPTIONS]
----

=== Глобальные опции

[cols="1,1,3",options="header"]
|===
| Опция | Короткая форма | Описание

| `--help`
| `-h`
| Показать справку по команде

| `--version`
| `-V`
| Показать версию CLI

| `--verbose`
| `-v`
| Подробный вывод (можно повторять: -vv, -vvv)

| `--quiet`
| `-q`
| Тихий режим (только ошибки)

| `--config`
| `-c`
| Путь к конфигурационному файлу

| `--no-color`
|
| Отключить цветной вывод

| `--json`
|
| Вывод в JSON формате (для скриптов)
|===

'''

[[cli-init]]
== dependanger init

Создаёт новый файл `metadata.json` с начальной структурой.

**Источник:** xref:../02-features/F002-metadata-serialization.adoc[F002: Metadata Serialization]

=== Синтаксис

[source,bash]
----
dependanger init [OPTIONS]
----

=== Опции

[cols="1,1,2,1",options="header"]
|===
| Опция | Короткая форма | Описание | По умолчанию

| `--output`
| `-o`
| Путь к выходному файлу
| `metadata.json`

| `--format`
|
| Формат: `json`, `yaml`
| `json`

| `--schema-version`
|
| Версия схемы
| Последняя

| `--force`
| `-f`
| Перезаписать существующий файл
| `false`
|===

=== Примеры

[source,bash]
----
# Создать metadata.json в текущей директории
dependanger init

# Создать в указанном пути
dependanger init --output config/deps.json

# Создать в YAML формате
dependanger init --format yaml --output deps.yaml

# Перезаписать существующий
dependanger init --force
----

=== Выходные данные

Создаёт файл с минимальной структурой:

[source,json]
----
{
  "schemaVersion": "1.0",
  "versions": {},
  "libraries": {},
  "plugins": {},
  "bundles": {}
}
----

'''

[[cli-add-library]]
== dependanger add-library

Добавляет библиотеку в `metadata.json`.

**Источник:** xref:../02-features/F002-metadata-serialization.adoc[F002: Metadata Serialization]

=== Синтаксис

[source,bash]
----
dependanger add-library <alias> <coordinates> [OPTIONS]
----

=== Аргументы

[cols="1,3",options="header"]
|===
| Аргумент | Описание

| `<alias>`
| Уникальный идентификатор библиотеки (например: `guava`, `kotlin-stdlib`)

| `<coordinates>`
| Maven координаты: `group:artifact` или `group:artifact:version`
|===

=== Опции

[cols="1,1,2,1",options="header"]
|===
| Опция | Короткая форма | Описание | По умолчанию

| `--version`
| `-v`
| Версия библиотеки
|

| `--version-ref`
|
| Ссылка на именованную версию
|

| `--tags`
| `-t`
| Теги (через запятую)
|

| `--input`
| `-i`
| Входной файл metadata
| `metadata.json`

| `--output`
| `-o`
| Выходной файл (если отличается от input)
| То же, что input
|===

=== Примеры

[source,bash]
----
# Добавить библиотеку с версией
dependanger add-library guava com.google.guava:guava --version 33.0.0-jre

# Добавить с тегами
dependanger add-library guava com.google.guava:guava -v 33.0.0-jre --tags core,utils

# Использовать ссылку на версию
dependanger add-library kotlin-stdlib org.jetbrains.kotlin:kotlin-stdlib --version-ref kotlin

# Версия inline в coordinates
dependanger add-library slf4j org.slf4j:slf4j-api:2.0.9

# Указать входной/выходной файл
dependanger add-library guava com.google.guava:guava -v 33.0.0-jre \
  --input config/deps.json --output config/deps.json
----

'''

[[cli-validate]]
== dependanger validate

Валидирует metadata.json или DSL конфигурацию.

**Источник:** xref:../02-features/F001-registry-dsl.adoc[F001: Registry DSL]

=== Синтаксис

[source,bash]
----
dependanger validate [OPTIONS]
----

=== Опции

[cols="1,1,2,1",options="header"]
|===
| Опция | Короткая форма | Описание | По умолчанию

| `--input`
| `-i`
| Входной файл
| `metadata.json`

| `--strict`
|
| Fail on warnings
| `false`

| `--format`
|
| Формат вывода: `text`, `json`
| `text`
|===

=== Проверки валидации

[cols="1,3",options="header"]
|===
| Проверка | Описание

| Уникальность alias
| Все alias в libraries, plugins, bundles уникальны

| Валидность coordinates
| Maven coordinates соответствуют формату group:artifact[:version]

| Разрешимость ссылок
| Все versionRef указывают на существующие versions

| Отсутствие циклов
| Bundles и fallback conditions не имеют циклических зависимостей

| Целостность bundles
| Все библиотеки в bundles существуют

| Deprecated references
| Валидность replacedBy ссылок
|===

=== Примеры

[source,bash]
----
# Валидация metadata.json
dependanger validate

# Валидация с strict режимом
dependanger validate --strict

# JSON вывод для CI/CD
dependanger validate --format json
----

=== Коды возврата

[cols="1,3",options="header"]
|===
| Код | Описание

| `0`
| Валидация успешна

| `1`
| Есть ошибки валидации

| `2`
| Есть предупреждения (только с `--strict`)

| `3`
| Файл не найден или невалидный JSON
|===

'''

[[cli-process]]
== dependanger process

Запускает processing pipeline для преобразования metadata в effective model.

**Источник:** xref:../02-features/F003-processing-pipeline.adoc[F003: Processing Pipeline]

=== Синтаксис

[source,bash]
----
dependanger process [OPTIONS]
----

=== Опции

[cols="1,1,2,1",options="header"]
|===
| Опция | Короткая форма | Описание | По умолчанию

| `--input`
| `-i`
| Входной файл metadata
| `metadata.json`

| `--output`
| `-o`
| Выходной файл effective metadata
| `effective.json`

| `--preset`
|
| Processing preset: `DEFAULT`, `MINIMAL`, `STRICT`, `DISTRIBUTION`
| `DEFAULT`

| `--distribution`
| `-d`
| Активный Distribution
|

| `--import-boms`
|
| Включить импорт версий из BOM
| (зависит от preset)

| `--no-import-boms`
|
| Отключить импорт BOM
|

| `--target-jdk`
|
| Целевая версия JDK для fallback
| Текущая JDK
|===

=== Processing Presets

[cols="1,3",options="header"]
|===
| Preset | Включённые процессоры

| `DEFAULT`
| profile, metadata-conversion, extracted-versions, bom-import, library-filter, version-fallback, version-resolver, bundle-filter, plugin-filter, plugin, used-versions, validation

| `MINIMAL`
| profile, metadata-conversion, extracted-versions, library-filter, version-fallback, version-resolver, bundle-filter, plugin-filter, plugin, used-versions

| `STRICT`
| Все процессоры + strictVersionResolution

| `DISTRIBUTION`
| DEFAULT + profile-фильтрация
|===

=== Примеры

[source,bash]
----
# Базовая обработка
dependanger process

# С импортом BOM
dependanger process --import-boms

# Для конкретного distribution
dependanger process --distribution android

# Строгий режим
dependanger process --preset STRICT

# Указать целевую JDK
dependanger process --target-jdk 17
----

'''

== dependanger report

Генерирует комплексный отчёт о зависимостях.

**Источник:** xref:../02-features/F009-report-generation.adoc[F009: Report Generation]

=== Синтаксис

[source,bash]
----
dependanger report [OPTIONS]
----

=== Опции

[cols="1,1,2,1",options="header"]
|===
| Опция | Короткая форма | Описание | По умолчанию

| `--input`
| `-i`
| Входной файл metadata/effective
| `metadata.json`

| `--output-dir`
| `-o`
| Директория для отчётов
| `.`

| `--format`
|
| Формат: `json`, `yaml`, `markdown`, `html`
| `markdown`

| `--sections`
|
| Секции (через запятую)
| все

| `--include-transitives`
|

| `false`

[cols="1,3",options="header"]

[[cli-configuration-file]]
== Файл конфигурации

CLI поддерживает файл конфигурации для настроек по умолчанию.

1. `./dependanger.json` (в текущей директории)

[source,json]
----
{
  "defaults": {
    "input": "config/metadata.json",
    "format": "json",
    "verbose": true
  },
  "generate": {
    "toml": {
      "filename": "libs.versions.toml",
      "comments": true,
      "sort": true
    },
    "bom": {
      "groupId": "com.example",
      "artifactId": "dependencies-bom"
    }
  },
  "check-updates": {
    "exclude": ["org.internal:*"],
    "includePrerelease": false
  },
  "security-check": {
    "failOn": "HIGH",
    "ignore": ["CVE-2023-XXXXX"]
  },
  "license-check": {
    "allow": ["Apache-2.0", "MIT", "BSD-3-Clause"],
    "deny": ["GPL-3.0", "AGPL-3.0"]
  },
  "repositories": [
    "https://repo.maven.apache.org/maven2",
    "https://maven.example.com/releases"
  ]
}
----
