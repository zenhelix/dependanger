== Матрица приоритетов

=== P1 — Критично для MVP

Без этих фич продукт не имеет ценности.

* **xref:F001-registry-dsl.adoc[F001: Registry DSL]** — основа всей системы, Kotlin DSL (включая settings)
* **xref:F002-metadata-serialization.adoc[F002: Metadata Serialization]** — промежуточное хранение
* **xref:F003-processing-pipeline.adoc[F003: Processing Pipeline]** — трансформация данных
* **xref:F004-toml-generation.adoc[F004: TOML Generation]** — основной выходной формат

=== P2 — Важно

Значительно увеличивают ценность, но MVP возможен и без них.

* **xref:F005-bom-generation.adoc[F005: BOM Generation]** — выходной формат для Maven
* **xref:F006-bom-import.adoc[F006: BOM Import]** — импорт версий из Spring BOM, Kotlin BOM и др.

=== P3 — Желательно

Улучшают user experience, но не критичны.

* **xref:F007-update-check.adoc[F007: Update Check]** — проверка обновлений
* **xref:F008-compatibility-analysis.adoc[F008: Compatibility Analysis]** — анализ совместимости
* **xref:F009-report-generation.adoc[F009: Report Generation]** — единый отчёт
* **xref:F010-security-scanning.adoc[F010: Security Scanning]** — проверка уязвимостей (CVE)
* **xref:F011-license-compliance.adoc[F011: License Compliance]** — проверка лицензий

=== P4 — Будущее

Фичи для будущих версий.

* **xref:F012-transitive-resolution.adoc[F012: Transitive Resolution]** — резолвинг транзитивных зависимостей и constraints

== SPI точки расширения

Каждая фича определяет точки расширения:

[cols="1,2,2",options="header"]
|===
| Фича | SPI | Описание

| F002
| `SerializationFormat<T>`
| Добавление новых форматов сериализации

| F003
| `EffectiveMetadataProcessor`
| Добавление процессоров в pipeline

| F004, F005
| `ArtifactGenerator<T>`
| Добавление новых генераторов
|===

== Как добавить новую фичу

. Скопируйте шаблон: `docs/_templates/feature-template.adoc`
. Переименуйте: `F###-краткое-название.adoc`
. Заполните все секции
. Добавьте в таблицу выше
. Обновите матрицу приоритетов и зависимости
