== Описание

Compatibility Analysis проверяет совместимость библиотек между собой и с целевым окружением (JDK, Kotlin).
Использует правила, определённые в `compatibility { }` блоке DSL.

== User Stories

=== US-001: Проверка требований JDK

[quote]
____
Как разработчик, я хочу получать предупреждения если библиотека требует более новую версию JDK, чем указана в проекте.
____

**Acceptance Criteria:**

[cols="1,3,3,3",options="header"]
|===
| ID | Given | When | Then

| AC-001-1
| Правило `jdkRequirement("spring-jdk17") { matches = "org.springframework:*"; minJdk = 17 }`
| Анализирую с `targetJdk = 11`, есть библиотека `spring-web`
| `CompatibilityIssue` с `severity = ERROR`, message содержит "requires JDK 17+"

| AC-001-2
| Правило `jdkRequirement` с `minJdk = 17`
| Анализирую с `targetJdk = 21`
| Нет ошибок (JDK 21 >= 17)

| AC-001-3
| Правило `jdkRequirement` с `severity = WARNING`
| Нарушение правила
| `CompatibilityIssue` с `severity = WARNING`, pipeline продолжается

| AC-001-4
| Правило `jdkRequirement` с паттерном `matches = "org.springframework:*"`
| Библиотеки `spring-web`, `spring-data-jpa` нарушают правило
| `affectedLibraries = ["spring-web", "spring-data-jpa"]`
|===

=== US-002: Взаимоисключающие библиотеки

[quote]
____
Как team lead, я хочу определить что Jackson и Gson не должны быть в проекте одновременно, чтобы избежать конфликтов JSON библиотек.
____

**Acceptance Criteria:**

[cols="1,3,3,3",options="header"]
|===
| ID | Given | When | Then

| AC-002-1
| Правило `mutualExclusion("json-libs") { libraries("jackson-core", "gson") }`
| Обе библиотеки присутствуют в effective metadata
| `CompatibilityIssue` с `affectedLibraries = ["jackson-core", "gson"]`

| AC-002-2
| Правило `mutualExclusion` для `jackson-core` и `gson`
| Только `jackson-core` присутствует
| Нет ошибок (нет конфликта)

| AC-002-3
| Правило `mutualExclusion` с `severity = WARNING`
| Обе библиотеки присутствуют
| WARNING в diagnostics, pipeline продолжается

| AC-002-4
| Правило `mutualExclusion` для 3 библиотек: A, B, C
| Присутствуют A и C (но не B)
| `CompatibilityIssue` с `affectedLibraries = ["A", "C"]`
|===

=== US-003: Проверка совместимости версий

[quote]
____
Как разработчик, я хочу убедиться что связанные библиотеки используют совместимые версии.
____

**Acceptance Criteria:**

[cols="1,3,3,3",options="header"]
|===
| ID | Given | When | Then

| AC-003-1
| Правило `versionConstraint("kotlin-sync") { libraries("kotlin-stdlib", "kotlin-reflect"); constraint = SAME_MAJOR_MINOR }`
| `kotlin-stdlib:2.1.0`, `kotlin-reflect:2.0.0`
| `CompatibilityIssue`: minor версии не совпадают (2.1 vs 2.0)

| AC-003-2
| Правило `versionConstraint` с `SAME_MAJOR_MINOR`
| `kotlin-stdlib:2.1.0`, `kotlin-reflect:2.1.20`
| Нет ошибок (2.1 == 2.1)

| AC-003-3
| Правило `versionConstraint` с `SAME_MAJOR`
| `lib-a:1.0.0`, `lib-b:1.9.0`
| Нет ошибок (major = 1)

| AC-003-4
| Правило `versionConstraint` с `EXACT`
| `lib-a:1.0.0`, `lib-b:1.0.1`
| `CompatibilityIssue`: версии не идентичны
|===

=== US-004: Кастомные правила

[quote]
____
Как архитектор, я хочу определять произвольные правила совместимости через код.
____

**Acceptance Criteria:**

[cols="1,3,3,3",options="header"]
|===
| ID | Given | When | Then

| AC-004-1
| `customRule("spring-kotlin") { condition = { ctx -> ctx.hasLibrary("spring-boot") && ctx.kotlinVersion < "1.9" } }`
| `spring-boot` присутствует, Kotlin 1.8
| `CompatibilityIssue` срабатывает

| AC-004-2
| Кастомное правило с `condition = { false }`
| Анализ
| Правило не срабатывает

| AC-004-3
| Кастомное правило с доступом к `ctx.libraries`, `ctx.jdkVersion`
| Анализ
| Правило имеет полный доступ к контексту
|===

== DSL синтаксис

[source,kotlin]
----
dependanger {
    compatibility {
        // Требование к JDK
        jdkRequirement("spring-jdk17") {
            matches = "org.springframework:*"
            minJdk = 17
            message = "Spring 6.x requires JDK 17+"
            severity = ERROR
        }

        // Взаимоисключающие библиотеки
        mutualExclusion("json-libs") {
            libraries("jackson-core", "gson")
            severity = WARNING
            message = "Consider using only one JSON library"
        }

        // Требование одинаковых версий
        versionConstraint("kotlin-sync") {
            libraries("kotlin-stdlib", "kotlin-reflect", "kotlinx-coroutines-core")
            constraint = SAME_MAJOR_MINOR
            message = "Kotlin libraries should use compatible versions"
        }

        // Кастомное правило
        customRule("spring-boot-kotlin") {
            condition = { ctx ->
                ctx.hasLibrary("spring-boot-starter") &&
                ctx.kotlinVersion?.let { it < "1.9" } == true
            }
            message = "Spring Boot 3.x works best with Kotlin 1.9+"
            severity = INFO
        }
    }

    // Включение процессора
    processing {
        analyzeCompatibility = true
    }
}
----

== Типы правил

[cols="1,3",options="header"]
|===
| Тип | Описание

| `jdkRequirement`
| Проверка минимальной версии JDK для библиотек

| `mutualExclusion`
| Проверка что взаимоисключающие библиотеки не используются вместе

| `versionConstraint`
| Проверка что версии связанных библиотек совместимы

| `customRule`
| Произвольное условие на основе context
|===

== API использование

[source,kotlin]
----
val dependanger = Dependanger.fromDsl {
    // ... DSL config
}
.preset(ProcessingPreset.STRICT)  // STRICT включает compatibility-analysis
.jdkVersion(17)                   // Целевой JDK
.build()

val result = dependanger.process()

// Получить проблемы совместимости (через extraction extension)
val issues: List<CompatibilityIssue> = result.compatibilityIssues

issues.forEach { issue ->
    println("[$\{issue.severity\}] $\{issue.ruleId\}: $\{issue.message\}")
}
----

== CompatibilityIssue

Информация о проблеме совместимости содержит:

* `ruleId` — идентификатор сработавшего правила
* `ruleType` — тип правила (jdkRequirement, mutualExclusion, versionConstraint)
* `message` — сообщение об ошибке
* `severity` — уровень серьёзности (ERROR, WARNING, INFO)
* `affectedLibraries` — затронутые библиотеки
* `suggestion` — предложение по исправлению

== SPI: CompatibilityCheckProcessor

`CompatibilityCheckProcessor` — опциональный процессор (order: 110, phase: COMPATIBILITY_ANALYSIS), который:

1. Получает правила совместимости из metadata
2. Проверяет каждое правило против effective библиотек
3. Добавляет `CompatibilityIssue` в diagnostics
4. При ERROR в strict mode — останавливает pipeline

Включается через `withCompatibilityAnalysis()` в builder API.

NOTE: Правила проверяются только против **effective** (отфильтрованных) библиотек.
Отфильтрованные библиотеки не триггерят правила.

== Edge Cases

[cols="1,3,2",options="header"]
|===
| Сценарий | Описание | Ожидаемое поведение

| Неполное правило (Incomplete Rule)
| `mutualExclusion("jackson", "gson")`, но `gson` отфильтрован distribution
| WARNING: "Rule 'json-libs' is incomplete: library 'gson' not in effective metadata"

| Несуществующая библиотека в правиле
| Правило ссылается на `unknown-lib` которой нет в DSL
| ERROR: "Unknown library 'unknown-lib' in rule 'rule-id'"

| Пустой список библиотек
| `mutualExclusion("empty") { libraries() }`
| ERROR: "Rule 'empty' has no libraries defined"

| Одна библиотека в mutualExclusion
| `mutualExclusion("single") { libraries("only-one") }`
| WARNING: "mutualExclusion rule 'single' has only one library"

| Нет правил совместимости
| Блок `compatibility { }` пуст или отсутствует
| Процессор пропускается, INFO: "No compatibility rules defined"

| Все правила проходят
| Ни одно правило не нарушено
| Пустой список CompatibilityIssue, INFO: "All compatibility checks passed"

| targetJdk не указан
| `withCompatibilityAnalysis()` без `targetJdk`, есть `jdkRequirement` правила
| WARNING: "targetJdk not specified, jdkRequirement rules will be skipped"

| Невалидный паттерн matches
| `matches = "[invalid"`
| ERROR: "Invalid pattern '[invalid' in rule 'rule-id'"

| Циклическая зависимость в customRule
| customRule вызывает другое правило (не поддерживается)
| Не применимо (правила независимы)

| Exception в customRule condition
| `condition = { throw RuntimeException() }`
| ERROR: "Exception in custom rule 'rule-id': [message]", pipeline продолжается

| Конфликтующие правила
| Два правила с одинаковым `ruleId`
| ERROR: "Duplicate rule ID 'rule-id'"

| Версии с разным форматом
| `kotlin-stdlib:2.1.0` vs `kotlin-reflect:2.1` (без patch)
| Нормализация версий перед сравнением: `2.1` → `2.1.0`

| Prerelease версии в versionConstraint
| `lib-a:1.0.0-RC1`, `lib-b:1.0.0`
| По умолчанию RC считается < stable; configurable

| Много нарушений
| 50 библиотек нарушают одно правило
| Все 50 в `affectedLibraries`, возможно группировка в отчёте

| Strict mode + ERROR
| `severity = ERROR`, strict mode включён
| Pipeline останавливается после первого ERROR

| Strict mode + WARNING
| `severity = WARNING`, strict mode включён
| Pipeline продолжается, WARNING накапливаются
|===

== CLI Command

=== dependanger analyze

Проверка совместимости через CLI.

[source,bash]
----
dependanger analyze [options] <input>
----

**Аргументы:**

[cols="1,2,1",options="header"]
|===
| Аргумент | Описание | Default

| `<input>`
| Путь к `effective-*.json` файлу
| `effective-metadata.json`
|===

**Опции:**

[cols="1,2,1",options="header"]
|===
| Опция | Описание | Default

| `--output`, `-o`
| Путь для выходного файла (для --format file)
| `./compatibility.json`

| `--format`, `-f`
| Формат вывода: `text`, `json`, `file`
| `text`

| `--target-jdk`
| Целевая версия JDK для проверки
| (обязательный для jdkRequirement)

| `--target-kotlin`
| Целевая версия Kotlin для проверки
| —

| `--strict`
| Exit code 1 при любых ERROR
| `false`

| `--min-severity`
| Минимальный уровень: `info`, `warning`, `error`
| `info`

| `--rules`
| Путь к файлу с правилами (если не в metadata)
| —
|===

**Примеры:**

[source,bash]
----
# Базовый анализ
dependanger analyze --target-jdk 17 effective-metadata.json

# JSON формат
dependanger analyze --target-jdk 17 --format json effective-metadata.json

# Strict mode (для CI)
dependanger analyze --target-jdk 17 --strict effective-metadata.json

# Только ERROR и WARNING
dependanger analyze --target-jdk 17 --min-severity warning effective-metadata.json
----

**Пример вывода (text):**

[source]
----
Analyzing compatibility for 15 libraries...

ERRORS:
  [spring-jdk17] Spring 6.x requires JDK 17+
    Affected: spring-web, spring-data-jpa
    Suggestion: Upgrade to JDK 17 or use Spring 5.x

WARNINGS:
  [json-libs] Consider using only one JSON library
    Affected: jackson-core, gson

INFO:
  [spring-boot-kotlin] Spring Boot 3.x works best with Kotlin 1.9+
    Affected: spring-boot-starter

Summary: 1 error, 1 warning, 1 info
----

== Gradle Task

=== dependangerAnalyze

Задача анализа совместимости в Gradle.

**Зависимости:**

* `dependangerGenerateEffective` — требуется EffectiveMetadata

**Inputs:**

* `effective-\{distribution\}.json` от `dependangerGenerateEffective`
* Правила из `compatibility { }` блока DSL

**Outputs:**

* `build/dependanger/compatibility.json` — JSON с результатами
* Console output — текстовый отчёт

**Опции:**

* `targetJdk` — целевая версия JDK
* `targetKotlin` — целевая версия Kotlin
* `minSeverity` — минимальный уровень severity (`INFO`, `WARNING`, `ERROR`)
* `failOnError` — exit code 1 при ERROR issues

[source,bash]
----
# Базовый анализ
./gradlew dependangerAnalyze

# С указанием JDK
./gradlew dependangerAnalyze -PtargetJdk=17

# Strict mode
./gradlew dependangerAnalyze -PfailOnError=true
----

== Settings DSL

Полная конфигурация анализа совместимости в Gradle DSL.

[source,kotlin]
----
dependanger {
    settings {
        compatibilityAnalysis {
            // Включение процессора
            enabled = true  // или через processing { analyzeCompatibility = true }

            // Целевое окружение
            targetJdk = 17
            targetKotlin = "2.1.0"
            targetGradle = "8.5"

            // Фильтрация результатов
            minSeverity = INFO  // INFO, WARNING, ERROR

            // CI интеграция
            failOnError = false  // default
            failOnWarning = false  // default

            // Встроенные правила
            enableBuiltinRules = true  // default

            // Настройки отчёта
            includePassedRules = false  // показывать правила которые прошли
            groupBySeverity = true  // группировать по severity
        }
    }

    // Или короткая форма через processing
    processing {
        analyzeCompatibility {
            enabled = true
            targetJdk = 17
            failOnErrors = true
        }
    }
}
----
