= Доменная модель
:toc: left
:toclevels: 3
:sectnums:
:icons: font

[abstract]
--
Документ описывает ключевые сущности системы Dependanger на трёх уровнях абстракции:
DSL Model (в памяти), Metadata Model (сериализованная), Effective Model (готовая к генерации).

Краткие определения терминов: xref:../00-overview.adoc#_глоссарий[Глоссарий в Overview].
--

== Уровни моделей

Система использует три уровня представления данных:

=== DSL Model

[quote]
____
То, что пишет пользователь в Kotlin DSL.
Модель в памяти, создаваемая через builder pattern.
____

* **Модуль:** `dependanger-core`
* **Главная сущность:** `DependangerDsl`
* **Характеристики:**
** Fluent API для удобного определения
** Поддержка DSL блоков (`@DependangerDslMarker`)
** Версии могут быть неразрешёнными (ссылки на alias)
** Валидация на уровне DSL

=== Metadata Model

[quote]
____
Сериализуемое представление модели.
Промежуточный формат для хранения и передачи.
____

* **Модуль:** `dependanger-metadata-json` (и другие форматы через SPI)
* **Главная сущность:** `DependangerMetadata`
* **Характеристики:**
** Абстрагирован через `SerializationFormat<T>` SPI
** JSON как основной формат (расширяемо до YAML)
** Хранит полную информацию из DSL
** Файл: `metadata.json`

=== Effective Model

[quote]
____
Результат обработки pipeline.
Все версии разрешены, фильтры применены, готов для генерации.
____

* **Модуль:** `dependanger-effective`
* **Главная сущность:** `EffectiveMetadata`
* **Характеристики:**
** Все версии разрешены с указанием `VersionSource`
** Применены фильтры (distribution profile)
** Содержит результаты валидации
** Опционально: update report, compatibility report
** Файл: `effective-{distribution}.json`

== Сущности

=== DependangerDsl

[quote]
____
Корневая точка входа Kotlin DSL для конфигурации зависимостей.
Содержит все определения для генерации артефактов (TOML, BOM).
____

NOTE: Полная спецификация DSL синтаксиса: xref:../02-features/F001-registry-dsl.adoc[F001: Registry DSL].

==== DSL блоки

[cols="1,2,2",options="header"]
|===
| Блок | Тип | Описание

| `versions { }`
| VersionsDsl
| Именованные версии с fallback условиями

| `libraries { }`
| LibrariesDsl
| Библиотеки с Maven координатами

| `plugins { }`
| PluginsDsl
| Gradle plugins

| `bundles { }`
| BundlesDsl
| Группы библиотек

| `bomImports { }`
| BomImportsDsl
| Импорт версий из внешних Maven BOM

| `targetPlatforms { }`
| TargetPlatformsDsl
| Целевые платформы с ограничениями (JDK, Kotlin targets)

| `distributions { }`
| DistributionsDsl
| Профили генерации с фильтрацией

| `compatibility { }`
| CompatibilityDsl
| Правила совместимости

| `settings { }`
| SettingsDsl
| Глобальные настройки валидации и генерации
|===

NOTE: Канонический порядок блоков: `versions` → `libraries` → `plugins` → `bundles` → `bomImports` → `targetPlatforms` → `distributions` → `compatibility` → `settings`

==== Пример DSL

[source,kotlin]
----
val dsl = DependangerDsl()

dsl.versions {
    version("kotlin", "2.1.20")
    version("spring-boot", "3.4.0") {
        fallback("2.7.18") { jdkBelow(17) }
    }
}

dsl.libraries {
    library("kotlin-stdlib", "org.jetbrains.kotlin:kotlin-stdlib",
        version = versionRef("kotlin")) {
        tags("core", "kotlin")
    }
}

dsl.bundles {
    bundle("kotlin-core") {
        libraries("kotlin-stdlib", "kotlin-reflect")
    }
}

dsl.plugins {
    plugin("kotlin-jvm", "org.jetbrains.kotlin.jvm",
        version = versionRef("kotlin"))
}

dsl.bomImports {
    bom("spring-boot-bom", "org.springframework.boot:spring-boot-dependencies",
        version = versionRef("spring-boot"))
    bom("kotlin-bom", "org.jetbrains.kotlin:kotlin-bom:2.1.20")
}

dsl.targetPlatforms {
    platform("jvm17") {
        jdk { min = 17; max = 21 }
    }
    platform("android-min21") {
        android { minSdk = 21; targetSdk = 34 }
    }
}

dsl.distributions {
    distribution("android") {
        spec {
            filter {
                byTags { include { anyOf("android", "mobile") } }
            }
        }
    }
}

dsl.settings {
    validation {
        onCompatibilityViolation = ValidationAction.FAIL
        onUnresolvedVersion = ValidationAction.WARN
    }
    toml {
        filename = "libs.versions.toml"
    }
}
----

==== Правила валидации

* Все alias (библиотек, плагинов, версий, бандлов, BOM) должны быть уникальны
* Alias должен соответствовать паттерну `[a-z][a-z0-9-]*`
* Ссылки на версии должны указывать на существующие version aliases

'''

=== Library

[quote]
____
Зависимость с Maven координатами и метаданными для фильтрации.
____

==== Атрибуты

[cols="1,1,1,3",options="header"]
|===
| Атрибут | Тип | Обязательность | Описание

| alias
| String
| Да
| Уникальный идентификатор для ссылок в DSL и генерируемом TOML

| group
| String
| Да
| Maven groupId (например: `org.jetbrains.kotlin`)

| name
| String
| Да
| Maven artifactId (например: `kotlin-stdlib`)

| version
| VersionReference
| Нет
| Ссылка на version alias (`Reference`), inline значение (`Literal`), или диапазон (`Range`). Может быть унаследована из BomImport

| tags
| Set<String>
| Нет
| Теги для фильтрации и группировки (например: `testing`, `database`, `spring`)

| requires
| Requirements
| Нет
| Требования к окружению (JDK версия, Kotlin версия)
|===

==== VersionReference

[source,kotlin]
----
sealed class VersionReference {
    data class Literal(val version: String) : VersionReference()
    data class Reference(val name: String) : VersionReference()
    data class Range(val range: VersionRange) : VersionReference()
}
----

==== DSL синтаксис

[source,kotlin]
----
libraries {
    // С ссылкой на version alias
    library("kotlin-stdlib", "org.jetbrains.kotlin:kotlin-stdlib",
        version = versionRef("kotlin")) {
        tags("core", "kotlin")
    }

    // С inline версией
    library("slf4j-api", "org.slf4j:slf4j-api:2.0.9") {
        tags("logging")
    }

    // Без версии (будет взята из BomImport)
    library("spring-web", "org.springframework:spring-web") {
        tags("web", "spring")
        requires {
            jdk { min = 17 }
        }
    }
}
----

==== Правила валидации

* `alias` уникален, соответствует паттерну `[a-z][a-z0-9-]*`
* `group:name` формируют валидные Maven координаты
* `versionRef()` должен ссылаться на существующий version alias

[quote]
____
Gradle plugin — определяется через alias, plugin ID и версию.
Публикуется в Version Catalog секции `[plugins]`.
____

==== Атрибуты

[cols="1,1,1,3",options="header"]
|===
| Атрибут | Тип | Обязательность | Описание

| alias
| String
| Да
| Уникальный идентификатор для ссылок в DSL и генерируемом TOML

| id
| String
| Да
| Gradle Plugin ID (например: `org.jetbrains.kotlin.jvm`)

| version
| VersionReference
| Нет
| Ссылка на version alias или inline значение
|===

==== DSL синтаксис

[source,kotlin]

=== Version

[quote]
____
Именованная версия для переиспользования несколькими библиотеками.
Поддерживает условные fallback'и.
____

==== Атрибуты

[cols="1,1,1,3",options="header"]
|===
| Атрибут | Тип | Обязательность | Описание

| name
| String
| Да
| Уникальный идентификатор версии (например: `kotlin`, `spring-boot`)

| value
| String
| Да
| Значение версии по умолчанию (например: `2.1.20`)

| fallbacks
| List<VersionFallback>
| Нет
| Условные альтернативные версии
|===

==== VersionFallback

[cols="1,1,3",options="header"]
|===
| Поле | Тип | Описание

| value
| String
| Альтернативное значение версии

| condition
| FallbackCondition
| Условие, при котором используется fallback
|===

==== DSL синтаксис

[source,kotlin]
----
versions {
    version("kotlin", "2.1.20")

    version("spring-boot", "3.4.0") {
        fallback("2.7.18") { jdkBelow(17) }
        fallback("3.2.0") { kotlinVersionBelow("1.9") }
    }
}
----

==== Доступные условия

[cols="1,3",options="header"]
|===
| Условие | Описание

| `jdkBelow(version)`
| JDK версия < указанной

| `jdkAtLeast(version)`
| JDK версия >= указанной

| `jdkBetween(min, max)`
| JDK версия в диапазоне [min, max]

| `kotlinVersionBelow(version)`
| Kotlin версия < указанной

| `kotlinVersionAtLeast(version)`
| Kotlin версия >= указанной

| `gradleVersionAtLeast(version)`
| Gradle версия >= указанной

| `gradleVersionBelow(version)`
| Gradle версия < указанной

| `distribution(name)`
| Активен указанный distribution profile

| `environment(key, value)`
| Переменная окружения равна значению

| `all(conditions...)`
| Все условия выполнены (AND)

| `any(conditions...)`
| Хотя бы одно условие выполнено (OR)

| `not(condition)`
| Условие не выполнено (NOT)
|===

==== Правила валидации

* `name` уникален
* `value` должен быть валидной версией
* Fallback условия оцениваются по порядку, первое подходящее выигрывает

=== Bundle

[quote]
____
Именованная группа библиотек для переиспользования.
Может наследовать библиотеки из других бандлов.
Публикуется в Version Catalog секции `[bundles]`.
____

==== Атрибуты

[cols="1,1,1,3",options="header"]
|===
| Атрибут | Тип | Обязательность | Описание

| name
| String
| Да
| Уникальный идентификатор бандла

| libraries
| List<String>
| Нет
| Alias'ы включаемых библиотек

| extends
| List<String>
| Нет
| Имена бандлов для наследования (композиция)
|===

==== DSL синтаксис

[source,kotlin]
----
bundles {
    bundle("kotlin-core") {
        libraries("kotlin-stdlib", "kotlin-reflect")
    }

    bundle("kotlin-serialization") {
        libraries("kotlinx-serialization-core", "kotlinx-serialization-json")
        extends("kotlin-core")  // Включает библиотеки из kotlin-core
    }

    bundle("spring-backend") {
        libraries("spring-web", "spring-data-jpa")
        extends("kotlin-core", "kotlin-serialization")
    }
}
----

==== Правила валидации

* `name` уникален
* Все ссылки в `libraries` должны указывать на существующие библиотеки
* Все ссылки в `extends` должны указывать на существующие бандлы
* Не допускаются циклические зависимости между бандлами

=== BomImport

[quote]
____
Импорт версий из внешнего Maven BOM/Platform.
Позволяет наследовать версии из Spring Boot BOM, Kotlin BOM и других.
____

==== Атрибуты

[cols="1,1,1,3",options="header"]
|===
| Атрибут | Тип | Обязательность | Описание

| alias
| String
| Да
| Уникальный идентификатор BOM

| group
| String
| Да
| Maven groupId BOM (например: `org.springframework.boot`)

| name
| String
| Да
| Maven artifactId BOM (например: `spring-boot-dependencies`)

| version
| VersionReference
| Да
| Версия BOM (ссылка или literal)
|===

==== DSL синтаксис

[source,kotlin]
----
bomImports {
    // С ссылкой на version alias
    bom("spring-boot-bom", "org.springframework.boot:spring-boot-dependencies",
        version = versionRef("spring-boot"))

    // С inline версией
    bom("kotlin-bom", "org.jetbrains.kotlin:kotlin-bom:2.1.20")

----

==== Поведение

* Версии из BOM импортируются во время processing pipeline (BomImportProcessor, order: 15)
* Приоритет: DECLARED (явная версия в DSL) > BOM_IMPORT (из BomImport)
* Порядок BomImport имеет значение: первый BOM с версией для библиотеки выигрывает

=== Distribution

[quote]
____
Профиль генерации артефактов с фильтрацией библиотек.
Определяет какие библиотеки включить в генерируемый TOML/BOM.
____

NOTE: **Поведение по умолчанию:** Если distribution не указан (или используется `DistributionSelection.Default`), все библиотеки включаются без фильтрации.

==== Атрибуты

[cols="1,1,1,3",options="header"]
|===
| Атрибут | Тип | Обязательность | Описание

| name
| String
| Да
| Уникальный идентификатор профиля

| spec
| FilterSpec
| Нет
| Спецификация фильтрации
|===

==== DSL синтаксис

[source,kotlin]
----
distributions {
    distribution("android") {
        spec {
            filter {
                byTags { include { anyOf("android", "mobile") } }
                byGroups { exclude { matching("org.springframework:*") } }
            }
        }
    }

    distribution("backend") {
        spec {
            filter {
                byTags { include { anyOf("server", "web", "database") } }
                byBundles { include("spring-backend", "kotlin-core") }
            }
        }
    }

    distribution("minimal") {
        spec {
            filter {
                byBundles { include("kotlin-core") }
            }
        }
    }
}
----

==== Типы фильтров

[cols="1,3",options="header"]
|===
| Фильтр | Описание

| `byTags`
| Фильтрация по тегам библиотек

| `byGroups`
| Фильтрация по Maven group pattern

| `byAliases`
| Фильтрация по alias библиотек

| `byBundles`
| Включение содержимого бандлов
|===

==== Логика комбинирования фильтров

[cols="1,3",options="header"]
|===
| Комбинация | Логика

| Несколько `include` в одном фильтре
| **OR** — библиотека включается, если подходит под любой include

| Несколько `exclude` в одном фильтре
| **OR** — библиотека исключается, если подходит под любой exclude

| `include` + `exclude` в одном фильтре
| `include` применяется первым, затем из результата удаляются `exclude`

| Несколько разных фильтров (`byTags` + `byGroups`)
| **AND** — библиотека должна пройти все фильтры
|===

===== Алгоритм фильтрации

1. Если нет `include` фильтров → все библиотеки потенциально включены
2. Применяются `include` фильтры → только matching библиотеки остаются
3. Применяются `exclude` фильтры → matching библиотеки удаляются
4. Результаты всех фильтров (`byTags`, `byGroups`, `byAliases`) объединяются через **AND**

=== CompatibilityRule

[quote]
____
Правило проверки совместимости между библиотеками или с окружением.
____

==== Типы правил

[cols="1,2,2",options="header"]
|===
| Тип | Описание | Пример

| `jdkRequirement`
| Требование к версии JDK
| Spring Boot 3.x требует JDK 17+

| `mutualExclusion`
| Взаимоисключающие библиотеки
| Jackson и Gson не должны быть вместе

| `versionConstraint`
| Ограничение версий между библиотеками
| Kotlin stdlib и reflect должны иметь одинаковую версию
|===

==== DSL синтаксис

[source,kotlin]
----
compatibility {
    jdkRequirement("spring-jdk17") {
        matches = "org.springframework:*"
        minJdk = 17
        message = "Spring 6.x requires JDK 17+"
    }

    mutualExclusion("json-libs") {
        libraries("jackson-core", "gson")
        severity = WARNING
        message = "Consider using only one JSON library"
    }

    versionConstraint("kotlin-version-sync") {
        libraries("kotlin-stdlib", "kotlin-reflect", "kotlinx-coroutines-core")
        constraint = SAME_MAJOR_MINOR
    }
}
----

==== Severity

[cols="1,3",options="header"]
|===
| Значение | Описание

| ERROR
| Нарушение блокирует генерацию

| WARNING
| Предупреждение, генерация продолжается

| INFO
| Информационное сообщение
|===

==== Поведение с отфильтрованными библиотеками

Правила проверяются только против **effective** (отфильтрованных) библиотек:

* Отфильтрованные библиотеки не триггерят правила
* Если правило ссылается на отфильтрованные библиотеки — генерируется WARNING о неполном правиле
* Правило всё равно проверяется для оставшихся библиотек

=== Settings

[quote]
____
Глобальные настройки валидации и поведения системы.
____

==== DSL синтаксис

[source,kotlin]
----
settings {
    // Настройки валидации
    validation {
        onCompatibilityViolation = ValidationAction.FAIL
        onDeprecatedLibrary = ValidationAction.WARN
        onJdkMismatch = ValidationAction.FAIL
        onUnresolvedVersion = ValidationAction.WARN
    }

    // Дефолтная дистрибуция
    defaultDistribution = "core"

    // Строгий режим резолюции версий
    strictVersionResolution = true

    // Настройки генерации TOML
    toml {
        filename = "libs.versions.toml"
        includeComments = true
        sortSections = true
    }

    // Настройки генерации BOM
    bom {
        groupId = "com.example"
        artifactId = "my-bom"
        version = "1.0.0"  // или versionRef("bom")
        includeOptionalDependencies = false
    }
}
----

=== Platform Library

[quote]
____
Библиотека типа Platform/BOM для экспорта в TOML Version Catalog.
Отличается от обычной библиотеки: экспортируется с типом `pom` и используется потребителями через `platform()`.
____

NOTE: Не путать с `bomImports {}`, который используется для *импорта* версий из внешних BOM.
`platformLibrary()` — для *экспорта* BOM-зависимости в генерируемый TOML.

==== DSL синтаксис

[source,kotlin]
----
libraries {
    // Обычная библиотека
    library("kotlin-stdlib", "org.jetbrains.kotlin:kotlin-stdlib",
        version = versionRef("kotlin"))

    // Platform library — экспортируется в TOML как library с type=pom
    platformLibrary("spring-boot-bom", "org.springframework.boot:spring-boot-dependencies",
        version = versionRef("spring-boot"))
}
----

==== Отличия от library()

* В генерируемом TOML отображается с типом `pom`
* Потребители подключают через `implementation(platform(libs.spring.boot.bom))`
* Не содержит `supportedTargets` (это не runtime dependency)

=== EffectiveMetadata

[quote]
____
Результат обработки pipeline — полностью разрешённая и готовая к генерации модель.
Создаётся для каждого distribution profile отдельно.
____

==== Ключевые атрибуты

[cols="1,1,3",options="header"]
|===
| Атрибут | Тип | Описание

| distribution
| String
| Имя distribution profile

| libraries
| Map<String, EffectiveLibrary>
| Библиотеки с разрешёнными версиями (после фильтрации)

| bundles
| Map<String, EffectiveBundle>
| Бандлы (после фильтрации)

| plugins
| Map<String, EffectivePlugin>
| Плагины с разрешёнными версиями

| versions
| Map<String, ResolvedVersion>
| Используемые версии с указанием источника

| diagnostics
| ProcessingDiagnostics
| Результаты валидации, предупреждения, ошибки
|===

---

=== ResolvedVersion

[quote]
____
Версия с информацией об источнике.
____

==== Атрибуты

[cols="1,1,3",options="header"]
|===
| Атрибут | Тип | Описание

| value
| String
| Значение версии (например: `3.4.0`)

| source
| VersionSource
| Источник версии
|===

==== VersionSource (приоритет)

[cols="1,1,3",options="header"]
|===
| Приоритет | Значение | Описание

| 1 (высший)
| DECLARED
| Явно указана в DSL конфигурации через `version = "1.0.0"`, `versionRef("alias")` или inline в coordinates

| 2 (низший)
| BOM_IMPORT
| Импортирована из BomImport через BomImportProcessor
|===

==== Примеры VersionSource

[source,kotlin]
----
// DECLARED — явно указана версия
library("guava", "com.google.guava:guava", version = "33.0.0-jre")

// DECLARED — через versionRef
library("kotlin-stdlib", "org.jetbrains.kotlin:kotlin-stdlib",
    version = versionRef("kotlin"))

// DECLARED — inline версия в coordinates
library("slf4j", "org.slf4j:slf4j-api:2.0.9")

// BOM_IMPORT — версия будет из Spring Boot BOM
library("spring-web", "org.springframework:spring-web")  // без версии
----

== Связи между сущностями

[cols="1,1,3",options="header"]
|===
| Связь | Кардинальность | Описание

| DependangerDsl → Library
| 1:N
| DSL содержит множество библиотек

| DependangerDsl → Plugin
| 1:N
| DSL содержит множество plugins

| DependangerDsl → Version
| 1:N
| DSL определяет именованные версии

| DependangerDsl → Bundle
| 1:N
| DSL содержит множество бандлов

| DependangerDsl → BomImport
| 1:N
| DSL импортирует внешние BOM

| DependangerDsl → TargetPlatform
| 1:N
| DSL определяет целевые платформы

| DependangerDsl → Distribution
| 1:N
| DSL содержит профили генерации

| DependangerDsl → CompatibilityRule
| 1:N
| DSL содержит правила совместимости

| DependangerDsl → Settings
| 1:1
| DSL содержит глобальные настройки

| Bundle → Library
| N:M
| Бандл включает библиотеки по alias

| Bundle → Bundle
| N:M
| Бандл наследует другие бандлы (extends)

| Library → Version
| N:1
| Библиотека ссылается на Version через `versionRef()`

| Plugin → Version
| N:1
| Plugin ссылается на Version через `versionRef()`
|===
