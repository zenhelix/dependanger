[[overview]]
== Обзор

Модуль `dependanger-generator-toml` отвечает за генерацию файлов Gradle Version Catalog в формате TOML из `EffectiveMetadata`.
Класс `TomlGenerator` реализует интерфейс `ArtifactGenerator<String>` и производит текстовое содержимое TOML-файла, готового для использования Gradle.

Gradle Version Catalog -- стандартный механизм централизованного управления зависимостями в Gradle, описанный в файле `libs.versions.toml`.

[[generate-algorithm]]
== Алгоритм generate()

=== Назначение

Преобразовать `EffectiveMetadata` в строку, содержащую валидный Gradle Version Catalog в формате TOML.

=== Высокоуровневый алгоритм

[source,text]
----
1. Извлечь данные из EffectiveMetadata:
   - versions: Map<String, ResolvedVersion>
   - libraries: Map<String, EffectiveLibrary>
   - plugins: Map<String, EffectivePlugin>
   - bundles: Map<String, EffectiveBundle>

2. Построить секцию [versions]:
   - Для каждой ResolvedVersion -> "alias = \"value\""
   - Если sortSections -> сортировать по alias

3. Построить секцию [libraries]:
   - Для каждой EffectiveLibrary:
     a. Если isDeprecated && includeDeprecationComments -> добавить комментарий
     b. Если useInlineVersions -> использовать инлайн-версию
     c. Иначе -> использовать version.ref, указывая на [versions]
   - Если sortSections -> сортировать по alias

4. Построить секцию [bundles]:
   - Для каждого EffectiveBundle -> "alias = [\"lib1\", \"lib2\"]"
   - Если sortSections -> сортировать по alias

5. Построить секцию [plugins]:
   - Для каждого EffectivePlugin:
     a. Если useInlineVersions -> инлайн-версия
     b. Иначе -> version.ref
   - Если sortSections -> сортировать по alias

6. Собрать все секции в единый TOML-текст:
   - Пустые секции не включаются
   - Между секциями -- пустая строка

7. Вернуть результирующую строку
----

=== Детальная реализация

[source,kotlin]
----
public class TomlGenerator(private val config: TomlConfig) : ArtifactGenerator<String> {
    override val generatorId: String = "toml-version-catalog"
    override val description: String = "Generates Gradle Version Catalog TOML file"
    override val fileExtension: String = "toml"

    private val logger = KotlinLogging.logger {}

    override fun generate(effective: EffectiveMetadata): String {
        logger.info { "Generating TOML version catalog (generatorId=$generatorId)" }

        val builder = StringBuilder()

        // Заголовочный комментарий
        if (config.includeComments) {
            builder.appendLine("# Generated by Dependanger (toml-version-catalog)")
            builder.appendLine()
        }

        // Секция [versions]
        val versionsSection = generateVersionsSection(effective.versions)
        if (versionsSection.isNotEmpty()) {
            builder.appendLine("[versions]")
            builder.appendLine(versionsSection)
        }

        // Секция [libraries]
        val librariesSection = generateLibrariesSection(effective.libraries, effective.versions)
        if (librariesSection.isNotEmpty()) {
            builder.appendLine("[libraries]")
            builder.appendLine(librariesSection)
        }

        // Секция [bundles]
        val bundlesSection = generateBundlesSection(effective.bundles)
        if (bundlesSection.isNotEmpty()) {
            builder.appendLine("[bundles]")
            builder.appendLine(bundlesSection)
        }

        // Секция [plugins]
        val pluginsSection = generatePluginsSection(effective.plugins, effective.versions)
        if (pluginsSection.isNotEmpty()) {
            builder.appendLine("[plugins]")
            builder.appendLine(pluginsSection)
        }

        val result = builder.toString().trimEnd() + "\n"
        logger.info { "TOML generation complete: ${effective.versions.size} versions, ${effective.libraries.size} libraries, ${effective.bundles.size} bundles, ${effective.plugins.size} plugins" }
        return result
    }
}
----

=== Генерация секции [versions]

[source,kotlin]
----
private fun generateVersionsSection(versions: Map<String, ResolvedVersion>): String {
    if (versions.isEmpty()) return ""

    val entries = versions.values.let { vals ->
        if (config.sortSections) vals.sortedBy { it.alias } else vals.toList()
    }

    return entries.joinToString("\n") { version ->
        "${version.alias} = \"${escapeTomlValue(version.value)}\""
    } + "\n"
}
----

*Логика:*

1. Если map пустой -- вернуть пустую строку (секция не будет добавлена).
2. При `sortSections = true` -- отсортировать записи по `alias` лексикографически.
3. Для каждой `ResolvedVersion` -- сформировать строку `alias = "value"`.
4. Значение версии экранируется через `escapeTomlValue()`.

=== Генерация секции [libraries]

[source,kotlin]
----
private fun generateLibrariesSection(
    libraries: Map<String, EffectiveLibrary>,
    versions: Map<String, ResolvedVersion>,
): String {
    if (libraries.isEmpty()) return ""

    val entries = libraries.values.let { vals ->
        if (config.sortSections) vals.sortedBy { it.alias } else vals.toList()
    }

    val sb = StringBuilder()
    for (lib in entries) {
        // Комментарий о deprecated
        if (lib.isDeprecated && config.includeDeprecationComments) {
            sb.appendLine(buildDeprecationComment(lib))
        }

        // Формирование записи библиотеки
        sb.appendLine(buildLibraryEntry(lib, versions))
    }
    return sb.toString()
}
----

==== Формирование записи библиотеки

[source,kotlin]
----
private fun buildLibraryEntry(
    lib: EffectiveLibrary,
    versions: Map<String, ResolvedVersion>,
): String {
    val group = escapeTomlValue(lib.group)
    val name = escapeTomlValue(lib.artifact)

    return if (config.useInlineVersions) {
        // Инлайн-версия
        val versionValue = resolveVersionValue(lib, versions)
        if (versionValue != null) {
            "${lib.alias} = { group = \"$group\", name = \"$name\", version = \"${escapeTomlValue(versionValue)}\" }"
        } else {
            // Библиотека без версии (managed dependency)
            "${lib.alias} = { group = \"$group\", name = \"$name\" }"
        }
    } else {
        // version.ref (по умолчанию, рекомендуемый)
        val versionRef = findVersionRef(lib, versions)
        if (versionRef != null) {
            "${lib.alias} = { group = \"$group\", name = \"$name\", version.ref = \"$versionRef\" }"
        } else {
            // Библиотека без версии или с версией, не вынесенной в [versions]
            val versionValue = resolveVersionValue(lib, versions)
            if (versionValue != null) {
                // Fallback: инлайн если нет ref
                "${lib.alias} = { group = \"$group\", name = \"$name\", version = \"${escapeTomlValue(versionValue)}\" }"
            } else {
                "${lib.alias} = { group = \"$group\", name = \"$name\" }"
            }
        }
    }
}
----

*Логика выбора версии:*

1. **`useInlineVersions = false` (по умолчанию):** Искать `version.ref` -- alias версии в `[versions]`, на которую ссылается `lib.version.alias`.
Если найдена -- использовать `version.ref`.
Если нет -- fallback на инлайн-версию.
2. **`useInlineVersions = true`:** Всегда использовать инлайн-версию, подставляя значение напрямую.
3. **Библиотека без версии:** Генерировать запись без поля version (managed dependency).

[source,kotlin]
----
private fun findVersionRef(lib: EffectiveLibrary, versions: Map<String, ResolvedVersion>): String? {
    val resolvedVersion = lib.version ?: return null
    // Проверяем, есть ли версия с этим alias в секции [versions]
    return if (versions.containsKey(resolvedVersion.alias)) resolvedVersion.alias else null
}

private fun resolveVersionValue(lib: EffectiveLibrary, versions: Map<String, ResolvedVersion>): String? {
    return lib.version?.value
}
----

==== Формирование комментария о deprecated

[source,kotlin]
----
private fun buildDeprecationComment(lib: EffectiveLibrary): String {
    val deprecation = lib.deprecation
    val parts = mutableListOf<String>()

    parts.add("DEPRECATED")

    if (deprecation?.message != null) {
        parts[0] = "DEPRECATED: ${deprecation.message}"
    }

    if (deprecation?.replacedBy != null) {
        parts.add("Use ${deprecation.replacedBy} instead")
    }

    if (deprecation?.removalVersion != null) {
        parts.add("Removal: ${deprecation.removalVersion}")
    }

    return "# ${parts.joinToString(". ")}."
}
----

[source,toml]
----

# DEPRECATED: Use new-lib instead.

private fun generateBundlesSection(bundles: Map<String, EffectiveBundle>): String {
    if (bundles.isEmpty()) return ""

    val entries = bundles.values.let { vals ->
        if (config.sortSections) vals.sortedBy { it.alias } else vals.toList()
    }

    return entries.joinToString("\n") { bundle ->
        val libs = bundle.libraries.joinToString(", ") { "\"$it\"" }
        "${bundle.alias} = [$libs]"
    } + "\n"
}
----

=== Генерация секции [plugins]

[source,kotlin]
----
private fun generatePluginsSection(
    plugins: Map<String, EffectivePlugin>,
    versions: Map<String, ResolvedVersion>,
): String {
    if (plugins.isEmpty()) return ""

    val entries = plugins.values.let { vals ->
        if (config.sortSections) vals.sortedBy { it.alias } else vals.toList()
    }

    return entries.joinToString("\n") { plugin ->
        buildPluginEntry(plugin, versions)
    } + "\n"
}

private fun buildPluginEntry(
    plugin: EffectivePlugin,
    versions: Map<String, ResolvedVersion>,
): String {
    val id = escapeTomlValue(plugin.id)

    return if (config.useInlineVersions) {
        val versionValue = plugin.version?.value
        if (versionValue != null) {
            "${plugin.alias} = { id = \"$id\", version = \"${escapeTomlValue(versionValue)}\" }"
        } else {
            "${plugin.alias} = { id = \"$id\" }"
        }
    } else {
        val versionRef = plugin.version?.let { rv ->
            if (versions.containsKey(rv.alias)) rv.alias else null
        }
        if (versionRef != null) {
            "${plugin.alias} = { id = \"$id\", version.ref = \"$versionRef\" }"
        } else {
            val versionValue = plugin.version?.value
            if (versionValue != null) {
                "${plugin.alias} = { id = \"$id\", version = \"${escapeTomlValue(versionValue)}\" }"
            } else {
                "${plugin.alias} = { id = \"$id\" }"
            }
        }
    }
}
----

=== Экранирование значений TOML

[source,kotlin]
----
private fun escapeTomlValue(value: String): String {
    return value
        .replace("\\", "\\\\")
        .replace("\"", "\\\"")
        .replace("\n", "\\n")
        .replace("\r", "\\r")
        .replace("\t", "\\t")
}
----

*TOML-спецификация требует экранирования:*

- `\` -> `\\`
- `"` -> `\"`
- Переводы строк -> `\n`, `\r`
- Табуляции -> `\t`

=== Детальная реализация

[source,kotlin]
----
override fun write(artifact: String, output: Path) {
    val resolvedFilename = resolveFilename(config.filename)
    val targetFile = output.resolve(resolvedFilename)

    logger.info { "Writing TOML to: $targetFile" }

    // Создать родительские директории если не существуют
    Files.createDirectories(targetFile.parent)

    // Записать файл с UTF-8 кодировкой
    Files.writeString(targetFile, artifact, StandardCharsets.UTF_8)

    logger.info { "TOML written successfully: $targetFile (${artifact.length} chars)" }
}
----

=== Резолвинг имени файла

[source,kotlin]
----
private fun resolveFilename(template: String): String {
    // Плейсхолдер {distribution} подставляется из контекста,
    // но на уровне write() мы работаем с уже готовым filename.
    // Подстановка {distribution} происходит на уровне API/CLI,
    // которые вызывают generate() для каждой дистрибуции
    // и передают корректный output path.
    return template
}
----

*Примечание:* Мульти-дистрибуция обрабатывается на уровне вызывающего кода (`DependangerResult`, CLI, Gradle-плагин), который генерирует `EffectiveMetadata` для каждой дистрибуции с подставленным `distribution` полем и вызывает `write()` с соответствующим путём.

Однако библиотеки и плагины будут использовать инлайн-версии.

=== Пример с platform-библиотекой

Platform-библиотеки (`isPlatform = true`) генерируются как обычные записи в TOML.
Gradle обрабатывает `platform()` на стороне build-скрипта, а не в Version Catalog.

[source,toml]
----
[libraries]
spring-boot-bom = { group = "org.springframework.boot", name = "spring-boot-dependencies", version.ref = "spring-boot" }
----

=== Пример без комментариев (includeComments = false)

Выходной файл не содержит ни заголовочного комментария, ни комментариев секций.
Если также `includeDeprecationComments = false`, то deprecated-комментарии тоже не включаются.

[[edge-cases]]
== Граничные случаи

[cols="1,2,2"]
|===
| Случай | Поведение | Обоснование

| Пустая `EffectiveMetadata` | Генерируется файл только с заголовочным комментарием (если `includeComments = true`), иначе -- пустой файл с одним переводом строки. | Валидный TOML. Gradle принимает пустой Version Catalog.
| Библиотека без версии (`version = null`) | Запись без поля `version`: `alias = \{ group = "...", name = "..." \}` | Managed dependency -- версия приходит из BOM.
| Версия содержит спецсимволы | Экранирование через `escapeTomlValue()`: кавычки, бэкслеши, переводы строк. | Соответствие TOML-спецификации.
| Алиас содержит точки | Алиасы в Gradle Version Catalog могут содержать `-` и `.`. В TOML точка в ключе допустима в кавычках, но Gradle ожидает ключ без кавычек. Алиасы с точками -- `kotlin.stdlib` -- допустимы, Gradle их поддерживает нативно. | Совместимость с Gradle.
| Deprecated-библиотека без `DeprecationInfo` | Комментарий `# DEPRECATED.` | `isDeprecated = true`, но `deprecation = null`.
| Deprecated-библиотека с полным `DeprecationInfo` | `# DEPRECATED: \{message\}. Use \{replacedBy\} instead. Removal: \{removalVersion\}.` | Полная информация для разработчика.
| Platform-библиотека (`isPlatform = true`) | Генерируется как обычная library-запись. | Gradle обрабатывает `platform()` в build-скрипте. В TOML нет семантики platform.
| Библиотека ссылается на несуществующую версию | Fallback на инлайн-версию из `lib.version.value`. Если value отсутствует -- запись без версии. Диагностическое предупреждение в логе. | Defensive: не падать, но зафиксировать аномалию.
| Дублирующиеся алиасы | Невозможно: `EffectiveMetadata` использует `Map<String, ...>`, ключи уникальны. | Гарантия модели.
| Очень длинные значения версий | Экранируются стандартно. Ограничений на длину нет. | TOML не ограничивает длину строки.
| Пустой bundle (нет библиотек) | `alias = []` | Валидный TOML. Gradle принимает пустой bundle.
| Bundle с одной библиотекой | `alias = ["lib"]` | Валидный TOML.
|===

[[error-handling]]
== Обработка ошибок

[cols="1,2,2"]
|===
| Ситуация | Поведение | Тип исключения

| `generate()` с пустой metadata | Успешно генерирует файл (возможно пустой). Не является ошибкой. | Нет
| `write()` -- невалидный путь | Пробрасывается `IOException` от `Files.writeString()`. | `java.io.IOException`
| `write()` -- нет прав на запись | Пробрасывается `IOException` или `AccessDeniedException`. | `java.nio.file.AccessDeniedException`
| `write()` -- родительская директория не существует | `Files.createDirectories()` создаёт. Если невозможно -- `IOException`. | `java.io.IOException`
| Некорректный `config.filename` (пустая строка) | Пробрасывается `InvalidPathException` при `output.resolve("")`. | `java.nio.file.InvalidPathException`
| `null` в обязательных полях модели | Невозможно: `EffectiveLibrary.group`, `EffectiveLibrary.artifact` -- non-nullable String. | Нет
|===

=== Стратегия логирования

[source,text]
----
INFO  - начало и завершение генерации (количество записей по секциям)
WARN  - библиотека с версией, не найденной в [versions] (fallback на inline)
WARN  - deprecated-библиотека без DeprecationInfo
DEBUG - детали каждой сгенерированной записи
----

== Тестовые сценарии

=== Юнит-тесты для generate()

[cols="1,3"]
|===
| Тест | Описание

| `empty metadata produces minimal output` | `EffectiveMetadata()` -> только заголовочный комментарий (или пустой файл при `includeComments = false`).
| `versions section generated correctly` | Проверить формат `alias = "value"` для каждой версии.
| `versions sorted alphabetically` | С `sortSections = true` -- алфавитный порядок. С `false` -- порядок итерации Map.
| `library with version.ref` | `version.ref = "alias"` указывает на `[versions]`.
| `library with inline version` | При `useInlineVersions = true` -- `version = "value"`.
| `library without version` | Запись без поля version.
| `library version.ref fallback to inline` | Когда `version.alias` нет в `versions` map -- инлайн.
| `platform library as normal entry` | `isPlatform = true` не влияет на формат TOML-записи.
| `deprecated library with full info` | Комментарий с message, replacedBy, removalVersion.
| `deprecated library with partial info` | Только `isDeprecated = true`, `deprecation = null` -> `# DEPRECATED.`
| `deprecated comment disabled` | `includeDeprecationComments = false` -> нет комментариев.
| `bundles section` | `alias = ["lib1", "lib2"]` формат.
| `empty bundle` | `alias = []`.
| `plugins with version.ref` | `id = "...", version.ref = "alias"`.
| `plugins with inline version` | При `useInlineVersions = true`.
| `plugin without version` | `alias = \{ id = "..." \}`.
| `include comments disabled` | Без заголовочного комментария.
| `special characters escaped` | Кавычки, бэкслеши, переводы строк в значениях.
| `empty sections omitted` | Пустая секция (0 записей) не включается в вывод.
| `full metadata end-to-end` | Все секции, множество записей, deprecated, platform.
|===
