== Описание

Metadata Serialization преобразует `DependangerDsl` (модель в памяти) в сериализуемый `DependangerMetadata` для:

* Хранения между запусками
* Передачи между этапами pipeline
* Кеширования для инкрементальной обработки

== User Stories

=== US-001: Сохранение метаданных в JSON

[quote]
____
Как пользователь Gradle plugin, я хочу чтобы результат DSL сохранялся в файл, чтобы другие таски могли использовать его без повторного парсинга DSL.
____

**Acceptance Criteria:**

[cols="1,3,3,3",options="header"]
|===
| ID | Given | When | Then

| AC-001-1
| DSL с определёнными библиотеками, версиями, бандлами
| Вызываю `dsl.toMetadata()` и `format.write(metadata, path)`
| Создаётся файл `metadata.json` с валидным JSON

| AC-001-2
| Файл `metadata.json` существует
| Вызываю `format.read(path)`
| Возвращается `DependangerMetadata` с теми же данными

| AC-001-3
| DSL с 10 библиотеками и 5 версиями
| Сериализую и десериализую обратно
| Все 10 библиотек и 5 версий сохранены без потерь (round-trip)

| AC-001-4
| DSL с fallback условиями в версиях
| Сериализую в JSON
| Условия сохранены в структурированном виде: `{"type": "jdkBelow", "version": 17}`

| AC-001-5
| Gradle task `dependangerGenerateMetadata`
| Запускаю task
| Файл создан в `build/dependanger/metadata.json`
|===

== SPI: SerializationFormat

=== Описание интерфейса

Интерфейс `SerializationFormat<T>` предоставляет точку расширения для добавления новых форматов сериализации.

**Основные возможности:**

* `formatId` — уникальный идентификатор формата (например: "json", "yaml")
* `fileExtension` — расширение файла (например: ".json", ".yaml")
* `serialize()` / `deserialize()` — преобразование модели в формат и обратно
* `write()` / `read()` — работа с файловой системой

=== Реализации

== Модель сериализации

=== DependangerMetadata

Корневая структура модели сериализации содержит:

* `schemaVersion` — версия схемы для обратной совместимости
* `versions` — именованные версии
* `libraries` — библиотеки с метаданными
* `plugins` — Gradle plugins
* `bundles` — группы библиотек
* `platforms` — целевые платформы и BOM imports
* `distributions` — профили фильтрации
* `compatibility` — правила совместимости
* `presets` — именованные presets настроек
* `settings` — глобальные настройки

=== Вложенные структуры

[cols="1,2",options="header"]
|===
| Структура | Описание

| `VersionMetadata`
| Версия с value, deprecation статусом и fallback условиями

| `LibraryMetadata`
| Maven координаты, теги, требования к платформе, поддерживаемые targets

| `PluginMetadata`
| Gradle plugin ID и версия

| `BundleMetadata`
| Список библиотек и включаемых бандлов

| `PlatformMetadata`
| Constraints для JDK, Android, поддерживаемые Kotlin targets

| `DistributionMetadata`
| Spec фильтрации, publishing настройки

| `Preset`
| Именованный preset с предустановленными настройками

| `SettingsMetadata`
| Глобальные настройки валидации
|===

== Gradle Task

=== dependangerGenerateMetadata

Задача `dependangerGenerateMetadata` сериализует DSL модель в файл.

**Inputs:**

* `DependangerMetadata` из DSL extension

**Outputs:**

* `build/dependanger/metadata.json`

== Версионирование схемы

[cols="1,2",options="header"]
|===
| Версия | Изменения

| 1.0
| Начальная версия

| 1.1
| (планируется) Добавлены kmpTargets в Library
|===

При десериализации:

* Проверяется `schemaVersion`
* Поддерживается миграция между версиями
* Неизвестные поля игнорируются (forward compatibility)

== Edge Cases

[cols="1,3,2",options="header"]
|===
| Сценарий | Описание | Ожидаемое поведение

| Невалидный JSON
| Файл `metadata.json` содержит синтаксическую ошибку JSON
| ERROR: "Failed to parse metadata.json: [parse error details]"

| Неизвестная версия схемы
| `schemaVersion` в файле = "99.0", текущая поддерживаемая = "1.1"
| ERROR: "Unsupported schema version '99.0'. Supported: 1.0, 1.1"

| Старая версия схемы
| `schemaVersion` = "1.0", текущая = "1.1"
| Успешная миграция, WARNING: "Migrating from schema 1.0 to 1.1"

| Отсутствует schemaVersion
| JSON файл без поля `schemaVersion`
| ERROR: "Missing required field 'schemaVersion'"

| Неизвестные поля в JSON
| JSON содержит поле `"unknownField": "value"`
| Поле игнорируется (forward compatibility), десериализация успешна

| Пустой JSON файл
| Файл существует, но содержит `{}`
| ERROR: "Missing required field 'schemaVersion'"

| Файл не существует
| Путь указывает на несуществующий файл
| ERROR: "File not found: [path]"

| Нет прав на запись
| Директория существует, но нет прав на запись
| ERROR: "Cannot write to [path]: Permission denied"

| Очень большой файл
| metadata.json содержит 10000+ библиотек
| Успешная обработка (возможно медленнее)

| Специальные символы в значениях
| Библиотека с description содержащим unicode, emoji, переносы строк
| Корректная сериализация с экранированием

| Циклические ссылки в модели
| (не должно быть возможно по дизайну DSL)
| Валидация в DSL предотвращает, но если попало — ERROR при сериализации

| Null значения
| Опциональные поля (tags, requires) = null
| Поля опускаются в JSON или сериализуются как null (configurable)

| Дубликаты ключей
| JSON с дубликатами: `{"kotlin": {...}, "kotlin": {...}}`
| ERROR: "Duplicate key 'kotlin' in libraries"

| Кодировка файла
| Файл в UTF-16 вместо UTF-8
| ERROR или автодетект (implementation-specific)

| Concurrent writes
| Два процесса пишут в один файл одновременно
| Использовать file locking или atomic write (rename)
|===
