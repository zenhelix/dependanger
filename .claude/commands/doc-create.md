# /doc-create

Создать новый документ из шаблона и помочь с первой итерацией.

## Использование

```
/doc-create [тип] [название]
```

## Типы документов

- `feature` — функциональное требование (F###)
- `adr` — Architecture Decision Record (ADR-###)
- `concept` — концептуальный документ
- `interface` — документация интерфейса (CLI, API)

## Примеры

```
/doc-create feature metadata-caching
/doc-create adr version-resolution-strategy
/doc-create concept error-handling
/doc-create interface cli
```

## Инструкции для агента

$ARGUMENTS содержит тип и название через пробел.

### Шаги выполнения

1. **Распарсить аргументы:**
   - Первое слово = тип (feature/adr/concept/interface)
   - Остальное = название (для формирования имени файла)

2. **Определить путь и шаблон:**

   | Тип | Шаблон | Путь назначения |
   |-----|--------|-----------------|
   | feature | `docs/_templates/feature-template.adoc` | `docs/02-features/F###-{название}.adoc` |
   | adr | `docs/_templates/adr-template.adoc` | `docs/05-decisions/ADR-###-{название}.adoc` |
   | concept | `docs/_templates/concept-template.adoc` | `docs/01-concepts/{название}.adoc` |
   | interface | `docs/_templates/interface-template.adoc` | `docs/03-interfaces/{название}.adoc` |

3. **Определить номер** (для feature и adr):
   - Найти существующие файлы в директории
   - Взять следующий свободный номер

4. **Скопировать шаблон** в целевую директорию

5. **Заполнить базовые метаданные:**
   - ID (F### или ADR-###)
   - Дата создания (текущая)
   - Статус: 🔴 Draft (L0)

6. **Начать первую итерацию (L0):**

   Задать пользователю вопросы для заполнения:

   **Для Feature:**
   ```
   Документ создан: docs/02-features/F011-{название}.adoc

   Давайте заполним первую итерацию (L0 Draft).

   1. **Цель фичи** (1-2 предложения):
      Какую проблему пользователя решает эта фича?

   2. **Основная User Story**:
      Как [роль], я хочу [что], чтобы [зачем]?

   3. **Открытые вопросы**:
      Что нужно уточнить прежде чем начать?
   ```

   **Для ADR:**
   ```
   Документ создан: docs/05-decisions/ADR-006-{название}.adoc

   Давайте заполним контекст решения.

   1. **Проблема**:
      Какая ситуация привела к необходимости принять решение?

   2. **Решение**:
      Что вы решили сделать? (формулировка: "Мы будем использовать X")

   3. **Альтернативы**:
      Какие другие варианты рассматривались?
   ```

7. **После получения ответов:**
   - Заполнить соответствующие секции документа
   - Показать результат
   - Предложить следующие шаги

### Формат ответа при создании

```markdown
## Создан документ: [путь]

**Тип:** [Feature / ADR / Concept / Interface]
**Статус:** 🔴 Draft (L0)

---

Давайте заполним первую итерацию.

[Вопросы для заполнения L0]
```

### Формат ответа после заполнения

```markdown
## Документ обновлён

**Текущий уровень:** L0 (Draft)

### Что заполнено:
- ✅ Цель / Контекст
- ✅ Основная User Story / Решение
- ✅ Открытые вопросы

### Следующие шаги для L1:
1. [Что нужно добавить]
2. [Что нужно добавить]

Хотите продолжить итерацию до L1?
```
