== Описание

Processing Pipeline выполняет многоэтапную обработку метаданных:

* Резолвинг версий (из alias, BomImport, Maven)
* Применение фильтров (distribution profile)
* Валидация
* Опциональные проверки (updates, compatibility)

Результат — `EffectiveMetadata`, готовый для генерации артефактов.

== User Stories

=== US-001: Резолвинг версий

[quote]
____
Как разработчик, я хочу чтобы версии резолвились автоматически из разных источников, чтобы не указывать версию для каждой библиотеки вручную.
____

**Acceptance Criteria:**

* Версии из alias резолвятся в конкретные значения
* Версии из BomImport импортируются автоматически
* Указывается источник версии (VersionSource)

=== US-002: Фильтрация по профилю

[quote]
____
Как team lead, я хочу генерировать разные наборы библиотек для разных профилей, чтобы production и development использовали разные зависимости.
____

**Acceptance Criteria:**

* Distribution profile фильтрует библиотеки и бандлы
* excludeLibraries исключает конкретные библиотеки
* includeBundles включает только указанные бандлы

=== US-003: Расширение pipeline

[quote]
____
Как продвинутый пользователь, я хочу добавить кастомный процессор в pipeline, чтобы обогатить метаданные своими данными.
____

**Acceptance Criteria:**

* Можно добавить процессор через SPI
* Процессор имеет приоритет (order)
* Процессор получает и возвращает ProcessingContext

== SPI: EffectiveMetadataProcessor

=== Описание интерфейса

Интерфейс `EffectiveMetadataProcessor` предоставляет точку расширения для добавления кастомных процессоров в pipeline.

**Основные возможности:**

* `order` — порядок выполнения (меньше = раньше)
* `processorId` — уникальный идентификатор процессора
* `isOptional` — можно ли отключить процессор (по умолчанию `false`)
* `process()` — suspend-функция обработки метаданных

== Processing Context

`ProcessingContext` содержит информацию об окружении, которая используется процессорами для принятия решений.

=== Параметры контекста

* `jdkVersion` — версия JDK (для условий fallback)
* `kotlinVersion` — версия Kotlin (опционально)
* `gradleVersion` — версия Gradle (опционально)
* `distributionProfile` — активный distribution profile
* `environment` — переменные окружения
* `strictVersionResolution` — режим строгой резолюции версий
* `enabledProcessors` / `disabledProcessors` — управление процессорами

== Version Source Priority

При резолвинге версий применяется приоритет:

[source]
----
DECLARED > BOM_IMPORT
----

[cols="1,1,3",options="header"]
|===
| Приоритет | Source | Описание

| 1 (высший)
| DECLARED
| Явно указана в DSL (включая inline версию в coordinates)

| 2 (низший)
| BOM_IMPORT
| Импортирована из BomImport через BomImportProcessor
|===

== Strict Mode

[source,kotlin]
----
dependanger {
    settings {
        strictVersionResolution = true  // ERROR при неразрешённых версиях
    }
}
----

При `strictVersionResolution = true`:

* Неразрешённая версия → ERROR (остановка pipeline)
* При `false` → WARNING (продолжение с пустой версией)

== Parallel Pipeline Execution

Pipeline поддерживает параллельное выполнение процессоров через фазы:

* **SEQUENTIAL** — строго последовательное выполнение
* **PARALLEL_IO** — параллельно I/O-bound операции (сетевые запросы)
* **PARALLEL_COMPUTE** — параллельно CPU-bound операции (валидация)

=== Фазы выполнения

[cols="1,2,2",options="header"]
|===
| Фаза | Процессоры | Описание

| SEQUENTIAL
| profile, metadata-conversion, extracted-versions, version-fallback, version-resolver
| Требуют последовательного выполнения (зависят от результатов предыдущих)

| PARALLEL_IO
| bom-import, update-check
| I/O-bound операции (сетевые запросы к Maven Central)

| PARALLEL_COMPUTE
| validation, compat-rules, compatibility-analysis
| CPU-bound операции (могут выполняться параллельно)
|===

== Pipeline Presets

Предустановленные конфигурации pipeline:

[cols="1,2,2",options="header"]
|===
| Preset | Описание | Включённые процессоры

| `DEFAULT`
| Стандартная конфигурация
| Все обязательные + bom-import

| `MINIMAL`
| Минимальный набор без опциональных
| Только обязательные процессоры

| `STRICT`
| Все проверки включены
| Все процессоры + strictVersionResolution

| `DISTRIBUTION`
| Для генерации артефактов
| Обязательные + profile фильтрация
|===

[source,kotlin]
----
dependanger {
    processing {
        preset = ProcessingPreset.STRICT
        // или кастомная настройка:
        withBomImport()
        withUpdateCheck()
        withoutCompatibilityAnalysis()
    }
}
----
