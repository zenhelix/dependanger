[[version-types]]
== Типы версий

[[version-reference]]
=== VersionReference

Ссылка на версию — может быть литералом, ссылкой на alias или диапазоном.

[source,kotlin]
----
sealed class VersionReference {
    data class Literal(val version: String) : VersionReference()
    data class Reference(val name: String) : VersionReference()
    data class Range(val range: VersionRange) : VersionReference()
}
----

[cols="1,3,2",options="header"]
|===
| Тип | Описание | Пример DSL

| `Literal`
| Явно указанная версия
| `"2.0.9"` или `version = "2.0.9"`

| `Reference`
| Ссылка на именованную версию
| `versionRef("kotlin")`

| `Range`
| Диапазон версий
| `versionRange("[1.0, 2.0)")`
|===

[[version-fallback]]
=== VersionFallback

Условная альтернативная версия.

[cols="1,1,3",options="header"]
|===
| Поле | Тип | Описание

| `value`
| String
| Альтернативное значение версии

| `condition`
| FallbackCondition
| Условие, при котором используется fallback
|===

[[version-source]]
=== VersionSource

Источник разрешённой версии (в Effective Model).

[cols="1,1,3",options="header"]
|===
| Приоритет | Значение | Описание

| 1 (высший)
| DECLARED
| Явно указана в DSL конфигурации через `version = "1.0.0"`, `versionRef("alias")` или inline в coordinates

| 2 (низший)
| BOM_IMPORT
| Импортирована из BomImport через BomImportProcessor
|===

==== Примеры VersionSource

[source,kotlin]
----
// DECLARED — явно указана версия
library("guava", "com.google.guava:guava", version = "33.0.0-jre")

// DECLARED — через versionRef
library("kotlin-stdlib", "org.jetbrains.kotlin:kotlin-stdlib",
    version = versionRef("kotlin"))

// DECLARED — inline версия в coordinates
library("slf4j", "org.slf4j:slf4j-api:2.0.9")

// BOM_IMPORT — версия будет из Spring Boot BOM
library("spring-web", "org.springframework:spring-web")  // без версии
----

[[resolved-version]]
=== ResolvedVersion

Версия с информацией об источнике (в Effective Model).

[cols="1,1,3",options="header"]
|===
| Атрибут | Тип | Описание

| `alias`
| String
| Alias версии из DSL (PK в map)

| `value`
| String
| Значение версии (например: `3.4.0`)

| `source`
| VersionSource
| Источник версии (DECLARED или BOM_IMPORT)

| `originalRef`
| VersionReference?
| Оригинальная ссылка на версию из DSL (до резолвинга)
|===

[[constraint-types]]
== Типы ограничений

[[constraint]]
=== Constraint

Ограничение для библиотеки — sealed class.

[source,kotlin]
----
sealed class Constraint {
    data class VersionConstraintDef(val constraint: String) : Constraint()
    data class Exclude(val group: String?, val module: String?) : Constraint()
    data class Substitute(val with: String) : Constraint()
}
----

[cols="1,3",options="header"]
|===
| Подтип | Описание

| `VersionConstraintDef`
| Ограничение версии (например: `strictly`, `require`, `prefer`)

| `Exclude`
| Исключение транзитивной зависимости по group/module

| `Substitute`
| Подмена зависимости на другую библиотеку (alias)
|===
