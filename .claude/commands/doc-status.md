# /doc-status

Показать статус всей документации проекта — сводка по уровням зрелости.

## Использование

```
/doc-status
/doc-status [директория]    # только указанная директория
```

## Примеры

```
/doc-status
/doc-status docs/02-features
/doc-status docs/05-decisions
```

## Инструкции для агента

$ARGUMENTS может содержать путь к директории. Если пустой — проверить всю `docs/`.

### Шаги выполнения

1. **Найти все .adoc файлы** в указанной директории (рекурсивно)

2. **Исключить:**
    - `docs/_templates/` (шаблоны)
    - `docs/index.adoc` (индексные файлы)
    - Файлы начинающиеся с `_`

3. **Для каждого документа определить:**
    - Тип (Feature, ADR, Concept, Interface, Other)
    - Уровень зрелости (L0-L3)
    - Статус из метаданных если есть

4. **Сформировать сводку** по категориям и уровням

5. **Выделить проблемные документы:**
    - L0 (Draft) — требуют внимания
    - Документы со статусом "В работе" давно не обновлявшиеся
    - Документы с большим количеством TODO

### Как определить уровень зрелости

**L0 (Draft):**

- Много TODO
- Нет user stories / контекста
- Только идея

**L1 (Minimal Viable):**

- Есть цель
- Есть хотя бы 1 user story
- Есть зависимости

**L2 (Development Ready):**

- Есть AC в Given/When/Then
- Есть edge cases
- Есть FR

**L3 (Complete):**

- Все AC testable
- Нет TODO
- Все секции заполнены

### Формат ответа

```markdown
## Статус документации проекта

**Всего документов:** [N]
**Дата проверки:** [дата]

### Сводка по уровням

| Уровень | Количество | Документы |
|---------|------------|-----------|
| 🟢 L3 Complete | N | F003, F004, ADR-001 |
| 🟡 L2 Development Ready | N | F001, F002 |
| 🟠 L1 Minimal Viable | N | F007 |
| 🔴 L0 Draft | N | F010, ADR-004 |

### По категориям

**Features (docs/02-features/):**
| Документ | Уровень | Последнее обновление |
|----------|---------|---------------------|
| F001-registry-dsl | L2 | 2024-01-15 |
| F003-processing-pipeline | L3 | 2024-01-20 |
| ... | ... | ... |

**ADR (docs/05-decisions/):**
| Документ | Статус | Дата |
|----------|--------|------|
| ADR-001-kotlinx-serialization | 🟢 Принято | 2024-01-10 |
| ADR-002-pipeline-architecture | 🟢 Принято | 2024-01-12 |
| ... | ... | ... |

**Concepts (docs/01-concepts/):**
| Документ | Уровень |
|----------|---------|
| architecture | L2 |
| domain-model | L2 |
| constitution | L2 |
| ... | ... |

### Требуют внимания

🔴 **Документы на L0 (Draft):**
1. `F010-new-feature.adoc` — нет user stories
2. `ADR-004-caching.adoc` — только контекст

🟠 **Документы на L1 давно не обновлялись:**
1. `F007-update-check.adoc` — последнее обновление 30+ дней назад

⚠️ **Много TODO:**
1. `F005-bom-generation.adoc` — 12 TODO
2. `interface-cli.adoc` — 8 TODO

### Рекомендации

1. **Приоритет 1:** Довести F010 до L1 (нужно для планирования)
2. **Приоритет 2:** Закрыть TODO в F005 (блокирует разработку)
3. **Приоритет 3:** Обновить F007 (устаревшая информация)
```

### Краткий формат (для CI)

Если запрошен краткий формат:

```
Documentation Status: 15 docs
L3: 3 | L2: 5 | L1: 4 | L0: 3

⚠️ Attention needed:
- F010 (L0) - no user stories
- F007 (L1) - stale (30+ days)
```
