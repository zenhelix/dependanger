== Обзор

CLI предоставляет standalone интерфейс для Dependanger без зависимости от Gradle.
Построен на базе https://ajalt.github.io/clikt/[Clikt].

[cols="1,2",options="header"]
|===
| Характеристика | Значение

| Бинарное имя
| `dependanger`

| Формат входных данных
| JSON (metadata.json)

| Целевая аудитория
| DevOps, CI/CD
|===

== Установка

=== Homebrew (macOS/Linux)

[source,bash]
----
brew install zenhelix/tap/dependanger
----

=== SDKMAN

[source,bash]
----
sdk install dependanger
----

=== Manual

[source,bash]
----
# Скачать релиз
curl -LO https://github.com/zenhelix/dependanger/releases/latest/download/dependanger.zip
unzip dependanger.zip
chmod +x dependanger/bin/dependanger

# Добавить в PATH
export PATH="$PWD/dependanger/bin:$PATH"
----

== Команды

=== generate

Генерация TOML Version Catalog и/или Maven BOM из metadata.

[source,bash]
----
dependanger generate [OPTIONS]
----

==== Опции

[cols="1,2,1",options="header"]
|===
| Опция | Описание | Default

| `--toml`
| Генерировать TOML Version Catalog
| `false` (если не указан ни один формат, генерируется TOML)

| `--bom`
| Генерировать Maven BOM
| `false`

| `-i, --input <path>`
| Входной metadata.json файл
| обязательный

| `-o, --output <path>`
| Директория для выходных файлов
| `.`

| `-d, --distribution <name>`
| Генерировать только для указанной дистрибуции
| все

| `--all`
| Генерировать для всех дистрибуций
| `false`

| `--toml-name <name>`
| Имя выходного TOML файла
| `libs.versions.toml`

| `--group-id <id>`
| Group ID для BOM
| `io.github.zenhelix`

| `--artifact-id <id>`
| Artifact ID для BOM
| `dependanger-bom`
|===

==== Exit Codes

[cols="1,2",options="header"]
|===
| Код | Описание

| `0`
| Генерация успешна

| `1`
| Ошибка валидации входных данных

| `2`
| Ошибка записи выходных файлов
|===

==== Примеры

[source,bash]
----
# Генерация TOML (default)
dependanger generate --input metadata.json --output ./build

# Генерация BOM
dependanger generate --bom --input metadata.json \
    --group-id com.example --artifact-id my-bom

# Генерация TOML и BOM
dependanger generate --toml --bom --input metadata.json

# Для конкретной дистрибуции
dependanger generate --toml --distribution android --input metadata.json

# Для всех дистрибуций
dependanger generate --toml --all --input metadata.json --output ./catalogs
----

=== validate

Валидация структуры metadata JSON и проверка на ошибки.

[source,bash]
----
dependanger validate [OPTIONS]
----

==== Опции

[cols="1,2,1",options="header"]
|===
| Опция | Описание | Default

| `-i, --input <path>`
| Входной metadata.json файл
| обязательный

| `--strict`
| Строгий режим — ошибка при нерезолвленных версиях
| `false`

| `--format <format>`
| Формат вывода: `text` или `json`. Алиас: `--json` эквивалентен `--format json`
| `text`
|===

==== Exit Codes

[cols="1,2",options="header"]
|===
| Код | Описание

| `0`
| Валидация успешна

| `1`
| Есть предупреждения (warnings)

| `2`
| Есть ошибки (errors)
|===

==== Примеры

[source,bash]
----
# Базовая валидация
dependanger validate --input metadata.json

# Строгий режим
dependanger validate --strict --input metadata.json

# JSON вывод для CI
dependanger validate --json --input metadata.json
----

=== check-updates

Проверка доступных обновлений версий зависимостей.

[source,bash]
----
dependanger check-updates [OPTIONS]
----

==== Опции

[cols="1,2,1",options="header"]
|===
| Опция | Описание | Default

| `-i, --input <path>`
| Входной metadata.json файл
| обязательный

| `--major, --show-major`
| Показывать major обновления
| `true`

| `--minor, --show-minor`
| Показывать minor обновления
| `true`

| `--patch, --show-patch`
| Показывать patch обновления
| `true`

| `--pre-release, --include-pre-release`
| Включить pre-release версии
| `false`

| `--format <format>`
| Формат вывода: `text` или `json`. Алиас: `--json` эквивалентен `--format json`
| `text`

| `--fail, --fail-on-major`
| Exit code 1 при наличии major обновлений
| `false`
|===

==== Exit Codes

[cols="1,2",options="header"]
|===
| Код | Описание

| `0`
| Проверка завершена, major обновлений нет (или `--fail-on-major` не указан)

| `1`
| Есть major обновления (при `--fail-on-major`)

| `2`
| Ошибка при проверке (невалидный input, сетевая ошибка)
|===

==== Примеры

[source,bash]
----
# Проверка всех обновлений
dependanger check-updates --input metadata.json

# Только major обновления
dependanger check-updates --major --input metadata.json

# С pre-release версиями
dependanger check-updates --include-pre-release --input metadata.json

# CI: fail при major обновлениях
dependanger check-updates --fail-on-major --input metadata.json

# JSON вывод
dependanger check-updates --json --input metadata.json
----

=== security-check

Проверка библиотек на известные уязвимости (CVE) через OSV API.

[source,bash]
----
dependanger security-check [OPTIONS]
----

==== Опции

[cols="1,2,1",options="header"]
|===
| Опция | Описание | Default

| `-i, --input <path>`
| Входной metadata.json файл
| обязательный

| `--min-severity <level>`
| Минимальный severity: `critical`, `high`, `medium`, `low`, `all`
| `high`

| `--min-cvss <score>`
| Минимальный CVSS score (0.0 - 10.0)
| `7.0`

| `--fail-on <level>`
| Fail при severity >= level: `critical`, `high`, `medium`, `low`, `none`
| `none`

| `--ignore-cve <cve>`
| Игнорировать CVE (можно указать несколько раз)
| —

| `--ignore-library <alias>`
| Игнорировать библиотеку (можно указать несколько раз)
| —

| `--format <format>`
| Формат вывода: `text` или `json`. Алиас: `--json` эквивалентен `--format json`
| `text`

| `-o, --output <path>`
| Записать результат в файл
| —
|===

==== Exit Codes

[cols="1,2",options="header"]
|===
| Код | Описание

| `0`
| Проверка завершена, критичных уязвимостей нет (или `--fail-on none`)

| `1`
| Найдены уязвимости >= указанного severity (при `--fail-on`)

| `2`
| Ошибка выполнения (невалидный input, OSV API недоступен)
|===

==== Примеры

[source,bash]
----
# Базовая проверка (HIGH+ уязвимости)
dependanger security-check --input metadata.json

# Все уязвимости включая LOW
dependanger security-check --min-severity all --input metadata.json

# Fail при CRITICAL
dependanger security-check --fail-on critical --input metadata.json

# Игнорировать известный CVE
dependanger security-check --ignore-cve CVE-2023-ACCEPTED --input metadata.json

# JSON вывод для CI
dependanger security-check --json --output security-report.json --input metadata.json
----

=== license-check

Проверка лицензий библиотек на соответствие политикам.

| обязательный

| Добавить лицензию в whitelist (можно указать несколько раз)

| `text`

| —
|===

[cols="1,2",options="header"]

=== resolve-transitives

== Управление метаданными

Команды для создания и редактирования metadata.json без Gradle.

=== init

Создание нового metadata.json файла.

[source,bash]
----
dependanger init [OPTIONS]
----

==== Опции

[cols="1,2,1",options="header"]
|===
| Опция | Описание | Default

| `-o, --output <path>`
| Путь к выходному файлу
| `./metadata.json`

| `--name <name>`
| Название проекта
| —

| `--kotlin-version <version>`
| Версия Kotlin (добавляет version alias)
| —

| `--jdk-version <version>`
| Целевая версия JDK
| `17`

| `--force`
| Перезаписать существующий файл
| `false`
|===

==== Примеры

[source,bash]
----
# Базовая инициализация
dependanger init --output metadata.json

# С начальными настройками
dependanger init --name "my-project" --kotlin-version 2.1.20 --jdk-version 21

# Перезаписать существующий
dependanger init --force --output metadata.json
----

dependanger add-version spring-boot 3.4.1 --fallback "jdkBelow(17)=2.7.18"
----

=== add-library

Добавить новую библиотеку в metadata.

[source,bash]
----
dependanger add-library <alias> <coordinates> [OPTIONS]
----

==== Опции

[cols="1,2,1",options="header"]
|===
| Опция | Описание | Default

| `-i, --input <path>`
| Путь к metadata.json
| `./metadata.json`

| `--version <version>`
| Inline версия или ссылка на alias (ref:name)
| —

| `--tags <tags>`
| Теги через запятую
| —

| `--requires-jdk <version>`
| Минимальная версия JDK
| —
|===

==== Примеры

[source,bash]
----
# С inline версией
dependanger add-library guava com.google.guava:guava --version 33.0.0-jre --tags core,utils

# С ссылкой на version alias
dependanger add-library kotlin-stdlib org.jetbrains.kotlin:kotlin-stdlib --version ref:kotlin --tags core,kotlin

# Без версии (будет из BOM)
dependanger add-library spring-web org.springframework:spring-web --tags web,spring
----

=== add-plugin

Добавить новый Gradle plugin в metadata.

[source,bash]
----
dependanger add-plugin <alias> <plugin-id> [OPTIONS]
----

==== Примеры

[source,bash]
----
dependanger add-plugin kotlin-jvm org.jetbrains.kotlin.jvm --version ref:kotlin

=== add-bundle

Добавить новый bundle в metadata.

[source,bash]
----
dependanger add-bundle <name> [OPTIONS]
----

==== Опции

[cols="1,2,1",options="header"]
|===
| Опция | Описание | Default

| `--libraries <aliases>`
| Alias'ы библиотек через запятую
| —

| `--extends <bundles>`
| Имена бандлов для наследования через запятую
| —
|===

==== Примеры

[source,bash]
----
dependanger add-bundle kotlin-core --libraries kotlin-stdlib,kotlin-reflect
dependanger add-bundle kotlin-full --extends kotlin-core --libraries kotlinx-coroutines-core
----

=== add-bom

Добавить новый BOM import в metadata.

[source,bash]
----
dependanger add-bom <coordinates> [OPTIONS]
----

==== Опции

[cols="1,2,1",options="header"]
|===
| Опция | Описание | Default

| `-i, --input <path>`
| Путь к metadata.json
| `./metadata.json`

| `--alias <alias>`
| Alias для BOM (если не указан, генерируется из artifactId)
| —

| `--version <version>`
| Версия BOM или ссылка на alias (ref:name)
| —
|===

==== Примеры

[source,bash]
----
dependanger add-bom org.springframework.boot:spring-boot-dependencies:3.4.1
dependanger add-bom org.jetbrains.kotlin:kotlin-bom --version ref:kotlin
dependanger add-bom org.junit:junit-bom:5.11.0 --alias junit-bom
----

NOTE: Команда соответствует DSL блоку `bomImports { bom(...) }`.
В терминологии Gradle/Maven это называется "platform", но в Dependanger используется термин "BOM import" для избежания путаницы с `targetPlatforms`.

=== add-distribution

Добавить новый distribution profile в metadata.

[source,bash]
----
dependanger add-distribution <name> [OPTIONS]
----

==== Опции

[cols="1,2,1",options="header"]
|===
| Опция | Описание | Default

| `--include-tags <tags>`
| Теги для включения (anyOf)
| —

| `--exclude-tags <tags>`
| Теги для исключения
| —

| `--include-bundles <bundles>`
| Бандлы для включения
| —
|===

==== Примеры

[source,bash]
----
dependanger add-distribution android --include-tags android,mobile --exclude-tags server
dependanger add-distribution backend --include-bundles spring-backend,kotlin-core
----

=== update-*

Обновить существующий элемент в metadata.

[source,bash]
----
dependanger update-version <alias> [OPTIONS]
dependanger update-library <alias> [OPTIONS]
----

Опции аналогичны соответствующим `add-*` командам.

[source,bash]

dependanger update-version kotlin 2.1.21

# Добавить теги к библиотеке
dependanger update-library guava --tags core,utils,google
----

=== migrate-deprecated

Автоматическая миграция deprecated библиотек.

[source,bash]
----
dependanger migrate-deprecated [OPTIONS]
----

==== Опции

[cols="1,2",options="header"]
|===
| Опция | Описание

| `--dry-run`
| Показать план миграции без выполнения

| `--replace`
| Заменить deprecated на replacedBy в бандлах (default: true)

| `--remove`
| Удалить deprecated библиотеки из metadata (default: false)

| `--remove-from-bundles`
| Удалить deprecated из бандлов вместо замены (default: false)

| `-i, --input <FILE>`
| Входной файл (default: `./dependanger-metadata.json`)

| `-o, --output <FILE>`
| Выходной файл (default: перезаписать input)

| `--backup`
| Создать backup файла перед изменением
|===

==== Примеры

[source,bash]
----
# Показать план миграции
dependanger migrate-deprecated --dry-run

# Заменить deprecated на replacedBy в бандлах
dependanger migrate-deprecated --replace

# Удалить deprecated библиотеки полностью
dependanger migrate-deprecated --remove

# Комбинировать: заменить в бандлах и удалить определения
dependanger migrate-deprecated --replace --remove
----

==== Вывод --dry-run

[source]
----
Migration plan for: dependanger-metadata.json

Deprecated libraries found: 3

1. old-http -> new-http
   - Used in bundles: legacy-bundle, http-utils
   - Action: Replace with 'new-http' in bundles

2. legacy-json -> modern-json
   - Used in bundles: utils
   - Action: Replace with 'modern-json' in bundles

3. abandoned-lib (no replacement)
   - Used in bundles: misc
   - Action: WARNING - no replacement available, manual migration required

Summary:
- Libraries to replace: 2
- Libraries requiring manual migration: 1

Run without --dry-run to apply changes.
----

==== Вывод после миграции

[source]
----
Migration completed:

✓ Replaced 'old-http' with 'new-http' in bundle 'legacy-bundle'
✓ Replaced 'old-http' with 'new-http' in bundle 'http-utils'
✓ Replaced 'legacy-json' with 'modern-json' in bundle 'utils'
⚠ Skipped 'abandoned-lib' - no replacement available

Updated: dependanger-metadata.json
Backup: dependanger-metadata.json.bak

Review changes and commit.
----

== Конфигурационный файл

CLI поддерживает конфигурационный файл `.dependanger.yaml`:

[source,yaml]
----
# .dependanger.yaml
defaults:
  input: ./dependanger-metadata.json
  output: ./build/dependanger

generate:
  toml:
    filename: libs.versions.toml
  bom:
    groupId: com.example
    artifactId: my-bom

check-updates:
  include-pre-release: false
  fail-on-major: true

analyze:
  target-jdk: 17
  fail-on-error: true
----

== CI/CD Интеграция

=== GitHub Actions

[source,yaml]
----
name: Dependency Check

on:
  schedule:
    - cron: '0 8 * * 1'  # Каждый понедельник в 8:00
  workflow_dispatch:

jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Install Dependanger
        run: |
          curl -LO https://github.com/zenhelix/dependanger/releases/latest/download/dependanger.zip
          unzip dependanger.zip
          echo "$PWD/dependanger/bin" >> $GITHUB_PATH
      
      - name: Validate metadata
        run: dependanger validate --strict --input metadata.json
      
      - name: Check for updates
        run: dependanger check-updates --json --input metadata.json > updates.json
      
      - name: Analyze compatibility
        run: dependanger analyze --fail-on-error --target-jdk 17 --input metadata.json
      
      - name: Upload report
        uses: actions/upload-artifact@v4
        with:
          name: dependency-report
          path: updates.json
----

dependency-check:
