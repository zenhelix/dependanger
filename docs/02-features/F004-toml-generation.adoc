:toc: left

[abstract]
--
Генерация Gradle Version Catalog (TOML) из EffectiveMetadata.
Основной выходной формат для Gradle-проектов.
Артефакт публикуется для использования в других Gradle-проектах организации.
--

== Описание

TOML Generation преобразует `EffectiveMetadata` в Gradle Version Catalog — стандартный формат для управления зависимостями в Gradle 7.0+.

== User Stories

=== US-001: Генерация Version Catalog

[quote]
____
Как Gradle разработчик, я хочу получить libs.versions.toml из моего каталога зависимостей, чтобы использовать централизованное управление зависимостями.
____

**Acceptance Criteria:**

[cols="1,3,3,3",options="header"]
|===
| ID | Given | When | Then

| AC-001-1
| EffectiveMetadata с 3 библиотеками и 2 версиями
| Вызываю `TomlGenerator.generate(metadata)`
| Создаётся TOML с секциями `[versions]` (2 записи), `[libraries]` (3 записи)

| AC-001-2
| EffectiveMetadata с бандлом `kotlin-core` содержащим 2 библиотеки
| Генерирую TOML
| Секция `[bundles]` содержит `kotlin-core = ["kotlin-stdlib", "kotlin-reflect"]`

| AC-001-3
| EffectiveMetadata с плагином `kotlin-jvm`
| Генерирую TOML
| Секция `[plugins]` содержит `kotlin-jvm = { id = "...", version.ref = "..." }`

| AC-001-4
| Сгенерированный TOML файл
| Подключаю в Gradle проект через `versionCatalogs { from(files(...)) }`
| Gradle успешно парсит каталог, `libs.kotlin.stdlib` доступен

| AC-001-5
| Gradle task `dependangerGenerateToml`
| Запускаю task без параметров
| Файл создан в `gradle/libs.versions.toml`
|===

=== US-002: Генерация для профиля

[quote]
____
Как team lead, я хочу генерировать разные TOML файлы для разных профилей, чтобы production и development проекты использовали разные зависимости.
____

**Acceptance Criteria:**

[cols="1,3,3,3",options="header"]
|===
| ID | Given | When | Then

| AC-002-1
| EffectiveMetadata с distribution `backend` (5 из 10 библиотек)
| Генерирую TOML для distribution `backend`
| TOML содержит только 5 библиотек из профиля `backend`

| AC-002-2
| Distribution profile `android` с фильтром `byTags { include("android") }`
| Генерирую TOML
| Только библиотеки с тегом `android` включены

| AC-002-3
| Настройка `toml { filename = "libs-\{distribution\}.versions.toml" }`
| Генерирую для distribution `production`
| Файл называется `libs-production.versions.toml`

| AC-002-4
| Два distribution: `backend` и `android`
| Генерирую TOML для обоих
| Создаются два файла с разным содержимым
|===

=== US-003: Комментарии для deprecated библиотек

[quote]
____
Как разработчик, я хочу видеть информацию о deprecated библиотеках в сгенерированном TOML, чтобы понимать какие зависимости нужно обновить.
____

**Acceptance Criteria:**

[cols="1,3,3,3",options="header"]
|===
| ID | Given | When | Then

| AC-003-1
| Deprecated библиотека `old-http` с `replacedBy = "new-http"`
| Генерирую TOML с `includeComments = true`
| Перед библиотекой комментарий: `# DEPRECATED: Use new-http`

| AC-003-2
| Deprecated библиотека с `removalVersion = "2.0.0"`
| Генерирую TOML
| Комментарий содержит: `Will be removed in 2.0.0`

| AC-003-3
| Deprecated библиотека с `message = "Use new-http which supports HTTP/2"`
| Генерирую TOML
| Комментарий содержит сообщение из `message`

| AC-003-4
| Генерация с `includeComments = false`
| Генерирую TOML
| Deprecated библиотеки включены, но без комментариев
|===

=== US-004: Platform Library в TOML

[quote]
____
Как Gradle разработчик, я хочу использовать platform libraries (BOM) из Version Catalog, чтобы управлять версиями через BOM.
____

**Acceptance Criteria:**

[cols="1,3,3,3",options="header"]
|===
| ID | Given | When | Then

| AC-004-1
| EffectiveMetadata с `platformLibrary("spring-bom", "org.springframework.boot:spring-boot-dependencies")`
| Генерирую TOML
| Библиотека добавлена в `[libraries]`: `spring-bom = { group = "...", name = "..." }`

| AC-004-2
| TOML с platform library `spring-bom`
| Использую в build.gradle.kts
| Можно использовать: `implementation(platform(libs.spring.bom))`

| AC-004-3
| Platform library с inline версией
| Генерирую TOML
| Версия включена inline: `spring-bom = { group = "...", name = "...", version = "3.4.0" }`
|===

NOTE: Gradle Version Catalogs не имеют отдельного типа для platform libraries.
Platform libraries генерируются как обычные библиотеки, а пользователь оборачивает их в `platform()` при использовании.

== SPI: ArtifactGenerator

=== Описание интерфейса

Интерфейс `ArtifactGenerator<T>` предоставляет точку расширения для добавления новых форматов генерации артефактов.

**Основные возможности:**

* `generatorId` — уникальный идентификатор генератора (например: "toml", "bom")
* `generate()` — создание артефакта из EffectiveMetadata
* `write()` — запись артефакта в файл

=== TomlGenerator

`TomlGenerator` реализует генерацию Gradle Version Catalog формата TOML. Принимает `EffectiveMetadata` и создаёт структуру с секциями `[versions]`, `[libraries]`, `[bundles]`, `[plugins]`.

== Формат TOML

=== Структура

[source,toml]
----
# Generated by Dependanger
# Distribution: production
# Generated at: 2026-01-26T10:30:00Z

[versions]
kotlin = "2.1.0"
spring-boot = "3.2.0"
ktor = "3.0.0"

[libraries]
kotlin-stdlib = { group = "org.jetbrains.kotlin", name = "kotlin-stdlib", version.ref = "kotlin" }
spring-boot-starter = { group = "org.springframework.boot", name = "spring-boot-starter", version.ref = "spring-boot" }
ktor-server-core = { group = "io.ktor", name = "ktor-server-core", version.ref = "ktor" }

# DEPRECATED: Use new-http which supports HTTP/2
# Replacement: new-http
# Will be removed in: 2.0.0
old-http = { group = "com.example", name = "old-http", version.ref = "old-http" }

[bundles]
kotlin-core = ["kotlin-stdlib", "kotlin-reflect"]
spring-backend = ["spring-boot-starter", "spring-boot-starter-web"]

[plugins]
kotlin-jvm = { id = "org.jetbrains.kotlin.jvm", version.ref = "kotlin" }
spring-boot = { id = "org.springframework.boot", version.ref = "spring-boot" }
----

=== Именование

[cols="1,2,2",options="header"]
|===
| Сущность | DSL alias | TOML key

| Library
| `kotlin-stdlib`
| `kotlin-stdlib`

| Version
| `spring-boot`
| `spring-boot`

| Bundle
| `spring-backend`
| `spring-backend`

| Plugin
| `kotlin-jvm`
| `kotlin-jvm`
|===

TOML ключи = DSL alias без изменений (нижний регистр, дефисы).

== Gradle Task

=== dependangerGenerateToml

Задача `dependangerGenerateToml` генерирует TOML Version Catalog из EffectiveMetadata.

**Inputs:**

* `effective-\{distribution\}.json` от предыдущего таска

**Outputs:**

* `gradle/libs.versions.toml` (или кастомный путь)

**Зависимости:**

* Зависит от `dependangerGenerateEffective`

== CLI Command

[source,bash]
----
dependanger generate --toml [OPTIONS]
----

=== Опции

[cols="1,2,1",options="header"]
|===
| Опция | Описание | Default

| `-i, --input <path>`
| Входной metadata.json файл
| обязательный

| `-o, --output <path>`
| Директория для выходных файлов
| `.`

| `-d, --distribution <name>`
| Генерировать для указанной дистрибуции
| все

| `--all`
| Генерировать для всех дистрибуций
| `false`

| `--toml-name <name>`
| Имя выходного TOML файла
| `libs.versions.toml`

| `--include-comments`
| Добавлять комментарии (deprecated, metadata)
| `true`

| `--sort`
| Сортировать секции алфавитно
| `true`
|===

=== Примеры

[source,bash]
----
# Генерация TOML (default)
dependanger generate --toml --input metadata.json

# Для конкретного distribution
dependanger generate --toml --distribution backend --input metadata.json

# Для всех distributions
dependanger generate --toml --all --input metadata.json --output ./catalogs

# Без комментариев
dependanger generate --toml --no-include-comments --input metadata.json
----

NOTE: Подробное описание CLI см. в xref:../03-interfaces/cli.adoc[CLI Interface].

== Настройки

=== DSL конфигурация

[source,kotlin]
----
dependanger {
    settings {
        toml {
            // Имя файла (поддерживает placeholder {distribution})
            filename = "libs.versions.toml"
            // или для multi-distribution:
            filename = "libs-{distribution}.versions.toml"

            // Добавлять комментарии о deprecated библиотеках
            includeDeprecationComments = true

            // Добавлять метаданные (generated at, distribution)
            includeMetadata = true

            // Сортировать секции алфавитно
            sortAlphabetically = true

            // Использовать inline версии вместо version.ref
            // (не рекомендуется, но поддерживается)
            useInlineVersions = false

            // Выходная директория
            outputDirectory = file("gradle")
        }
    }
}
----

=== Inline версии

По умолчанию генерируются ссылки на версии через `version.ref`.
Можно включить inline версии:

[source,toml]
----
# С version.ref (default)
kotlin-stdlib = { group = "org.jetbrains.kotlin", name = "kotlin-stdlib", version.ref = "kotlin" }

# С inline версией (useInlineVersions = true)
kotlin-stdlib = { group = "org.jetbrains.kotlin", name = "kotlin-stdlib", version = "2.1.20" }
----

WARNING: Inline версии не рекомендуются, так как затрудняют централизованное управление версиями.

== Использование в Gradle

=== settings.gradle.kts

[source,kotlin]
----
dependencyResolutionManagement {
    versionCatalogs {
        create("libs") {
            from(files("gradle/libs.versions.toml"))
        }
    }
}
----

=== build.gradle.kts

[source,kotlin]
----
dependencies {
    implementation(libs.kotlin.stdlib)
    implementation(libs.spring.boot.starter)
    implementation(libs.bundles.kotlin.core)
}

plugins {
    alias(libs.plugins.kotlin.jvm)
}
----

== Валидация

[cols="1,2,1",options="header"]
|===
| Правило | Описание | Severity

| Уникальность ключей
| Все ключи в секции уникальны
| ERROR

| Валидные ссылки
| version.ref указывает на существующую версию
| ERROR

| Валидные бандлы
| Библиотеки в бандле существуют
| ERROR

| TOML синтаксис
| Ключи соответствуют TOML spec

== Функциональные требования

[cols="1,1,4",options="header"]
|===
| ID | Приоритет | Требование

| FR-001
| MUST
| Система ДОЛЖНА генерировать валидный TOML файл, совместимый с Gradle Version Catalogs

| FR-002
| MUST
| Система ДОЛЖНА включать секции `[versions]`, `[libraries]`, `[bundles]`, `[plugins]`

| FR-003
| MUST
| Система ДОЛЖНА использовать `version.ref` для ссылок на версии (не inline версии)

| FR-004
| MUST
| Система ДОЛЖНА сохранять DSL alias как TOML ключи без трансформации

| FR-005
| MUST
| Система ДОЛЖНА валидировать TOML перед записью (ключи, ссылки, синтаксис)

| FR-006
| SHOULD
| Система ДОЛЖНА поддерживать генерацию для конкретного Distribution

| FR-007
| SHOULD
| Система ДОЛЖНА поддерживать кастомное имя выходного файла

| FR-008
| SHOULD
| Система ДОЛЖНА добавлять комментарии с метаданными (generated at, distribution)

| FR-009
| SHOULD
| Система ДОЛЖНА сортировать секции алфавитно (опционально)

| FR-010
| MAY
| Система МОЖЕТ поддерживать inline версии для библиотек без versionRef

| FR-011
| SHOULD
| Система ДОЛЖНА добавлять комментарии о deprecation для deprecated библиотек (replacedBy, message, removalVersion)

| FR-012
| SHOULD
| Система ДОЛЖНА поддерживать генерацию TOML через CLI команду `dependanger generate --toml`

| FR-013
| SHOULD
| Система ДОЛЖНА поддерживать placeholder `\{distribution\}` в имени файла для multi-distribution

| FR-014
| SHOULD
| Система ДОЛЖНА генерировать platform libraries как обычные библиотеки (пользователь оборачивает в `platform()`)

| FR-015
| MAY
| Система МОЖЕТ поддерживать опцию `useInlineVersions` для inline версий вместо version.ref

== Edge Cases

[cols="1,3,2",options="header"]
|===
| Сценарий | Описание | Ожидаемое поведение

| Пустой каталог
| EffectiveMetadata без библиотек, версий, бандлов, плагинов
| Генерируется TOML с пустыми секциями (валидный файл)

| Библиотека без версии
| Библиотека с `version = null` после pipeline
| ERROR: "Library 'xxx' has no resolved version"

| Невалидный TOML ключ
| Alias содержит символы недопустимые в TOML: `my.lib` или `my lib`
| ERROR: "Invalid TOML key 'my.lib'. Keys must match [a-zA-Z0-9_-]+"

| Очень длинный alias
| Alias длиной 500+ символов
| WARNING, но генерация продолжается (Gradle может не поддержать)

| Циклическая ссылка version.ref
| version.ref указывает на несуществующую версию
| ERROR: "Unknown version reference 'xxx' in library 'yyy'"

| Бандл с несуществующей библиотекой
| Bundle содержит alias библиотеки которой нет в effective
| ERROR: "Unknown library 'xxx' in bundle 'yyy'"

| Пустой бандл
| Bundle без библиотек: `bundle("empty") { }`
| WARNING: "Empty bundle 'empty'", бандл пропускается или включается пустым

| Дубликаты после фильтрации
| (не должно быть возможно — alias уникальны)
| Валидация в pipeline предотвращает

| Специальные символы в версии
| Версия `1.0.0-SNAPSHOT+build.123`
| Корректная сериализация в кавычках: `version = "1.0.0-SNAPSHOT+build.123"`

| Platform library
| Библиотека типа `platform` (BOM)
| Генерируется как обычная библиотека (Gradle сам определит тип)

| Конфликт имён секций
| (невозможно — секции фиксированы)
| N/A

| Нет прав на запись
| Директория `gradle/` не существует или read-only
| ERROR: "Cannot write to gradle/libs.versions.toml: [причина]"

| Файл уже существует
| `libs.versions.toml` уже есть
| Перезаписывается (с WARNING если `overwrite = false`)

| Gradle < 7.0
| Пользователь использует Gradle 6.x
| WARNING: "Version Catalogs require Gradle 7.0+", генерация продолжается

| Unicode в description/comments
| Комментарий с emoji или кириллицей
| Корректная сериализация в UTF-8

| Deprecated библиотека
| Библиотека помечена как deprecated с replacedBy
| Добавляется комментарий перед библиотекой: `# DEPRECATED: ...`

| Deprecated с многострочным message
| `message` содержит перенос строки
| Каждая строка message становится отдельным комментарием

| Deprecated без replacedBy
| Только message указан
| Комментарий: `# DEPRECATED: {message}` без строки Replacement
|===
