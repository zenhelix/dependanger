== Описание

Registry DSL предоставляет Kotlin DSL (`DependangerDsl`) для определения централизованного каталога зависимостей.
Пользователь описывает библиотеки, версии, бандлы, импорты BOM, целевые платформы и правила совместимости в декларативном стиле.

== User Stories

=== US-001: Определение библиотеки

[quote]
____
Как разработчик, я хочу определить библиотеку с Maven координатами и метаданными, чтобы она была включена в генерируемый каталог.
____

**Acceptance Criteria:**

[cols="1,3,3,3",options="header"]
|===
| ID | Given | When | Then

| AC-001-1
| DSL с пустым блоком `libraries { }`
| Добавляю `library("my-lib", "com.example:my-lib:1.0.0")`
| Библиотека создана с `alias=my-lib`, `group=com.example`, `name=my-lib`, `version=1.0.0`

| AC-001-2
| DSL с библиотекой без версии
| Добавляю `library("spring-web", "org.springframework:spring-web")`
| Библиотека создана с `version=null`, версия будет резолвиться из BOM или останется null

| AC-001-3
| DSL с библиотекой
| Добавляю `tags("web", "spring")` в блок библиотеки
| Библиотека имеет `tags=["web", "spring"]`

| AC-001-4
| DSL с библиотекой
| Добавляю `requires { jdk { min = 17 } }`
| Библиотека имеет `requires.jdk.min=17`

| AC-001-5
| DSL с библиотекой
| Добавляю `deprecated(replacedBy = "new-lib", message = "Use new-lib")`
| Библиотека помечена как deprecated с указанием замены
|===

=== US-002: Определение версии с fallback

[quote]
____
Как разработчик, я хочу определить версию с условными fallback, чтобы использовать разные версии в зависимости от окружения.
____

**Acceptance Criteria:**

[cols="1,3,3,3",options="header"]
|===
| ID | Given | When | Then

| AC-002-1
| DSL с пустым блоком `versions { }`
| Добавляю `version("kotlin", "2.1.20")`
| Версия создана с `name=kotlin`, `value=2.1.20`, `fallbacks=[]`

| AC-002-2
| DSL с версией `spring-boot = 3.4.0`
| Добавляю `fallback("2.7.18") { jdkBelow(17) }`
| Версия имеет fallback: при JDK < 17 используется `2.7.18`

| AC-002-3
| Версия с fallback `jdkBelow(17)`, контекст с JDK 11
| Pipeline резолвит версию
| Используется fallback версия `2.7.18`

| AC-002-4
| Версия с fallback `jdkBelow(17)`, контекст с JDK 21
| Pipeline резолвит версию
| Используется основная версия `3.4.0`

| AC-002-5
| Версия с несколькими fallback
| Pipeline резолвит версию
| Fallback проверяются по порядку, первый matching выигрывает
|===

=== US-003: Определение бандла

[quote]
____
Как team lead, я хочу определить бандл — группу библиотек для конкретной роли, чтобы разработчики могли подключить готовый набор.
____

**Acceptance Criteria:**

[cols="1,3,3,3",options="header"]
|===
| ID | Given | When | Then

| AC-003-1
| DSL с библиотеками `kotlin-stdlib`, `kotlin-reflect`
| Добавляю `bundle("kotlin-core") { libraries("kotlin-stdlib", "kotlin-reflect") }`
| Бандл создан с `name=kotlin-core`, содержит 2 библиотеки

| AC-003-2
| DSL с бандлом `kotlin-core`
| Добавляю `bundle("spring-backend") { extends("kotlin-core") }`
| Бандл `spring-backend` включает все библиотеки из `kotlin-core`

| AC-003-3
| DSL с бандлами `A` и `B`
| Добавляю `bundle("C") { extends("A", "B") }`
| Бандл `C` включает библиотеки из `A` и `B` (объединение)

| AC-003-4
| DSL с бандлом `A` содержащим `lib1`
| Добавляю `bundle("B") { libraries("lib2"); extends("A") }`
| Бандл `B` содержит `lib1` (из extends) и `lib2` (явно)
|===

=== US-004: Импорт версий из BOM

[quote]
____
Как разработчик, я хочу импортировать версии из внешнего Maven BOM (Spring Boot, Kotlin), чтобы не дублировать их определение.
____

**Acceptance Criteria:**

[cols="1,3,3,3",options="header"]
|===
| ID | Given | When | Then

| AC-004-1
| DSL с пустым блоком `bomImports { }`
| Добавляю `bom("spring-bom", "org.springframework.boot:spring-boot-dependencies:3.4.0")`
| BomImport создан с координатами, готов к резолвингу в pipeline

| AC-004-2
| BOM указан, библиотека `spring-web` без версии
| Pipeline обрабатывает библиотеку
| Версия берётся из BOM, `versionSource = BOM_IMPORT`

| AC-004-3
| BOM указан, библиотека `spring-web` с явной версией `5.0.0`
| Pipeline обрабатывает библиотеку
| Используется явная версия `5.0.0`, `versionSource = DECLARED`
|===

NOTE: Подробное описание BOM Import см. в xref:F006-bom-import.adoc[F006: BOM Import].

[cols="1,3,3,3",options="header"]

=== US-006: Настройка глобальных параметров

[quote]
____
Как team lead, я хочу настроить глобальные параметры валидации и генерации, чтобы контролировать поведение системы.
____

**Acceptance Criteria:**

[cols="1,3,3,3",options="header"]
|===
| ID | Given | When | Then

| AC-006-1
| DSL с пустым блоком `settings { }`
| Добавляю `validation { onUnresolvedVersion = FAIL }`
| При неразрешённой версии pipeline завершается с ERROR

| AC-006-2
| DSL с пустым блоком `settings { }`
| Добавляю `defaultDistribution = "backend"`

| AC-006-3
| DSL с пустым блоком `settings { }`
| Добавляю `toml { filename = "custom.versions.toml" }`
| TOML генерируется в файл `custom.versions.toml`

| AC-006-4
| DSL с пустым блоком `settings { }`
| Добавляю `bom { groupId = "com.example"; artifactId = "my-bom" }`
| BOM генерируется с указанными координатами
|===

== Спецификация DSL

=== Базовая структура

[source,kotlin]
----
val dsl = DependangerDsl()

dsl.versions {
    version("kotlin", "2.1.20")
    version("spring-boot", "3.4.0") {
        fallback("2.7.18") { jdkBelow(17) }
    }
}

dsl.libraries {
    library("kotlin-stdlib", "org.jetbrains.kotlin:kotlin-stdlib",
        version = versionRef("kotlin")) {
        tags("kotlin", "core")
    }

    library("spring-web", "org.springframework:spring-web") {
        tags("spring", "web")
        requires { jdk { min = 17 } }
    }
}

dsl.bundles {
    bundle("kotlin-core") {
        libraries("kotlin-stdlib", "kotlin-reflect")
    }

    bundle("spring-backend") {
        libraries("spring-web", "spring-data-jpa")
        extends("kotlin-core")
    }
}

dsl.plugins {
    plugin("kotlin-jvm", "org.jetbrains.kotlin.jvm",
        version = versionRef("kotlin"))
}

dsl.bomImports {
    bom("spring-boot-bom", "org.springframework.boot:spring-boot-dependencies",
        version = versionRef("spring-boot"))
    bom("kotlin-bom", "org.jetbrains.kotlin:kotlin-bom:2.1.20")
}

dsl.targetPlatforms {
    platform("jvm17") {
        jdk { min = 17; max = 21 }
    }
    platform("android-min21") {
        android { minSdk = 21; targetSdk = 34 }
    }
}

dsl.distributions {
    distribution("android") {
        spec {
            filter {
                byTags { include { anyOf("android", "mobile") } }
            }
        }
    }

    distribution("backend") {
        spec {
            filter {
                byBundles { include("spring-backend") }
            }
        }
    }
}

dsl.compatibility {
    jdkRequirement("spring-jdk17") {
        matches = "org.springframework:*"
        minJdk = 17
    }

    mutualExclusion("json-libs") {
        libraries("jackson-core", "gson")
        severity = WARNING
    }
}

dsl.settings {
    validation {
        onCompatibilityViolation = ValidationAction.FAIL
        onUnresolvedVersion = ValidationAction.WARN
    }
    defaultDistribution = "core"
    toml {
        filename = "libs.versions.toml"
        includeComments = true
    }
    bom {
        groupId = "com.example"
        artifactId = "my-bom"
        version = "1.0.0"
    }
}
----

== Блок versions

=== Синтаксис

[source,kotlin]
----
versions {
    version("kotlin", "2.1.20")

    version("spring-boot", "3.4.0") {
        fallback("2.7.18") { jdkBelow(17) }
        fallback("3.2.0") { kotlinVersionBelow("1.9") }
    }
}
----

=== Доступные условия

[cols="1,3",options="header"]
|===
| Условие | Описание

| `jdkBelow(version)`
| JDK версия < указанной

| `jdkAtLeast(version)`
| JDK версия >= указанной

| `jdkBetween(min, max)`
| JDK версия в диапазоне

| `kotlinVersionBelow(version)`
| Kotlin версия < указанной

| `kotlinVersionAtLeast(version)`
| Kotlin версия >= указанной

| `gradleVersionAtLeast(version)`
| Gradle версия >= указанной

| `gradleVersionBelow(version)`
| Gradle версия < указанной

| `distribution(name)`
| Активен указанный distribution profile

| `environment(key, value)`
| Переменная окружения равна значению

| `all(conditions...)`
| Все условия выполнены

| `any(conditions...)`
| Хотя бы одно условие выполнено

| `not(condition)`
| Условие не выполнено
|===

== Блок libraries

=== Синтаксис

[source,kotlin]
----
libraries {
    // С ссылкой на version alias
    library("kotlin-stdlib", "org.jetbrains.kotlin:kotlin-stdlib",
        version = versionRef("kotlin")) {
        tags("core", "kotlin")
    }

    // С inline версией
    library("slf4j-api", "org.slf4j:slf4j-api:2.0.9") {
        tags("logging")
    }

    // Без версии (будет взята из BomImport)
    library("spring-web", "org.springframework:spring-web") {
        tags("web", "spring")
        requires { jdk { min = 17 } }
    }

    // Platform library для экспорта в TOML
    platformLibrary("spring-boot-bom", "org.springframework.boot:spring-boot-dependencies",
        version = versionRef("spring-boot"))
}
----

== Блок bomImports

=== Синтаксис

[source,kotlin]
----
bomImports {
    // С ссылкой на version alias
    bom("spring-boot-bom", "org.springframework.boot:spring-boot-dependencies",
        version = versionRef("spring-boot"))

    // С inline версией
    bom("kotlin-bom", "org.jetbrains.kotlin:kotlin-bom:2.1.20")

----

=== Поведение

* Версии из BOM импортируются во время processing pipeline (BomImportProcessor)
* Приоритет: DECLARED (явная версия в DSL) > BOM_IMPORT (из bomImports)
* Порядок BomImport имеет значение: первый BOM с версией для библиотеки выигрывает

== Блок targetPlatforms

=== Синтаксис

[source,kotlin]
----
targetPlatforms {
    // JDK constraints
    platform("jvm17") {
        jdk { min = 17; max = 21 }
        targets { jvm() }
    }

    // Android constraints
    platform("android-min21") {
        android { minSdk = 21; targetSdk = 34 }
        targets { android() }
    }

    // KMP targets
    platform("kmp-mobile") {
        targets { android(); ios() }
    }
}
----

== Блок distributions

=== Синтаксис

[source,kotlin]
----
distributions {
    distribution("android") {
        spec {
            filter {
                byTags { include { anyOf("android", "mobile") } }
                byGroups { exclude { matching("org.springframework:*") } }
            }
        }
    }

    distribution("backend") {
        spec {
            filter {
                byTags { include { anyOf("server", "web", "database") } }
                byBundles { include("spring-backend", "kotlin-core") }
            }
        }
    }
}
----

=== Типы фильтров

[cols="1,3",options="header"]
|===
| Фильтр | Описание

| `byTags`
| Фильтрация по тегам библиотек

| `byGroups`
| Фильтрация по Maven group pattern

| `byAliases`
| Фильтрация по alias библиотек

| `byBundles`
| Включение содержимого бандлов
|===

NOTE: Если distribution не указан, все библиотеки включаются без фильтрации.

== Блок settings

=== Синтаксис

[source,kotlin]
----
settings {
    // Настройки валидации
    validation {
        onCompatibilityViolation = ValidationAction.FAIL
        onDeprecatedLibrary = ValidationAction.WARN
        onJdkMismatch = ValidationAction.FAIL
        onUnresolvedVersion = ValidationAction.WARN
    }

    // Дефолтная дистрибуция
    defaultDistribution = "core"

    // Строгий режим резолюции версий
    strictVersionResolution = true

    // Настройки генерации TOML
    toml {
        filename = "libs.versions.toml"
        includeComments = true
        sortSections = true
    }

    // Настройки генерации BOM
    bom {
        groupId = "com.example"
        artifactId = "my-bom"
        version = "1.0.0"
        includeOptionalDependencies = false
    }
}
----

=== ValidationAction

[cols="1,3",options="header"]
|===
| Значение | Описание

| FAIL
| Ошибка, блокирует обработку

| WARN
| Предупреждение, обработка продолжается

| INFO
| Информационное сообщение

| IGNORE
| Игнорировать

== Блок compatibility

Правила проверки совместимости библиотек.

=== Синтаксис

[source,kotlin]
----
compatibility {
    // Требование к JDK для библиотек
    jdkRequirement("spring-boot-3x") {
        libraries("spring-boot-starter")
        minJdk = 17
    }

    // Взаимоисключающие библиотеки
    mutualExclusion("logging") {
        libraries("slf4j-simple", "logback-classic", "log4j-core")
        message = "Only one logging implementation should be used"
    }

    // Ограничение версий
    versionConstraint("kotlin-compat") {
        library = "kotlinx-coroutines-core"
        versionRange = "[1.7.0,)"  // >= 1.7.0
    }

    // Кастомное правило
    customRule("security-check") {
        condition { metadata ->
            // Проверка условия
            true
        }
        message = "Security check failed"
        severity = WARNING
    }
}
----

=== Типы правил

[cols="1,3",options="header"]
|===
| Правило | Описание

| `jdkRequirement`
| Проверка минимальной/максимальной версии JDK для указанных библиотек

| `mutualExclusion`
| Проверка что только одна библиотека из списка включена

| `versionConstraint`
| Проверка версии библиотеки по Maven version range

| `customRule`
| Кастомное правило с произвольной логикой
|===

=== Severity

[cols="1,3",options="header"]
|===
| Значение | Описание

| ERROR
| Останавливает pipeline

| WARNING
| Предупреждение, pipeline продолжается

| INFO
| Информационное сообщение
|===

== Блок processing

Настройки пайплайна обработки метаданных.

=== Синтаксис

[source,kotlin]
----
processing {
    // Включить импорт версий из BOM
    importBoms = true

    // Включить проверку обновлений
    checkUpdates = true

    // Настройки анализа совместимости
    analyzeCompatibility {
        enabled = true
        targetJdk = 17
        failOnErrors = true
    }

    // Использовать preset конфигурации pipeline
    preset = ProcessingPreset.STRICT

    // Кастомная настройка процессоров
    withBomImport()
    withUpdateCheck()
    withoutCompatibilityAnalysis()

    // Отключить конкретные процессоры
    disabledProcessors = listOf("update-check")
}
----

=== ProcessingPreset

[cols="1,3",options="header"]
|===
| Preset | Описание

| DEFAULT
| Стандартная конфигурация: обязательные процессоры + bom-import

| MINIMAL
| Минимальный набор: только обязательные процессоры

| STRICT
| Все проверки включены + strictVersionResolution

| DISTRIBUTION
| Для генерации артефактов: обязательные + profile фильтрация
|===

NOTE: Подробное описание pipeline см. в xref:F003-processing-pipeline.adoc[F003: Processing Pipeline].

== Модель данных

=== DependangerDsl

Корневая точка входа DSL, содержащая блоки:

* `versions { }` — именованные версии с fallback
* `libraries { }` — библиотеки с Maven координатами
* `plugins { }` — Gradle plugins
* `bundles { }` — группы библиотек
* `bomImports { }` — импорт версий из внешних BOM
* `targetPlatforms { }` — целевые платформы
* `distributions { }` — профили генерации с фильтрацией
* `compatibility { }` — правила совместимости
* `settings { }` — глобальные настройки
* `presets { }` — именованные пресеты конфигурации
* `processing { }` — настройки пайплайна обработки

=== Ключевые сущности

* **Library** — `alias`, `group:name`, `version`, `tags`, `requires`
* **Plugin** — `alias`, `id`, `version`
* **Version** — `name`, `value`, `fallbacks`
* **Bundle** — `name`, `libraries`, `extends`
* **BomImport** — `alias`, `group:name:version`
* **TargetPlatform** — `name`, `jdk`, `android`, `targets`
* **Distribution** — `name`, `spec` (фильтры)
* **Settings** — `validation`, `toml`, `bom`, `defaultDistribution`

См. xref:../01-concepts/domain-model.adoc[Доменная модель] для полного описания.

== Валидация

[cols="1,2,1",options="header"]
|===
| Правило | Описание | Severity

| Уникальность alias
| Все alias уникальны в рамках Registry
| ERROR

| Валидные Maven координаты
| group и artifact соответствуют формату
| ERROR

| Существующие ссылки
| versionRef ссылается на существующий alias
| ERROR

| Нет циклов в бандлах
| Бандлы не создают циклических зависимостей
| ERROR

| Deprecated замена существует
| replacedBy указывает на существующую библиотеку
| WARNING
|===

== Edge Cases

[cols="1,3,2",options="header"]
|===
| Сценарий | Описание | Ожидаемое поведение

| Дубликат alias
| Две библиотеки с одинаковым alias `my-lib`
| ERROR: "Duplicate alias 'my-lib'"

| Невалидный alias
| Alias содержит недопустимые символы: `My_Lib`, `123lib`, `lib.name`
| ERROR: "Invalid alias format. Expected: [a-z][a-z0-9-]*"

| Несуществующий versionRef
| `versionRef("unknown")` ссылается на несуществующую версию
| ERROR: "Unknown version reference 'unknown'"

| Циклическая зависимость бандлов
| Бандл A extends B, бандл B extends A
| ERROR: "Circular bundle dependency detected: A → B → A"

| Несуществующая библиотека в бандле
| `bundle("test") { libraries("unknown-lib") }`
| ERROR: "Unknown library 'unknown-lib' in bundle 'test'"

| Несуществующий бандл в extends
| `bundle("test") { extends("unknown-bundle") }`
| ERROR: "Unknown bundle 'unknown-bundle' in extends"

| Пустой alias
| `library("", "com.example:lib:1.0")`
| ERROR: "Alias cannot be empty"

| Невалидные Maven координаты
| `library("lib", "invalid-coordinates")`
| ERROR: "Invalid Maven coordinates format. Expected: group:artifact or group:artifact:version"

| Deprecated без replacedBy
| `deprecated(message = "Old lib")` без указания замены
| WARNING: допустимо, но рекомендуется указать replacedBy

| Deprecated replacedBy несуществующий
| `deprecated(replacedBy = "unknown-lib")`
| WARNING: "Replacement library 'unknown-lib' not found"

| Пустой DSL
| DSL без определений (пустые блоки)
| Валидно, создаётся пустой DependangerMetadata

| Дубликат fallback условия
| Две одинаковые fallback условия для версии
| WARNING: первый fallback выигрывает

| Конфликт fallback условий
| Fallback условия перекрываются (jdkBelow(17) и jdkBelow(11))
| Используется первый matching fallback по порядку определения

| Отрицательные JDK constraints
| `jdk { min = -1 }`
| ERROR: "JDK version must be positive"

| min > max в constraints
| `jdk { min = 21; max = 17 }`
| ERROR: "Invalid JDK range: min (21) > max (17)"

| Пустой distribution spec
| `distribution("empty") { }` без фильтров
| Все библиотеки включаются (эквивалент отсутствия фильтрации)
|===
