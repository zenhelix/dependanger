[[overview]]
== Обзор

Модуль `dependanger-updates` проверяет доступные обновления для библиотек, запрашивая Maven metadata XML из репозиториев.
Результаты хранятся в `EffectiveMetadata.extensions` через `ExtensionKey` (`UpdatesExtensionKey`).

Приоритет: **P3** (желательно).

[[spec-update-check]]
== SPEC-07.1: UpdateCheckProcessor.process()

[[purpose-update-check]]
=== Назначение

Параллельно запрашивает Maven репозитории для определения доступных обновлений каждой библиотеки.
Классифицирует обновления по типу (PATCH, MINOR, MAJOR, PRERELEASE) и сохраняет результаты в extensions.

[[algorithm-update-check]]
=== Алгоритм

[source,text]
----
1. Получить настройки updateCheck из context.settings
2. Получить список repositories: `settings.repositories` -> `context.settings.repositories` -> Maven Central (см. xref:index.adoc#repository-convention[конвенция репозиториев])
3. Получить фильтры: excludePatterns, includePrerelease
4. Отфильтровать библиотеки:
   a. Исключить библиотеки с ignoreUpdates = true
   b. Исключить библиотеки matching excludePatterns
   c. Исключить библиотеки без version (version == null)
5. Создать VersionCache (файловый кеш, TTL 1h)
6. Параллельно (Semaphore с concurrency = parallelism, default 10):
   Для каждой отфильтрованной библиотеки:
   a. Проверить кеш -> если есть и не истёк -> использовать
   b. Запросить maven-metadata.xml из репозиториев
   c. Распарсить XML: извлечь <versioning><versions><version>
   d. Отфильтровать prerelease (alpha, beta, rc, snapshot, M, CR, ea)
   e. Сравнить текущую версию с найденными
   f. Определить latestStable, latestAny, updateType
   g. Если есть обновление -> создать UpdateAvailableInfo
   h. Сохранить в кеш
7. Собрать все UpdateAvailableInfo в список
8. Записать в metadata через withExtension(UpdatesExtensionKey, updates)
9. Добавить INFO в diagnostics: "X updates available"
----

==== MavenMetadataFetcher (новый класс)

[source,kotlin]
----
public class MavenMetadataFetcher(
    private val repositories: List<MavenRepository>,
    private val httpClient: HttpClient,
    private val timeout: Long = 30_000,
    private val credentialsProvider: CredentialsProvider? = null,
) {
    /**
     * Запрашивает maven-metadata.xml для артефакта.
     * URL: {repoUrl}/{groupPath}/{artifactId}/maven-metadata.xml
     */
    public suspend fun fetchAvailableVersions(
        group: String,
        artifact: String
    ): VersionFetchResult {
        val groupPath = group.replace('.', '/')
        for (repo in repositories) {
            val url = "${repo.url.trimEnd('/')}/$groupPath/$artifact/maven-metadata.xml"
            try {
                val response = httpClient.get(url) {
                    timeout { requestTimeoutMillis = this@MavenMetadataFetcher.timeout }
                    credentialsProvider?.getCredentials(repo.url)?.let { creds ->
                        basicAuth(creds.username, creds.password)
                    }
                }
                if (response.status == HttpStatusCode.OK) {
                    val versions = parseMavenMetadataXml(response.bodyAsText())
                    return VersionFetchResult(versions, repo.url)
                }
            } catch (e: Exception) {
                logger.warn { "Failed to fetch metadata for $group:$artifact from ${repo.url}: ${e.message}" }
            }
        }
        return VersionFetchResult(emptyList(), null)
    }

    private fun parseMavenMetadataXml(xml: String): List<String> {
        // Парсинг: <metadata><versioning><versions><version>X.Y.Z</version>...
        val doc = DocumentBuilderFactory.newInstance().newDocumentBuilder()
            .parse(xml.byteInputStream())
        val versionNodes = doc.getElementsByTagName("version")
        return (0 until versionNodes.length).map { versionNodes.item(it).textContent }
    }
}

public data class VersionFetchResult(
    val versions: List<String>,
    val repository: String?,
)
----

==== VersionComparator

NOTE: `VersionComparator` и `SemanticVersion` определены в `dependanger-core` (см. xref:SPEC-01-dependanger-core.adoc#core-utilities[SPEC-01: Общие утилиты]).
Модуль `dependanger-updates` использует каноническую реализацию из `io.github.zenhelix.dependanger.core.util.VersionComparator`.

Используемые методы:

* `VersionComparator.parse(version): SemanticVersion?` — парсинг строки в семантическую версию
* `VersionComparator.isPrerelease(version): Boolean` — определение prerelease
* `VersionComparator.compare(a, b): Int` — сравнение двух версий
* `VersionComparator.classifyUpdate(current, latest): UpdateType` — классификация обновления (MAJOR/MINOR/PATCH/PRERELEASE)

==== UpdateCheckProcessor.process()

[source,kotlin]
----
override suspend fun process(
    metadata: EffectiveMetadata,
    context: ProcessingContext
): EffectiveMetadata {
    val settings = context.settings.updateCheck ?: return metadata
    val repos = settings.repositories.ifEmpty {
        context.settings.repositories.ifEmpty {
            listOf(MavenRepository("https://repo.maven.apache.org/maven2", "Maven Central"))
        }
    }
    val fetcher = MavenMetadataFetcher(repos, createHttpClient(), settings.timeout)
    val cache = VersionCache(settings.cacheDirectory, settings.cacheTtlHours)
    val includePrerelease = settings.includePrerelease
    val parallelism = settings.parallelism

    // Отфильтровать библиотеки
    val librariesToCheck = metadata.libraries.values
        .filter { it.version != null }
        .filter { lib -> settings.excludePatterns.none { pattern -> GlobMatcher.matches(pattern, lib.group, lib.artifact) } }

    // GlobMatcher из dependanger-core (см. xref:SPEC-01-dependanger-core.adoc#core-utilities[SPEC-01])

    val semaphore = Semaphore(parallelism)
    val updates = coroutineScope {
        librariesToCheck.map { lib ->
            async {
                semaphore.withPermit {
                    checkUpdate(lib, fetcher, cache, includePrerelease)
                }
            }
        }.awaitAll().filterNotNull()
    }

    return metadata
        .withExtension(UpdatesExtensionKey, updates)
        .let { m ->
            if (updates.isNotEmpty()) {
                val summary = "Updates available: ${updates.size} " +
                    "(${updates.count { it.updateType == UpdateType.MAJOR }} major, " +
                    "${updates.count { it.updateType == UpdateType.MINOR }} minor, " +
                    "${updates.count { it.updateType == UpdateType.PATCH }} patch)"
                m.withDiagnostic(
                    Diagnostics.info("UPDATES_AVAILABLE", summary, "update-check")
                )
            } else m
        }
}

private suspend fun checkUpdate(
    lib: EffectiveLibrary,
    fetcher: MavenMetadataFetcher,
    cache: VersionCache,
    includePrerelease: Boolean,
): UpdateAvailableInfo? {
    val currentVersion = lib.version?.value ?: return null
    val currentParsed = VersionComparator.parse(currentVersion) ?: return null

    // Кеш
    val cachedVersions = cache.get(lib.group, lib.artifact)
    val result = cachedVersions ?: fetcher.fetchAvailableVersions(lib.group, lib.artifact)

    if (cachedVersions == null && result.versions.isNotEmpty()) {
        cache.put(lib.group, lib.artifact, result)
    }

    if (result.versions.isEmpty()) return null

    // Фильтрация и сортировка
    val stableVersions = result.versions
        .filter { !VersionComparator.isPrerelease(it) }
        .mapNotNull { v -> VersionComparator.parse(v)?.let { v to it } }
        .sortedByDescending { it.second }

    val allVersions = result.versions
        .mapNotNull { v -> VersionComparator.parse(v)?.let { v to it } }
        .sortedByDescending { it.second }

    val latestStable = stableVersions.firstOrNull()
    val latestAny = allVersions.firstOrNull()

    // Определить обновление
    val targetVersion = if (includePrerelease) latestAny else latestStable
    if (targetVersion == null || targetVersion.second <= currentParsed) return null

    // UpdateType enum определён в dependanger-core (см. xref:SPEC-01-dependanger-core.adoc#core-utilities[SPEC-01])
    val updateType = VersionComparator.classifyUpdate(currentParsed, targetVersion.second)

    return UpdateAvailableInfo(
        alias = lib.alias,
        group = lib.group,
        artifact = lib.artifact,
        currentVersion = currentVersion,
        latestVersion = targetVersion.first,
        latestStable = latestStable?.first,
        latestAny = latestAny?.first,
        updateType = updateType,
        repository = result.repository,
    )
}

[[cache-impl]]
== Кеш версий

=== Структура

[source,text]
----
~/.dependanger/cache/versions/
  org.jetbrains.kotlin/
    kotlin-stdlib.json    # {"versions":["2.1.20",...], "fetchedAt":"...", "repository":"..."}
----

NOTE: Ключ кеша: `"{groupId}/{artifactId}.json"`.
Dots в groupId сохраняются как есть (НЕ заменяются на слеши).
Например: `org.jetbrains.kotlin/kotlin-stdlib.json`.

=== TTL

- Default: 1 час (настраивается через `settings.updateCheck.cache.ttl`)
- При недоступности сети: используется стейл кеш с WARNING

[[edge-cases]]
== Edge Cases и обработка ошибок

[cols="1,2,2"]
|===
| Ситуация | Поведение | Диагностический код

| Репозиторий недоступен | WARNING, библиотека пропускается | `UPDATE_REPO_UNREACHABLE`
| Таймаут запроса | WARNING, retry 1 раз | `UPDATE_TIMEOUT`
| Библиотека не найдена | WARNING, пропуск | `UPDATE_LIB_NOT_FOUND`
| Невалидный формат версии | WARNING, сравнение как строки | `UPDATE_INVALID_VERSION`
| Пустой каталог | Процессор завершается, 0 обновлений | --
| Все актуальны | INFO "All libraries are up to date" | `ALL_UP_TO_DATE`
| Prerelease текущая версия | Показывать обновление до стабильной | --
| Downgrade (max < current) | Не показывать | --
| Rate limiting (429) | Exponential backoff + WARNING | `UPDATE_RATE_LIMITED`
| 500+ библиотек | Semaphore(10), параллельная обработка | --
|===
