= Доменная модель
:toc: left
:toclevels: 3
:sectnums:
:icons: font

[abstract]
--
Документ описывает ключевые сущности системы Dependanger на трёх уровнях абстракции:
DSL Model (в памяти), Metadata Model (сериализованная), Effective Model (готовая к генерации).

Краткие определения терминов: xref:../00-overview.adoc#glossary[Глоссарий в Overview].

**Справочники:**

* xref:../06-reference/dsl-syntax.adoc[DSL Syntax Reference] — полная спецификация DSL синтаксиса
* xref:../06-reference/model-schema.adoc[Model Schema Reference] — структуры данных
--

=== DSL Model

[quote]
____
То, что пишет пользователь в Kotlin DSL.
Модель в памяти, создаваемая через builder pattern.
____

* **Модуль:** `dependanger-core`
* **Главная сущность:** `DependangerDsl`
* **Характеристики:**
** Fluent API для удобного определения
** Поддержка DSL блоков (`@DependangerDslMarker`)
** Версии могут быть неразрешёнными (ссылки на alias)
** Валидация на уровне DSL

=== Metadata Model

[quote]
____
Сериализуемое представление модели.
Промежуточный формат для хранения и передачи.
____

* **Модуль:** `dependanger-metadata-json` (и другие форматы через SPI)
* **Главная сущность:** `DependangerMetadata`
* **Характеристики:**
** Абстрагирован через `SerializationFormat<T>` SPI
** JSON как основной формат (расширяемо до YAML)
** Хранит полную информацию из DSL
** Файл: `metadata.json`

=== Effective Model

[quote]
____
Результат обработки pipeline.
Все версии разрешены, фильтры применены, готов для генерации.
____

* **Модуль:** `dependanger-effective`
* **Главная сущность:** `EffectiveMetadata`
* **Характеристики:**
** Все версии разрешены с указанием `VersionSource`
** Применены фильтры (Distribution)
** Содержит результаты валидации
** Опционально: update report, compatibility report
** Файл: `effective-\{distribution\}.json`

== Сущности

=== DependangerDsl

[quote]
____
Корневая точка входа Kotlin DSL для конфигурации зависимостей.
Содержит все определения для генерации артефактов (TOML, BOM).
____

NOTE: Полная спецификация DSL синтаксиса: xref:../06-reference/dsl-syntax.adoc[DSL Syntax Reference].

==== DSL блоки

[cols="1,2,2",options="header"]
|===
| Блок | Тип | Описание

| `versions { }`
| VersionsDsl
| Именованные версии с fallback условиями

| `libraries { }`
| LibrariesDsl
| Библиотеки с Maven координатами

| `plugins { }`
| PluginsDsl
| Gradle plugins

| `bundles { }`
| BundlesDsl
| Группы библиотек

| `bomImports { }`
| BomImportsDsl
| Импорт версий из внешних Maven BOM

| `targetPlatforms { }`
| TargetPlatformsDsl
| Целевые платформы с ограничениями (JDK, Kotlin targets)

| `distributions { }`
| DistributionsDsl
| Профили генерации с фильтрацией

| `compatibility { }`
| CompatibilityDsl
| Правила совместимости

| `settings { }`
| SettingsDsl
| Глобальные настройки валидации и генерации

| `presets { }`
| PresetsDsl
| Именованные пресеты конфигурации для переиспользования

| `constraints { }`
| ConstraintsDsl
| Ограничения для зависимостей (версии, исключения, подстановки)
|===

NOTE: Канонический порядок блоков: `versions` → `libraries` → `plugins` → `bundles` → `bomImports` → `targetPlatforms` → `distributions` → `compatibility` → `settings` → `presets` → `processing`

==== Пример DSL

[source,kotlin]
----
val dsl = DependangerDsl()

dsl.versions {
    version("kotlin", "2.1.20")
    version("spring-boot", "3.4.0") {
        fallback("2.7.18") { jdkBelow(17) }
    }
}

dsl.libraries {
    library("kotlin-stdlib", "org.jetbrains.kotlin:kotlin-stdlib",
        version = versionRef("kotlin")) {
        tags("core", "kotlin")
    }
}

dsl.bundles {
    bundle("kotlin-core") {
        libraries("kotlin-stdlib", "kotlin-reflect")
    }
}

dsl.plugins {
    plugin("kotlin-jvm", "org.jetbrains.kotlin.jvm",
        version = versionRef("kotlin"))
}

dsl.bomImports {
    bom("spring-boot-bom", "org.springframework.boot:spring-boot-dependencies",
        version = versionRef("spring-boot"))
    bom("kotlin-bom", "org.jetbrains.kotlin:kotlin-bom:2.1.20")
}

dsl.targetPlatforms {
    platform("jvm17") {
        jdk { min = 17; max = 21 }
    }
    platform("android-min21") {
        android { minSdk = 21; targetSdk = 34 }
    }
}

dsl.distributions {
    distribution("android") {
        spec {
            filter {
                byTags { include { anyOf("android", "mobile") } }
            }
        }
    }
}

dsl.settings {
    validation {
        onCompatibilityViolation = ValidationAction.FAIL
        onUnresolvedVersion = ValidationAction.WARN
    }
    toml {
        filename = "libs.versions.toml"
    }
}
----

==== Правила валидации

* Все alias (библиотек, плагинов, версий, бандлов, BOM) должны быть уникальны
* Alias должен соответствовать паттерну `[a-z][a-z0-9-]*`
* Ссылки на версии должны указывать на существующие version aliases

'''

=== Library

[quote]
____
Зависимость с Maven координатами и метаданными для фильтрации.
____

==== Атрибуты

[cols="1,1,1,3",options="header"]
|===
| Атрибут | Тип | Обязательность | Описание

| alias
| String
| Да
| Уникальный идентификатор для ссылок в DSL и генерируемом TOML

| group
| String
| Да
| Maven groupId (например: `org.jetbrains.kotlin`)

| artifact
| String
| Да
| Maven artifactId (например: `kotlin-stdlib`)

| version
| VersionReference
| Нет
| Ссылка на version alias (`Reference`), inline значение (`Literal`), или диапазон (`Range`). Может быть унаследована из BomImport

| tags
| Set<String>
| Нет
| Теги для фильтрации и группировки (например: `testing`, `database`, `spring`)

| license
| LicenseInfo?
| Нет
| Информация о лицензии библиотеки (SPDX идентификатор, категория)

| constraints
| List<Constraint>
| Нет
| Ограничения для библиотеки (версионные, исключения, подстановки). См. <<constraint>>

| requires
| Requirements
| Нет
| Требования к окружению (JDK версия, Kotlin версия)

| deprecation
| DeprecationInfo
| Нет
| Информация об устаревании библиотеки (замена, сообщение, версия удаления)

| description
| String
| Нет
| Описание библиотеки

| isPlatform
| Boolean
| Нет (default: false)
| Является ли библиотека платформой (BOM)

| ignoreUpdates
| Boolean
| Нет (default: false)
| Пропускать проверку обновлений для этой библиотеки
|===

==== DeprecationInfo

Информация об устаревшей библиотеке.

[cols="1,1,1,3",options="header"]
|===
| Поле | Тип | Обязательность | Описание

| replacedBy
| String
| Нет
| Alias библиотеки-замены. Должен указывать на существующую библиотеку

| message
| String
| Нет
| Сообщение для разработчиков с пояснением причины и инструкциями по миграции

| since
| String
| Нет
| Версия каталога Dependanger, начиная с которой библиотека помечена как deprecated

| removalVersion
| String
| Нет
| Версия каталога, в которой библиотека будет удалена
|===

[[constraint]]
==== Constraint

Ограничение для библиотеки — sealed class с тремя подтипами.

[source,kotlin]
----
sealed class Constraint {
    data class VersionConstraintDef(
        val coordinates: String,
        val version: VersionReference?,
        val because: String?
    ) : Constraint()
    data class Exclude(
        val group: String,
        val artifact: String
    ) : Constraint()
    data class Substitute(
        val from: String,
        val to: String,
        val because: String?
    ) : Constraint()
}
----

[cols="1,3",options="header"]
|===
| Подтип | Описание

| `VersionConstraintDef`
| Ограничение версии по координатам (`coordinates`), с опциональной целевой версией и причиной (`because`)

| `Exclude`
| Исключение транзитивной зависимости по `group` и `artifact`

| `Substitute`
| Подмена зависимости `from` на `to` с опциональной причиной
|===

==== LicenseInfo

Информация о лицензии библиотеки.

[cols="1,1,3",options="header"]
|===
| Поле | Тип | Описание

| `id`
| String?
| Идентификатор лицензии (SPDX формат, например: `Apache-2.0`, `MIT`)

| `url`
| String?
| URL текста лицензии
|===

==== VersionReference

Ссылка на версию — может быть литералом, ссылкой на alias или диапазоном.
Подробности: xref:../06-reference/model-schema.adoc#version-reference[Model Schema: VersionReference].

==== DSL синтаксис

[source,kotlin]
----
libraries {
    // С ссылкой на version alias
    library("kotlin-stdlib", "org.jetbrains.kotlin:kotlin-stdlib",
        version = versionRef("kotlin")) {
        tags("core", "kotlin")
    }

    // С inline версией
    library("slf4j-api", "org.slf4j:slf4j-api:2.0.9") {
        tags("logging")
    }

    // Без версии (будет взята из BomImport)
    library("spring-web", "org.springframework:spring-web") {
        tags("web", "spring")
        requires {
            jdk { min = 17 }
        }
    }

    // Deprecated библиотека с полной информацией
    library("old-http", "com.example:old-http:1.0") {
        tags("http", "legacy")
        deprecated {
            replacedBy = "new-http"
            message = "Use new-http which supports HTTP/2"
            since = "1.2.0"
            removalVersion = "2.0.0"
        }
    }

    // Deprecated библиотека — короткая форма
    library("legacy-json", "com.example:legacy-json:1.0") {
        deprecated(replacedBy = "modern-json")
    }

    // Deprecated без замены (abandoned)
    library("abandoned-lib", "com.example:abandoned:1.0") {
        deprecated(message = "Library is no longer maintained")
    }
}
----

==== Правила валидации

* `alias` уникален, соответствует паттерну `[a-z][a-z0-9-]*`
* `group:name` формируют валидные Maven координаты
* `versionRef()` должен ссылаться на существующий version alias
* `deprecation.replacedBy` должен указывать на существующую не-deprecated библиотеку (WARNING если на deprecated)

'''

[quote]
____
Gradle plugin — определяется через alias, plugin ID и версию.
Публикуется в Version Catalog секции `[plugins]`.
____

==== Атрибуты

[cols="1,1,1,3",options="header"]
|===
| Атрибут | Тип | Обязательность | Описание

| alias
| String
| Да
| Уникальный идентификатор для ссылок в DSL и генерируемом TOML

| id
| String
| Да
| Gradle Plugin ID (например: `org.jetbrains.kotlin.jvm`)

| version
| VersionReference
| Нет
| Ссылка на version alias или inline значение

| tags
| Set<String>
| Нет
| Теги для фильтрации и группировки (аналогично Library.tags)

==== DSL синтаксис

[source,kotlin]

* `id` должен быть валидным Gradle Plugin ID

'''

=== Version

[quote]
____
Именованная версия для переиспользования несколькими библиотеками.
Поддерживает условные fallback'и.
____

==== Атрибуты

[cols="1,1,1,3",options="header"]
|===
| Атрибут | Тип | Обязательность | Описание

| name
| String
| Да
| Уникальный идентификатор версии (например: `kotlin`, `spring-boot`)

| value
| String
| Да
| Значение версии по умолчанию (например: `2.1.20`)

| fallbacks
| List<VersionFallback>
| Нет
| Условные альтернативные версии
|===

==== VersionFallback

[cols="1,1,3",options="header"]
|===
| Поле | Тип | Описание

| value
| String
| Альтернативное значение версии

| condition
| FallbackCondition
| Условие, при котором используется fallback
|===

==== DSL синтаксис

[source,kotlin]
----
versions {
    version("kotlin", "2.1.20")

    version("spring-boot", "3.4.0") {
        fallback("2.7.18") { jdkBelow(17) }
        fallback("3.2.0") { kotlinVersionBelow("1.9") }
    }
}
----

==== Доступные условия

[cols="1,3",options="header"]
|===
| Условие | Описание

| `jdkBelow(version)`
| JDK версия < указанной

| `jdkAtLeast(version)`
| JDK версия >= указанной

| `jdkBetween(min, max)`
| JDK версия в диапазоне [min, max]

| `kotlinVersionBelow(version)`
| Kotlin версия < указанной

| `kotlinVersionAtLeast(version)`
| Kotlin версия >= указанной

| `gradleVersionAtLeast(version)`
| Gradle версия >= указанной

| `gradleVersionBelow(version)`
| Gradle версия < указанной

| `distribution(name)`
| Активен указанный Distribution

| `environment(key, value)`
| Переменная окружения равна значению

| `all(conditions...)`
| Все условия выполнены (AND)

| `any(conditions...)`
| Хотя бы одно условие выполнено (OR)

| `not(condition)`
| Условие не выполнено (NOT)
|===

==== Правила валидации

* `name` уникален
* `value` должен быть валидной версией
* Fallback условия оцениваются по порядку, первое подходящее выигрывает

'''

=== Bundle

[quote]
____
Именованная группа библиотек для переиспользования.
Может наследовать библиотеки из других бандлов.
Публикуется в Version Catalog секции `[bundles]`.
____

==== Атрибуты

[cols="1,1,1,3",options="header"]
|===
| Атрибут | Тип | Обязательность | Описание

| alias
| String
| Да
| Уникальный идентификатор бандла

| libraries
| List<String>
| Нет
| Alias'ы включаемых библиотек

| extends
| List<String>
| Нет
| Имена бандлов для наследования (композиция)
|===

==== DSL синтаксис

[source,kotlin]
----
bundles {
    bundle("kotlin-core") {
        libraries("kotlin-stdlib", "kotlin-reflect")
    }

    bundle("kotlin-serialization") {
        libraries("kotlinx-serialization-core", "kotlinx-serialization-json")
        extends("kotlin-core")  // Включает библиотеки из kotlin-core
    }

    bundle("spring-backend") {
        libraries("spring-web", "spring-data-jpa")
        extends("kotlin-core", "kotlin-serialization")
    }
}
----

==== Правила валидации

* `name` уникален
* Все ссылки в `libraries` должны указывать на существующие библиотеки
* Все ссылки в `extends` должны указывать на существующие бандлы
* Не допускаются циклические зависимости между бандлами

'''

[[bom-import]]
=== BomImport

[quote]
____
Импорт версий из внешнего Maven BOM/Platform.
Позволяет наследовать версии из Spring Boot BOM, Kotlin BOM и других.
____

==== Атрибуты

[cols="1,1,1,3",options="header"]
|===
| Атрибут | Тип | Обязательность | Описание

| alias
| String
| Да
| Уникальный идентификатор BOM

| group
| String
| Да
| Maven groupId BOM (например: `org.springframework.boot`)

| name
| String
| Да
| Maven artifactId BOM (например: `spring-boot-dependencies`)

| version
| VersionReference
| Да
| Версия BOM (ссылка или literal)
|===

==== DSL синтаксис

[source,kotlin]
----
bomImports {
    // С ссылкой на version alias
    bom("spring-boot-bom", "org.springframework.boot:spring-boot-dependencies",
        version = versionRef("spring-boot"))

    // С inline версией
    bom("kotlin-bom", "org.jetbrains.kotlin:kotlin-bom:2.1.20")
}
----

==== Поведение

* Версии из BOM импортируются во время processing pipeline (BomImportProcessor, order: 15)
* Приоритет: DECLARED (явная версия в DSL) > BOM_IMPORT (из BomImport)
* Порядок BomImport имеет значение: первый BOM с версией для библиотеки выигрывает

'''

=== TargetPlatform

[quote]
____
Целевая платформа с ограничениями по JDK, Android и поддерживаемыми Kotlin targets.
Используется для валидации совместимости библиотек с целевым окружением.
____

==== Атрибуты

[cols="1,1,1,3",options="header"]
|===
| Атрибут | Тип | Обязательность | Описание

| name
| String
| Да
| Уникальный идентификатор платформы

| jdkConstraints
| JdkConstraints
| Нет
| Ограничения по версии JDK (min, max)

| androidConstraints
| AndroidConstraints
| Нет
| Ограничения для Android (minSdk, targetSdk)

| supportedTargets
| SupportedTargets
| Нет
| Поддерживаемые Kotlin Multiplatform targets
|===

==== DSL синтаксис

[source,kotlin]
----
targetPlatforms {
    // Платформа с JDK constraints
    platform("jvm17") {
        jdk { min = 17; max = 21 }
        targets { jvm() }
    }

    // Платформа для Android
    platform("android-min21") {
        android { minSdk = 21; targetSdk = 34 }
        targets { android() }
    }

    // KMP платформа
    platform("kmp-mobile") {
        targets { android(); ios() }
    }
}

=== Distribution

[quote]
____
Профиль генерации артефактов с фильтрацией библиотек.
Определяет какие библиотеки включить в генерируемый TOML/BOM.
____

NOTE: **Поведение по умолчанию:** Если distribution не указан (`null`), все библиотеки включаются без фильтрации.
Для предварительного просмотра результата фильтрации используйте `Dependanger.previewFilter(distribution)`.

==== Атрибуты

[cols="1,1,1,3",options="header"]
|===
| Атрибут | Тип | Обязательность | Описание

| name
| String
| Да
| Уникальный идентификатор профиля

| spec
| FilterSpec
| Нет
| Спецификация фильтрации
|===

==== DSL синтаксис

[source,kotlin]
----
distributions {
    distribution("android") {
        spec {
            filter {
                byTags { include { anyOf("android", "mobile") } }
                byGroups { exclude { matching("org.springframework:*") } }
            }
        }
    }

    distribution("backend") {
        spec {
            filter {
                byTags { include { anyOf("server", "web", "database") } }
                byBundles { include("spring-backend", "kotlin-core") }
            }
        }
    }

    distribution("minimal") {
        spec {
            filter {
                byBundles { include("kotlin-core") }
            }
        }
    }

    // Production без deprecated библиотек
    distribution("production") {
        spec {
            filter {
                byDeprecated { exclude() }
            }
        }
    }
}
----

==== Типы фильтров

[cols="1,3",options="header"]
|===
| Фильтр | Описание

| `byTags`
| Фильтрация по тегам библиотек

| `byGroups`
| Фильтрация по Maven group pattern

| `byAliases`
| Фильтрация по alias библиотек

| `byBundles`
| Включение содержимого бандлов

| `byDeprecated`
| Фильтрация deprecated библиотек. По умолчанию deprecated **включаются** (с WARNING). Используйте `exclude()` для исключения

| `customFilters`
| `Map<String, JsonElement>` — расширяемые фильтры для custom SPI-процессоров. Ключ — идентификатор фильтра, значение — конфигурация в JSON. Сериализуется в metadata.json
|===

==== Логика комбинирования фильтров

[cols="1,3",options="header"]
|===
| Комбинация | Логика

| Несколько `include` в одном фильтре
| **OR** — библиотека включается, если подходит под любой include

| Несколько `exclude` в одном фильтре
| **OR** — библиотека исключается, если подходит под любой exclude

| `include` + `exclude` в одном фильтре
| `include` применяется первым, затем из результата удаляются `exclude`

| Несколько разных фильтров (`byTags` + `byGroups`)
| **AND** — библиотека должна пройти все фильтры
|===

===== Алгоритм фильтрации

1. Для каждого типа фильтра (`byTags`, `byGroups`, `byAliases`) независимо:
a. Если нет `include` → все библиотеки проходят этот фильтр
b. Если есть `include` → только matching библиотеки проходят
c. Из результата удаляются matching `exclude`
2. Библиотека включается в distribution, только если прошла **ВСЕ** фильтры (AND)

===== Пример фильтрации

[source,kotlin]
----
// Библиотеки:
// - "retrofit"     tags=["network", "android"]  group="com.squareup"
// - "okhttp"       tags=["network"]             group="com.squareup"
// - "room"         tags=["database", "android"] group="androidx.room"
// - "postgres"     tags=["database", "server"]  group="org.postgresql"

distributions {
    distribution("android") {
        byTags { include("android") }                    // retrofit, room
        byGroups { exclude("androidx.room") }            // исключить room
    }
}
// Результат: только "retrofit"
// - retrofit: byTags=✓ (android), byGroups=✓ (не исключён) → AND=✓
// - okhttp:   byTags=✗ (нет android)                     → AND=✗
// - room:     byTags=✓ (android), byGroups=✗ (исключён)  → AND=✗
// - postgres: byTags=✗ (нет android)                     → AND=✗
----

'''

[quote]
____
Правило проверки совместимости между библиотеками или с окружением.
____

==== Типы правил

[cols="1,2,2",options="header"]
|===
| Тип | Описание | Пример

        libraries("jackson-core", "gson")
        severity = WARNING

        libraries("kotlin-stdlib", "kotlin-reflect", "kotlinx-coroutines-core")

==== Severity

[cols="1,3",options="header"]
|===
| Значение | Описание

| ERROR
| Нарушение блокирует генерацию

| WARNING
| Предупреждение, генерация продолжается

| INFO
| Информационное сообщение
|===

==== Поведение с отфильтрованными библиотеками

Правила проверяются только против **effective** (отфильтрованных) библиотек:

* Отфильтрованные библиотеки не триггерят правила

=== Platform Library

[quote]
____
Библиотека типа Platform/BOM для экспорта в TOML Version Catalog.
Отличается от обычной библиотеки: экспортируется с типом `pom` и используется потребителями через `platform()`.
____

NOTE: Не путать с `bomImports {}`, который используется для *импорта* версий из внешних BOM.
`platformLibrary()` — для *экспорта* BOM-зависимости в генерируемый TOML.

==== DSL синтаксис

[source,kotlin]
----
libraries {
    // Обычная библиотека
    library("kotlin-stdlib", "org.jetbrains.kotlin:kotlin-stdlib",
        version = versionRef("kotlin"))

    // Platform library — экспортируется в TOML как library с type=pom
    platformLibrary("spring-boot-bom", "org.springframework.boot:spring-boot-dependencies",
        version = versionRef("spring-boot"))
}
----

==== Отличия от library()

* В генерируемом TOML отображается с типом `pom`
* Потребители подключают через `implementation(platform(libs.spring.boot.bom))`
* Не содержит `supportedTargets` (это не runtime dependency)

'''

[[runtime-configuration]]
== Runtime Configuration

Настройки, которые влияют на поведение системы во время выполнения, но **НЕ сериализуются** в metadata.json.

TIP: DSL делится на две части: (1) **Данные** — `versions`, `libraries`, `bundles` и др. — сериализуются в `metadata.json`; (2) **Runtime конфигурация** — `processing {}` — используется только при выполнении pipeline и не сохраняется.

=== ProcessingConfig

[quote]
____
Конфигурация пайплайна обработки метаданных.
Определяет какие процессоры включены, их настройки и режим работы.

    preset = ProcessingPreset.STRICT

    // Или кастомная настройка процессоров

[cols="1,1,3",options="header"]
|===
| Атрибут | Тип | Описание

| distribution
| String
| Имя Distribution

| libraries
| Map<String, EffectiveLibrary>
| Библиотеки с разрешёнными версиями (после фильтрации)

| bundles
| Map<String, EffectiveBundle>
| Бандлы (после фильтрации)

| plugins
| Map<String, EffectivePlugin>
| Плагины с разрешёнными версиями

| versions
| Map<String, ResolvedVersion>

==== Расширения (Extensions)

`EffectiveMetadata` содержит встроенные поля для результатов built-in фич (`updates`, `vulnerabilities`, `licenseViolations` и др.), обеспечивая удобный прямой доступ.

Для custom SPI-процессоров предусмотрен механизм расширений через типобезопасный `ExtensionKey<T>`:

[source,kotlin]
----
// Объявление ключа (в модуле custom-процессора)
val MY_RESULT_KEY = ExtensionKey<MyCustomResult>("my-custom-result")

// Запись результата в процессоре
override suspend fun process(
    metadata: EffectiveMetadata,
    context: ProcessingContext
): EffectiveMetadata {
    val result = MyCustomResult(...)
    return metadata.withExtension(MY_RESULT_KEY, result)
}

// Чтение результата
val result = effectiveMetadata.getExtension(MY_RESULT_KEY)
----

NOTE: Extensions имеют аннотацию `@Transient` и не участвуют в JSON-сериализации.
Это runtime-only механизм для передачи данных между процессорами и генераторами.

===== Вспомогательные функции

[cols="1,3",options="header"]
|===
| Функция | Описание

| `getExtension(key: ExtensionKey<T>): T?`
| Получить значение расширения по типобезопасному ключу

| `withExtension(key: ExtensionKey<T>, value: T): EffectiveMetadata`
| Создать копию `EffectiveMetadata` с добавленным расширением (immutable)
|===

'''

=== ResolvedVersion

[quote]
____
Версия с информацией об источнике.
____

==== Атрибуты

[cols="1,1,3",options="header"]
|===
| Атрибут | Тип | Описание

| alias
| String
| Alias версии из DSL (PK в map)

| value
| String
| Значение версии (например: `3.4.0`)

| source
| VersionSource
| Источник версии

| originalRef
| VersionReference?
| Оригинальная ссылка на версию из DSL (до резолвинга)
|===

==== VersionSource (приоритет)

[cols="1,1,3",options="header"]
|===
| Приоритет | Значение | Описание

| 1 (высший)
| DECLARED
| Явно указана в DSL конфигурации через `version = "1.0.0"`, `versionRef("alias")` или inline в coordinates

| 2 (низший)
| BOM_IMPORT
| Импортирована из BomImport через BomImportProcessor
|===

==== Примеры VersionSource

[source,kotlin]
----
// DECLARED — явно указана версия
library("guava", "com.google.guava:guava", version = "33.0.0-jre")

// DECLARED — через versionRef
library("kotlin-stdlib", "org.jetbrains.kotlin:kotlin-stdlib",
    version = versionRef("kotlin"))

// DECLARED — inline версия в coordinates
library("slf4j", "org.slf4j:slf4j-api:2.0.9")

// BOM_IMPORT — версия будет из Spring Boot BOM
library("spring-web", "org.springframework:spring-web")  // без версии
----
