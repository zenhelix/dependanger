[[overview]]
== Обзор

Модуль `dependanger-maven-resolver` обеспечивает интеграцию с Maven-репозиториями: скачивание POM-файлов, парсинг `<dependencyManagement>`, кеширование BOM и резолвинг property-placeholder'ов.
Это базовый модуль, который используется другими feature-модулями (`dependanger-updates`, `dependanger-analysis`, `dependanger-license`, `dependanger-transitive`).

Приоритет: **P2** (важно).

[[spec-bom-import-processor]]
== SPEC-06.1: BomImportProcessor.process()

[[purpose-bom-import]]
=== Назначение

Импортировать версии из внешних Maven BOM (Bill of Materials) для библиотек, у которых нет явно заданной версии.
Процессор работает в фазе `BOM_IMPORT` (order=15, `SEQUENTIAL`) после `EXTRACTED_VERSIONS` (order=12) и перед `LIBRARY_FILTER` (order=20).
Параллельная загрузка нескольких BOM выполняется внутри процессора через coroutines.

[[algorithm-bom-import]]
=== Алгоритм

[source,text]
----
1. Получить список bomImports из context.originalMetadata.bomImports
2. Если bomImports пуст -> вернуть metadata без изменений
3. Получить настройки из context.settings (bomCache, timeouts). Repositories: `context.settings.repositories` -> Maven Central (см. xref:index.adoc#repository-convention[конвенция репозиториев])
4. Создать BomCache на основе настроек
5. Создать MavenPomDownloader (Ktor HTTP client)
6. Для каждого BOM (в порядке объявления):
   a. Попытаться загрузить из кеша -> если есть и TTL не истёк, использовать
   b. Иначе скачать POM из репозиториев (пробовать по порядку)
   c. Распарсить XML: извлечь <properties>, <dependencyManagement><dependencies>
   c1. Синтезировать built-in project.* properties из координат POM и parent
   d. Рекурсивно резолвить parent BOM (если есть <parent>), path-based cycle detection
   e. Рекурсивно обработать <dependency scope="import"> (вложенные BOM)
   f. Резолвить property placeholders: ${property.name} -> значения из merged properties
   f1. При неразрешённом property -> UNRESOLVED_BOM_PROPERTY diagnostic, пропустить dependency
   g. Сохранить BomContent в кеш (файловый + in-memory resolvedCache)
   h. При сетевой ошибке: если есть кеш (даже устаревший) -> использовать с WARNING
7. Объединить все BomContent в единую карту group:artifact -> version (первый BOM побеждает)
8. Для каждой библиотеки в metadata.libraries:
   a. Если version != null (DECLARED) -> пропустить
   b. Найти version в объединённой карте по group:artifact
   c. Если найдена -> создать ResolvedVersion(source=BOM_IMPORT, originalRef=bomAlias)
   d. Если не найдена -> оставить как есть (WARNING или ERROR в strict mode)
9. Вернуть обновлённую EffectiveMetadata с новыми версиями
----

==== MavenPomDownloader (новый класс)

Возвращает `DownloadResult` (sealed interface) вместо `String?` для типизированной обработки различных ошибок:

[source,kotlin]
----
public sealed interface DownloadResult {
    public data class Success(val content: String) : DownloadResult
    public data object NotFound : DownloadResult
    public data class AuthRequired(val url: String, val statusCode: Int) : DownloadResult
    public data class Failed(val error: String) : DownloadResult
}

public class MavenPomDownloader(
    private val repositories: List<MavenRepository>,
    private val httpClient: HttpClient,
    private val credentialsProvider: CredentialsProvider?,
    private val connectTimeoutMs: Long,
    private val readTimeoutMs: Long,
) {
    public suspend fun downloadPom(
        group: String,
        artifact: String,
        version: String,
    ): DownloadResult {
        val groupPath = group.replace('.', '/')
        val pomFileName = "$artifact-$version.pom"

        var lastAuthRequired: DownloadResult.AuthRequired? = null
        var lastFailure: DownloadResult.Failed? = null
        var allNotFound = true

        for (repo in repositories) {
            val url = "${repo.url.trimEnd('/')}/$groupPath/$artifact/$version/$pomFileName"
            when (val result = downloadWithRetry(url, repo.url)) {
                is DownloadResult.Success      -> return result
                is DownloadResult.AuthRequired -> {
                    allNotFound = false
                    lastAuthRequired = result
                }
                is DownloadResult.NotFound     -> { /* continue to next repo */ }
                is DownloadResult.Failed       -> {
                    allNotFound = false
                    lastFailure = result
                }
            }
        }

        return when {
            lastAuthRequired != null -> lastAuthRequired
            allNotFound              -> DownloadResult.NotFound
            lastFailure != null      -> lastFailure
            else                     -> DownloadResult.NotFound
        }
    }
}

==== Retry Logic

Retry логика делегирована в `getWithRetry()` extension function из модуля `dependanger-http-client`:

[source,kotlin]
----
private suspend fun downloadWithRetry(url: String, repoUrl: String): DownloadResult {
    val httpResult = httpClient.getWithRetry(url, RetryConfig()) {
        credentialsProvider?.getCredentials(repoUrl)?.let { creds ->
            basicAuth(creds.username, creds.password)
        }
        timeout {
            connectTimeoutMillis = connectTimeoutMs
            requestTimeoutMillis = readTimeoutMs
        }
    }

    return when (httpResult) {
        is HttpResult.Success      -> DownloadResult.Success(httpResult.data)
        is HttpResult.NotFound     -> DownloadResult.NotFound
        is HttpResult.AuthRequired -> DownloadResult.AuthRequired(httpResult.url, httpResult.statusCode)
        is HttpResult.RateLimited  -> DownloadResult.Failed("Rate limited for $url, retry after ${httpResult.retryAfterMs}ms")
        is HttpResult.Failed       -> DownloadResult.Failed(httpResult.error)
    }
}

==== PomXmlParser (новый класс)

Парсинг XML делегирован в `PomParser.parse()` из модуля `dependanger-maven-pom`.
Property resolution делегирован в `PropertyResolver` из того же модуля.

[source,kotlin]
----
public class PomXmlParser {

    public fun parseBomContent(pomXml: String): PomParseResult {
        val project = PomParser.parse(pomXml)
        return toPomParseResult(project)
    }

    public fun resolveProperty(
        value: String,
        properties: Map<String, String>,
    ): String {
        val resolver = PropertyResolver(properties)
        return try {
            resolver.resolve(value)
        } catch (e: PropertyResolutionException) {
            throw IllegalStateException("Unresolved property in BOM: ${e.message}", e)
        }
    }

    private fun toPomParseResult(project: PomProject): PomParseResult {
        val parent = project.parent?.let {
            ParentPom(
                group = it.coordinates.groupId,
                artifact = it.coordinates.artifactId,
                version = it.coordinates.version,
            )
        }

        val syntheticProperties = buildSyntheticProperties(project)
        val explicitProperties = project.properties.entries
        val properties = syntheticProperties + explicitProperties

        val rawDependencies = project.dependencyManagement?.dependencies?.map {
            RawBomDependency(
                group = it.groupId,
                artifact = it.artifactId,
                version = it.version ?: "",
                scope = it.scope,
                type = it.type,
            )
        } ?: emptyList()

        return PomParseResult(
            properties = properties,
            parent = parent,
            dependencies = rawDependencies,
        )
    }

    private fun buildSyntheticProperties(project: PomProject): Map<String, String> {
        val props = mutableMapOf<String, String>()

        props["project.groupId"] = project.coordinates.groupId
        props["project.artifactId"] = project.coordinates.artifactId
        props["project.version"] = project.coordinates.version

        project.parent?.let { p ->
            props["project.parent.groupId"] = p.coordinates.groupId
            props["project.parent.artifactId"] = p.coordinates.artifactId
            props["project.parent.version"] = p.coordinates.version
        }

        return props
    }
}

==== Property Resolution Precedence

Приоритет свойств (от высшего к низшему):

1. **Explicit `<properties>`** из текущего (child) POM
2. **Synthetic `project.*` properties** из текущего (child) POM (из координат)
3. **Все properties из parent POM** (рекурсивно, explicit + synthetic)

NOTE: Это упрощённая двухуровневая модель: child ALL > parent ALL.
В Maven приоритет формально четырёхуровневый (child explicit > parent explicit > child synthetic > parent synthetic), но на практике разница проявляется только если parent явно переопределяет `project.*` свойства через `<properties>`, что является антипаттерном.
Реализация через `parentProperties + childProperties` (Kotlin Map `+`, правая сторона побеждает) корректна для всех реальных сценариев.

* Child POM properties имеют приоритет над parent POM properties
* При конфликте -- child побеждает
* `project.groupId`, `project.version` наследуются от parent, если не заданы явно в child
* Nested properties (`$\{$\{var\}.version\}`) НЕ поддерживаются в первой итерации

Поддерживаемые built-in (synthetic) properties:

[cols="1,2"]
|===
| Property | Источник

| `project.groupId` | `<groupId>` или parent `<groupId>`
| `project.artifactId` | `<artifactId>`
| `project.version` | `<version>` или parent `<version>`
| `project.parent.groupId` | `<parent><groupId>`
| `project.parent.artifactId` | `<parent><artifactId>`
| `project.parent.version` | `<parent><version>`
|===

IMPORTANT: Nested property resolution (например `$\{$\{framework\}.version\}`) -- ограничение первой итерации.
При обнаружении вложенного placeholder'а -- WARNING в diagnostics, значение оставляется как есть.

==== BomImportProcessor.process()

[source,kotlin]
----
override suspend fun process(
    metadata: EffectiveMetadata,
    context: ProcessingContext
): EffectiveMetadata {
    val bomImports = context.originalMetadata.bomImports
    if (bomImports.isEmpty()) return metadata

    val settings = context.settings
    val cacheDir = settings.bomCache.directory
        ?: DependangerPaths.resolveInUserHome(DependangerPaths.BOM_CACHE_DIR)
    val repositories = settings.repositories
        .filterIsInstance<MavenRepository>()
        .ifEmpty {
            listOf(MavenRepository(url = "https://repo.maven.apache.org/maven2", name = "Maven Central"))
        }
    val credentialsProvider = context[CredentialsProviderKey]

    // BomResolutionContext (AutoCloseable) управляет HTTP-клиентом, кешем, downloader'ом и парсером
    BomResolutionContext(
        repositories = repositories,
        credentialsProvider = credentialsProvider,
        cacheDirectory = cacheDir,
        ttlHours = settings.bomCache.ttlHours,
        ttlSnapshotHours = settings.bomCache.ttlSnapshotHours,
    ).use { resolutionCtx ->

    // Резолвить все BOM (с path-based обнаружением циклов + кеш уже разрешённых)
    val resolvedBoms = mutableListOf<BomContent>()
    val resolvedCache = mutableMapOf<String, BomContent>() // optimization: не скачивать одно и то же дважды
    var diagnostics = metadata.diagnostics

    for (bomImport in bomImports) {
        // Разрешить VersionReference в строку версии
        val resolvedVersion = when (val v = bomImport.version) {
            is VersionReference.Literal -> v.version
            is VersionReference.Reference -> {
                val found = context.originalMetadata.versions.find { it.name == v.name }
                if (found == null) {
                    diagnostics = diagnostics + Diagnostics.error("BOM_UNRESOLVED_VERSION",
                        "Version reference '${v.name}' not found for BOM '${bomImport.alias}'")
                    continue
                }
                found.value
            }
            is VersionReference.Range -> {
                diagnostics = diagnostics + Diagnostics.warning("BOM_VERSION_RANGE",
                    "Version ranges not supported for BOM imports, skipping '${bomImport.alias}'")
                continue
            }
        }
        val pathStack = mutableSetOf<String>() // per-BOM path для cycle detection
        val result = resolveBomRecursive(
            bomImport.group, bomImport.artifact, resolvedVersion,
            cache, downloader, parser, pathStack, resolvedCache, diagnostics, depth = 0
        )
        resolvedBoms.add(result.content)
        diagnostics = result.diagnostics
    }

    // Объединить: первый BOM имеет приоритет
    val versionMap = mutableMapOf<String, Pair<String, String>>() // "group:artifact" -> (version, bomAlias)
    for ((index, bom) in resolvedBoms.withIndex()) {
        val bomAlias = bomImports[index].alias
        for (dep in bom.dependencies) {
            val key = "${dep.group}:${dep.artifact}"
            versionMap.putIfAbsent(key, dep.version to bomAlias)
        }
    }

    // Обогатить библиотеки
    val updatedLibraries = metadata.libraries.mapValues { (_, lib) ->
        if (lib.version != null) return@mapValues lib // DECLARED имеет приоритет
        val key = "${lib.group}:${lib.artifact}"
        val (version, bomAlias) = versionMap[key] ?: return@mapValues lib
        lib.copy(
            version = ResolvedVersion(
                alias = lib.alias,
                value = version,
                source = VersionSource.BOM_IMPORT,
                originalRef = bomAlias,
            ),
        )
    }

    return metadata.copy(
        libraries = updatedLibraries,
        diagnostics = diagnostics,
    )
}

==== Обнаружение циклов (path-based)

[source,kotlin]
----
internal data class BomResolveResult(
    val content: BomContent,
    val diagnostics: Diagnostics,
)
----

Детекция циклов использует **path-based** подход: `pathStack` содержит только узлы на текущем пути рекурсии.
Узел добавляется при входе и **удаляется при выходе** (`try/finally`).
Это корректно обрабатывает diamond-зависимости (BOM-A и BOM-B оба импортируют jackson-bom) без ложных CIRCULAR_BOM.

Дополнительно, `resolvedCache` хранит уже разрешённые BOM для оптимизации -- не скачивать и не парсить одно и то же дважды.

[source,kotlin]
----
private suspend fun resolveBomRecursive(
    group: String, artifact: String, version: String,
    cache: BomCache, downloader: MavenPomDownloader, parser: PomXmlParser,
    pathStack: MutableSet<String>,          // cycle detection: текущий путь рекурсии
    resolvedCache: MutableMap<String, BomContent>, // optimization: уже разрешённые BOM
    diagnostics: Diagnostics, depth: Int
): BomResolveResult {
    val key = "$group:$artifact:$version"

    // Цикл: этот BOM уже на текущем пути рекурсии
    if (key in pathStack) {
        return BomResolveResult(BomContent(), diagnostics + Diagnostics.error(
            "CIRCULAR_BOM", "Circular BOM dependency detected: $key", "bom-import"))
    }
    if (depth > 10) {
        return BomResolveResult(BomContent(), diagnostics + Diagnostics.warning(
            "BOM_DEPTH_EXCEEDED", "BOM parent hierarchy > 10 levels for $key", "bom-import"))
    }

    // Оптимизация: уже разрешено ранее (diamond dependency)
    resolvedCache[key]?.let { return BomResolveResult(it, diagnostics) }

    pathStack.add(key)
    try {
        // Попробовать файловый кеш
        val cached = cache.get(group, artifact, version)
        if (cached != null) {
            resolvedCache[key] = cached
            return BomResolveResult(cached, diagnostics)
        }

        // Скачать POM
        val pomXml = downloader.downloadPom(group, artifact, version)
        if (pomXml == null) {
            val stale = cache.getStale(group, artifact, version)
            if (stale != null) {
                resolvedCache[key] = stale
                return BomResolveResult(stale, diagnostics + Diagnostics.warning(
                    "BOM_STALE_CACHE", "Using stale cache for BOM $key", "bom-import"))
            }
            return BomResolveResult(BomContent(), diagnostics + Diagnostics.error(
                "BOM_FETCH_FAILED", "Cannot fetch BOM: $key", "bom-import"))
        }

        // Парсинг (включает синтетические project.* properties)
        val parseResult = parser.parseBomContent(pomXml)

        // Резолвить parent
        var mergedProperties = parseResult.properties
        var mergedDependencies = mutableListOf<BomDependency>()
        var currentDiagnostics = diagnostics

        parseResult.parent?.let { parent ->
            val parentResult = resolveBomRecursive(
                parent.group, parent.artifact, parent.version,
                cache, downloader, parser, pathStack, resolvedCache, currentDiagnostics, depth + 1
            )
            mergedProperties = parentResult.content.properties + mergedProperties // child overrides
            mergedDependencies.addAll(parentResult.content.dependencies)
            currentDiagnostics = parentResult.diagnostics
        }

        // Резолвить property placeholders и добавить зависимости
        for (rawDep in parseResult.dependencies) {
            if (rawDep.scope == "import" && rawDep.type == "pom") {
                // Рекурсивный BOM import -- resolveProperty с обработкой ошибок
                val resolvedVersion = try {
                    parser.resolveProperty(rawDep.version, mergedProperties)
                } catch (e: IllegalStateException) {
                    currentDiagnostics = currentDiagnostics + Diagnostics.error(
                        "UNRESOLVED_BOM_PROPERTY", "${e.message} in import ${rawDep.group}:${rawDep.artifact}", "bom-import")
                    continue // пропустить этот BOM import
                }
                val importResult = resolveBomRecursive(
                    rawDep.group, rawDep.artifact, resolvedVersion,
                    cache, downloader, parser, pathStack, resolvedCache, currentDiagnostics, depth + 1
                )
                mergedDependencies.addAll(importResult.content.dependencies)
                currentDiagnostics = importResult.diagnostics
            } else {
                // Обычная dependency -- resolveProperty с обработкой ошибок
                try {
                    mergedDependencies.add(BomDependency(
                        group = parser.resolveProperty(rawDep.group, mergedProperties),
                        artifact = parser.resolveProperty(rawDep.artifact, mergedProperties),
                        version = parser.resolveProperty(rawDep.version, mergedProperties),
                    ))
                } catch (e: IllegalStateException) {
                    currentDiagnostics = currentDiagnostics + Diagnostics.error(
                        "UNRESOLVED_BOM_PROPERTY", "${e.message} in ${rawDep.group}:${rawDep.artifact}:${rawDep.version}", "bom-import")
                    // пропустить эту dependency
                }
            }
        }

        val content = BomContent(mergedDependencies, mergedProperties)
        try { cache.put(group, artifact, version, content) } catch (_: Exception) { /* WARNING */ }
        resolvedCache[key] = content

        return BomResolveResult(content, currentDiagnostics)
    } finally {
        pathStack.remove(key) // path-based: убрать узел при выходе из рекурсии
    }
}

== SPEC-06.2: BomCache

[[purpose-bom-cache]]
=== Назначение

Файловый кеш для скачанных BOM-файлов.
Хранит сериализованные `BomContent` в локальной файловой системе.

[[algorithm-bom-cache]]
=== Алгоритм кеширования

==== Структура директории кеша

[source,text]
----
{user.home}/.dependanger/cache/bom/   <-- System.getProperty("user.home")
  org/
    springframework/
      boot/
        spring-boot-dependencies/
          3.4.0/
            bom-content.json      # Сериализованный BomContent
            metadata.json         # {fetchedAt, ttl, isSnapshot}

==== BomCache.get()

[source,text]
----
1. Построить путь к файлу: {cacheDir}/{groupPath}/{artifact}/{version}/bom-content.json
2. Проверить существование файла -> если нет, вернуть null
3. Прочитать metadata.json -> fetchedAt, isSnapshot
4. Рассчитать TTL: если isSnapshot -> ttlSnapshotHours, иначе ttlHours
5. Если (now - fetchedAt) > ttl -> вернуть null (expired)
6. Десериализовать bom-content.json в BomContent
7. Вернуть BomContent?
----

[source,kotlin]
----
public fun get(group: String, artifact: String, version: String): BomContent? {
    val dir = resolveCacheDir(group, artifact, version)
    val contentFile = dir.resolve("bom-content.json")
    val metaFile = dir.resolve("metadata.json")

    if (!contentFile.exists() || !metaFile.exists()) return null

    val meta = Json.decodeFromString<CacheMetadata>(metaFile.readText())
    val ttl = if (version.endsWith("-SNAPSHOT")) ttlSnapshotHours else ttlHours
    val ageHours = (Clock.System.now() - meta.fetchedAt).inWholeHours

    if (ageHours > ttl) return null

    return Json.decodeFromString<BomContent>(contentFile.readText())
}

public fun getStale(group: String, artifact: String, version: String): BomContent? {
    val dir = resolveCacheDir(group, artifact, version)
    val contentFile = dir.resolve("bom-content.json")
    if (!contentFile.exists()) return null
    return try {
        Json.decodeFromString<BomContent>(contentFile.readText())
    } catch (_: Exception) {
        null
    }
}

public fun put(group: String, artifact: String, version: String, content: BomContent) {
    val dir = resolveCacheDir(group, artifact, version)
    dir.mkdirs()

    val contentFile = dir.resolve("bom-content.json")
    val metaFile = dir.resolve("metadata.json")

    // Atomic write через temp file
    val tempContent = File.createTempFile("bom-", ".tmp", dir)
    tempContent.writeText(Json.encodeToString(content))
    tempContent.renameTo(contentFile)

    val meta = CacheMetadata(
        fetchedAt = Clock.System.now(),
        isSnapshot = version.endsWith("-SNAPSHOT"),
    )
    val tempMeta = File.createTempFile("meta-", ".tmp", dir)
    tempMeta.writeText(Json.encodeToString(meta))
    tempMeta.renameTo(metaFile)
}

=== Обработка HTTP ответов

[cols="1,2"]
|===
| Код | Поведение

| 200 OK | Успех, парсить тело
| 404 Not Found | Пробовать следующий репозиторий
| 401/403 | ERROR "Authentication required for {url}"
| 429 | Retry с Retry-After header
| 500+ | Retry 3 раза с exponential backoff (1s, 2s, 4s)
| Timeout | Retry 1 раз, затем ERROR
|===

[[pom-xml-parsing]]
== Парсинг POM XML

=== Структура Maven POM (BOM)

[source,xml]
----
<project>
    <parent>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-parent</artifactId>
        <version>3.4.0</version>
    </parent>

    <properties>
        <spring.version>6.2.1</spring.version>
        <jackson.version>2.17.0</jackson.version>
    </properties>

    <dependencyManagement>
        <dependencies>
            <!-- Обычная зависимость с property placeholder -->
            <dependency>
                <groupId>org.springframework</groupId>
                <artifactId>spring-web</artifactId>
                <version>${spring.version}</version>
            </dependency>

            <!-- Вложенный BOM import -->
            <dependency>
                <groupId>com.fasterxml.jackson</groupId>
                <artifactId>jackson-bom</artifactId>
                <version>${jackson.version}</version>
                <type>pom</type>
                <scope>import</scope>
            </dependency>
        </dependencies>
    </dependencyManagement>
</project>
----

=== Алгоритм парсинга

[source,text]
----
1. Распарсить XML через DocumentBuilderFactory (stdlib javax.xml)
2. Найти /project/parent -> ParentPom? (groupId, artifactId, version)
3. Синтезировать project.* properties из /project/groupId, /project/artifactId, /project/version и parent
4. Найти /project/properties -> Map<String, String>, объединить с synthetic (explicit overrides)
5. Найти /project/dependencyManagement/dependencies/dependency[]
6. Для каждой dependency:
   a. Извлечь groupId, artifactId, version, scope, type
   b. Если scope="import" && type="pom" -> отметить как вложенный BOM
   c. Иначе -> обычная BOM dependency
7. Property resolution: заменить ${name} на значение из merged properties (includes project.*)
8. Parent resolution: рекурсивно загрузить parent, объединить properties и dependencies
----

[[edge-cases]]
== Edge Cases и обработка ошибок

[cols="1,2,2"]
|===
| Ситуация | Поведение | Диагностический код

| Пустой bomImports | Процессор пропускается, возврат без изменений | --
| Невалидный XML | ERROR, pipeline останавливается | `INVALID_BOM_XML`
| BOM без dependencyManagement | WARNING, BOM игнорируется | `BOM_NO_DEPS`
| Неразрешённый property | ERROR с именем property | `UNRESOLVED_BOM_PROPERTY`
| Циклическая зависимость BOM | ERROR, обнаружение через path-based stack | `CIRCULAR_BOM`
| Diamond BOM dependency (shared parent/import) | OK, resolvedCache возвращает ранее вычисленный результат | --
| Глубокая parent иерархия (>10) | WARNING, остановка рекурсии | `BOM_DEPTH_EXCEEDED`
| `${project.version}` и другие built-in | Синтезируются из координат POM в parseBomContent() | --
| Сеть недоступна, кеш есть | WARNING, использовать стейл кеш | `BOM_STALE_CACHE`
| Сеть недоступна, кеша нет | ERROR, pipeline останавливается | `BOM_FETCH_FAILED`
| Snapshot BOM | TTL = ttlSnapshotHours (1h default), WARNING | --
| Кеш повреждён | WARNING, удалить и перекачать | `BOM_CACHE_CORRUPT`
| Нет прав на запись в кеш | WARNING, работать без кеша | `BOM_CACHE_READONLY`
| Дубликат в BOM | WARNING, последний побеждает (как Maven) | `BOM_DUPLICATE_ENTRY`
| HTTP 401/403 | ERROR с подсказкой настроить credentials | `BOM_AUTH_REQUIRED`
|===

[[diagnostic-codes]]
== Диагностические коды

[cols="1,2,1"]
|===
| Код | Описание | Уровень

| `BOM_VERSION_IMPORTED` | Версия импортирована из BOM | INFO
| `BOM_NO_DEPS` | BOM не содержит dependencyManagement | WARNING
| `BOM_STALE_CACHE` | Используется устаревший кеш | WARNING
| `BOM_DUPLICATE_ENTRY` | Дублирующаяся dependency в BOM | WARNING
| `BOM_CACHE_CORRUPT` | Повреждённый кеш, перекачка | WARNING
| `BOM_CACHE_READONLY` | Нет прав на запись в кеш | WARNING
| `BOM_SNAPSHOT_WARNING` | Snapshot BOM версия нестабильна | WARNING
| `INVALID_BOM_XML` | Невалидный XML в BOM | ERROR
| `UNRESOLVED_BOM_PROPERTY` | Неразрешённый property placeholder | ERROR
| `CIRCULAR_BOM` | Циклическая зависимость BOM | ERROR
| `BOM_DEPTH_EXCEEDED` | Слишком глубокая parent иерархия | WARNING
| `BOM_FETCH_FAILED` | Невозможно скачать BOM | ERROR
| `BOM_AUTH_REQUIRED` | Требуется аутентификация | ERROR
|===

[[extension-keys]]
== ExtensionKey

BomImportProcessor не использует ExtensionKey напрямую -- результат записывается непосредственно в `EffectiveMetadata.libraries` (поле `version: ResolvedVersion?`).

[[testing]]
== Тестовые сценарии

=== Юнит-тесты BomImportProcessor

[cols="1,3"]
|===
| Тест | Описание

| `empty bomImports returns unchanged metadata` | Пустой список BOM -> metadata без изменений
| `single BOM resolves library version` | Spring BOM -> spring-web получает версию
| `declared version not overridden by BOM` | library с explicit version -> BOM не перезаписывает
| `multiple BOMs first wins` | BOM-A и BOM-B содержат одну library -> версия из BOM-A
| `BOM with property placeholders` | `${spring.version}` -> резолвится из `<properties>`
| `parent BOM inheritance` | BOM наследует dependencies от parent
| `nested BOM import (scope=import)` | BOM импортирует другой BOM
| `circular BOM detection` | BOM-A -> BOM-B -> BOM-A -> CIRCULAR_BOM ERROR
| `diamond BOM is not circular` | BOM-A и BOM-B оба импортируют BOM-C -> OK, не CIRCULAR_BOM
| `shared parent is not circular` | BOM-A и BOM-B оба имеют parent P -> OK, не CIRCULAR_BOM
| `unresolved property produces diagnostic` | `${unknown}` в dependency -> UNRESOLVED_BOM_PROPERTY ERROR, dependency пропускается
| `project.version resolves` | BOM с `${project.version}` -> резолвится из координат POM
| `network failure with stale cache` | Сеть недоступна, есть кеш -> WARNING + кеш
| `network failure without cache` | Сеть недоступна, кеша нет -> ERROR
| `strict mode unresolved version` | Библиотека без версии, strict -> ERROR
| `non-strict mode unresolved version` | Библиотека без версии, non-strict -> WARNING
|===

=== Юнит-тесты BomCache

[cols="1,3"]
|===
| Тест | Описание

| `put and get returns content` | Запись и чтение из кеша
| `expired TTL returns null` | TTL истёк -> null, getStale() возвращает контент

=== Юнит-тесты PomXmlParser

[cols="1,3"]
|===
| Тест | Описание

| `parse valid BOM POM` | Корректный BOM -> PomParseResult
| `parse properties` | `<properties>` -> Map
| `resolveProperty with placeholders` | `${spring.version}` -> "6.2.1"
| `resolveProperty unresolved throws` | `${unknown}` -> IllegalStateException
| `resolveProperty no placeholders` | Строка без `${}` -> возвращается как есть
| `parse parent POM` | `<parent>` -> ParentPom
| `nested import scope` | `scope=import type=pom` -> отмечается как вложенный BOM
| `invalid XML throws` | Невалидный XML -> парсинг-ошибка
| `synthetic project.version` | POM с `<version>3.4.0</version>` -> properties содержит `project.version=3.4.0`
| `synthetic project.groupId from parent` | POM без `<groupId>` с parent -> `project.groupId` = parent groupId
| `synthetic project.parent.*` | POM с parent -> `project.parent.groupId`, `project.parent.version` заполнены
| `explicit property overrides synthetic` | `<properties><project.version>custom</...>` -> `project.version=custom`
|===

[[priority]]
== Приоритет

**P2** (важно) -- BOM Import является ключевой feature для продуктивного использования.
Используется в DSL через `bomImports { }` и включается через `withBomImport()` или `processing { importBoms = true }`.
