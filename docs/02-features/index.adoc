== Матрица приоритетов

=== P1 — Критично для MVP

Без этих фич продукт не имеет ценности.

* **F001: Registry DSL** — основа всей системы, Kotlin DSL (включая settings)
* **F002: Metadata Serialization** — промежуточное хранение
* **F003: Processing Pipeline** — трансформация данных
* **F004: TOML Generation** — основной выходной формат

=== P2 — Важно

Значительно увеличивают ценность, но MVP возможен и без них.

* **F005: BOM Generation** — выходной формат для Maven
* **F006: BOM Import** — импорт версий из Spring BOM, Kotlin BOM и др.

=== P3 — Желательно

Улучшают user experience, но не критичны.

* **F007: Update Check** — проверка обновлений
* **F008: Compatibility Analysis** — анализ совместимости
* **F009: Report Generation** — единый отчёт

== SPI точки расширения

Каждая фича определяет точки расширения:

[cols="1,2,2",options="header"]
|===
| Фича | SPI | Описание

| F002
| `SerializationFormat<T>`
| Добавление новых форматов сериализации

| F003
| `EffectiveMetadataProcessor`
| Добавление процессоров в pipeline

| F004, F005
| `ArtifactGenerator<T>`
| Добавление новых генераторов
|===

== Как добавить новую фичу

. Скопируйте шаблон: `docs/_templates/feature-template.adoc`
. Переименуйте: `F###-краткое-название.adoc`
. Заполните все секции
. Добавьте в таблицу выше
. Обновите матрицу приоритетов и зависимости
