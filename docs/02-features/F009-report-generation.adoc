Генерация единого JSON отчёта о зависимостях, обновлениях и совместимости.
--

== Метаданные

[cols="1,2"]
|===
| ID | F009
| Название | Report Generation
| Приоритет | P3 (желательно)
| Модуль | `dependanger-report`
| Зависит от | F003, F007, F008
| Блокирует | —
|===

== Описание

Report Generation создаёт единый JSON отчёт, объединяющий информацию о:

* Всех библиотеках с их версиями и источниками
* Доступных обновлениях (из F007)
* Проблемах совместимости (из F008)
* Валидационных ошибках и предупреждениях

Отчёт предназначен для машинного парсинга в CI/CD, дашбордах и инструментах анализа.

== User Stories

=== US-001: Единый отчёт для CI

[quote]
____
Как DevOps инженер, я хочу получать единый JSON отчёт о состоянии зависимостей, чтобы интегрировать его в CI/CD pipeline и дашборды.
____

**Acceptance Criteria:**

[cols="1,3,3,3",options="header"]
|===
| ID | Given | When | Then

| AC-001-1
| EffectiveMetadata с 10 библиотеками, 3 плагинами, 2 бандлами
| Вызываю `result.generateReport()`
| JSON содержит `libraries` (10), `plugins` (3), `bundles` (2)

| AC-001-2
| Pipeline выполнен с `withUpdateCheck()`
| Генерирую отчёт
| Секция `updates` содержит информацию об обновлениях

| AC-001-3
| Pipeline выполнен с `withCompatibilityAnalysis()`
| Генерирую отчёт
| Секция `compatibility` содержит найденные проблемы

| AC-001-4
| Сгенерированный отчёт
| Парсю через `jq` или любой JSON parser
| Валидный JSON, соответствует схеме

| AC-001-5
| Gradle task `dependangerReport`
| Запускаю task
| Файл создан в `build/dependanger/dependanger-report.json`
|===

=== US-002: Секция summary

[quote]
____
Как менеджер, я хочу видеть сводку состояния зависимостей, чтобы быстро оценить ситуацию.
____

**Acceptance Criteria:**

[cols="1,3,3,3",options="header"]
|===
| ID | Given | When | Then

| AC-002-1
| 45 библиотек, 12 с обновлениями, 2 проблемы совместимости
| Генерирую отчёт
| `summary.libraryCount = 45`, `summary.updatesAvailable = 12`, `summary.compatibilityIssues = 2`

| AC-002-2
| 3 validation errors, 5 warnings
| Генерирую отчёт
| `summary.validationErrors = 3`, `summary.validationWarnings = 5`

| AC-002-3
| Пустой каталог (0 библиотек)
| Генерирую отчёт
| `summary.libraryCount = 0`, все счётчики = 0
|===

=== US-003: Фильтрация содержимого отчёта

[quote]
____
Как DevOps инженер, я хочу выбирать какие секции включать в отчёт, чтобы уменьшить его размер.
____

**Acceptance Criteria:**

[cols="1,3,3,3",options="header"]
|===
| ID | Given | When | Then

| AC-003-1
| Опция `--include-updates=false`
| Генерирую отчёт

| AC-003-2
| Опция `--include-compatibility=false`
| Генерирую отчёт
| Секция `compatibility` отсутствует или пуста

| AC-003-3
| Опция `--summary-only`
| Генерирую отчёт
| Только секции `schemaVersion`, `generatedAt`, `distribution`, `summary`
|===

[quote]
____
Как team lead, я хочу видеть список deprecated библиотек в отчёте, чтобы планировать миграцию на актуальные зависимости.
____

**Acceptance Criteria:**

[cols="1,3,3,3",options="header"]
|===
| ID | Given | When | Then

| AC-004-1
| 3 deprecated библиотеки в каталоге
| Генерирую отчёт
| Секция `deprecated` содержит 3 записи, `summary.deprecatedCount = 3`

| AC-004-2
| Deprecated библиотека с `replacedBy`, `message`, `removalVersion`
| Генерирую отчёт
| Запись содержит все поля deprecation info

| AC-004-3
| Deprecated библиотека используется в бандле `legacy`
| Генерирую отчёт
| Запись содержит `usedInBundles: ["legacy"]`

| AC-004-4
| Нет deprecated библиотек
| Генерирую отчёт
| `summary.deprecatedCount = 0`, секция `deprecated` пустой массив
|===

=== US-005: Секция vulnerabilities в отчёте

[quote]
____
Как security engineer, я хочу видеть список уязвимостей в отчёте, чтобы отслеживать безопасность зависимостей.
____

**Acceptance Criteria:**

[cols="1,3,3,3",options="header"]
|===
| ID | Given | When | Then

| AC-005-1
| Pipeline выполнен с `withSecurityCheck()`, найдено 3 уязвимости
| Генерирую отчёт
| Секция `vulnerabilities` содержит 3 записи, `summary.vulnerabilitiesCount = 3`

| AC-005-2
| Уязвимость с CVE, severity, CVSS, fixedVersion
| Генерирую отчёт
| Запись содержит все поля VulnerabilityInfo

| AC-005-3
| 1 CRITICAL + 2 HIGH уязвимости
| Генерирую отчёт
| `summary.criticalVulnerabilities = 1`, `summary.highVulnerabilities = 2`

| AC-005-4
| Security check не выполнен (без `withSecurityCheck()`)
| Генерирую отчёт
| Секция `vulnerabilities` отсутствует или `null`
|===

== API использование

[source,kotlin]
----
val result = Dependanger.fromDsl {
    // ... DSL config
}
.withUpdateCheck()
.withCompatibilityAnalysis(targetJdk = 17)
.build()

// Генерация отчёта
val report = result.generateReport()

// Запись в файл
result.writeReportTo(Path.of("dependanger-report.json"))
----

== Версионирование схемы

[cols="1,3",options="header"]
|===
| Версия | Изменения

| 1.0
| Начальная версия схемы

| 1.1
| Добавлено поле `vulnerabilities` с информацией о CVE

| 1.2
| Добавлено поле `licenses` с информацией о лицензиях
|===

При парсинге отчёта:

* Клиент ДОЛЖЕН проверять `schemaVersion`
* Неизвестные поля ДОЛЖНЫ игнорироваться (forward compatibility)
* Старые версии схемы поддерживаются (backward compatibility)

== Функциональные требования

[cols="1,1,4",options="header"]
|===
| ID | Приоритет | Требование

| FR-001
| MUST
| Система ДОЛЖНА генерировать отчёт в JSON формате

| FR-002
| MUST
| Система ДОЛЖНА включать `schemaVersion` для версионирования схемы

| FR-003
| MUST
| Система ДОЛЖНА включать `generatedAt` с timestamp генерации (ISO 8601)

| FR-004
| MUST
| Система ДОЛЖНА включать секцию `summary` со счётчиками

| FR-005
| MUST
| Система ДОЛЖНА включать секцию `libraries` со всеми библиотеками

| FR-006
| SHOULD
| Система ДОЛЖНА включать секцию `updates` если выполнен UpdateCheck

| FR-007
| SHOULD
| Система ДОЛЖНА включать секцию `compatibility` если выполнен CompatibilityAnalysis

| FR-008
| SHOULD
| Система ДОЛЖНА включать секцию `validation` с ошибками и предупреждениями

| FR-009
| SHOULD
| Система ДОЛЖНА поддерживать фильтрацию секций через опции

| FR-010
| MAY
| Система МОЖЕТ поддерживать YAML формат вывода

| FR-011
| MAY
| Система МОЖЕТ поддерживать Markdown формат вывода (human-readable)

| FR-012
| MAY
| Система МОЖЕТ поддерживать режим `--summary-only` для краткого отчёта

| FR-013
| SHOULD
| Система ДОЛЖНА включать секцию `deprecated` со списком deprecated библиотек (replacedBy, message, since, removalVersion, usedInBundles)

| FR-014
| SHOULD
| Система ДОЛЖНА включать `deprecatedCount` в секцию `summary`

| FR-015
| SHOULD
| Система ДОЛЖНА включать секцию `vulnerabilities` если выполнен SecurityCheck (CVE, severity, CVSS, fixedVersion)

| FR-016
| SHOULD
| Система ДОЛЖНА включать `vulnerabilitiesCount`, `criticalVulnerabilities`, `highVulnerabilities` в секцию `summary`

| FR-017
| SHOULD

== Edge Cases

[cols="1,3,2",options="header"]
|===
| Сценарий | Описание | Ожидаемое поведение

| Пустой каталог
| EffectiveMetadata без библиотек, плагинов, бандлов
| Валидный JSON с пустыми массивами, `summary` с нулями

| Нет deprecated библиотек
| Ни одна библиотека не помечена как deprecated
| `summary.deprecatedCount = 0`, `deprecated = []`

| SecurityCheck не выполнен
| Pipeline без `withSecurityCheck()`
| Секция `vulnerabilities` отсутствует или `"vulnerabilities": null`

| Нет уязвимостей
| Все библиотеки безопасны
| `summary.vulnerabilitiesCount = 0`, `vulnerabilities = []`

| LicenseCheck не выполнен
| Pipeline без `withLicenseCheck()`
| Секция `licenses` отсутствует или `"licenses": null`

| Все лицензии allowed
| Все библиотеки с разрешёнными лицензиями
| `summary.deniedLicenses = 0`, `summary.unknownLicenses = 0`

| Библиотека с неизвестной лицензией
| Лицензия не определена ни в одном источнике
| `policyStatus = "UNKNOWN"`, `source = "UNKNOWN"`

| Deprecated без usedInBundles
| Deprecated библиотека не используется ни в одном бандле
| `usedInBundles = []`

| UpdateCheck не выполнен
| Pipeline без `withUpdateCheck()`
| Секция `updates` отсутствует или `"updates": null`

| CompatibilityAnalysis не выполнен
| Pipeline без `withCompatibilityAnalysis()`
| Секция `compatibility` отсутствует или `"compatibility": null`

| Нет ошибок валидации
| Все проверки прошли успешно
| `validation.errors = []`, `validation.warnings = []`

| Очень большой каталог
| 1000+ библиотек
| Отчёт генерируется (возможно большой файл), рекомендуется `--summary-only`

| Библиотека без версии
| Библиотека с `version = null`
| `"version": null` в JSON, WARNING в validation

| Специальные символы
| Библиотека с description содержащим `"`, `\n`, unicode
| Корректное экранирование в JSON

| Нет прав на запись
| Директория read-only
| ERROR: "Cannot write report to [path]: Permission denied"

| Файл уже существует
| `dependanger-report.json` уже есть
| Перезаписывается

| distribution не указан
| Генерация без указания distribution
| `"distribution": null` или `"distribution": "default"`

| Циклическая ссылка
| (не должно быть в EffectiveMetadata)
| Валидация в pipeline предотвращает

| Невалидный формат вывода
| `--format=xml` (не поддерживается)
| ERROR: "Unsupported format 'xml'. Supported: json, yaml, markdown"

| Timestamp с timezone
| Генерация в разных timezone
| Всегда UTC: `"generatedAt": "2026-01-27T10:30:00Z"`

| Прерывание генерации
| Ctrl+C во время записи файла
| Частичный файл удаляется (atomic write через temp file)

| Concurrent генерация
| Два процесса генерируют в один файл
| File locking или последний выигрывает с WARNING
|===
