[[overview]]
== Обзор

Модуль `dependanger-generator-bom` отвечает за генерацию Maven BOM (Bill of Materials) файлов из `EffectiveMetadata`.
Класс `BomGenerator` реализует интерфейс `ArtifactGenerator<String>` и производит текстовое содержимое POM XML-файла с секцией `<dependencyManagement>`.

Maven BOM -- стандартный механизм централизованного управления версиями зависимостей в Maven/Gradle экосистеме.
BOM-файл представляет собой POM с `<packaging>pom</packaging>` и блоком `<dependencyManagement>`, перечисляющим зависимости с зафиксированными версиями.

[[generate-algorithm]]
== Алгоритм generate()

=== Назначение

Преобразовать `EffectiveMetadata` в строку, содержащую валидный Maven POM XML с `<dependencyManagement>`.

=== Высокоуровневый алгоритм

[source,text]
----
1. Валидация BomConfig:
   - groupId, artifactId, version не пустые

2. Извлечь данные из EffectiveMetadata:
   - libraries: Map<String, EffectiveLibrary>

3. Подготовить список зависимостей:
   - Отфильтровать библиотеки: только с version != null (BOM требует конкретных версий)
   - Сортировать по groupId:artifactId
   - Platform-библиотеки (isPlatform=true): добавить <type>pom</type> и <scope>import</scope>

4. Построить XML-документ:
   a. XML-декларация: <?xml version="1.0" encoding="UTF-8"?>
   b. <project> с Maven namespace
   c. <modelVersion>4.0.0</modelVersion>
   d. Координаты из BomConfig: groupId, artifactId, version
   e. <packaging>pom</packaging>
   f. <dependencyManagement> с <dependencies>
   g. Для каждой библиотеки:
      - Если deprecated && includeDeprecationComments -> XML-комментарий
      - <dependency> с groupId, artifactId, version
      - Если isPlatform -> <type>pom</type>, <scope>import</scope>
      - Если includeOptionalDependencies -> <optional>true</optional>

5. Сериализовать XML в строку:
   - Если prettyPrint -> с отступами и переводами строк
   - Иначе -> минифицированный XML

6. Вернуть результирующую строку
----

=== Детальная реализация

[source,kotlin]
----
public class BomGenerator(private val config: BomConfig) : ArtifactGenerator<String> {
    override val generatorId: String = "maven-bom"
    override val description: String = "Generates Maven BOM (Bill of Materials) POM file"
    override val fileExtension: String = "xml"

    private val logger = KotlinLogging.logger {}

    companion object {
        private const val MAVEN_POM_NS = "http://maven.apache.org/POM/4.0.0"
        private const val MAVEN_XSI_NS = "http://www.w3.org/2001/XMLSchema-instance"
        private const val MAVEN_XSD_LOCATION =
            "http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd"
        private const val MODEL_VERSION = "4.0.0"
        private const val INDENT = "    " // 4 пробела
    }

    override fun generate(effective: EffectiveMetadata): String {
        validateConfig()

        logger.info { "Generating Maven BOM (generatorId=$generatorId) for ${config.groupId}:${config.artifactId}:${config.version}" }

        val dependencies = prepareDependencies(effective)
        val xml = buildXml(dependencies)

        logger.info { "BOM generation complete: ${dependencies.size} dependencies" }
        return xml
    }
}
----

=== Подготовка списка зависимостей

[source,kotlin]
----
private fun prepareDependencies(effective: EffectiveMetadata): List<EffectiveLibrary> {
    return effective.libraries.values
        // Только библиотеки с явной версией
        .filter { it.version != null }
        // Сортировка по groupId:artifactId
        .sortedWith(compareBy({ it.group }, { it.artifact }))
}
----

*Логика фильтрации:*

1. Библиотеки без версии (`version = null`) исключаются -- BOM требует конкретные версии.
2. Библиотеки без версии в EffectiveMetadata означают, что версия не была разрешена (managed dependency без источника).
3. Логируется предупреждение для исключённых библиотек.

[source,kotlin]
----
// Внутри prepareDependencies, перед фильтрацией:
val noVersion = effective.libraries.values.filter { it.version == null }
if (noVersion.isNotEmpty()) {
    logger.warn { "Skipping ${noVersion.size} libraries without version: ${noVersion.map { it.alias }}" }
}
----

=== Построение XML

[source,kotlin]
----
private fun buildXml(dependencies: List<EffectiveLibrary>): String {
    val sb = StringBuilder()
    val indent = if (config.prettyPrint) INDENT else ""
    val nl = if (config.prettyPrint) "\n" else ""

    // XML-декларация
    sb.append("<?xml version=\"1.0\" encoding=\"UTF-8\"?>$nl")

    // <project>
    sb.append("<project xmlns=\"$MAVEN_POM_NS\"$nl")
    sb.append("${indent.repeat(2)}xmlns:xsi=\"$MAVEN_XSI_NS\"$nl")
    sb.append("${indent.repeat(2)}xsi:schemaLocation=\"$MAVEN_XSD_LOCATION\">$nl")

    // Координаты
    sb.append("$indent<modelVersion>$MODEL_VERSION</modelVersion>$nl")
    sb.append("$indent<groupId>${escapeXml(config.groupId)}</groupId>$nl")
    sb.append("$indent<artifactId>${escapeXml(config.artifactId)}</artifactId>$nl")
    sb.append("$indent<version>${escapeXml(config.version)}</version>$nl")
    sb.append("$indent<packaging>pom</packaging>$nl")

    // dependencyManagement
    if (dependencies.isNotEmpty()) {
        sb.append("${nl}$indent<dependencyManagement>$nl")
        sb.append("$indent$indent<dependencies>$nl")

        for (lib in dependencies) {
            // Deprecated-комментарий
            if (lib.isDeprecated && config.includeDeprecationComments) {
                sb.append(buildDeprecationXmlComment(lib, indent))
            }
            // <dependency>
            sb.append(buildDependencyElement(lib, indent))
        }

        sb.append("$indent$indent</dependencies>$nl")
        sb.append("$indent</dependencyManagement>$nl")
    }

    // </project>
    sb.append("</project>$nl")

    return sb.toString()
}
----

=== Формирование элемента <dependency>

[source,kotlin]
----
private fun buildDependencyElement(lib: EffectiveLibrary, baseIndent: String): String {
    val nl = if (config.prettyPrint) "\n" else ""
    val indent3 = if (config.prettyPrint) baseIndent.repeat(3) else ""
    val indent4 = if (config.prettyPrint) baseIndent.repeat(4) else ""

    val sb = StringBuilder()
    sb.append("$indent3<dependency>$nl")
    sb.append("$indent4<groupId>${escapeXml(lib.group)}</groupId>$nl")
    sb.append("$indent4<artifactId>${escapeXml(lib.artifact)}</artifactId>$nl")

    // Версия -- всегда инлайн (BOM не поддерживает version.ref)
    val versionValue = lib.version?.value
    if (versionValue != null) {
        sb.append("$indent4<version>${escapeXml(versionValue)}</version>$nl")
    }

    // Platform (BOM import): <type>pom</type> <scope>import</scope>
    if (lib.isPlatform) {
        sb.append("$indent4<type>pom</type>$nl")
        sb.append("$indent4<scope>import</scope>$nl")
    }

    // Optional
    if (config.includeOptionalDependencies) {
        sb.append("$indent4<optional>true</optional>$nl")
    }

    sb.append("$indent3</dependency>$nl")
    return sb.toString()
}
----

=== Формирование deprecated-комментария (XML)

[source,kotlin]
----
private fun buildDeprecationXmlComment(lib: EffectiveLibrary, baseIndent: String): String {
    val nl = if (config.prettyPrint) "\n" else ""
    val indent3 = if (config.prettyPrint) baseIndent.repeat(3) else ""

    val deprecation = lib.deprecation
    val parts = mutableListOf<String>()

    parts.add("DEPRECATED")

    if (deprecation?.message != null) {
        parts[0] = "DEPRECATED: ${deprecation.message}"
    }

    if (deprecation?.replacedBy != null) {
        parts.add("Use ${deprecation.replacedBy} instead")
    }

    if (deprecation?.removalVersion != null) {
        parts.add("Removal: ${deprecation.removalVersion}")
    }

    val commentText = parts.joinToString(". ")
    // XML-комментарий не должен содержать "--"
    val safeComment = commentText.replace("--", "- -")
    return "$indent3<!-- $safeComment -->$nl"
}
----

=== Экранирование XML

[source,kotlin]
----
private fun escapeXml(value: String): String {
    return value
        .replace("&", "&amp;")
        .replace("<", "&lt;")
        .replace(">", "&gt;")
        .replace("\"", "&quot;")
        .replace("'", "&apos;")
}
----

*Обязательное экранирование по XML-спецификации:*

- `&` -> `&amp;` (должно быть первым!)
- `<` -> `&lt;`
- `>` -> `&gt;`
- `"` -> `&quot;`
- `'` -> `&apos;`

override fun write(artifact: String, output: Path) {
    val targetFile = output.resolve(config.filename)

    logger.info { "Writing BOM to: $targetFile" }

    // Создать родительские директории если не существуют
    Files.createDirectories(targetFile.parent)

    // Записать файл с UTF-8 кодировкой
    Files.writeString(targetFile, artifact, StandardCharsets.UTF_8)

    logger.info { "BOM written successfully: $targetFile (${artifact.length} chars)" }
}
----

[[edge-cases]]
== Граничные случаи

[cols="1,2,2"]
|===
| Случай | Поведение | Обоснование

| Пустая `EffectiveMetadata` | Генерируется BOM без блока `<dependencyManagement>`. Файл содержит только координаты BOM-артефакта. | Валидный POM. Maven принимает пустой BOM.
| Библиотека без версии (`version = null`) | Исключается из BOM. Логируется предупреждение. | BOM обязан содержать конкретные версии. Зависимость без версии бессмысленна в BOM.
| Все библиотеки без версий | Блок `<dependencyManagement>` отсутствует (аналогично пустой metadata). | Нет зависимостей для включения.
| Спецсимволы XML в значениях | Экранирование: `&` -> `&amp;`, `<` -> `&lt;`, `>` -> `&gt;`, `"` -> `&quot;`, `'` -> `&apos;`. | XML-спецификация.
| Deprecated-библиотека без `DeprecationInfo` | XML-комментарий `<!-- DEPRECATED -->`. | `isDeprecated = true`, но `deprecation = null`.
| Deprecated-библиотека с полным `DeprecationInfo` | `<!-- DEPRECATED: \{message\}. Use \{replacedBy\} instead. Removal: \{removalVersion\} -->`. | Полная информация в комментарии.
| Deprecated-комментарий содержит `--` | Заменяется на `- -` перед вставкой в XML-комментарий. | XML-спецификация запрещает `--` внутри комментариев (`<!-- ... -->`).
| Platform-библиотека (`isPlatform = true`) | Добавляется `<type>pom</type>` и `<scope>import</scope>`. | Maven-семантика: импорт BOM-зависимости.
| Platform-библиотека + optional | Содержит и `<type>pom</type>`, и `<scope>import</scope>`, и `<optional>true</optional>`. | Конфиг-опция применяется ко всем зависимостям.
| Дублирующиеся алиасы | Невозможно: `EffectiveMetadata.libraries` -- `Map<String, EffectiveLibrary>`, ключи уникальны. | Гарантия модели.
| Одинаковые groupId:artifactId с разными версиями | Невозможно: алиасы уникальны в map. Если два алиаса ссылаются на одну координату с разными версиями, обе записи будут в BOM (это валидно, хоть и нежелательно). Выявляется на этапе валидации (pipeline). | Генератор не валидирует семантику.
| Пустые строки в groupId/artifactId/version BomConfig | `IllegalArgumentException` при вызове `generate()`. | Fail-fast валидация.
| Очень большой BOM (тысячи зависимостей) | Работает корректно. `StringBuilder` эффективен для больших строк. | Нет искусственных ограничений.
| Версия с SNAPSHOT | Генерируется как есть: `<version>1.0.0-SNAPSHOT</version>`. | Валидная Maven-версия.
|===

[[error-handling]]
== Обработка ошибок

[cols="1,2,2"]
|===
| Ситуация | Поведение | Тип исключения

| Пустые `groupId`, `artifactId` или `version` в BomConfig | `require()` в `validateConfig()` выбрасывает исключение. | `IllegalArgumentException`
| `generate()` с пустой metadata | Успешно генерирует BOM без `<dependencyManagement>`. Не является ошибкой. | Нет
| `write()` -- невалидный путь | Пробрасывается `IOException` от `Files.writeString()`. | `java.io.IOException`
| `write()` -- нет прав на запись | Пробрасывается `AccessDeniedException`. | `java.nio.file.AccessDeniedException`
| `write()` -- родительская директория не существует | `Files.createDirectories()` создаёт автоматически. Если невозможно -- `IOException`. | `java.io.IOException`
| Некорректный `config.filename` (пустая строка) | `InvalidPathException` при `output.resolve("")`. | `java.nio.file.InvalidPathException`
|===

=== Стратегия логирования

[source,text]
----
INFO  - начало генерации с указанием координат BOM (groupId:artifactId:version)
INFO  - завершение генерации (количество зависимостей)
WARN  - библиотеки без версии, исключённые из BOM (список алиасов)
WARN  - deprecated-библиотека без DeprecationInfo
DEBUG - детали каждой сгенерированной зависимости
----

[[toml-vs-bom]]
== Отличия BOM от TOML-генератора

[cols="1,2,2"]
|===
| Аспект | TOML Generator (SPEC-04) | BOM Generator (SPEC-05)

| Формат вывода | TOML (key-value) | XML (Maven POM)
| Секции | [versions], [libraries], [bundles], [plugins] | `<dependencyManagement>` с `<dependencies>`
| Версии | `version.ref` (DRY через [versions]) или инлайн | Всегда инлайн (XML не поддерживает ref)
| Плагины | Поддерживаются в секции [plugins] | Не поддерживаются (BOM -- только зависимости)
| Bundles | Поддерживаются в секции [bundles] | Не поддерживаются (нет аналога в Maven BOM)
| Platform-библиотеки | Как обычные записи (Gradle обрабатывает `platform()`) | `<type>pom</type>` + `<scope>import</scope>`
| Библиотеки без версии | Записи без поля version (managed dependency) | Исключаются полностью (BOM требует конкретные версии)
| Координаты артефакта | Нет (TOML-файл не имеет координат) | groupId, artifactId, version из BomConfig
| Deprecated-комментарии | `# DEPRECATED: ...` | `<!-- DEPRECATED: ... -->`
| Конфиг по умолчанию | Все поля имеют дефолты | `groupId`, `artifactId`, `version` -- обязательны
|===

[[dependencies]]
== Зависимости модуля

[source,text]
----
dependanger-generator-bom зависит от:
- dependanger-core (api) -- модели: DeprecationInfo, Diagnostics
- dependanger-effective (api) -- EffectiveMetadata, EffectiveLibrary, ResolvedVersion
- kotlin-logging-jvm (implementation) -- логирование

Транзитивно через dependanger-core:
- kotlinx-serialization-core
- kotlinx-serialization-json
----

Внешних зависимостей для XML-генерации не требуется -- XML формируется через `StringBuilder` без использования XML-библиотек (DOM, SAX, StAX).
Это осознанное решение: формат BOM достаточно прост, а отсутствие внешних XML-зависимостей уменьшает footprint модуля.

[[xml-library-consideration]]
=== Альтернатива: использование javax.xml

Если в будущем потребуется более сложная XML-генерация (namespaces, CDATA, сложная вложенность), можно рассмотреть переход на `javax.xml.stream.XMLStreamWriter` (StAX).
Это встроенный API JDK, не требующий дополнительных зависимостей.
