// ============================================================================
// Нефункциональные требования
// Описывают КАК система должна работать, а не ЧТО делать
// ============================================================================

[abstract]
--
Документ описывает нефункциональные требования к системе Dependanger:
производительность, надежность, удобство использования и другие качественные характеристики.
--

== Производительность (Performance)

=== Время отклика

[cols="1,2,2,2",options="header"]
|===
| ID | Операция | Целевое значение | Условия

| NFR-P001
| Генерация TOML каталога
| < 5 секунд
| Каталог из 100 библиотек, SSD, JDK 17

| NFR-P002
| Проверка обновлений
| < 30 секунд
| 100 библиотек, стабильное сетевое соединение

| NFR-P003
| Валидация конфигурации
| < 1 секунда
| Любой размер каталога

| NFR-P004
| Cold start CLI
| < 2 секунды
| Первый запуск без кэша
|===

=== Использование ресурсов

[cols="1,2,2",options="header"]
|===
| ID | Ресурс | Ограничение

| NFR-P010
| Heap memory (генерация)
| < 512 MB

| NFR-P011
| Дисковый кэш
| < 100 MB

| NFR-P012
| Сетевой трафик (update check)
| ≤ 1 HTTP запрос на 10 библиотек (batch API); timeout 10 сек/запрос
|===

=== Масштабируемость

[cols="1,2,2",options="header"]
|===
| ID | Требование | Приоритет

| NFR-P020
| Система ДОЛЖНА обрабатывать каталоги до 1000 библиотек без ошибок
| MUST

| NFR-P021
| Время генерации ДОЛЖНО расти линейно O(n) от количества библиотек
| SHOULD

| NFR-P022
| Параллельная проверка обновлений ДОЛЖНА использовать configurable пул потоков (default=10)
| SHOULD
|===

== Надежность (Reliability)

=== Обработка ошибок

[cols="1,3,2",options="header"]
|===
| ID | Требование | Приоритет

| NFR-R001
| Система ДОЛЖНА корректно обрабатывать недоступность Maven репозиториев
| MUST

| NFR-R002
| При частичном сбое (часть библиотек недоступна) система ДОЛЖНА продолжать работу и сообщать об ошибках
| MUST

| NFR-R003
| Все ошибки валидации ДОЛЖНЫ быть собраны и показаны вместе (не fail-fast)
| MUST

| NFR-R004
| Сообщения об ошибках ДОЛЖНЫ содержать: error code, file:line (если применимо), описание причины, suggestion для исправления
| MUST
|===

=== Восстановление

[cols="1,3,2",options="header"]
|===
| ID | Требование | Приоритет

| NFR-R010
| Прерванная генерация НЕ ДОЛЖНА оставлять частично записанные файлы
| MUST

| NFR-R011
| Система ДОЛЖНА поддерживать --dry-run для предпросмотра изменений
| SHOULD
|===

== Удобство использования (Usability)

=== Документация

[cols="1,3,2",options="header"]
|===
| ID | Требование | Приоритет

| NFR-U001
| Новый пользователь (опыт Gradle, без опыта Dependanger) ДОЛЖЕН создать работающий каталог за ≤ 10 минут по Quick Start Guide
| MUST

| NFR-U002
| Все команды CLI ДОЛЖНЫ иметь встроенную справку (--help)
| MUST

| NFR-U003
| Сообщения об ошибках ДОЛЖНЫ включать подсказки по исправлению
| SHOULD
|===

=== Интерфейс

[cols="1,3,2",options="header"]
|===
| ID | Требование | Приоритет

| NFR-U010
| CLI ДОЛЖЕН поддерживать цветной вывод (с возможностью отключения)
| SHOULD

| NFR-U011
| Длительные операции ДОЛЖНЫ показывать прогресс
| SHOULD

| NFR-U012
| Вывод ДОЛЖЕН быть структурирован для удобного чтения человеком и машиной (--json)
| MUST
|===

== Совместимость (Compatibility)

=== Платформы

[cols="1,3,2",options="header"]
|===
| ID | Требование | Приоритет

| NFR-C001
| Система ДОЛЖНА работать на JDK 17+
| MUST

| NFR-C002
| Gradle plugin ДОЛЖЕН поддерживать Gradle 8.0+
| MUST

| NFR-C003
| CLI ДОЛЖЕН работать на Linux, macOS, Windows
| MUST
|===

=== Форматы

[cols="1,3,2",options="header"]
|===
| ID | Требование | Приоритет

| NFR-C010
| Генерируемый TOML ДОЛЖЕН соответствовать Gradle Version Catalog spec
| MUST

| NFR-C011
| Генерируемый BOM ДОЛЖЕН соответствовать Maven BOM spec
| MUST

| NFR-C012
| JSON output ДОЛЖЕН быть валидным JSON
| MUST
|===

=== Обратная совместимость

[cols="1,3,2",options="header"]
|===
| ID | Требование | Приоритет

| NFR-C020
| Мажорная версия = breaking changes в DSL
| MUST

| NFR-C021
| Минорная версия = новые возможности, обратно совместимые
| MUST

| NFR-C022
| Патч версия = исправления багов
| MUST
|===

== Поддерживаемость (Maintainability)

=== Код

[cols="1,3,2",options="header"]
|===
| ID | Требование | Приоритет

| NFR-M001
| Покрытие тестами критических путей > 80%
| SHOULD

| NFR-M002
| Все публичные API ДОЛЖНЫ иметь KDoc документацию
| MUST

| NFR-M003
| Архитектура ДОЛЖНА позволять добавление новых генераторов без изменения core
| SHOULD
|===

=== Логирование

[cols="1,3,2",options="header"]
|===
| ID | Требование | Приоритет

| NFR-M010
| Система ДОЛЖНА поддерживать уровни логирования (ERROR, WARN, INFO, DEBUG)
| MUST

| NFR-M011
| DEBUG уровень ДОЛЖЕН логировать: timestamp, operation name, input params, output/error, duration (ms)
| SHOULD
|===

== Безопасность (Security)

[cols="1,3,2",options="header"]
|===
| ID | Требование | Приоритет

| NFR-S001
| Система НЕ ДОЛЖНА хранить credentials в открытом виде
| MUST

| NFR-S002
| Система ДОЛЖНА использовать HTTPS для всех внешних запросов
| MUST

| NFR-S003
| Зависимости ДОЛЖНЫ регулярно проверяться на уязвимости
| SHOULD
|===
