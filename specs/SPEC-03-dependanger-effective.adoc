== Обзор

Модуль `dependanger-effective` отвечает за трансформацию `DependangerMetadata` в `EffectiveMetadata` через упорядоченный конвейер процессоров.

=== Текущее состояние

[cols="1,2"]
|===
| Компонент | Статус

| Инфраструктура пайплайна | *Реализовано:* ProcessingPipeline, ProcessingPhase, ProcessingContext, PipelineBuilder, ProcessingPresetConfigurator, ProcessingCallback, ProcessingEvent
| Модели effective | *Реализовано:* EffectiveMetadata, EffectiveLibrary, EffectivePlugin, EffectiveBundle, ResolvedVersion, ProcessingInfo, ExtensionKey, CompatibilityIssue
| EffectiveMetadataBuilder.build() | *TODO-заглушка*
| 12 Core Processors | *Все TODO-заглушки*
|===

=== SPI: ExtensionSerializerProvider

Модуль определяет SPI-интерфейс `ExtensionSerializerProvider` в пакете `io.github.zenhelix.dependanger.effective.spi` для ServiceLoader-based регистрации serializer'ов extension-моделей.
Подробности см. в xref:index.adoc#extension-serializer-spi[cross-cutting секции index.adoc] и xref:SPEC-02-dependanger-metadata-json.adoc#spec-effective-json[SPEC-02.5].

=== Архитектура пайплайна

Определения ProcessingPreset (DEFAULT, STRICT, MINIMAL) см. в xref:index.adoc#processing-preset-definitions[cross-cutting секции index.adoc].

[source,text]
----
DependangerMetadata
  |
  v
ProcessingContext (metadata + settings + environment + distribution)
  |
  v
ProcessingPipeline (List<EffectiveMetadataProcessor>, sorted by order)
  |
  +-- [order=5]  ProfileProcessor
  +-- [order=10] MetadataConversionProcessor
  +-- [order=12] ExtractedVersionsProcessor
  +-- [order=20] LibraryFilterProcessor
  +-- [order=25] VersionFallbackProcessor
  +-- [order=30] VersionResolverProcessor
  +-- [order=40] BundleFilterProcessor
  +-- [order=45] PluginFilterProcessor
  +-- [order=50] PluginProcessor
  +-- [order=60] UsedVersionsProcessor
  +-- [order=65] ValidationProcessor
  +-- [order=66] CompatRulesProcessor
  |
  v
EffectiveMetadata (versions, libraries, plugins, bundles, diagnostics)
----

=== Режимы выполнения

Все 12 core-процессоров работают в режиме `ExecutionMode.SEQUENTIAL`.
Параллельные фазы (`PARALLEL_IO`) используются опциональными feature-процессорами (UPDATE_CHECK, SECURITY_CHECK, LICENSE_CHECK, TRANSITIVE_RESOLVER).
BOM_IMPORT работает в `SEQUENTIAL` режиме, т.к. модифицирует `libraries` (внутренняя параллельность через coroutines).

При параллельном выполнении процессоры могут модифицировать только `diagnostics` и `extensions`.
Модификация `versions`, `libraries`, `plugins`, `bundles` в параллельном режиме вызывает `IllegalStateException`.

=== Обработка ошибок

[cols="1,2,2"]
|===
| Ситуация | Поведение | Обоснование

| Процессор бросает исключение | PhaseError event, исключение пробрасывается выше | Fail-fast
| `supports()` возвращает false | Процессор пропускается, metadata передается дальше без изменений | Условное выполнение
| Ошибка валидации (не критическая) | Добавляется DiagnosticMessage в diagnostics | Мягкая деградация
| Процессор отключен через PipelineBuilder | Не включается в список active processors | Конфигурируемость
|===

[[spec-builder]]
== SPEC-03.0: EffectiveMetadataBuilder.build()

=== Назначение

Точка входа для трансформации `DependangerMetadata` -> `EffectiveMetadata`.
Создает пайплайн на основе preset-конфигурации и запускает обработку.

=== Текущий код (TODO)

[source,kotlin]
----
// EffectiveMetadataBuilder.kt
public suspend fun build(metadata: DependangerMetadata): EffectiveMetadata = TODO()
----

=== Алгоритм

[source,text]
----
1. Создать PipelineBuilder
2. Зарегистрировать все 12 core-процессоров
3. Применить preset-конфигурацию через ProcessingPreset.configure(builder)
4. Собрать pipeline через builder.build()
5. Создать ProcessingContext из параметров:
   - originalMetadata = metadata
   - settings = metadata.settings
   - environment = this.environment
   - activeDistribution = this.distribution ?: metadata.settings.defaultDistribution
6. Вызвать pipeline.process(context)
7. Вернуть результат
----

=== Детальная реализация

[source,kotlin]
----
public suspend fun build(metadata: DependangerMetadata): EffectiveMetadata {
    val builder = PipelineBuilder()

    // Регистрация всех core-процессоров
    builder.addProcessors(
        listOf(
            ProfileProcessor(),
            MetadataConversionProcessor(),
            ExtractedVersionsProcessor(),
            LibraryFilterProcessor(),
            VersionFallbackProcessor(),
            VersionResolverProcessor(),
            BundleFilterProcessor(),
            PluginFilterProcessor(),
            PluginProcessor(),
            UsedVersionsProcessor(),
            ValidationProcessor(),
            CompatRulesProcessor(),
        )
    )

    // Применяем конфигурацию preset
    preset.configure(builder)

    // Собираем pipeline
    val pipeline = builder.build()

    // Определяем активную дистрибуцию
    val activeDistribution = distribution ?: metadata.settings.defaultDistribution

    // Создаем контекст
    val context = ProcessingContext(
        originalMetadata = metadata,
        settings = metadata.settings,
        environment = environment,
        activeDistribution = activeDistribution,
    )

    // Запускаем обработку
    return pipeline.process(context)
}
----

=== Edge cases

- `metadata` с пустыми списками -- пайплайн отработает, вернет пустой `EffectiveMetadata`
- `preset = MINIMAL` -- оставляет 5 процессоров: `profile`(5), `metadata-conversion`(10), `extracted-versions`(12), `version-fallback`(25), `version-resolver`(30).
Отключены: `library-filter`(20), `bundle-filter`(40), `plugin-filter`(45), `plugin`(50), `used-versions`(60), `validation`(65), `compat-rules`(66)
- `distribution` не задан и `defaultDistribution` тоже null -- `activeDistribution = null`, ProfileProcessor просто пропускает установку
- Дублирование процессоров -- `PipelineBuilder.build()` выбросит `PipelineConfigurationException`

`components/core/dependanger-effective/src/main/kotlin/io/github/zenhelix/dependanger/effective/builder/EffectiveMetadataBuilder.kt`

[[spec-profile]]
== SPEC-03.1: ProfileProcessor

=== Идентификация

[cols="1,2"]
|===
| Поле | Значение
| id | `profile`
| phase | `ProcessingPhase.PROFILE`
| order | `5`
| isOptional | `false`
| executionMode | `SEQUENTIAL`
|===

=== Назначение

Выбрать и валидировать активную дистрибуцию из метаданных.
Устанавливает поле `distribution` в `EffectiveMetadata`.

=== Зависимости от других процессоров

Нет -- первый процессор в цепочке.

=== Алгоритм

[source,text]
----
1. Получить activeDistribution из context.activeDistribution
2. ЕСЛИ activeDistribution == null:
   a. Вернуть metadata без изменений (distribution остается null или тем что пришло из pipeline init)
3. ЕСЛИ activeDistribution != null:
   a. Найти Distribution по имени в context.originalMetadata.distributions
   b. ЕСЛИ найдена:
      - Вернуть metadata.copy(distribution = activeDistribution) + info diagnostic PROFILE_APPLIED
   c. ЕСЛИ не найдена:
      - Добавить error diagnostic PROFILE_NOT_FOUND
      - Вернуть metadata.copy(distribution = activeDistribution) с ошибкой в diagnostics
----

=== Детальная реализация

[source,kotlin]
----
override suspend fun process(
    metadata: EffectiveMetadata,
    context: ProcessingContext,
): EffectiveMetadata {
    val distName = context.activeDistribution
        ?: return metadata

    val distributions = context.originalMetadata.distributions
    val found = distributions.find { it.name == distName }

    return if (found != null) {
        metadata.copy(
            distribution = distName,
            diagnostics = metadata.diagnostics + Diagnostics.info(
                code = "PROFILE_APPLIED",
                message = "Distribution '$distName' applied",
                processorId = id,
            ),
        )
    } else {
        val available = distributions.map { it.name }
        metadata.copy(
            distribution = distName,
            diagnostics = metadata.diagnostics + Diagnostics.error(
                code = "PROFILE_NOT_FOUND",
                message = "Distribution '$distName' not found. Available: $available",
                processorId = id,
            ),
        )
    }
}
----

=== Input/Output пример

[source,kotlin]
----
// Input
val metadata = DependangerMetadata(
    distributions = listOf(
        Distribution("backend", FilterSpec(byTags = TagFilter(includes = listOf(TagInclude(anyOf = setOf("backend")))))),
        Distribution("frontend", FilterSpec(byTags = TagFilter(includes = listOf(TagInclude(anyOf = setOf("frontend")))))),
    )
)
val context = ProcessingContext(..., activeDistribution = "backend")

// Output
EffectiveMetadata(
    distribution = "backend",
    diagnostics = Diagnostics(infos = listOf(
        DiagnosticMessage("PROFILE_APPLIED", "Distribution 'backend' applied", INFO, "profile")
    ))
)
----

=== Edge cases

- `activeDistribution = null` -- процессор no-op, возвращает metadata как есть
- Дистрибуция не найдена -- `PROFILE_NOT_FOUND` error, но обработка продолжается (другие процессоры получат metadata с ошибкой)
- Пустой список `distributions` при заданном `activeDistribution` -- error diagnostic
- `distribution` уже установлен в metadata (из pipeline init) -- ProfileProcessor ПЕРЕЗАПИСЫВАЕТ любое существующее значение `distribution` в EffectiveMetadata, если `activeDistribution != null`

=== Тестовые сценарии

[cols="1,3"]
|===
| Тест | Описание

| `no active distribution` | context.activeDistribution = null -> metadata без изменений
| `valid distribution` | Найдена -> distribution установлена + info diagnostic
| `invalid distribution` | Не найдена -> error diagnostic PROFILE_NOT_FOUND
| `empty distributions list` | Нет дистрибуций -> error если activeDistribution задан
| `distribution name case-sensitive` | "Backend" != "backend"
|===

[[spec-metadata-conversion]]
== SPEC-03.2: MetadataConversionProcessor

=== Идентификация

[cols="1,2"]
|===
| Поле | Значение
| id | `metadata-conversion`
| phase | `ProcessingPhase.METADATA_CONVERSION`
| order | `10`
| isOptional | `false`
| executionMode | `SEQUENTIAL`
|===

=== Назначение

Основной процессор -- конвертирует `DependangerMetadata` в начальное заполнение `EffectiveMetadata`.
Трансформирует списковые структуры в map-структуры с ключом по alias.

=== Зависимости от других процессоров

- *Зависит от:* ProfileProcessor (order=5) -- distribution уже установлена
- *Блокирует:* Все последующие процессоры зависят от заполненных maps

=== Алгоритм

[source,text]
----
1. КОНВЕРТАЦИЯ ВЕРСИЙ:
   a. Для каждой Version в originalMetadata.versions:
      - Создать ResolvedVersion(alias=name, value=value, source=DECLARED)
      - Ключ map = version.name
   b. Результат: Map<String, ResolvedVersion>

2. КОНВЕРТАЦИЯ БИБЛИОТЕК:
   a. Для каждой Library в originalMetadata.libraries:
      - Создать EffectiveLibrary:
        - alias = library.alias
        - group = library.group
        - artifact = library.artifact
        - version = конвертировать VersionReference в ResolvedVersion (или null)
        - description = library.description
        - tags = library.tags
        - requires = library.requires
        - isDeprecated = (library.deprecation != null)
        - deprecation = library.deprecation
        - license = library.license
        - constraints = library.constraints
        - isPlatform = library.isPlatform
      - Ключ map = library.alias
   b. Конвертация VersionReference -> ResolvedVersion:
      - Literal(v) -> ResolvedVersion(alias="", value=v, source=DECLARED)
      - Reference(name) -> null (будет разрешен VersionResolverProcessor)
      - Range(...) -> null (оставляем для специальной обработки)
      - null -> null

3. КОНВЕРТАЦИЯ ПЛАГИНОВ:
   a. Для каждого Plugin в originalMetadata.plugins:
      - Создать EffectivePlugin(alias=alias, id=id, version=...)
      - Конвертация версии аналогично библиотекам
      - Ключ map = plugin.alias

4. КОНВЕРТАЦИЯ БАНДЛОВ:
   a. Построить полный граф extends для бандлов
   b. Для каждого Bundle в originalMetadata.bundles:
      - Разрешить extends: рекурсивно собрать библиотеки из родительских бандлов
      - Создать EffectiveBundle(alias=alias, libraries=merged_libraries)
      - Ключ map = bundle.alias
   c. Алгоритм разрешения extends:
      - Для бандла B с extends=[A]:
        1. Найти бандл A в списке
        2. Рекурсивно разрешить extends бандла A
        3. Объединить: A.libraries + B.libraries (без дубликатов, порядок: сначала extends, потом собственные)

5. Собрать результат:
   metadata.copy(
       versions = versionsMap,
       libraries = librariesMap,
       plugins = pluginsMap,
       bundles = bundlesMap,
   )
----

=== Детальная реализация

[source,kotlin]
----
override suspend fun process(
    metadata: EffectiveMetadata,
    context: ProcessingContext,
): EffectiveMetadata {
    val original = context.originalMetadata

    // 1. Конвертация версий
    val versions = original.versions.associate { version ->
        version.name to ResolvedVersion(
            alias = version.name,
            value = version.value,
            source = VersionSource.DECLARED,
        )
    }

    // 2. Конвертация библиотек
    val libraries = original.libraries.associate { lib ->
        lib.alias to EffectiveLibrary(
            alias = lib.alias,
            group = lib.group,
            artifact = lib.artifact,
            version = convertVersionReference(lib.version),
            description = lib.description,
            tags = lib.tags,
            requires = lib.requires,
            isDeprecated = lib.deprecation != null,
            deprecation = lib.deprecation,
            license = lib.license,
            constraints = lib.constraints,
            isPlatform = lib.isPlatform,
        )
    }

    // 3. Конвертация плагинов
    val plugins = original.plugins.associate { plugin ->
        plugin.alias to EffectivePlugin(
            alias = plugin.alias,
            id = plugin.id,
            version = convertVersionReference(plugin.version),
        )
    }

    // 4. Конвертация бандлов с разрешением extends
    val bundleIndex = original.bundles.associateBy { it.alias }
    val bundles = original.bundles.associate { bundle ->
        val resolvedLibraries = resolveBundleExtends(bundle, bundleIndex, mutableSetOf())
        bundle.alias to EffectiveBundle(
            alias = bundle.alias,
            libraries = resolvedLibraries,
        )
    }

    return metadata.copy(
        versions = versions,
        libraries = libraries,
        plugins = plugins,
        bundles = bundles,
    )
}

private fun convertVersionReference(ref: VersionReference?): ResolvedVersion? = when (ref) {
    is VersionReference.Literal -> ResolvedVersion(
        alias = "",
        value = ref.version,
        source = VersionSource.DECLARED,
    )
    is VersionReference.Reference -> null  // разрешится в VersionResolverProcessor
    is VersionReference.Range -> null      // range не поддерживает resolved value
    null -> null
}

private fun resolveBundleExtends(
    bundle: Bundle,
    index: Map<String, Bundle>,
    visited: MutableSet<String>,
): List<String> {
    if (bundle.alias in visited) return emptyList() // защита от циклов
    visited.add(bundle.alias)

    val parentLibraries = bundle.extends.flatMap { parentAlias ->
        val parent = index[parentAlias]
        if (parent != null) resolveBundleExtends(parent, index, visited)
        else emptyList()
    }

    return (parentLibraries + bundle.libraries).distinct()
}
----

=== Edge cases

- Дублирующиеся alias в библиотеках -- последний в списке побеждает (поведение `associate`)
- Бандл extends несуществующий бандл -- библиотеки из отсутствующего бандла просто не добавляются
- Циклический extends (A extends B, B extends A) -- защита через `visited` set, цикл разрывается
- Пустой `originalMetadata` -- все maps будут пустыми, это валидный результат
- Библиотека без версии (`version = null`) -- `EffectiveLibrary.version = null`
- `VersionReference.Range` -- `version = null` в EffectiveLibrary (range обрабатывается отдельно)

=== Тестовые сценарии

[cols="1,3"]
|===
| Тест | Описание

| `empty metadata conversion` | Пустые списки -> пустые maps
| `versions converted to map` | List<Version> -> Map<String, ResolvedVersion>
| `literal version resolved immediately` | Literal("1.0") -> ResolvedVersion(value="1.0")
| `reference version left null` | Reference("kotlin") -> null
| `range version left null` | Range(...) -> null
| `libraries with all fields` | Все поля Library конвертированы
| `deprecation sets isDeprecated` | deprecation != null -> isDeprecated = true
| `plugins converted` | List<Plugin> -> Map<String, EffectivePlugin>
| `bundle extends resolved` | extends собирает библиотеки из родителей
| `bundle extends dedup` | Дубликаты в libraries после extends удалены
| `circular bundle extends` | Цикл не вызывает бесконечную рекурсию
| `duplicate alias last wins` | Два Library с alias="x" -> последний в map
| `bundle extends missing parent` | extends ссылается на несуществующий -> пропускается
|===

[[spec-extracted-versions]]
== SPEC-03.3: ExtractedVersionsProcessor

=== Идентификация

[cols="1,2"]
|===
| Поле | Значение
| id | `extracted-versions`
| phase | `ProcessingPhase.EXTRACTED_VERSIONS`
| order | `12`
| isOptional | `false`
| executionMode | `SEQUENTIAL`
|===

=== Назначение

Извлечь inline-литеральные версии из библиотек и плагинов и вынести их в карту `versions`.
Библиотеки и плагины обновляются для ссылки на новую именованную версию.
Это обеспечивает DRY -- в TOML version catalog предпочтительно использовать `version.ref`.

=== Зависимости от других процессоров

- *Зависит от:* MetadataConversionProcessor (order=10) -- заполнены maps
- *Блокирует:* VersionFallbackProcessor (order=25), VersionResolverProcessor (order=30)

=== Алгоритм

[source,text]
----
1. Инициализировать newVersions = mutableMapOf из metadata.versions

2. ДЛЯ КАЖДОЙ библиотеки в metadata.libraries:
   a. ЕСЛИ library.version != null И library.version.alias == "" И library.version.value != "":
      (это литерал, созданный MetadataConversionProcessor из VersionReference.Literal)
      b. Генерировать имя версии: versionName = "${library.alias}-version"
      c. ЕСЛИ versionName уже есть в newVersions:
         - ЕСЛИ значение совпадает -> пропустить (уже извлечено)
         - ЕСЛИ значение отличается -> добавить warning EXTRACTED_VERSION_CONFLICT,
           использовать суффикс: versionName = "${library.alias}-version-2"
      d. Создать ResolvedVersion(alias=versionName, value=library.version.value, source=DECLARED)
      e. Добавить в newVersions
      f. Обновить library: version = ResolvedVersion(alias=versionName, value=library.version.value, source=DECLARED)
      g. Добавить info diagnostic EXTRACTED_VERSION_CREATED

3. ДЛЯ КАЖДОГО плагина в metadata.plugins:
   a. Аналогичная логика: если version литеральный -> извлечь в versions map
   b. versionName = "${plugin.alias}-version"

4. Вернуть metadata.copy(versions=newVersions, libraries=updatedLibraries, plugins=updatedPlugins, diagnostics=...)
----

=== Детальная реализация

[source,kotlin]
----
override suspend fun process(
    metadata: EffectiveMetadata,
    context: ProcessingContext,
): EffectiveMetadata {
    val newVersions = metadata.versions.toMutableMap()
    var diagnostics = metadata.diagnostics

    // Извлечение из библиотек
    val updatedLibraries = metadata.libraries.mapValues { (alias, lib) ->
        val version = lib.version
        if (version != null && version.alias.isEmpty() && version.value.isNotEmpty()) {
            val versionName = generateVersionName(alias, newVersions)
            val resolved = ResolvedVersion(
                alias = versionName,
                value = version.value,
                source = VersionSource.DECLARED,
            )
            newVersions[versionName] = resolved
            diagnostics = diagnostics + Diagnostics.info(
                code = "EXTRACTED_VERSION_CREATED",
                message = "Extracted version '$versionName' = '${version.value}' from library '$alias'",
                processorId = id,
            )
            lib.copy(version = resolved)
        } else {
            lib
        }
    }

    // Извлечение из плагинов
    val updatedPlugins = metadata.plugins.mapValues { (alias, plugin) ->
        val version = plugin.version
        if (version != null && version.alias.isEmpty() && version.value.isNotEmpty()) {
            val versionName = generateVersionName(alias, newVersions)
            val resolved = ResolvedVersion(
                alias = versionName,
                value = version.value,
                source = VersionSource.DECLARED,
            )
            newVersions[versionName] = resolved
            diagnostics = diagnostics + Diagnostics.info(
                code = "EXTRACTED_VERSION_CREATED",
                message = "Extracted version '$versionName' = '${version.value}' from plugin '$alias'",
                processorId = id,
            )
            plugin.copy(version = resolved)
        } else {
            plugin
        }
    }

    return metadata.copy(
        versions = newVersions,
        libraries = updatedLibraries,
        plugins = updatedPlugins,
        diagnostics = diagnostics,
    )
}

private fun generateVersionName(
    alias: String,
    existing: Map<String, ResolvedVersion>,
): String {
    val base = "$alias-version"
    if (base !in existing) return base
    var counter = 2
    while ("$base-$counter" in existing) counter++
    return "$base-$counter"
}

[[spec-library-filter]]
== SPEC-03.4: LibraryFilterProcessor

=== Идентификация

[cols="1,2"]
|===
| Поле | Значение
| id | `library-filter`
| phase | `ProcessingPhase.LIBRARY_FILTER`
| order | `20`
| isOptional | `false`
| executionMode | `SEQUENTIAL`
|===

=== Назначение

Фильтрация библиотек на основе `FilterSpec` активной дистрибуции.
Если дистрибуция не активна или не имеет FilterSpec -- все библиотеки проходят без фильтрации.

=== Зависимости от других процессоров

- *Зависит от:* ProfileProcessor (order=5) -- определена активная дистрибуция; MetadataConversionProcessor (order=10) -- заполнены libraries
- *Блокирует:* BundleFilterProcessor (order=40) -- фильтрация бандлов зависит от оставшихся библиотек

=== Алгоритм

[source,text]
----
1. Получить activeDistribution из metadata.distribution
2. ЕСЛИ activeDistribution == null -> вернуть metadata без изменений
3. Найти Distribution по имени в context.originalMetadata.distributions
4. ЕСЛИ distribution.spec == null -> вернуть metadata без изменений
5. Применить фильтры из FilterSpec последовательно:

   a. byTags (TagFilter):
      - ДЛЯ КАЖДОГО TagInclude в includes:
        - anyOf: библиотека включена если пересечение tags и anyOf непустое
        - allOf: библиотека включена если tags содержит все элементы allOf
        - Если includes пустой -> включить все
      - ДЛЯ КАЖДОГО TagExclude в excludes:
        - anyOf: библиотека исключена если пересечение tags и anyOf непустое
      - Логика: (match ANY include) AND (NOT match ANY exclude)

   b. byGroups (GroupFilter):
      - includes: библиотека включена если group+":"+artifact match любой из includes (glob)
      - excludes: библиотека исключена если group+":"+artifact match любой из excludes (glob)
      - Glob matching: "*" соответствует любой подстроке
        - "org.springframework:*" matches "org.springframework:spring-core"
        - "org.springframework*:*" matches "org.springframework.boot:spring-boot"
      - Если includes пуст -> включить все (если excludes не исключает)
      - Если excludes пуст -> ничего не исключать

   c. byAliases (AliasFilter):
      - includes: библиотека включена если alias в includes
      - excludes: библиотека исключена если alias в excludes
      - Exact match

   d. byBundles (BundleFilter):
      - includes: библиотека включена если она входит в любой из указанных бандлов
      - excludes: библиотека исключена если она входит в любой из указанных бандлов
      - Проверка: library.alias in bundle.libraries для всех бандлов в metadata.bundles

   e. byDeprecated (DeprecatedFilter):
      - include=false -> удалить библиотеки с isDeprecated=true
      - include=true (default) -> оставить все

6. Общая логика комбинирования фильтров:
   - Фильтры применяются как AND: библиотека проходит ТОЛЬКО если проходит ВСЕ фильтры
   - Внутри каждого фильтра: includes (OR) -> excludes (NOT)

7. Для каждой отфильтрованной библиотеки -> info diagnostic LIBRARY_FILTERED
8. Вернуть metadata.copy(libraries = filteredLibraries, diagnostics = ...)
----

=== Детальная реализация (псевдокод)

[source,kotlin]
----
override suspend fun process(
    metadata: EffectiveMetadata,
    context: ProcessingContext,
): EffectiveMetadata {
    val distName = metadata.distribution ?: return metadata
    val distribution = context.originalMetadata.distributions.find { it.name == distName }
    val spec = distribution?.spec ?: return metadata

    var diagnostics = metadata.diagnostics
    val bundleIndex = metadata.bundles

    val filtered = metadata.libraries.filter { (alias, lib) ->
        val passes = passesTagFilter(lib, spec.byTags)
                && passesGroupFilter(lib, spec.byGroups)
                && passesAliasFilter(alias, spec.byAliases)
                && passesBundleFilter(alias, bundleIndex, spec.byBundles)
                && passesDeprecatedFilter(lib, spec.byDeprecated)

        if (!passes) {
            diagnostics = diagnostics + Diagnostics.info(
                code = "LIBRARY_FILTERED",
                message = "Library '$alias' filtered out by distribution '$distName'",
                processorId = id,
            )
        }
        passes
    }

    return metadata.copy(libraries = filtered, diagnostics = diagnostics)
}

private fun passesTagFilter(lib: EffectiveLibrary, filter: TagFilter?): Boolean {
    if (filter == null) return true
    val tags = lib.tags

    // Includes: библиотека должна пройти хотя бы один include
    val passesIncludes = if (filter.includes.isEmpty()) true
    else filter.includes.any { include ->
        val anyOfOk = include.anyOf.isEmpty() || (tags intersect include.anyOf).isNotEmpty()
        val allOfOk = include.allOf.isEmpty() || tags.containsAll(include.allOf)
        anyOfOk && allOfOk
    }

    // Excludes: библиотека не должна попасть ни в один exclude
    val passesExcludes = filter.excludes.all { exclude ->
        exclude.anyOf.isEmpty() || (tags intersect exclude.anyOf).isEmpty()
    }

    return passesIncludes && passesExcludes
}

private fun passesGroupFilter(lib: EffectiveLibrary, filter: GroupFilter?): Boolean {
    if (filter == null) return true
    val coordinate = "${lib.group}:${lib.artifact}"

    val passesIncludes = filter.includes.isEmpty()
            || filter.includes.any { pattern -> globMatch(coordinate, pattern) }

    val passesExcludes = filter.excludes.isEmpty()
            || filter.excludes.none { pattern -> globMatch(coordinate, pattern) }

    return passesIncludes && passesExcludes
}

private fun passesAliasFilter(alias: String, filter: AliasFilter?): Boolean {
    if (filter == null) return true
    val passesIncludes = filter.includes.isEmpty() || alias in filter.includes
    val passesExcludes = filter.excludes.isEmpty() || alias !in filter.excludes
    return passesIncludes && passesExcludes
}

private fun passesBundleFilter(
    alias: String,
    bundles: Map<String, EffectiveBundle>,
    filter: BundleFilter?,
): Boolean {
    if (filter == null) return true
    val memberOf = bundles.filter { (_, b) -> alias in b.libraries }.keys

    val passesIncludes = filter.includes.isEmpty()
            || (memberOf intersect filter.includes).isNotEmpty()

    val passesExcludes = filter.excludes.isEmpty()
            || (memberOf intersect filter.excludes).isEmpty()

    return passesIncludes && passesExcludes
}

private fun passesDeprecatedFilter(lib: EffectiveLibrary, filter: DeprecatedFilter?): Boolean {
    if (filter == null) return true
    return filter.include || !lib.isDeprecated
}

=== Edge cases

- Нет активной дистрибуции -- все библиотеки проходят
- FilterSpec = null -- все проходят

- Пустой includes с непустым excludes -- excludes работает по всем библиотекам

[[spec-version-fallback]]
== SPEC-03.5: VersionFallbackProcessor

=== Идентификация

[cols="1,2"]
|===
| Поле | Значение
| id | `version-fallback`
| phase | `ProcessingPhase.VERSION_FALLBACK`
| order | `25`
| isOptional | `false`
| executionMode | `SEQUENTIAL`
|===

=== Назначение

Применить fallback-версии на основе условий окружения (`ProcessingEnvironment`).
Для каждой версии с fallbacks оценивает `FallbackCondition` -- первое совпадение заменяет значение версии.

=== Зависимости от других процессоров

- *Зависит от:* MetadataConversionProcessor (order=10) -- versions заполнены
- *Блокирует:* VersionResolverProcessor (order=30) -- fallback должен быть применен до разрешения ссылок

=== Алгоритм

[source,text]
----
1. Для каждой Version в context.originalMetadata.versions (содержащей fallbacks):
   a. Для каждого VersionFallback (в порядке объявления):
      - Оценить FallbackCondition против context.environment
      - ЕСЛИ condition == true:
        1. Заменить значение в metadata.versions[version.name]
        2. Добавить info diagnostic FALLBACK_APPLIED
        3. BREAK (первый match побеждает)
   b. Если ни один fallback не совпал -> оставить оригинальное значение

2. ОЦЕНКА FallbackCondition:
   - JdkBelow(v) -> environment.jdkVersion != null && environment.jdkVersion < v
   - JdkAtLeast(v) -> environment.jdkVersion != null && environment.jdkVersion >= v
   - JdkBetween(min, max) -> environment.jdkVersion != null && jdkVersion in min..max
   - KotlinVersionBelow(v) -> compareVersions(environment.kotlinVersion, v) < 0
   - KotlinVersionAtLeast(v) -> compareVersions(environment.kotlinVersion, v) >= 0
   - GradleVersionBelow(v) -> compareVersions(environment.gradleVersion, v) < 0
   - GradleVersionAtLeast(v) -> compareVersions(environment.gradleVersion, v) >= 0
   - DistributionCondition(name) -> context.activeDistribution == name
   - EnvironmentCondition(key, value) -> environment.environmentVariables[key] == value
   - All(conditions) -> conditions.all { evaluate(it) }
   - Any(conditions) -> conditions.any { evaluate(it) }
   - Not(condition) -> !evaluate(condition)

3. ЕСЛИ environment поле null (например jdkVersion) при оценке JdkBelow:
   - Condition возвращает false (условие не может быть оценено)
   - Добавить warning FALLBACK_ENV_MISSING
----

=== Детальная реализация

[source,kotlin]
----
override suspend fun process(
    metadata: EffectiveMetadata,
    context: ProcessingContext,
): EffectiveMetadata {
    val originalVersions = context.originalMetadata.versions
    val versionsWithFallbacks = originalVersions.filter { it.fallbacks.isNotEmpty() }

    if (versionsWithFallbacks.isEmpty()) return metadata

    val updatedVersions = metadata.versions.toMutableMap()
    var diagnostics = metadata.diagnostics

    for (version in versionsWithFallbacks) {
        val current = updatedVersions[version.name] ?: continue

        for (fallback in version.fallbacks) {
            if (evaluateCondition(fallback.condition, context)) {
                updatedVersions[version.name] = current.copy(value = fallback.value)
                diagnostics = diagnostics + Diagnostics.info(
                    code = "FALLBACK_APPLIED",
                    message = "Version '${version.name}': fallback '${fallback.value}' applied " +
                            "(was '${current.value}', condition: ${fallback.condition})",
                    processorId = id,
                )
                break  // first match wins
            }
        }
    }

    return metadata.copy(versions = updatedVersions, diagnostics = diagnostics)
}

private fun evaluateCondition(
    condition: FallbackCondition,
    context: ProcessingContext,
): Boolean {
    val env = context.environment
    return when (condition) {
        is FallbackCondition.JdkBelow ->
            env.jdkVersion != null && env.jdkVersion < condition.version

        is FallbackCondition.JdkAtLeast ->
            env.jdkVersion != null && env.jdkVersion >= condition.version

        is FallbackCondition.JdkBetween ->
            env.jdkVersion != null && env.jdkVersion in condition.min..condition.max

        is FallbackCondition.KotlinVersionBelow ->
            env.kotlinVersion != null && compareVersions(env.kotlinVersion, condition.version) < 0

        is FallbackCondition.KotlinVersionAtLeast ->
            env.kotlinVersion != null && compareVersions(env.kotlinVersion, condition.version) >= 0

        is FallbackCondition.GradleVersionBelow ->
            env.gradleVersion != null && compareVersions(env.gradleVersion, condition.version) < 0

        is FallbackCondition.GradleVersionAtLeast ->
            env.gradleVersion != null && compareVersions(env.gradleVersion, condition.version) >= 0

        is FallbackCondition.DistributionCondition ->
            context.activeDistribution == condition.name

        is FallbackCondition.EnvironmentCondition ->
            env.environmentVariables[condition.key] == condition.value

        is FallbackCondition.All ->
            condition.conditions.all { evaluateCondition(it, context) }

        is FallbackCondition.Any ->
            condition.conditions.any { evaluateCondition(it, context) }

        is FallbackCondition.Not ->
            !evaluateCondition(condition.condition, context)
    }
}

// import io.github.zenhelix.dependanger.core.util.VersionComparator
// VersionComparator.compare(a, b)

//   VersionFallback("1.9.25", JdkBelow(17)),

=== Edge cases

- Нет версий с fallbacks -- no-op
- `jdkVersion = null` при `JdkBelow` -- condition = false, fallback не применяется
- `kotlinVersion = null` -- аналогично

[cols="1,3"]
|===
| Тест | Описание

| `no fallbacks - no change` | Версии без fallbacks не изменяются

| `null jdk version` | jdkVersion=null -> JdkBelow -> false
| `first match wins` | Два fallback совпадают -> первый применяется

[[spec-version-resolver]]
== SPEC-03.6: VersionResolverProcessor

=== Идентификация

[cols="1,2"]
|===
| Поле | Значение
| id | `version-resolver`
| phase | `ProcessingPhase.VERSION_RESOLVER`
| order | `30`
| isOptional | `false`
| executionMode | `SEQUENTIAL`
|===

=== Назначение

Разрешить все `VersionReference.Reference` в библиотеках и плагинах -- подставить фактические значения версий из `metadata.versions`.

=== Зависимости от других процессоров

- *Зависит от:* MetadataConversionProcessor (order=10), ExtractedVersionsProcessor (order=12), VersionFallbackProcessor (order=25)
- *Блокирует:* Все последующие процессоры ожидают resolved versions

=== Алгоритм

[source,text]
----
1. ДЛЯ КАЖДОЙ библиотеки с version == null (VersionReference.Reference в оригинале):
   a. Найти оригинальный VersionReference из context.originalMetadata
   b. ЕСЛИ это Reference(name):
      - Найти name в metadata.versions
      - ЕСЛИ найден: установить library.version = ResolvedVersion(alias=name, value=found.value, source=found.source, originalRef=name)
      - ЕСЛИ не найден: добавить error diagnostic UNRESOLVED_VERSION
   c. ЕСЛИ это Range: оставить version = null (range не разрешается в конкретное значение)

2. ДЛЯ КАЖДОГО плагина с version == null (аналогично):
   a. Найти оригинальный VersionReference из context.originalMetadata
   b. Разрешить Reference -> ResolvedVersion

3. Вернуть metadata с обновленными libraries и plugins
----

=== Детальная реализация

[source,kotlin]
----
override suspend fun process(
    metadata: EffectiveMetadata,
    context: ProcessingContext,
): EffectiveMetadata {
    val originalLibraries = context.originalMetadata.libraries.associateBy { it.alias }
    val originalPlugins = context.originalMetadata.plugins.associateBy { it.alias }
    var diagnostics = metadata.diagnostics

    // Разрешение версий библиотек
    val resolvedLibraries = metadata.libraries.mapValues { (alias, lib) ->
        if (lib.version != null) return@mapValues lib

        val originalRef = originalLibraries[alias]?.version
        when (originalRef) {
            is VersionReference.Reference -> {
                val resolved = metadata.versions[originalRef.name]
                if (resolved != null) {
                    diagnostics = diagnostics + Diagnostics.info(
                        code = "VERSION_RESOLVED",
                        message = "Library '$alias': version ref '${originalRef.name}' -> '${resolved.value}'",
                        processorId = id,
                    )
                    lib.copy(
                        version = ResolvedVersion(
                            alias = originalRef.name,
                            value = resolved.value,
                            source = resolved.source,
                            originalRef = originalRef.name,
                        )
                    )
                } else {
                    diagnostics = diagnostics + Diagnostics.error(
                        code = "UNRESOLVED_VERSION",
                        message = "Library '$alias': version ref '${originalRef.name}' not found in versions map",
                        processorId = id,
                        context = mapOf("alias" to alias, "ref" to originalRef.name),
                    )
                    lib
                }
            }
            else -> lib
        }
    }

    // Разрешение версий плагинов
    val resolvedPlugins = metadata.plugins.mapValues { (alias, plugin) ->
        if (plugin.version != null) return@mapValues plugin

        val originalRef = originalPlugins[alias]?.version
        when (originalRef) {
            is VersionReference.Reference -> {
                val resolved = metadata.versions[originalRef.name]
                if (resolved != null) {
                    plugin.copy(
                        version = ResolvedVersion(
                            alias = originalRef.name,
                            value = resolved.value,
                            source = resolved.source,
                            originalRef = originalRef.name,
                        )
                    )
                } else {
                    diagnostics = diagnostics + Diagnostics.error(
                        code = "UNRESOLVED_VERSION",
                        message = "Plugin '$alias': version ref '${originalRef.name}' not found",
                        processorId = id,
                        context = mapOf("alias" to alias, "ref" to originalRef.name),
                    )
                    plugin
                }
            }
            else -> plugin
        }
    }

    return metadata.copy(
        libraries = resolvedLibraries,
        plugins = resolvedPlugins,
        diagnostics = diagnostics,
    )
}
----

[source,kotlin]
----
// Input
// versions: {"kotlin" -> ResolvedVersion("kotlin", "2.1.20", DECLARED)}
// libraries: {"kotlin-stdlib" -> EffectiveLibrary(..., version=null)}
//   original: Library("kotlin-stdlib", ..., version=Reference("kotlin"))

// Output
// libraries: {"kotlin-stdlib" -> EffectiveLibrary(...,
//   version=ResolvedVersion("kotlin", "2.1.20", DECLARED, originalRef="kotlin"))}
----

=== Edge cases

- Библиотека уже имеет resolved version (Literal) -- пропускается
- Reference на несуществующую версию -- error UNRESOLVED_VERSION, version остается null
- Библиотека отфильтрована LibraryFilterProcessor -- не в metadata, не обрабатывается
- Плагин с Literal версией -- уже resolved в MetadataConversionProcessor
- `VersionReference.Range` -- оставляется как null (range не разрешается)
- Библиотека в effective metadata но отсутствует в originalMetadata -- пропускается (нет originalRef для lookup)

[cols="1,3"]
|===
| Тест | Описание

| `reference resolved` | Reference("kotlin") + versions["kotlin"] -> resolved
| `reference not found` | Reference("missing") -> UNRESOLVED_VERSION error
| `literal already resolved` | Literal -> не изменяется
| `range stays null` | Range -> version остается null
| `plugin reference resolved` | Аналогично для плагинов
| `plugin reference not found` | Аналогично -> UNRESOLVED_VERSION
| `mixed versions` | Несколько библиотек с разными типами
| `no libraries to resolve` | Все version != null -> no changes

[[spec-bundle-filter]]
== SPEC-03.7: BundleFilterProcessor

=== Идентификация

[cols="1,2"]
|===
| Поле | Значение
| id | `bundle-filter`
| phase | `ProcessingPhase.BUNDLE_FILTER`
| order | `40`
| isOptional | `false`
| executionMode | `SEQUENTIAL`
|===

=== Назначение

Фильтрация бандлов: удалить ссылки на отфильтрованные библиотеки и пустые бандлы.
Применить BundleFilter дистрибуции, если задан.

=== Зависимости от других процессоров

- *Зависит от:* LibraryFilterProcessor (order=20) -- некоторые библиотеки могут быть удалены
- *Блокирует:* PluginFilterProcessor (order=45)

=== Алгоритм

[source,text]
----
1. Получить множество оставшихся alias библиотек: existingAliases = metadata.libraries.keys

2. Для каждого бандла в metadata.bundles:
   a. Отфильтровать libraries: оставить только те alias которые есть в existingAliases
   b. Для каждого удаленного -> warning BUNDLE_LIBRARY_MISSING

3. Применить BundleFilter дистрибуции (если есть):
   a. Получить FilterSpec из distribution
   b. byBundles.includes: оставить только бандлы из includes
   c. byBundles.excludes: удалить бандлы из excludes

4. Удалить пустые бандлы (libraries.isEmpty()):
   a. Для каждого -> warning BUNDLE_EMPTIED

5. Вернуть metadata.copy(bundles = filteredBundles, diagnostics = ...)
----

=== Детальная реализация

[source,kotlin]
----
override suspend fun process(
    metadata: EffectiveMetadata,
    context: ProcessingContext,
): EffectiveMetadata {
    val existingLibraryAliases = metadata.libraries.keys
    var diagnostics = metadata.diagnostics

    // 1. Очистка ссылок на несуществующие библиотеки
    var cleanedBundles = metadata.bundles.mapValues { (bundleAlias, bundle) ->
        val validLibraries = bundle.libraries.filter { libAlias ->
            val exists = libAlias in existingLibraryAliases
            if (!exists) {
                diagnostics = diagnostics + Diagnostics.warning(
                    code = "BUNDLE_LIBRARY_MISSING",
                    message = "Bundle '$bundleAlias': library '$libAlias' not found (filtered out?)",
                    processorId = id,
                )
            }
            exists
        }
        bundle.copy(libraries = validLibraries)
    }

    // 2. Применить BundleFilter дистрибуции
    val distName = metadata.distribution
    if (distName != null) {
        val distribution = context.originalMetadata.distributions.find { it.name == distName }
        val bundleFilter = distribution?.spec?.byBundles
        if (bundleFilter != null) {
            cleanedBundles = cleanedBundles.filter { (alias, _) ->
                val passesIncludes = bundleFilter.includes.isEmpty() || alias in bundleFilter.includes
                val passesExcludes = bundleFilter.excludes.isEmpty() || alias !in bundleFilter.excludes
                passesIncludes && passesExcludes
            }
        }
    }

    // 3. Удалить пустые бандлы
    val finalBundles = cleanedBundles.filter { (alias, bundle) ->
        val notEmpty = bundle.libraries.isNotEmpty()
        if (!notEmpty) {
            diagnostics = diagnostics + Diagnostics.warning(
                code = "BUNDLE_EMPTIED",
                message = "Bundle '$alias' removed: no libraries remaining after filtering",
                processorId = id,
            )
        }
        notEmpty
    }

    return metadata.copy(bundles = finalBundles, diagnostics = diagnostics)
}
----

=== Edge cases

- Нет бандлов -- no-op
- Бандл ссылается на отфильтрованную библиотеку -- ссылка удаляется + warning
- Бандл стал пустым -- удаляется + warning
- Нет активной дистрибуции -- только очистка невалидных ссылок, BundleFilter не применяется
- BundleFilter с пустыми includes/excludes -- все проходят

=== Тестовые сценарии

[cols="1,3"]
|===
| Тест | Описание

| `no bundles` | Пустой map -> пустой map
| `library references cleaned` | Отфильтрованные библиотеки удалены из бандлов
| `empty bundle removed` | Бандл без библиотек -> удален
| `bundle filter applied` | BundleFilter includes/excludes
| `no distribution - only cleanup` | Без BundleFilter, только очистка ссылок
| `all libraries in bundle exist` | Бандл не изменяется
| `diagnostics for each issue` | Каждое удаление генерирует warning
|===

[[spec-plugin-filter]]
== SPEC-03.8: PluginFilterProcessor

=== Идентификация

[cols="1,2"]
|===
| Поле | Значение
| id | `plugin-filter`
| phase | `ProcessingPhase.PLUGIN_FILTER`
| order | `45`
| isOptional | `false`
| executionMode | `SEQUENTIAL`
|===

=== Назначение

Фильтрация плагинов по тегам на основе дистрибуции.

=== Зависимости от других процессоров

- *Зависит от:* ProfileProcessor (order=5), MetadataConversionProcessor (order=10)
- *Блокирует:* PluginProcessor (order=50)

=== Алгоритм

[source,text]
----
1. Получить activeDistribution из metadata.distribution
2. ЕСЛИ null -> вернуть metadata (все плагины проходят)
3. Найти Distribution, получить FilterSpec
4. ЕСЛИ spec == null или byTags == null -> вернуть metadata
5. Для каждого плагина:
   a. Получить tags плагина из context.originalMetadata.plugins (по alias)
   b. Применить TagFilter (аналогично LibraryFilterProcessor)
   c. Если не проходит -> info diagnostic PLUGIN_FILTERED
6. Вернуть metadata.copy(plugins = filteredPlugins, diagnostics = ...)
----

=== Детальная реализация

[source,kotlin]
----
override suspend fun process(
    metadata: EffectiveMetadata,
    context: ProcessingContext,
): EffectiveMetadata {
    val distName = metadata.distribution ?: return metadata
    val distribution = context.originalMetadata.distributions.find { it.name == distName }
    val tagFilter = distribution?.spec?.byTags ?: return metadata

    val originalPluginsIndex = context.originalMetadata.plugins.associateBy { it.alias }
    var diagnostics = metadata.diagnostics

    val filteredPlugins = metadata.plugins.filter { (alias, _) ->
        val originalPlugin = originalPluginsIndex[alias]
        val tags = originalPlugin?.tags ?: emptySet()

        val passes = passesTagFilter(tags, tagFilter)
        if (!passes) {
            diagnostics = diagnostics + Diagnostics.info(
                code = "PLUGIN_FILTERED",
                message = "Plugin '$alias' filtered out by distribution '$distName'",
                processorId = id,
            )
        }
        passes
    }

    return metadata.copy(plugins = filteredPlugins, diagnostics = diagnostics)
}

private fun passesTagFilter(tags: Set<String>, filter: TagFilter): Boolean {
    val passesIncludes = if (filter.includes.isEmpty()) true
    else filter.includes.any { include ->
        val anyOfOk = include.anyOf.isEmpty() || (tags intersect include.anyOf).isNotEmpty()
        val allOfOk = include.allOf.isEmpty() || tags.containsAll(include.allOf)
        anyOfOk && allOfOk
    }

    val passesExcludes = filter.excludes.all { exclude ->
        exclude.anyOf.isEmpty() || (tags intersect exclude.anyOf).isEmpty()
    }

    return passesIncludes && passesExcludes
}
----

=== Edge cases

- Нет дистрибуции -- все плагины проходят
- Плагин без тегов -- не проходит includes (если есть)
- Плагин отсутствует в originalMetadata -- tags = emptySet()
- Только byGroups без byTags -- плагины не фильтруются (плагины не имеют group/artifact)

=== Тестовые сценарии

[cols="1,3"]
|===
| Тест | Описание

| `no distribution` | Все плагины остаются
| `tag filter applied` | Плагины фильтруются по тегам
| `no tag filter in spec` | Все плагины остаются
| `plugin without tags` | Не проходит если includes непуст
| `exclude applied` | Плагин с исключающим тегом удален
|===

| phase | `ProcessingPhase.PLUGIN`
| order | `50`
| isOptional | `false`

=== Назначение

Разрешить VersionReference.Reference для плагинов и обработать метаданные плагинов.

override suspend fun process(
    metadata: EffectiveMetadata,
    context: ProcessingContext,

                        message = "Plugin '$alias': version ref '${originalRef.name}' not found",

                        context = mapOf("alias" to alias, "ref" to originalRef.name),

                }
            }
            else -> plugin
        }

[[spec-used-versions]]
== SPEC-03.10: UsedVersionsProcessor

=== Идентификация

[cols="1,2"]
|===
| Поле | Значение
| id | `used-versions`
| phase | `ProcessingPhase.USED_VERSIONS`
| order | `60`
| isOptional | `false`
| executionMode | `SEQUENTIAL`
|===

=== Назначение

Удалить неиспользуемые версии из `metadata.versions`.
После фильтрации библиотек и плагинов некоторые версии могут остаться без ссылок.
Очистка важна для генерации TOML -- версии без ссылок создают "мертвый" код.

=== Зависимости от других процессоров

- *Зависит от:* Все фильтры (order 20-50) и resolver (order 30)
- *Блокирует:* ValidationProcessor (order=65) -- валидирует уже очищенные данные

=== Алгоритм

[source,text]
----
1. Собрать множество используемых имен версий:
   a. Из libraries: для каждой library.version?.alias (если не пустой)
   b. Из plugins: для каждой plugin.version?.alias (если не пустой)
   c. Также учесть originalRef: library.version?.originalRef, plugin.version?.originalRef

2. Отфильтровать versions: оставить только те, чей ключ в usedNames

3. Для каждой удаленной версии -> info diagnostic UNUSED_VERSION_REMOVED

4. Вернуть metadata.copy(versions = usedVersions, diagnostics = ...)
----

=== Детальная реализация

[source,kotlin]
----
override suspend fun process(
    metadata: EffectiveMetadata,
    context: ProcessingContext,
): EffectiveMetadata {
    val usedVersionNames = mutableSetOf<String>()

    // Собираем используемые версии из библиотек
    for ((_, lib) in metadata.libraries) {
        lib.version?.let { v ->
            if (v.alias.isNotEmpty()) usedVersionNames.add(v.alias)
            v.originalRef?.let { usedVersionNames.add(it) }
        }
    }

    // Собираем используемые версии из плагинов
    for ((_, plugin) in metadata.plugins) {
        plugin.version?.let { v ->
            if (v.alias.isNotEmpty()) usedVersionNames.add(v.alias)
            v.originalRef?.let { usedVersionNames.add(it) }
        }
    }

    var diagnostics = metadata.diagnostics

    val usedVersions = metadata.versions.filter { (name, _) ->
        val used = name in usedVersionNames
        if (!used) {
            diagnostics = diagnostics + Diagnostics.info(
                code = "UNUSED_VERSION_REMOVED",
                message = "Version '$name' removed: not referenced by any library or plugin",
                processorId = id,
            )
        }
        used
    }

    return metadata.copy(versions = usedVersions, diagnostics = diagnostics)
}
----

- Все версии используются -- без изменений
- Нет библиотек и плагинов -- все версии удаляются
- Версия используется плагином но не библиотекой -- сохраняется

[[spec-validation]]
== SPEC-03.11: ValidationProcessor

=== Идентификация

[cols="1,2"]
|===
| Поле | Значение
| id | `validation`
| phase | `ProcessingPhase.VALIDATION`
| order | `65`
| isOptional | `false`
| executionMode | `SEQUENTIAL`
|===

=== Назначение

Валидация итогового `EffectiveMetadata` на корректность.
Генерирует диагностические сообщения в зависимости от `ValidationSettings`.

=== Зависимости от других процессоров

- *Зависит от:* Все предшествующие процессоры (данные уже обработаны)
- *Блокирует:* CompatRulesProcessor (order=66)

=== Алгоритм

[source,text]
----
ВАЛИДАЦИЯ 1: Дублирующиеся алиасы
  a. Проверить уникальность ключей в libraries (уже обеспечивается Map, но проверить оригинал)
  b. Проверить уникальность в plugins
  c. Проверить уникальность в bundles
  d. Проверить пересечение namespaces: alias не должен использоваться одновременно как library и plugin
  e. При нарушении: VALIDATION_DUPLICATE_ALIAS (ERROR)

ВАЛИДАЦИЯ 2: Все ссылки на версии разрешены
  a. Для каждой библиотеки: если в оригинале VersionReference.Reference,
     то в effective должен быть ResolvedVersion
  b. Для каждого плагина: аналогично
  c. Действие зависит от settings.validation.onUnresolvedVersion (дефолт: **WARN**):
     - FAIL -> ERROR
     - WARN -> WARNING (дефолт)
     - IGNORE -> пропустить
  d. Код: VALIDATION_UNRESOLVED_REF

ВАЛИДАЦИЯ 3: Все ссылки бандлов на библиотеки корректны
  a. Для каждого бандла: все libraries[] должны существовать в metadata.libraries
  b. При нарушении: VALIDATION_BUNDLE_REF_MISSING (ERROR)

ВАЛИДАЦИЯ 4: Нет циклических extends бандлов
  a. Проверить originalMetadata.bundles на циклы в extends
  b. Алгоритм: DFS с detection of back edges
  c. При нарушении: VALIDATION_CIRCULAR_EXTENDS (ERROR)

ВАЛИДАЦИЯ 5: Deprecated replacedBy ссылается на существующее
  a. Для каждой deprecated библиотеки с deprecation.replacedBy:
     - replacedBy должен существовать в metadata.libraries
  b. Действие зависит от settings.validation.onDeprecatedLibrary (дефолт: **WARN**):
     - FAIL -> ERROR
     - WARN -> WARNING (дефолт)
     - IGNORE -> пропустить
  c. Код: VALIDATION_DEPRECATED_REF

ВАЛИДАЦИЯ 6: Корректность Maven-координат
  a. Для каждой библиотеки: group и artifact не пустые
  b. group не содержит ":" (валидный Maven groupId)
  c. artifact не содержит ":" (валидный Maven artifactId)
  d. При нарушении: VALIDATION_INVALID_COORDINATES (ERROR)
----

=== Дефолтные значения ValidationSettings

Дефолты определены в `Settings.kt`:

[cols="1,1,2"]
|===
| Настройка | Дефолт | Описание

| `onCompatibilityViolation` | `FAIL` | Нарушение совместимости -> ERROR по умолчанию
| `onDeprecatedLibrary` | `WARN` | Deprecated библиотека -> WARNING по умолчанию
| `onJdkMismatch` | `FAIL` | Несовместимость JDK -> ERROR по умолчанию
| `onUnresolvedVersion` | `WARN` | Неразрешённая версия -> WARNING по умолчанию

Эти значения применяются когда пользователь не переопределил настройки.
В `STRICT` preset все значения переопределяются на `FAIL`.

=== Детальная реализация

[source,kotlin]
----
override suspend fun process(
    metadata: EffectiveMetadata,
    context: ProcessingContext,
): EffectiveMetadata {
    val validationSettings = context.settings.validation
    var diagnostics = metadata.diagnostics

    // 1. Дублирующиеся алиасы (cross-namespace)
    diagnostics = diagnostics + validateDuplicateAliases(metadata)

    // 2. Неразрешенные версии
    diagnostics = diagnostics + validateUnresolvedVersions(
        metadata, context, validationSettings.onUnresolvedVersion,
    )

    // 3. Ссылки бандлов
    diagnostics = diagnostics + validateBundleReferences(metadata)

    // 4. Циклические extends
    diagnostics = diagnostics + validateCircularExtends(context.originalMetadata)

    // 5. Deprecated replacedBy
    diagnostics = diagnostics + validateDeprecatedReferences(
        metadata, validationSettings.onDeprecatedLibrary,
    )

    // 6. Maven-координаты
    diagnostics = diagnostics + validateCoordinates(metadata)

    return metadata.copy(diagnostics = diagnostics)
}

private fun validateDuplicateAliases(metadata: EffectiveMetadata): Diagnostics {
    val allAliases = mutableMapOf<String, MutableList<String>>()
    metadata.libraries.keys.forEach { allAliases.getOrPut(it) { mutableListOf() }.add("library") }
    metadata.plugins.keys.forEach { allAliases.getOrPut(it) { mutableListOf() }.add("plugin") }
    metadata.bundles.keys.forEach { allAliases.getOrPut(it) { mutableListOf() }.add("bundle") }

    val duplicates = allAliases.filter { it.value.size > 1 }
    return duplicates.map { (alias, types) ->
        DiagnosticMessage(
            code = "VALIDATION_DUPLICATE_ALIAS",
            message = "Alias '$alias' used in multiple namespaces: ${types.joinToString()}",
            severity = Severity.ERROR,
            processorId = "validation",
        )
    }.let { messages ->
        Diagnostics(errors = messages)
    }
}

private fun validateUnresolvedVersions(
    metadata: EffectiveMetadata,
    context: ProcessingContext,
    action: ValidationAction,
): Diagnostics {
    if (action == ValidationAction.IGNORE) return Diagnostics()

    val originalLibs = context.originalMetadata.libraries.associateBy { it.alias }
    val issues = mutableListOf<DiagnosticMessage>()

    for ((alias, lib) in metadata.libraries) {
        val originalRef = originalLibs[alias]?.version
        if (originalRef is VersionReference.Reference && lib.version == null) {
            issues.add(DiagnosticMessage(
                code = "VALIDATION_UNRESOLVED_REF",
                message = "Library '$alias': version reference '${originalRef.name}' is not resolved",
                severity = action.toSeverity(),
                processorId = "validation",
            ))
        }
    }
    return Diagnostics.of(*issues.toTypedArray())
}

private fun validateBundleReferences(metadata: EffectiveMetadata): Diagnostics {
    val issues = mutableListOf<DiagnosticMessage>()
    for ((bundleAlias, bundle) in metadata.bundles) {
        for (libAlias in bundle.libraries) {
            if (libAlias !in metadata.libraries) {
                issues.add(DiagnosticMessage(
                    code = "VALIDATION_BUNDLE_REF_MISSING",
                    message = "Bundle '$bundleAlias': library '$libAlias' does not exist",
                    severity = Severity.ERROR,
                    processorId = "validation",
                ))
            }
        }
    }
    return Diagnostics.of(*issues.toTypedArray())
}

private fun validateCircularExtends(metadata: DependangerMetadata): Diagnostics {
    val bundleIndex = metadata.bundles.associateBy { it.alias }
    val visited = mutableSetOf<String>()
    val inStack = mutableSetOf<String>()
    val cycles = mutableListOf<String>()

    fun dfs(alias: String) {
        if (alias in inStack) { cycles.add(alias); return }
        if (alias in visited) return
        visited.add(alias)
        inStack.add(alias)
        bundleIndex[alias]?.extends?.forEach { dfs(it) }
        inStack.remove(alias)
    }

    bundleIndex.keys.forEach { dfs(it) }

    return cycles.map { alias ->
        DiagnosticMessage(
            code = "VALIDATION_CIRCULAR_EXTENDS",
            message = "Bundle '$alias' has circular extends dependency",
            severity = Severity.ERROR,
            processorId = "validation",
        )
    }.let { Diagnostics(errors = it) }
}

private fun validateDeprecatedReferences(
    metadata: EffectiveMetadata,
    action: ValidationAction,
): Diagnostics {
    if (action == ValidationAction.IGNORE) return Diagnostics()

    val issues = mutableListOf<DiagnosticMessage>()
    for ((alias, lib) in metadata.libraries) {
        if (lib.isDeprecated && lib.deprecation?.replacedBy != null) {
            if (lib.deprecation.replacedBy !in metadata.libraries) {
                issues.add(DiagnosticMessage(
                    code = "VALIDATION_DEPRECATED_REF",
                    message = "Library '$alias': replacedBy '${lib.deprecation.replacedBy}' does not exist",
                    severity = action.toSeverity(),
                    processorId = "validation",
                ))
            }
        }
    }
    return Diagnostics.of(*issues.toTypedArray())
}

private fun validateCoordinates(metadata: EffectiveMetadata): Diagnostics {
    val issues = mutableListOf<DiagnosticMessage>()
    for ((alias, lib) in metadata.libraries) {
        if (lib.group.isBlank() || lib.artifact.isBlank()) {
            issues.add(DiagnosticMessage(
                code = "VALIDATION_INVALID_COORDINATES",
                message = "Library '$alias': empty group or artifact (group='${lib.group}', artifact='${lib.artifact}')",
                severity = Severity.ERROR,
                processorId = "validation",
            ))
        }
        if (":" in lib.group || ":" in lib.artifact) {
            issues.add(DiagnosticMessage(
                code = "VALIDATION_INVALID_COORDINATES",
                message = "Library '$alias': group or artifact contains ':' (group='${lib.group}', artifact='${lib.artifact}')",
                severity = Severity.ERROR,
                processorId = "validation",
            ))
        }
    }
    return Diagnostics.of(*issues.toTypedArray())
}

private fun ValidationAction.toSeverity(): Severity = when (this) {
    ValidationAction.FAIL -> Severity.ERROR
    ValidationAction.WARN -> Severity.WARNING
    ValidationAction.INFO -> Severity.INFO
    ValidationAction.IGNORE -> Severity.INFO // не должен вызываться
}

=== Edge cases

- Пустые данные -- нет ошибок валидации
- `onUnresolvedVersion = IGNORE` -- неразрешенные версии не генерируют диагностику
- Alias используется как library и bundle -- VALIDATION_DUPLICATE_ALIAS
- Циклические extends: A -> B -> A -- обнаруживается DFS
- replacedBy ссылается на саму себя -- формально валидно (alias существует)
- Пустой group -- VALIDATION_INVALID_COORDINATES

[cols="1,3"]
|===
| Тест | Описание

| `valid metadata - no diagnostics` | Корректные данные -> без новых ошибок
| `duplicate alias across namespaces` | library + plugin с одним alias -> error

[[spec-compat-rules]]
== SPEC-03.12: CompatRulesProcessor

=== Идентификация

[cols="1,2"]
|===
| Поле | Значение
| id | `compat-rules`
| phase | `ProcessingPhase.COMPAT_RULES`
| order | `66`
| isOptional | `false`
| executionMode | `SEQUENTIAL`
|===

=== Назначение

Проверка декларативных правил совместимости (`CompatibilityRule`) против эффективных метаданных.
Обрабатывает только статические проверки, не требующие I/O: `JdkRequirement`, `MutualExclusion`, `VersionConstraint`.
`CustomRule` делегируется `CompatibilityCheckProcessor` (order=110) из модуля `dependanger-analysis` (см. xref:SPEC-08-dependanger-analysis.adoc[SPEC-08]).
Результаты записываются в `extensions` через `CompatibilityIssuesExtensionKey`
и дублируются в `diagnostics`.

=== Зависимости от других процессоров

- *Зависит от:* Все предшествующие (данные должны быть финализированы)
- *Блокирует:* Ничего (последний core-процессор)

=== Алгоритм

[source,text]
----
1. Получить rules из context.originalMetadata.compatibility
2. ЕСЛИ rules пуст -> вернуть metadata

3. ДЛЯ КАЖДОГО rule:

   a. JdkRequirement:
      - Найти библиотеки matching rule.matches (glob по alias или "group:artifact")
      - Для каждой matching библиотеки:
        - ЕСЛИ environment.jdkVersion != null:
          - ЕСЛИ rule.minJdk != null && jdkVersion < rule.minJdk -> violation
          - ЕСЛИ rule.maxJdk != null && jdkVersion > rule.maxJdk -> violation
        - Создать CompatibilityIssue с severity из rule

   b. MutualExclusion:
      - Проверить, сколько библиотек из rule.libraries присутствуют в metadata.libraries
      - ЕСЛИ больше одной -> violation
      - Сообщение: "Libraries X and Y are mutually exclusive"

   c. VersionConstraint:
      - Получить resolved versions для библиотек из rule.libraries
      - В зависимости от constraint type:
        - SAME_VERSION: все версии равны?
        - SAME_MAJOR: все major-версии равны?
        - SAME_MAJOR_MINOR: все major.minor равны?
      - ЕСЛИ не совпадают -> violation

   d. CustomRule:
      - Пропустить -- обработка делегирована CompatibilityCheckProcessor (order=110, feature-модуль `dependanger-analysis`)
      - Добавить INFO diagnostic "Custom rule '{ruleId}' deferred to compatibility-analysis processor"

4. Собрать все CompatibilityIssue
5. Записать в extensions через CompatibilityIssuesExtensionKey
6. Добавить diagnostics на основе severity каждого issue
7. Вернуть обновленный metadata
----

=== Детальная реализация

[source,kotlin]
----
override suspend fun process(
    metadata: EffectiveMetadata,
    context: ProcessingContext,
): EffectiveMetadata {
    val rules = context.originalMetadata.compatibility
    if (rules.isEmpty()) return metadata

    val issues = mutableListOf<CompatibilityIssue>()
    var diagnostics = metadata.diagnostics

    for (rule in rules) {
        val ruleIssues = when (rule) {
            is CompatibilityRule.JdkRequirement ->
                checkJdkRequirement(rule, metadata, context.environment)

            is CompatibilityRule.MutualExclusion ->
                checkMutualExclusion(rule, metadata)

            is CompatibilityRule.VersionConstraint ->
                checkVersionConstraint(rule, metadata)

            is CompatibilityRule.CustomRule -> {
                // CustomRule делегируется CompatibilityCheckProcessor (order=110)
                diagnostics = diagnostics + Diagnostics.info(
                    code = "COMPAT_CUSTOM_RULE_DEFERRED",
                    message = "Custom rule '${rule.ruleId}' deferred to compatibility-analysis processor",
                    processorId = id,
                )
                emptyList()
            }
        }
        issues.addAll(ruleIssues)
    }

    // Записать issues в extensions
    val existingIssues = metadata.compatibilityIssues
    val allIssues = existingIssues + issues

    var result = metadata.withExtension(
        CompatibilityIssuesExtensionKey,
        allIssues,
    )

    // Добавить diagnostics
    for (issue in issues) {
        diagnostics = diagnostics + when (issue.severity) {
            Severity.ERROR -> Diagnostics.error(
                code = "COMPAT_${issue.ruleId.uppercase()}",
                message = issue.message,
                processorId = id,
            )
            Severity.WARNING -> Diagnostics.warning(
                code = "COMPAT_${issue.ruleId.uppercase()}",
                message = issue.message,
                processorId = id,
            )
            Severity.INFO -> Diagnostics.info(
                code = "COMPAT_${issue.ruleId.uppercase()}",
                message = issue.message,
                processorId = id,
            )
        }
    }

    return result.copy(diagnostics = diagnostics)
}

private fun checkJdkRequirement(
    rule: CompatibilityRule.JdkRequirement,
    metadata: EffectiveMetadata,
    environment: ProcessingEnvironment,
): List<CompatibilityIssue> {
    val jdkVersion = environment.jdkVersion ?: return emptyList()
    val matchingLibs = metadata.libraries.filter { (alias, lib) ->
        globMatch("${lib.group}:${lib.artifact}", rule.matches) || globMatch(alias, rule.matches)
    }

    val issues = mutableListOf<CompatibilityIssue>()
    for ((alias, _) in matchingLibs) {
        val violatesMin = rule.minJdk != null && jdkVersion < rule.minJdk
        val violatesMax = rule.maxJdk != null && jdkVersion > rule.maxJdk
        if (violatesMin || violatesMax) {
            issues.add(CompatibilityIssue(
                ruleId = rule.name,
                message = rule.message
                    ?: "Library '$alias' requires JDK ${rule.minJdk ?: "?"}-${rule.maxJdk ?: "?"}, current: $jdkVersion",
                severity = rule.severity,
                affectedLibraries = listOf(alias),
                suggestion = "Upgrade JDK or use a compatible library version",
            ))
        }
    }
    return issues
}

private fun checkMutualExclusion(
    rule: CompatibilityRule.MutualExclusion,
    metadata: EffectiveMetadata,
): List<CompatibilityIssue> {
    val presentLibs = rule.libraries.filter { it in metadata.libraries }
    return if (presentLibs.size > 1) {
        listOf(CompatibilityIssue(
            ruleId = rule.name,
            message = rule.message
                ?: "Mutually exclusive libraries present: ${presentLibs.joinToString()}",
            severity = rule.severity,
            affectedLibraries = presentLibs,
            suggestion = "Remove one of the conflicting libraries",
        ))
    } else emptyList()
}

private fun checkVersionConstraint(
    rule: CompatibilityRule.VersionConstraint,
    metadata: EffectiveMetadata,
): List<CompatibilityIssue> {
    val libraryVersions = rule.libraries.mapNotNull { alias ->
        metadata.libraries[alias]?.version?.value?.let { alias to it }
    }

    if (libraryVersions.size < 2) return emptyList()

    val versions = libraryVersions.map { it.second }
    val violation = when (rule.constraint) {
        VersionConstraintType.SAME_VERSION -> versions.distinct().size > 1
        VersionConstraintType.SAME_MAJOR -> versions.map { parseMajor(it) }.distinct().size > 1
        VersionConstraintType.SAME_MAJOR_MINOR -> versions.map { parseMajorMinor(it) }.distinct().size > 1
    }

    return if (violation) {
        val details = libraryVersions.joinToString { "${it.first}=${it.second}" }
        listOf(CompatibilityIssue(
            ruleId = rule.name,
            message = rule.message
                ?: "Version constraint ${rule.constraint} violated: $details",
            severity = rule.severity,
            affectedLibraries = libraryVersions.map { it.first },
            suggestion = "Align versions according to ${rule.constraint} constraint",
        ))
    } else emptyList()
}

=== Edge cases

- Нет правил -- no-op
- JdkRequirement без jdkVersion в environment -- пропускается
- MutualExclusion с 0 или 1 библиотекой в metadata -- нет нарушения
- VersionConstraint с библиотекой без версии -- пропускается (нет version для сравнения)
- CustomRule -- пропускается с INFO, обработка в CompatibilityCheckProcessor (order=110)
- glob `*` matches все библиотеки
- Версии с pre-release суффиксами ("1.0.0-RC1") -- major/minor парсится из числовых частей

=== Тестовые сценарии

[cols="1,3"]
|===
| Тест | Описание

| `no rules - no issues` | Пустой compatibility -> без issues
| `jdk requirement violation` | jdkVersion < minJdk -> issue
| `jdk requirement satisfied` | jdkVersion >= minJdk -> без issues
| `jdk null - skipped` | jdkVersion=null -> без проверки
| `mutual exclusion detected` | Оба library присутствуют -> issue
| `mutual exclusion ok` | Только одна -> без issues
| `same version violated` | Разные версии -> issue
| `same version ok` | Одинаковые версии -> без issues
| `same major violated` | Разные major -> issue
| `same major minor violated` | Разные major.minor -> issue
| `custom rule deferred` | CustomRule пропускается с INFO, делегирован CompatibilityCheckProcessor
| `issues stored in extensions` | CompatibilityIssue в extensions map
| `issues in diagnostics` | Дублируются как diagnostics
| `glob matching in jdk rule` | `org.springframework:*` matches spring libs

=== Канонический доступ к extensions

Доступ к типизированным extensions в `EffectiveMetadata` осуществляется через:

[source,kotlin]

// Запись (в процессорах):
