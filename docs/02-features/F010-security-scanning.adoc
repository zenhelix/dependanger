== Описание

Security Scanning проверяет библиотеки из каталога на наличие известных уязвимостей (CVE) через OSV API (https://osv.dev).
Результат добавляется в `ProcessingDiagnostics` и включается в отчёт.

== User Stories

=== US-001: Проверка уязвимостей

[quote]
____
Как security engineer, я хочу автоматически проверять библиотеки на известные уязвимости, чтобы своевременно обновлять опасные зависимости.
____

[cols="1,3,3,3",options="header"]

| ID | Given | When | Then

| AC-001-1
| Библиотека `log4j-core:2.14.0` с известной уязвимостью CVE-2021-44228
| Запускаю pipeline с `withSecurityCheck()`
| В diagnostics появляется `VulnerabilityInfo` с `severity = CRITICAL`

| AC-001-2
| Библиотека `spring-web:6.0.0` без известных уязвимостей
| Запускаю security check
| Для этой библиотеки нет записей в vulnerabilities

| AC-001-3
| 10 библиотек в каталоге, 2 с уязвимостями
| Запускаю security check
| В diagnostics ровно 2 `VulnerabilityInfo`

| AC-001-4
| Библиотека с уязвимостью, `fixedVersion = "2.17.0"`
| Запускаю security check
| `VulnerabilityInfo` содержит `fixedVersion` для быстрого исправления

| AC-001-5
| Уязвимость с CVSS score 9.8
| Запускаю security check
| `VulnerabilityInfo` содержит `cvssScore = 9.8`
|===

== API использование

[source,kotlin]
----
val result = Dependanger.fromDsl {
    // ... DSL config
}
.withSecurityCheck {
    minSeverity = Severity.HIGH
    failOnCritical = true
    failOnHigh = false
    ignoreCves = listOf("CVE-2023-ACCEPTED")
    ignoreLibraries = listOf("internal-test-lib")
}
.build()

// Получить информацию об уязвимостях
val vulnerabilities = result.diagnostics.all
    .filterIsInstance<VulnerabilityInfo>()

vulnerabilities.forEach { vuln ->
    println("${vuln.library}: ${vuln.cveId} (${vuln.severity})")
    vuln.fixedVersion?.let { println("  Fix: upgrade to $it") }
}
----

=== Batching

OSV API поддерживает batch запросы (до 1000 packages за раз).
Dependanger группирует библиотеки в batches для эффективности.

== VulnerabilityInfo

Информация об уязвимости содержит:

[cols="1,1,3",options="header"]
|===
| Поле | Тип | Описание

| `library`
| String
| Alias библиотеки в каталоге

| `group`
| String
| Maven groupId

| `name`
| String
| Maven artifactId

| `version`
| String
| Текущая версия с уязвимостью

| `cveId`
| String
| CVE идентификатор (например: CVE-2021-44228)

| `osvId`
| String
| OSV идентификатор (например: GHSA-jfh8-c2jp-5v3q)

| `severity`
| Severity
| CRITICAL, HIGH, MEDIUM, LOW, UNKNOWN

| `cvssScore`
| Double?
| CVSS v3 score (0.0 - 10.0), null если неизвестен

| `summary`
| String
| Краткое описание уязвимости

| `fixedVersion`
| String?
| Версия с исправлением (если известна)

| `references`
| List<String>
| Ссылки на advisory, patches

| `publishedAt`
| Instant
| Дата публикации уязвимости
|===

== Severity

[cols="1,1,3",options="header"]
|===
| Severity | CVSS Score | Описание

| CRITICAL
| 9.0 - 10.0
| Критическая уязвимость, требует немедленного исправления

| HIGH
| 7.0 - 8.9
| Высокая опасность, исправить в ближайшее время

| MEDIUM
| 4.0 - 6.9
| Средняя опасность, запланировать исправление

| LOW
| 0.1 - 3.9
| Низкая опасность, исправить при возможности

| UNKNOWN
| —
| CVSS score неизвестен
|===

== Конфигурация

[source,kotlin]
----
.withSecurityCheck {
    // Фильтрация
    minSeverity = Severity.HIGH        // Минимальный severity (default: HIGH)
    minCvssScore = 7.0                 // Минимальный CVSS score (default: 7.0)

    // Fail условия (по умолчанию только warn)
    failOnCritical = false             // Fail при CRITICAL (default: false)
    failOnHigh = false                 // Fail при HIGH (default: false)

    // Игнорирование
    ignoreCves = listOf("CVE-2023-OLD")      // Игнорировать конкретные CVE
    ignoreLibraries = listOf("test-lib")     // Игнорировать библиотеки

    // Транзитивные зависимости (требует F012)
    includeTransitives = false         // Проверять транзитивные (default: false)

    // Сеть
    timeout = 30.seconds               // Таймаут на запрос
    retries = 3                        // Количество retry при ошибках
}
----

== SPI: SecurityCheckProcessor

`SecurityCheckProcessor` — опциональный процессор (order: 80, phase: PARALLEL_IO), который:

1. Собирает все библиотеки из EffectiveMetadata
2. Batch-запросом отправляет в OSV API
3. Парсит ответ и создаёт `VulnerabilityInfo` для каждой уязвимости
4. Добавляет результаты в `ProcessingDiagnostics`
5. При `failOnCritical/failOnHigh` добавляет ERROR в diagnostics

Включается через `withSecurityCheck()` в builder API.

== Кеширование

* Результаты кешируются в `~/.dependanger/cache/security/`
* TTL: 24 часа (настраивается)
* Ключ кеша: `{group}:{artifact}:{version}`
* При отсутствии сети используется кеш (с WARNING)

== Gradle Task

=== dependangerSecurityCheck

Задача проверки уязвимостей в Gradle.

**Зависимости:**

* `dependangerGenerateEffective` — требуется EffectiveMetadata

**Inputs:**

* `effective-\{distribution\}.json` от `dependangerGenerateEffective`

**Outputs:**

* `build/dependanger/security-report.json` — JSON отчёт
* Console output — текстовый отчёт

**Опции:**

* `minSeverity` — минимальный severity (`CRITICAL`, `HIGH`, `MEDIUM`, `LOW`)
* `minCvssScore` — минимальный CVSS score
* `failOnCritical`, `failOnHigh` — fail при уязвимостях
* `ignoreCves`, `ignoreLibraries` — списки для игнорирования
* `includeTransitives` — проверять транзитивные зависимости

[source,bash]
----
# Базовая проверка
./gradlew dependangerSecurityCheck

# Fail при CRITICAL
./gradlew dependangerSecurityCheck -PfailOnCritical=true

# Только CRITICAL и HIGH
./gradlew dependangerSecurityCheck -PminSeverity=HIGH
----

== Settings DSL

Полная конфигурация проверки безопасности в Gradle DSL.

[source,kotlin]
----
dependanger {
    settings {
        securityCheck {
            // Включение процессора
            enabled = true  // или через processing { checkSecurity = true }

            // Фильтрация
            minSeverity = HIGH  // CRITICAL, HIGH, MEDIUM, LOW
            minCvssScore = 7.0  // 0.0 - 10.0

            // Fail условия
            failOnCritical = false  // default
            failOnHigh = false  // default

            // Игнорирование
            ignoreCves = listOf(
                "CVE-2023-ACCEPTED",
                "CVE-2023-KNOWN-RISK"
            )
            ignoreLibraries = listOf("internal-test-lib")

            // Транзитивные зависимости
            includeTransitives = false  // требует F012

            // Кеширование
            cache {
                enabled = true
                ttl = 24.hours  // default
                directory = file("~/.dependanger/cache/security")
            }

            // Сеть
            timeout = 30.seconds
            retries = 3
            batchSize = 1000  // OSV API batch limit

            // Отчёт
            includeIgnoredInReport = true  // показывать ignoredVulnerabilities
        }
    }

    // Или короткая форма через processing
    processing {
        checkSecurity = true
    }
}
----

== Edge Cases

[cols="1,3,2",options="header"]
|===
| Сценарий | Описание | Ожидаемое поведение

| OSV API недоступен
| api.osv.dev не отвечает, кеш пуст
| WARNING: "Cannot reach OSV API", проверка пропускается

| OSV API недоступен, есть кеш
| api.osv.dev не отвечает, есть кеш < TTL
| Используется кеш, WARNING в логе

| Таймаут запроса
| OSV API отвечает > timeout (30s default)
| WARNING: "Request timeout", retry или пропуск

| Библиотека не найдена в OSV
| Новая/редкая библиотека без данных в OSV
| Библиотека считается безопасной (нет данных != есть уязвимости)

| Rate limiting
| OSV API возвращает 429 Too Many Requests
| Exponential backoff, WARNING после N попыток

| Невалидный CVSS score
| OSV возвращает некорректный CVSS
| WARNING, severity определяется по текстовому полю

| Уязвимость без CVE
| OSV entry без aliases (только OSV ID)
| Используется `osvId`, `cveId = null`

| Множественные уязвимости для одной библиотеки
| log4j-core имеет CVE-2021-44228 и CVE-2021-45046
| Создаются отдельные `VulnerabilityInfo` для каждой

| fixedVersion недоступен
| OSV не содержит информацию о фиксе
| `fixedVersion = null`, рекомендация "check manually"

| Пустой каталог
| EffectiveMetadata без библиотек
| Процессор завершается успешно, 0 уязвимостей

| Все библиотеки безопасны
| Нет уязвимостей ни для одной библиотеки
| INFO: "No vulnerabilities found", пустой список

| Игнорируемая CRITICAL уязвимость
| CVE в `ignoreCves`, `failOnCritical = true`
| Pipeline продолжается, уязвимость в `ignoredVulnerabilities`

| Большой каталог
| 500+ библиотек
| Batch запросы (до 1000 за раз), параллельная обработка

| Приватная библиотека
| Библиотека не опубликована в Maven Central
| OSV не найдёт данных, считается безопасной

| CVSS v2 vs v3
| OSV возвращает только CVSS v2 score
| Конвертация в approximate v3 score или WARNING

| Уязвимость в конкретной версии range
| CVE затрагивает только версии 1.0-1.5, текущая 2.0
| Не показывается (OSV фильтрует по version)

| Concurrent security checks
| Два pipeline запущены параллельно
| Независимое выполнение, общий кеш (thread-safe)
|===
