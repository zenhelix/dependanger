= F012: Transitive Resolution
:toc: left
:toclevels: 3
:sectnums:
:icons: font

[abstract]
--
Резолвинг и визуализация транзитивных зависимостей с поддержкой constraints для контроля версий.
--

== Метаданные

[cols="1,2"]
|===
| ID | F012
| Название | Transitive Resolution
| Приоритет | P4 (будущее)
| Модуль | `dependanger-transitive`
| Зависит от | F003 (Processing Pipeline)
| Блокирует | —
|===

== Описание

Transitive Resolution предоставляет:

* **Резолвинг** транзитивных зависимостей из Maven repositories
* **Визуализация** дерева зависимостей в отчёте
* **Интеграция** с F010 (Security) и F011 (Licenses) для проверки транзитивных
* **Constraints** для принудительного контроля версий транзитивных зависимостей

NOTE: Это **информационная** фича.
Dependanger не заменяет dependency resolution в Gradle/Maven, а дополняет его visibility и constraints.

== DSL: Constraints

=== Синтаксис

[source,kotlin]
----
dependanger {
    // Прямые зависимости
    library("spring-web", "org.springframework:spring-web:6.2.0")

    // Constraints для транзитивных зависимостей
    constraints {
        // Принудительная версия
        constraint("log4j-core", "org.apache.logging.log4j:log4j-core:2.17.0") {
            reason = "Security fix for CVE-2021-44228"
        }

        // Короткая форма
        constraint("snakeyaml", "org.yaml:snakeyaml:2.0")

        // Исключение транзитивной зависимости
        exclude("commons-logging", "commons-logging:commons-logging") {
            reason = "Using SLF4J instead"
        }

        // Замена одной библиотеки на другую
        substitute("javax-annotation", "jakarta.annotation:jakarta.annotation-api:2.1.1") {
            from = "javax.annotation:javax.annotation-api"
            reason = "Jakarta EE migration"
        }
    }
}
----

=== Доменная модель

[source,kotlin]
----
data class Constraint(
    val alias: String,
    val coordinates: MavenCoordinates,
    val type: ConstraintType,
    val reason: String? = null,
    val substituteFrom: MavenCoordinates? = null  // только для SUBSTITUTE
)

enum class ConstraintType {
    VERSION,      // Принудительная версия
    EXCLUDE,      // Исключить зависимость
    SUBSTITUTE    // Заменить на другую
}
----

=== Алгоритм резолвинга

[source]
----
1. Для каждой прямой зависимости:
   a. Проверить кэш
   b. Если нет в кэше — скачать POM из repository
   c. Распарсить <dependencies> секцию
   d. Рекурсивно резолвить каждую зависимость (до maxDepth)

2. Построить дерево зависимостей

3. Обнаружить конфликты версий:
   a. Найти артефакты с разными версиями
   b. Применить стратегию HIGHEST
   c. Записать конфликты в отчёт

4. Применить constraints:
   a. VERSION — заменить resolved version
   b. EXCLUDE — удалить из дерева
   c. SUBSTITUTE — заменить артефакт

5. Передать результат следующим процессорам (security-check, license-check)
----

=== Conflict Resolution Strategy

[cols="1,3",options="header"]
|===
| Стратегия | Описание

| HIGHEST
| Использовать наивысшую версию (как Gradle по умолчанию). Конфликты записываются в отчёт.
|===

NOTE: В текущей версии поддерживается только стратегия HIGHEST.
Другие стратегии (FAIL, LOWEST, PREFER_PROJECT) могут быть добавлены в будущем.

== Gradle Task

=== dependangerResolveTransitives

Задача резолвинга транзитивных зависимостей в Gradle.

**Зависимости:**

* `dependangerGenerateEffective` — требуется EffectiveMetadata

**Inputs:**

* `effective-\{distribution\}.json` от `dependangerGenerateEffective`

**Outputs:**

* `build/dependanger/transitives.json` — JSON с деревом зависимостей
* `build/dependanger/dependanger-constraints.gradle.kts` — Gradle snippet для constraints
* Console output — текстовый отчёт

**Опции:**

* `maxDepth` — максимальная глубина резолвинга
* `showConflicts` — показать только конфликты
* `generateConstraints` — генерировать Gradle snippet

[source,bash]
----
# Базовый резолвинг
./gradlew dependangerResolveTransitives

# Только конфликты
./gradlew dependangerResolveTransitives -PshowConflicts=true

# С ограничением глубины
./gradlew dependangerResolveTransitives -PmaxDepth=5
----

== Settings DSL

Полная конфигурация резолвинга транзитивных зависимостей в Gradle DSL.

[source,kotlin]
----
dependanger {
    settings {
        transitiveResolution {
            // Включение процессора
            enabled = true  // или через processing { resolveTransitives = true }

            // Maven repositories для резолвинга
            repositories {
                mavenCentral()
                maven("https://repo.spring.io/milestone")
                maven("https://nexus.example.com/releases") {
                    credentials {
                        username = providers.environmentVariable("NEXUS_USER")
                        password = providers.environmentVariable("NEXUS_PASSWORD")
                    }
                }
            }

            // Глубина резолвинга
            maxDepth = 10  // default

            // Scopes для включения
            scopes = listOf("compile", "runtime")  // default: compile only
            includeOptional = false  // default

            // Кеширование
            cache {
                enabled = true
                ttl = 24.hours  // default
                directory = file("~/.dependanger/cache/transitives")
            }

            // Сеть
            timeout = 30.seconds  // per-library

            // Стратегия разрешения конфликтов
            conflictResolution = HIGHEST  // только HIGHEST в текущей версии

            // Генерация
            generateConstraintsSnippet = true  // default
            constraintsFilename = "dependanger-constraints.gradle.kts"
        }
    }

    // Или короткая форма через processing
    processing {
        resolveTransitives = true
    }
}
----

== Генерация Constraints

=== Gradle Snippet

При наличии constraints генерируется файл `dependanger-constraints.gradle.kts`:

[source,kotlin]
----
// Generated by Dependanger
// Apply: apply(from = "dependanger-constraints.gradle.kts")

dependencies {
    constraints {
        implementation("org.apache.logging.log4j:log4j-core:2.17.0") {
            because("Security fix for CVE-2021-44228")
        }
        implementation("org.yaml:snakeyaml:2.0")
    }
}

configurations.all {
    // Excludes
    exclude(group = "commons-logging", module = "commons-logging")

    // Substitutions
    resolutionStrategy.dependencySubstitution {
        substitute(module("javax.annotation:javax.annotation-api"))
            .using(module("jakarta.annotation:jakarta.annotation-api:2.1.1"))
            .because("Jakarta EE migration")
    }
}
----

=== BOM (Maven)

Constraints генерируются в `<dependencyManagement>` секции BOM:

[source,xml]
----
<dependencyManagement>
    <dependencies>
        <!-- Constraints -->
        <dependency>
            <groupId>org.apache.logging.log4j</groupId>
            <artifactId>log4j-core</artifactId>
            <version>2.17.0</version>
        </dependency>

        <!-- Exclusions через optional + provided scope -->
        <dependency>
            <groupId>commons-logging</groupId>
            <artifactId>commons-logging</artifactId>
            <version>99.0-does-not-exist</version>
            <scope>provided</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
----

NOTE: Maven BOM имеет ограниченную поддержку exclusions и substitutions.
Для полного контроля рекомендуется использовать Gradle.

== Edge Cases

[cols="1,3,2",options="header"]
|===
| Сценарий | Описание | Ожидаемое поведение

| POM не найден
| Артефакт существует, но POM отсутствует в repository
| WARNING, транзитивные для этой библиотеки не резолвятся

| Циклическая зависимость
| A -> B -> C -> A
| Цикл обнаруживается, прерывается, WARNING в логе

| Repository недоступен
| Сетевая ошибка или 5xx
| Retry с exponential backoff, затем WARNING и продолжение

| Очень глубокое дерево
| Глубина > maxDepth
| Резолвинг останавливается, WARNING: "Max depth reached"

| Очень много транзитивных
| 1000+ транзитивных зависимостей
| Резолвинг выполняется (медленнее), рекомендуется увеличить timeout

| Constraint на несуществующую библиотеку
| Constraint для артефакта, которого нет в дереве
| WARNING: "Constraint 'xxx' does not match any transitive dependency"

| Конфликтующие constraints
| Два constraint для одного артефакта с разными версиями
| ERROR: "Conflicting constraints for 'xxx': 2.17.0 vs 2.18.0"

| SNAPSHOT версии
| Транзитивная зависит от SNAPSHOT
| Резолвится как обычная версия, WARNING о нестабильности

| Optional dependencies
| POM содержит `<optional>true</optional>`
| По умолчанию не включаются, можно включить через `includeOptional = true`

| Provided/Test scope
| Зависимости с scope != compile
| По умолчанию только compile, можно настроить через `scopes = [...]`

| Parent POM
| Зависимости определены в parent POM
| Parent POM резолвится и парсится

| BOM в dependencyManagement
| POM импортирует BOM
| BOM резолвится, версии применяются

| Кэш устарел
| TTL кэша истёк
| Перезапрос из repository

| Прерывание резолвинга
| Ctrl+C во время резолвинга
| Graceful shutdown, частичные результаты не сохраняются

| Параллельный резолвинг
| Две библиотеки с общей транзитивной
| Резолвится один раз, результат переиспользуется
|===
