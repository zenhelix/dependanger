= F011: License Compliance
:toc: left
:toclevels: 3
:sectnums:
:icons: font

[abstract]
--
Проверка лицензий библиотек и соответствия политикам (whitelist/blacklist).
Опциональный процессор в pipeline (order: 130, phase: LICENSE_CHECK).
--

== Метаданные

[cols="1,2"]
|===
| ID | F011
| Название | License Compliance
| Приоритет | P3 (желательно)
| Модуль | `dependanger-license`
| Зависит от | F003 (Processing Pipeline)
| Блокирует | —
|===

== Описание

License Compliance проверяет лицензии библиотек из каталога и валидирует их соответствие политикам организации (whitelist/blacklist).
Информация о лицензиях получается из нескольких источников по приоритету.
Результат добавляется в `Diagnostics` и включается в отчёт.

== Источники лицензий

Информация о лицензиях получается из нескольких источников по приоритету (первый найденный выигрывает):

[cols="1,1,3",options="header"]
|===
| Приоритет | Источник | Описание

| 1 (высший)
| **User-defined**
| Явно указано пользователем в DSL через `license()` в блоке библиотеки

| 2
| **Maven POM**
| Секция `<licenses>` из POM файла библиотеки в Maven репозитории

| 3 (низший)
| **ClearlyDefined API**
| Агрегированные данные из https://clearlydefined.io/
|===

=== Логика резолвинга

[source]
----
для каждой библиотеки:
  1. если есть user-defined license → использовать её
  2. иначе запросить Maven POM:
     - если <licenses> заполнен → использовать
     - иначе → перейти к п.3
  3. запросить ClearlyDefined API:
     - если данные есть → использовать
     - иначе → license = UNKNOWN
----

== DSL: Явное указание лицензии

=== Синтаксис

[source,kotlin]
----
libraries {
    // Автоматическое определение (Maven POM → ClearlyDefined)
    library("kotlin-stdlib", "org.jetbrains.kotlin:kotlin-stdlib:2.1.20")

    // Явное указание лицензии (высший приоритет)
    library("internal-lib", "com.company:internal-lib:1.0.0") {
        license("MIT")
    }

    // С URL лицензии
    library("proprietary-sdk", "com.vendor:sdk:2.0.0") {
        license("Proprietary") {
            url = "https://vendor.com/license"
        }
    }

    // Dual-license
    library("dual-licensed", "com.example:dual:1.0.0") {
        license("MIT")
        license("Apache-2.0")
    }

    // SPDX expression
    library("complex-license", "com.example:complex:1.0.0") {
        license(spdxExpression = "MIT OR Apache-2.0")
    }
}
----

== API использование

[source,kotlin]
----
val dependanger = Dependanger.fromDsl {
    // ... DSL config
    settings {
        licenseCheck {
            // Whitelist (если указан — только эти разрешены)
            allowedLicenses = listOf("MIT", "Apache-2.0", "BSD-3-Clause", "ISC")

            // Blacklist (эти всегда запрещены)
            deniedLicenses = listOf("GPL-3.0", "AGPL-3.0")

            // Предупреждения
            warnOnCopyleft = true          // warn для LGPL, MPL, EPL
            warnOnUnknown = true           // warn если лицензия неизвестна

            // Fail условия (по умолчанию false — только warn)
            failOnDenied = false           // fail при denied лицензии
            failOnUnknown = false          // fail при unknown лицензии

            // Исключения
            ignoreLibraries = listOf("test-lib")  // пропустить проверку
        }
    }
}
.preset(ProcessingPreset.STRICT)  // STRICT включает license-check
.build()

val result = dependanger.process()

// Получить нарушения лицензий (через extraction extension)
val violations: List<LicenseViolation> = result.licenseViolations

violations.forEach { violation ->
    println("$\{violation.library\}: $\{violation.message\}")
}
----

== Категории лицензий

[cols="1,2,3",options="header"]
|===
| Категория | Примеры | Описание

| PERMISSIVE
| MIT, Apache-2.0, BSD-2-Clause, BSD-3-Clause, ISC, Unlicense, CC0-1.0
| Разрешительные лицензии с минимальными ограничениями

| WEAK_COPYLEFT
| LGPL-2.1, LGPL-3.0, MPL-2.0, EPL-1.0, EPL-2.0
| Copyleft распространяется только на изменения в самой библиотеке

| STRONG_COPYLEFT
| GPL-2.0, GPL-3.0, AGPL-3.0
| Copyleft распространяется на всё производное ПО

| PUBLIC_DOMAIN
| CC0-1.0, Unlicense, WTFPL
| Отказ от авторских прав

| PROPRIETARY
| Custom, Commercial
| Проприетарные лицензии

| UNKNOWN
| —
| Лицензия не определена
|===

== Конфигурация

[source,kotlin]
----
// Конфигурация через DSL settings
settings {
    licenseCheck {
        // === Политики ===

        // Whitelist — только эти лицензии разрешены
        // Если пустой — whitelist отключён (все разрешены кроме denied)
        allowedLicenses = listOf("MIT", "Apache-2.0", "BSD-3-Clause")

        // Blacklist — эти лицензии запрещены
        deniedLicenses = listOf("GPL-3.0", "AGPL-3.0")

        // === Предупреждения ===

        warnOnCopyleft = true          // warn для weak/strong copyleft (default: true)
        warnOnUnknown = true           // warn если лицензия неизвестна (default: true)

        // === Fail условия (по умолчанию false — только warn) ===

        failOnDenied = false           // fail при denied лицензии
        failOnUnknown = false          // fail при unknown лицензии
        failOnCopyleft = false         // fail при copyleft лицензии

        // === Исключения ===

        ignoreLibraries = listOf("internal-test-lib")  // пропустить проверку

        // === Транзитивные (требует F012) ===

        includeTransitives = false         // Проверять транзитивные (default: false)

        // Таймаут для сетевых запросов
        timeout = 30.seconds
    }
}
----

== SPI: LicenseCheckProcessor

`LicenseCheckProcessor` — опциональный процессор (order: 130, phase: LICENSE_CHECK, executionMode: PARALLEL_IO), который:

1. Собирает все библиотеки из EffectiveMetadata
2. Для каждой библиотеки резолвит лицензию по приоритету источников
3. Классифицирует лицензии по категориям
4. Проверяет соответствие whitelist/blacklist
5. Добавляет `LicenseInfo` и violations в `Diagnostics`

Включается через `withLicenseCheck()` в builder API.

== SPI: LicenseSource

Интерфейс для добавления кастомных источников лицензий:

[source,kotlin]
----
interface LicenseSourceProvider {
    val sourceId: String
    val priority: Int

    suspend fun resolve(
        group: String,
        artifact: String,
        version: String
    ): List<License>?
}
----

Можно добавить свой источник через ServiceLoader.

== Кеширование

* Результаты кешируются в `~/.dependanger/cache/licenses/`
* TTL: 7 дней (лицензии меняются редко)
* Ключ кеша: `{group}:{artifact}:{version}`
* При отсутствии сети используется кеш (с WARNING)

== Gradle Task

=== dependangerLicenseCheck

Задача проверки лицензий в Gradle.

**Зависимости:**

* `dependangerGenerateEffective` — требуется EffectiveMetadata

**Inputs:**

* `effective-\{distribution\}.json` от `dependangerGenerateEffective`

**Outputs:**

* `build/dependanger/license-report.json` — JSON отчёт
* Console output — текстовый отчёт

**Опции:**

* `allowedLicenses`, `deniedLicenses` — whitelist/blacklist
* `failOnDenied`, `failOnUnknown` — fail условия
* `warnOnCopyleft`, `warnOnUnknown` — предупреждения
* `includeTransitives` — проверять транзитивные зависимости

[source,bash]
----
# Базовая проверка
./gradlew dependangerLicenseCheck

# Fail при denied
./gradlew dependangerLicenseCheck -PfailOnDenied=true

# С whitelist
./gradlew dependangerLicenseCheck -PallowedLicenses=MIT,Apache-2.0
----

== Settings DSL

Полная конфигурация проверки лицензий в Gradle DSL.

[source,kotlin]
----
dependanger {
    settings {
        licenseCheck {
            // Включение процессора
            enabled = true  // или через processing { checkLicenses = true }

            // Whitelist (если указан — только эти разрешены)
            allowedLicenses = listOf("MIT", "Apache-2.0", "BSD-3-Clause", "ISC")

            // Blacklist (эти всегда запрещены)
            deniedLicenses = listOf("GPL-3.0", "AGPL-3.0")

            // Предупреждения
            warnOnCopyleft = true  // default: true
            warnOnUnknown = true  // default: true

            // Fail условия (по умолчанию false — только warn)
            failOnDenied = false  // default
            failOnUnknown = false  // default
            failOnCopyleft = false  // default

            // Исключения
            ignoreLibraries = listOf("internal-test-lib")

            // Транзитивные зависимости
            includeTransitives = false  // требует F012

            // Кеширование
            cache {
                enabled = true
                ttl = 7.days  // default: 7 дней (лицензии редко меняются)
                directory = file("~/.dependanger/cache/licenses")
            }

            // Сеть
            timeout = 30.seconds
        }
    }

    // Или короткая форма через processing
    processing {
        checkLicenses = true
    }
}
----

== Edge Cases

[cols="1,3,2",options="header"]
|===
| Сценарий | Описание | Ожидаемое поведение

| Maven POM недоступен
| Maven Central не отвечает, кеш пуст
| Переход к ClearlyDefined, WARNING в логе

| ClearlyDefined недоступен
| API не отвечает, Maven POM пуст
| `category = UNKNOWN`, WARNING

| Оба источника недоступны
| Сеть недоступна, кеш пуст
| `category = UNKNOWN`, WARNING или ERROR при `failOnUnknown`

| Кеш есть, сеть недоступна
| Результат в кеше < TTL
| Используется кеш, WARNING в логе

| Лицензия не в SPDX
| POM содержит "Company Proprietary License"
| `spdxId = null`, `name = "Company Proprietary License"`, `category = PROPRIETARY`

| Dual-license
| POM содержит две лицензии (MIT и Apache-2.0)
| `licenses` содержит обе, при проверке whitelist достаточно одной

| Dual-license с denied
| Лицензии: GPL-3.0 OR MIT, `deniedLicenses = ["GPL-3.0"]`
| OK — есть альтернативная разрешённая лицензия

| Все лицензии denied
| Dual-license: GPL-3.0 OR AGPL-3.0, обе в denied
| Violation — нет разрешённой альтернативы

| Пустой whitelist
| `allowedLicenses = []`
| Whitelist отключён, все лицензии разрешены (кроме denied)

| Whitelist + denied конфликт
| Лицензия в whitelist и denied одновременно
| Denied имеет приоритет — violation

| User-defined override
| DSL: `license("MIT")`, POM: Apache-2.0
| Используется MIT (user-defined приоритет)

| Невалидный SPDX ID
| User указал `license("InvalidLicense")`
| WARNING: "Unknown SPDX ID 'InvalidLicense'", используется как custom

| POM без секции licenses
| `<licenses>` отсутствует в POM
| Переход к ClearlyDefined

| Очень большой каталог
| 500+ библиотек
| Batch запросы, параллельная обработка (PARALLEL_IO)

| Private библиотека
| Библиотека не в Maven Central
| ClearlyDefined не найдёт, `category = UNKNOWN` (если нет user-defined)

| SPDX expression
| `license(spdxExpression = "MIT AND Apache-2.0")`
| Парсинг expression, проверка каждой лицензии

| Deprecated лицензия
| Библиотека имеет deprecated SPDX ID (GPL-2.0 → GPL-2.0-only)
| Нормализация к актуальному ID
|===
